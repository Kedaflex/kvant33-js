## 📖 Что такое формы и зачем они нужны?

Если до этого момента мы изучали, как **показывать** контент пользователю (текст, картинки, таблицы), то формы — это то, как пользователь **передает** данные нам (или нашему серверу).

Без форм интернет был бы «библиотекой» (можно только читать), а с формами он стал «сервисом». Любое действие, требующее ввода данных, — это форма:
- Поиск в Google
- Авторизация во ВКонтакте
- Покупка товара в интернет-магазине
- Написание комментария под видео
- Смена настроек в личном кабинете

Форма — это контейнер, который собирает данные из разных полей ввода и упаковывает их для отправки.

---

## 🏗️ Анатомия тега `<form>`

Тег **`<form>`** является парным. Он не только визуально объединяет элементы ввода, но и сообщает браузеру: «Всё, что находится внутри меня, должно быть отправлено вместе».

### Базовый синтаксис:

```html
<form action="/send-data" method="POST">
  <!-- Здесь будут поля ввода (input, select, textarea) -->
  <button type="submit">Отправить</button>
</form>
```

Важно понимать: сама форма ничего не отправляет «магическим» образом. Она лишь подготавливает данные. Для отправки всегда нужен обработчик на стороне сервера (скрипт на PHP, Python, Node.js и т.д.).

---

## ⚙️ Ключевые атрибуты тега `<form>`

Правильная настройка атрибутов — это 50% успеха. Разберем их максимально подробно.

### 1. `action` — Куда отправляем?
Атрибут `action` указывает **URL-адрес** страницы или скрипта на сервере, который примет и обработает ваши данные.

- Если `action` пустой (или не указан), данные отправятся на ту же самую страницу, где находится форма.
- Может быть относительным путем: `action="/register"`
- Может быть абсолютным URL: `action="https://api.mysite.com/v1/login"`

### 2. `method` — Как отправляем? (GET vs POST)
Это критически важный выбор. Существует два основных способа передачи данных.

| Характеристика | Метод `GET` | Метод `POST` |
|---|---|---|
| **Где данные?** | В адресной строке (URL) | В теле HTTP-запроса (скрыты) |
| **Видимость** | Все данные видны в ссылке | Данные не видны пользователю |
| **Объем данных** | Ограничен (около 2000-4000 символов) | Практически не ограничен |
| **Безопасность** | **Низкая** (нельзя для паролей!) | **Высокая** (подходит для секретных данных) |
| **Кэширование** | Запросы сохраняются в истории браузера | Запросы не сохраняются в истории |
| **Когда использовать?** | Поиск, фильтры, навигация | Регистрация, загрузка файлов, оплата |

> [!warning] Правило безопасности
> Никогда не используйте `method="GET"` для передачи паролей, номеров карт или личной информации. Любой, кто посмотрит в историю браузера или на экран пользователя, увидит эти данные прямо в адресной строке.

### 3. `enctype` — В каком формате?
Этот атрибут определяет, как именно данные будут закодированы перед отправкой.

- **`application/x-www-form-urlencoded`**: (По умолчанию). Данные превращаются в строку вида `name=Ivan&age=25`.
- **`multipart/form-data`**: **Обязателен**, если ваша форма содержит поле для загрузки файлов (`<input type="file">`). Без этого атрибута сервер получит только имя файла, но не сам файл.
- **`text/plain`**: Редко используется, отправляет данные как обычный текст без кодирования.

### 4. `target` — Где открыть результат?
Аналогично тегу `<a>`, определяет, где отобразится ответ сервера после нажатия кнопки «Отправить».

- `_self`: (По умолчанию). В текущей вкладке.
- `_blank`: В новой вкладке (полезно, если вы не хотите, чтобы пользователь уходил с текущей страницы).

### 5. `autocomplete`
Может принимать значения `on` или `off`. Позволяет браузеру подсказывать пользователю данные, которые он вводил ранее (имя, почта, адрес). Для форм с чувствительными данными (например, ввод ПИН-кода) лучше ставить `off`.

### 6. `novalidate`
Если этот атрибут присутствует, браузер не будет проверять правильность заполнения полей (например, наличие символа `@` в почте) перед отправкой. Обычно используется, когда вы хотите реализовать свою проверку на JavaScript.

## 🔄 Как данные попадают на сервер? (Клиент-серверная модель)

Давай разберем процесс по шагам на примере формы поиска:

1. **Заполнение**: Пользователь вводит слово «Котятки» в поле `<input name="query">`.
2. **Клик**: Пользователь нажимает кнопку «Найти» (`type="submit"`).
3. **Сбор**: Браузер находит все элементы ввода **внутри** тега `<form>`.
4. **Упаковка**: Браузер смотрит на атрибут `name` каждого поля. Если у поля нет атрибута `name`, его значение **не будет отправлено**. Это самая частая ошибка новичков!
5. **Отправка**: Если `method="GET"`, браузер сформирует URL: `mysite.com/search?query=Котятки` и перейдет по нему.
6. **Обработка**: Сервер принимает запрос, достает из него слово «Котятки», ищет в базе данных и отдает пользователю страницу с результатами.

## 🔑 Золотое правило: Атрибут `name`

Представь, что ты отправляешь посылку. Значение внутри поля ввода (например, "Ivan") — это содержимое посылки. А атрибут **`name`** (например, "first_name") — это подпись на коробке.
Без подписи сервер не поймет, что это за данные: имя это, фамилия или модель пылесоса.

```html
<!-- Сервер получит: username=jules -->
<input type="text" name="username" value="jules">

<!-- Это значение НЕ БУДЕТ отправлено, сервер его не увидит -->
<input type="text" value="я потеряюсь">
```

## ♿ Доступность форм (A11y)

Формы — самый сложный элемент для людей, использующих скринридеры.

- **Смысловые кнопки**: Всегда используйте тег `<button type="submit">` или `<input type="submit">` для отправки.
- **Группировка**: Если форма большая, разделяйте её на логические блоки (мы изучим это в уроке про `<fieldset>`).
- **Ясные инструкции**: Текст внутри формы должен быть понятным без визуального контекста.

> [!tip] Подсказка
> Всегда указывайте язык страницы в теге `<html lang="ru">`. Это поможет скринридерам правильно озвучивать названия полей формы.

---

## ⚠️ Типичные ошибки

1. **Вложенные формы**: Нельзя вкладывать один тег `<form>` в другой. Браузер запутается, и данные отправятся некорректно.
2. **Забытый `name`**: Если данные не приходят на сервер — первым делом проверяй наличие атрибута `name`.
3. **Отсутствие кнопки отправки**: Форма без кнопки `submit` бесполезна (если только вы не используете сложный JavaScript).
4. **Неверный `enctype` при загрузке файлов**: Если вы загружаете картинку, но забыли `enctype="multipart/form-data"`, файл не дойдет.
5. **Использование GET для паролей**: Это дыра в безопасности.

## 🧪 Практика

Создай файл `form-test.html`. Мы создадим каркас для двух разных типов форм.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Изучаем тег form</title>
    <style>
        body { font-family: sans-serif; padding: 20px; line-height: 1.6; }
        form { border: 1px solid #ccc; padding: 20px; margin-bottom: 20px; border-radius: 8px; }
        h2 { margin-top: 0; }
        .note { font-size: 0.9em; color: #666; }
    </style>
</head>
<body>

    <h1>Примеры структур форм</h1>

    <!-- Форма поиска (используем GET) -->
    <form action="https://www.google.com/search" method="GET" target="_blank">
      <h2>Поиск в Google</h2>
      <p>Эта форма отправит ваш запрос прямо в поисковик Google.</p>
      <input type="text" name="q" placeholder="Что ищем?">
      <button type="submit">Найти в Google</button>
      <p class="note">Обратите внимание на URL после нажатия кнопки.</p>
    </form>

    <!-- Форма регистрации (используем POST) -->
    <form action="/api/register" method="POST">
      <h2>Регистрация (Макет)</h2>
      <p>Данные из этой формы будут скрыты от глаз пользователя при отправке.</p>

      <div>
        <input type="text" name="user_login" placeholder="Придумайте логин">
      </div>
      <br>
      <div>
        <input type="password" name="user_pass" placeholder="Пароль">
      </div>
      <br>
      <button type="submit">Зарегистрироваться</button>
      <p class="note">Эта форма никуда реально не отправит данные, так как адрес /api/register не существует на вашем компьютере.</p>
    </form>

</body>
</html>
```

## ❓ Часто задаваемые вопросы

### «Можно ли сделать форму без тега `<form>`?»
Визуально — да. Вы можете просто расставить поля ввода на странице. Но чтобы отправить их на сервер стандартным способом (кнопкой), они **обязаны** быть внутри `<form>`. Если вы используете JavaScript (технология AJAX/Fetch), вы можете собирать данные из полей и без этого тега, но использование `<form>` всё равно считается хорошим тоном и стандартом доступности.

### «Что произойдет, если нажать Enter внутри формы?»
Браузер автоматически попытается отправить форму (сработает событие `submit`), если в форме есть хотя бы одно текстовое поле и кнопка отправки. Это стандартное поведение, удобное для пользователей.

### «Как отправить форму на электронную почту напрямую?»
Раньше использовали `action="mailto:admin@example.com"`. **Никогда так не делайте.** Это работает плохо: у пользователя должен быть установлен почтовый клиент (типа Outlook), он выдает предупреждения безопасности и часто вообще не срабатывает. Формы всегда должны отправляться на серверный скрипт, который уже сам отправит письмо.

### «Сколько форм может быть на одной странице?»
Сколько угодно. Главное — они не должны быть вложены друг в друга. Каждая форма — это независимый «пакет» данных.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **Тег `<form>`** — это контейнер, объединяющий данные для отправки.
> 2. **`action`** — это адрес "приемника" на сервере.
> 3. **`method="GET"`** — данные в URL, для поиска и фильтров.
> 4. **`method="POST"`** — данные скрыты, для паролей, файлов и важной информации.
> 5. **Атрибут `name`** — обязателен для каждого поля ввода, иначе данные не отправятся.
> 6. **`enctype="multipart/form-data"`** — необходим для загрузки файлов.
> 7. Формы делают сайты интерактивными сервисами, а не просто документами.
