## 📖 Выбор из вариантов

Часто нам не нужно, чтобы пользователь что-то печатал. Нам нужно, чтобы он **выбрал** один или несколько вариантов из предложенных. В HTML для этого есть две разновидности тега `<input>`:
1. **Чекбоксы** (`type="checkbox"`) — «флажки».
2. **Радиокнопки** (`type="radio"`) — «переключатели».

---

## ✅ Чекбоксы (`type="checkbox"`) — Множественный выбор

Чекбоксы используются, когда пользователь может выбрать **любое количество** вариантов (ноль, один или все сразу).

### Базовый синтаксис:

```html
<label>
  <input type="checkbox" name="hobbies" value="reading"> Чтение
</label>

<label>
  <input type="checkbox" name="hobbies" value="coding"> Программирование
</label>
```

### Ключевые атрибуты:
- **`value`**: Это значение, которое будет отправлено на сервер, если чекбокс отмечен. Если вы его забудете, сервер получит просто слово `"on"`, что крайне неудобно.
- **`checked`**: Если добавить этот атрибут, чекбокс будет отмечен по умолчанию при загрузке страницы.
- **`name`**: У группы связанных чекбоксов (например, список хобби) `name` обычно совпадает. Сервер получит массив выбранных значений.

---

## 🔘 Радиокнопки (`type="radio"`) — Одиночный выбор

Радиокнопки используются, когда пользователь должен выбрать **ровно один** вариант из списка. Выбор нового варианта автоматически отменяет предыдущий.

### Почему они называются «радио»?
Это аналогия со старыми кнопками на радиоприемниках: когда вы нажимали кнопку одной станции, кнопка другой станции физически «выпрыгивала» обратно.

### Базовый синтаксис:

```html
<p>Выберите ваш пол:</p>
<label>
  <input type="radio" name="gender" value="male"> Мужской
</label>
<label>
  <input type="radio" name="gender" value="female"> Женский
</label>
```

> [!important] Золотое правило радиокнопок
> Чтобы радиокнопки работали как группа (выбиралась только одна), у них **обязательно должен быть одинаковый атрибут `name`**.

Если вы дадите им разные имена, они будут работать как независимые элементы, и пользователь сможет выбрать все сразу, что является грубой ошибкой.

---

## 📊 Сравнение: Чекбокс vs Радиокнопка

| Характеристика | Чекбокс (`checkbox`) | Радиокнопка (`radio`) |
|---|---|---|
| **Форма** | Квадратная | Круглая |
| **Логика выбора** | Множественный (0, 1, много) | Строго один из группы |
| **Отмена выбора** | Можно кликнуть второй раз, чтобы снять галочку | Нельзя снять выбор (можно только переключить на другой) |
| **Атрибут `name`** | Одинаковый для группы (удобство сервера) | **Обязательно одинаковый** для работы группы |
| **Когда использовать?** | Согласие с правилами, выбор категорий | Выбор тарифа, способа оплаты, пола |

## ⚙️ Атрибут `value` — Что увидит сервер?

В текстовых полях `value` — это то, что ввел пользователь. В чекбоксах и радиокнопках пользователь ничего не вводит, он только переключает состояние. Поэтому вы, как разработчик, должны заранее прописать, какое «секретное слово» уйдет на сервер.

```html
<!-- Если отмечено, сервер получит: news_sub=yes -->
<input type="checkbox" name="news_sub" value="yes" checked> Подписаться на новости
```

Если у вас 10 чекбоксов с разными `value`, но одинаковыми `name`, сервер получит список всех отмеченных `value`.

## ♿ Интерактивность и `<label>`

Как мы учили в прошлом уроке, `<label>` критически важен. Сами по себе кружочки и квадратики инпутов очень маленькие. Попасть по ним мышкой (и тем более пальцем на телефоне) бывает трудно.

**Всегда оборачивайте чекбоксы и радиокнопки в `<label>`.** Это делает кликабельным не только сам инпут, но и текст рядом с ним.

```html
<!-- Плохо (только мелкий квадрат кликабелен) -->
<input type="checkbox"> Я согласен

<!-- Хорошо (весь текст кликабелен) -->
<label>
  <input type="checkbox"> Я согласен
</label>
```

## 🧪 Практика: Форма предпочтений

Давайте создадим форму, в которой используются оба типа элементов.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Выбор вариантов</title>
    <style>
        body { font-family: sans-serif; padding: 30px; line-height: 1.6; }
        .section { background: #f4f4f4; padding: 20px; border-radius: 8px; margin-bottom: 20px; }
        h2 { margin-top: 0; color: #2c3e50; }
        label { display: block; margin-bottom: 10px; cursor: pointer; }
        input[type="radio"], input[type="checkbox"] { transform: scale(1.2); margin-right: 10px; }
        .btn { background: #27ae60; color: white; border: none; padding: 10px 20px; border-radius: 5px; cursor: pointer; }
    </style>
</head>
<body>

    <form action="/submit-survey" method="POST">

        <!-- Группа Радиокнопок -->
        <div class="section">
            <h2>Как часто вы занимаетесь спортом?</h2>
            <label>
                <input type="radio" name="sport_freq" value="daily">
                Каждый день
            </label>
            <label>
                <input type="radio" name="sport_freq" value="weekly" checked>
                Несколько раз в неделю
            </label>
            <label>
                <input type="radio" name="sport_freq" value="monthly">
                Раз в месяц
            </label>
            <label>
                <input type="radio" name="sport_freq" value="never">
                Вообще не занимаюсь
            </label>
        </div>

        <!-- Группа Чекбоксов -->
        <div class="section">
            <h2>Какие языки вы знаете? (можно несколько)</h2>
            <label>
                <input type="checkbox" name="langs" value="html" checked>
                HTML (уже учу!)
            </label>
            <label>
                <input type="checkbox" name="langs" value="css">
                CSS
            </label>
            <label>
                <input type="checkbox" name="langs" value="js">
                JavaScript
            </label>
            <label>
                <input type="checkbox" name="langs" value="python">
                Python
            </label>
        </div>

        <!-- Одиночный чекбокс (согласие) -->
        <div class="section">
            <label>
                <input type="checkbox" name="terms" required>
                Я согласен с условиями обработки персональных данных
            </label>
        </div>

        <button type="submit" class="btn">Отправить анкету</button>

    </form>

</body>
```

## 🛠️ Трюки и современность

### 1. Как сделать кнопку неактивной по умолчанию?
Используйте атрибут `disabled`. Это полезно, например, если вы хотите запретить выбор варианта, которого нет в наличии.
```html
<label style="color: gray;">
  <input type="radio" name="color" value="gold" disabled> Золотой (нет в наличии)
</label>
```

### 2. Стилизация (Краткий обзор)
Стандартные чекбоксы и радиокнопки выглядят «старомодно» и по-разному в разных браузерах. Чтобы сделать их красивыми (например, превратить радиокнопки в стильные переключатели или плитки), разработчики часто скрывают настоящий инпут с помощью CSS и рисуют поверх него свой дизайн, используя псевдоэлементы `::before` и `::after`. Мы научимся этому в модуле CSS.

## ⚠️ Типичные ошибки

1. **Разные `name` у радиокнопок**: Это самая частая ошибка. Группа перестает быть группой, выбор не переключается.
2. **Отсутствие `value`**: Сервер получит `news=on` вместо `news=subscribed`, что затрудняет обработку данных.
3. **Отсутствие `<label>`**: Плохой UX, особенно на мобильных. Пользователь вынужден «целиться» в маленькую точку.
4. **Использование чекбокса там, где нужен радио**: Например, выбор страны доставки. Пользователь может выбрать 5 стран, а ваша логика доставки рассчитана только на одну.
5. **Радиокнопки без выбора по умолчанию**: С точки зрения UX, в группе радиокнопок всегда должен быть один выбранный вариант (`checked`). Если пользователь ничего не выберет в обязательной группе, это может вызвать ошибку.

## 🔍 Глубокое погружение: Состояние `indeterminate`

Существует «третье состояние» чекбокса — неопределенное. Визуально оно выглядит не как галочка, а как горизонтальная черточка.
Его **нельзя** задать через HTML, оно устанавливается только через JavaScript. Это часто используется в списках с вложенностью (например, вы выбрали часть файлов в папке — у самой папки появится «черточка»).

---

## ❓ Часто задаваемые вопросы

### «Можно ли сделать так, чтобы радиокнопку можно было снять вторым кликом?»
Стандартными средствами HTML — нет. Если выбор сделан, его можно только переключить. Если вам нужно иметь возможность отменить выбор совсем, используйте либо чекбоксы, либо добавьте вариант «Не выбрано» в группу радиокнопок.

### «Что если я хочу отправить форму, когда пользователь кликнет на радиокнопку (без нажатия кнопки 'Отправить')?»
Для этого понадобится JavaScript. HTML сам по себе отправляет форму только по нажатию на кнопку `submit` или клавишу `Enter`.

### «Как передать несколько значений с одним именем в чекбоксах?»
В таких языках как PHP принято добавлять квадратные скобки в конце имени: `name="hobbies[]"`. Тогда сервер автоматически упакует все выбранные варианты в удобный массив.

> [!abstract] Запомни
> 1. **Чекбоксы** — для любого количества ответов.
> 2. **Радиокнопки** — для выбора строго одного ответа из группы.

> 4. Атрибут **`value`** обязателен, чтобы сервер понял, какой именно вариант был выбран.

> 6. **`<label>`** — лучший друг элементов выбора, расширяющий зону клика.
