## 📖 Главный элемент действия

Форма без кнопки — это как машина без ключа зажигания. Вы можете заполнить все поля, но без финального клика данные никуда не отправятся. Кнопка — это сигнал браузеру собрать всю введенную информацию и передать её по адресу, указанному в `action` формы.

В HTML существует два основных способа создать кнопку, и очень важно понимать, какой из них выбрать.

---

## 🏗️ Способ 1: Тег `<button>` (Рекомендуемый)

Это современный и самый гибкий способ. Тег `<button>` является **парным**, что позволяет вкладывать в него не только текст, но и другие HTML-элементы (иконки, картинки, спаны).

### Базовый синтаксис:

```html
<button type="submit">Отправить данные</button>
```

---

## 🏗️ Способ 2: Тег `<input>` (Устаревающий)

Исторически кнопки создавались как разновидность инпута. Этот тег **одиночный**, и текст на кнопке задается через атрибут `value`.

```html
<input type="submit" value="Отправить данные">
```

### Почему `<button>` лучше?
1. **Гибкость**: В `<button>` можно вставить иконку (`<button><img src="icon.png"> OK</button>`), а в `<input>` — нет.
2. **Псевдоэлементы**: К тегу `<button>` легче применять сложные CSS-эффекты (`::before`, `::after`).
3. **Семантика**: Намного логичнее, когда кнопка — это отдельный тег `button`, а не еще один `input`.

---

## ⚙️ Атрибут `type` — Три жизни кнопки

Это самый важный атрибут. Если вы его не укажете, большинство современных браузеров по умолчанию посчитают кнопку как `type="submit"`.

### 1. `type="submit"` (Кнопка отправки)
Основной тип внутри формы. При нажатии:
- Происходит валидация полей (проверка `required`, `email` и т.д.).
- Данные собираются и отправляются на сервер.
- Страница перезагружается (или происходит переход по адресу в `action`).

### 2. `type="reset"` (Кнопка сброса)
Очищает все поля формы, возвращая им значения по умолчанию.
> [!warning] Мнение экспертов
> Дизайнеры и UX-специалисты **не рекомендуют** использовать эту кнопку. Пользователь может нажать её случайно и потерять все введенные данные, что вызовет только гнев. Если и ставить её, то очень далеко от основной кнопки отправки.

### 3. `type="button"` (Обычная кнопка)
Сама по себе **ничего не делает**. Она не отправляет форму и не сбрасывает её.
- Зачем она нужна? Она используется как «крючок» для JavaScript. Например, для открытия модального окна, запуска анимации или выполнения расчетов прямо на странице без её перезагрузки.

---

## 🖼️ Кнопки с изображениями

Вы можете сделать кнопку-картинку двумя способами:

1. **Внутри `<button>`**:
   ```html
   <button type="submit">
     <img src="send-icon.png" alt="" width="20"> Отправить
   </button>
   ```

2. **Специальный тип `<input type="image">`**:
   ```html
   <input type="image" src="submit-btn.png" alt="Отправить">
   ```
   *Интересный факт: При нажатии на такую кнопку на сервер отправятся еще и координаты клика (x и y), что иногда используется для создания интерактивных карт.*

---

## 🚫 Атрибут `disabled`

Если вы хотите запретить пользователю нажимать кнопку (например, пока он не поставил галочку «Согласен с правилами»), добавьте атрибут `disabled`.

```html
<button type="submit" disabled>Зарегистрироваться</button>
```

Кнопка станет серой (в большинстве браузеров) и перестанет реагировать на клики. Чтобы включить её обратно, обычно используют JavaScript.

---

## 🔗 Атрибут `form` (Кнопка вне формы)

Иногда по дизайну кнопка отправки должна находиться далеко от самой формы (например, форма в боковой колонке, а кнопка в шапке сайта). В HTML5 для этого появился атрибут `form`.

```html
<form id="register-form" action="/save">
  <!-- Поля ввода -->
</form>

<!-- Эта кнопка отправит форму выше, хотя находится вне её тегов -->
<button type="submit" form="register-form">Сохранить</button>
```

---

## 🧠 Психология дизайна: Call to Action (CTA)

Кнопка — это призыв к действию. Вот несколько правил хорошего тона:

1.  **Глаголы вместо существительных**: "Зарегистрироваться" лучше, чем "Регистрация". "Получить подарок" лучше, чем "Подарок".
2.  **Размер и цвет**: Главная кнопка действия (например, "Купить") должна быть яркой и заметной. Второстепенные кнопки (например, "Отмена") должны быть менее контрастными.
3.  **Состояние при наведении (`:hover`)**: Пользователь должен видеть, что кнопка живая. Она должна менять цвет или слегка приподниматься при наведении курсора.
4.  **Визуальная иерархия**: На одном экране не должно быть две одинаково яркие кнопки, иначе пользователь запутается, какая из них важнее.

## 🧪 Практика: Группа действий

Давайте создадим блок управления формой с разными типами кнопок.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Работа с кнопками</title>
    <style>
        body { font-family: sans-serif; padding: 50px; background-color: #f0f2f5; }
        .form-card { max-width: 400px; background: white; padding: 30px; border-radius: 12px; box-shadow: 0 4px 10px rgba(0,0,0,0.1); }
        input[type="text"] { width: 100%; padding: 10px; margin-bottom: 20px; border: 1px solid #ddd; border-radius: 6px; box-sizing: border-box; }

        .btn-group { display: flex; gap: 10px; flex-wrap: wrap; }

        /* Стилизация кнопок */
        button {
            padding: 10px 20px;
            border: none;
            border-radius: 6px;
            font-weight: bold;
            cursor: pointer;
            transition: 0.2s;
        }

        .btn-primary { background-color: #007bff; color: white; }
        .btn-primary:hover { background-color: #0056b3; }

        .btn-secondary { background-color: #6c757d; color: white; }
        .btn-secondary:hover { background-color: #5a6268; }

        .btn-danger { background-color: #dc3545; color: white; }
        .btn-danger:hover { background-color: #c82333; }

        button:disabled { background-color: #ccc; cursor: not-allowed; opacity: 0.6; }
    </style>
</head>
<body>

    <div class="form-card">
        <h3>Ваш профиль</h3>
        <form id="profile-form">
            <label>Имя:</label>
            <input type="text" value="Иван Иванов">

            <label>Никнейм:</label>
            <input type="text" value="ivan_pro">

            <div class="btn-group">
                <!-- Главная кнопка -->
                <button type="submit" class="btn-primary">Сохранить</button>

                <!-- Второстепенная (просто кнопка для JS) -->
                <button type="button" class="btn-secondary" onclick="alert('Данные не изменены')">Отмена</button>

                <!-- Опасная кнопка сброса -->
                <button type="reset" class="btn-danger">Очистить всё</button>

                <!-- Заблокированная кнопка -->
                <button type="button" disabled>Удалить аккаунт</button>
            </div>
        </form>
    </div>

</body>
</html>
```

---

## ⚠️ Типичные ошибки

1.  **Отсутствие `type`**: Как мы уже говорили, кнопка может случайно сработать как `submit`, когда вы этого не ждете (например, при нажатии на кнопку открытия галереи форма внезапно отправится).
2.  **Ссылка в виде кнопки**: Если нажатие должно просто перевести пользователя на другую страницу — используйте тег `<a>`, стилизованный под кнопку, а не `<button>`. Кнопка — для изменения данных на сервере, ссылка — для навигации.
3.  **Забытый `alt` для кнопок-картинок**: Скринридер должен понимать, что делает эта картинка.
4.  **Слишком много кнопок**: Не заставляйте пользователя думать, на что нажать. Одно главное действие — одна яркая кнопка.

---

## 🔍 Глубокое погружение: Формы с несколькими кнопками Submit

Иногда в одной форме нужно две кнопки отправки с разными действиями. Например: "Опубликовать статью" и "Сохранить в черновик".

Чтобы сервер понял, какая кнопка была нажата, им дают атрибут `name` и разные `value`.
```html
<button type="submit" name="action" value="publish">Опубликовать</button>
<button type="submit" name="action" value="draft">В черновик</button>
```
Сервер получит либо `action=publish`, либо `action=draft`.

## ❓ Часто задаваемые вопросы

### «Можно ли сделать так, чтобы кнопка `reset` не очищала одно конкретное поле?»
Стандартными средствами — нет. `reset` сбрасывает **всю** форму к её первоначальному состоянию (к тому, что было написано в атрибутах `value`).

### «Почему моя кнопка не отправляет форму?»
Проверьте:
1. Находится ли она **внутри** тега `<form>`.
2. Указан ли у неё `type="submit"`.
3. Нет ли ошибок в JavaScript, которые могут блокировать отправку.

### «Как сделать кнопку со скругленными углами?»
Это делается через CSS свойство `border-radius: 10px;`. Кнопки отлично поддаются стилизации.

### «Что делать, если я хочу кнопку с градиентом?»
Используйте CSS свойство `background: linear-gradient(...)`. Современный веб позволяет делать кнопки любого вида без использования картинок.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **`<button>`** — лучший выбор для создания кнопок благодаря своей гибкости.
> 2. **`type="submit"`** — отправляет данные на сервер.
> 3. **`type="button"`** — ничего не делает без JavaScript, безопасно для интерфейсных кнопок.
> 4. **`type="reset"`** — опасен, используйте с осторожностью.
> 5. **`disabled`** — выключает кнопку.
> 6. Текст на кнопке должен быть понятным и призывать к действию (используйте глаголы).
> 7. Разделяйте кнопки по визуальной значимости (основная и второстепенные).
