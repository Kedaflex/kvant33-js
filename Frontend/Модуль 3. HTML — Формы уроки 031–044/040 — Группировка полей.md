## 📖 Порядок в хаосе

Представьте форму заказа, в которой 30 полей: имя, адрес, индекс, данные карты, выбор доставки, комментарий... Если все эти поля идут просто сплошным списком, пользователь быстро устанет и может бросить заполнение.

В хорошем дизайне данные всегда разбиты на «порции». Например: «Личные данные», «Адрес доставки», «Оплата». В HTML для этого есть два специальных тега, которые работают в паре.

---

## 🏗️ Теги `<fieldset>` и `<legend>`

1.  **`<fieldset>`** (от англ. *field set* — набор полей) — это контейнер, который логически объединяет несколько элементов управления формой.
2.  **`<legend>`** — это заголовок (подпись) для этого набора полей.

### Базовый синтаксис:

```html
<form>
  <fieldset>
    <legend>Контактная информация</legend>

    <label for="name">Имя:</label>
    <input type="text" id="name" name="username">

    <label for="email">Email:</label>
    <input type="email" id="email" name="email">
  </fieldset>
</form>
```

---

## ⚙️ Зачем это нужно? (Больше, чем просто рамка)

На первый взгляд может показаться, что `<fieldset>` просто рисует серую рамку вокруг полей. Но его роль гораздо важнее:

### 1. Доступность (Accessibility) — Главная причина
Когда незрячий пользователь переходит к полю внутри группы, скринридер сначала прочитает текст из `<legend>`, а потом уже название самого поля.
*Например: «Группа: Способ оплаты. Выбрано: Банковская карта». Без легенды пользователь может не понять, к чему относятся данные радиокнопки.*

### 2. Массовое управление через `disabled`
Если вы добавите атрибут `disabled` к тегу `<fieldset>`, то **все** поля внутри него мгновенно станут неактивными. Это очень удобно, если вы хотите отключить целый блок формы (например, блок «Адрес доставки», если выбран «Самовывоз»).

```html
<fieldset disabled>
  <legend>Этот блок сейчас недоступен</legend>
  <input type="text">
  <select><option>...</option></select>
  <button type="submit">Кнопка тоже не нажмется</button>
</fieldset>
```

### 3. Семантика и SEO
Поисковые роботы лучше понимают структуру вашей страницы и то, какие данные связаны между собой.

## 🔘 Группировка Радиокнопок и Чекбоксов

Именно для элементов выбора `<fieldset>` используется чаще всего. Группа радиокнопок — это классический пример «набора полей».

```html
<fieldset>
  <legend>Выберите размер пиццы:</legend>

  <label><input type="radio" name="size" value="s"> Маленькая</label>
  <label><input type="radio" name="size" value="m"> Средняя</label>
  <label><input type="radio" name="size" value="l"> Большая</label>
</fieldset>
```

---

## 🎨 Стилизация `<fieldset>` и `<legend>`

По умолчанию `<fieldset>` выглядит как прямоугольник с тонкой рамкой, а `<legend>` «врезается» в эту рамку сверху. Это выглядит довольно старомодно, поэтому дизайнеры часто меняют стили.

### Популярные приемы:
- Убрать стандартную рамку (`border: none`).
- Сделать `<legend>` жирным и крупным.
- Добавить отступы (`padding`) внутри группы.
- Использовать фон для выделения блока.

```css
fieldset {
    border: 2px solid #3498db;
    border-radius: 10px;
    padding: 20px;
    margin-bottom: 30px;
}

legend {
    background-color: #3498db;
    color: white;
    padding: 5px 15px;
    border-radius: 20px;
    font-weight: bold;
    font-size: 0.9em;
}
```

## 🧪 Практика: Профессиональная многослойная форма

Давайте создадим форму регистрации волонтера, разделенную на логические этапы.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Группировка полей</title>
    <style>
        body { font-family: 'Segoe UI', sans-serif; padding: 40px; background-color: #f0f4f8; }
        form { max-width: 500px; margin: 0 auto; background: white; padding: 10px; border-radius: 15px; }

        fieldset {
            border: 1px solid #d1d5db;
            border-radius: 8px;
            padding: 20px;
            margin: 15px;
        }

        legend {
            font-weight: bold;
            color: #1f2937;
            padding: 0 10px;
            font-size: 1.1em;
        }

        .field { margin-bottom: 15px; }
        label { display: block; margin-bottom: 5px; font-size: 14px; }
        input[type="text"], input[type="email"], select {
            width: 100%;
            padding: 8px;
            border: 1px solid #ccc;
            border-radius: 4px;
            box-sizing: border-box;
        }

        .radio-group label {
            display: inline-block;
            margin-right: 15px;
            font-weight: normal;
        }

        button {
            display: block;
            width: calc(100% - 30px);
            margin: 20px auto;
            padding: 12px;
            background-color: #10b981;
            color: white;
            border: none;
            border-radius: 6px;
            font-weight: bold;
            cursor: pointer;
        }
    </style>
</head>
<body>

    <form>
        <h1>Анкета волонтера</h1>

        <!-- Блок 1 -->
        <fieldset>
            <legend>Личные данные</legend>
            <div class="field">
                <label for="fio">Полное имя:</label>
                <input type="text" id="fio" name="fullname" required>
            </div>
            <div class="field">
                <label for="email">E-mail:</label>
                <input type="email" id="email" name="email" required>
            </div>
        </fieldset>

        <!-- Блок 2 -->
        <fieldset>
            <legend>Опыт и навыки</legend>
            <div class="field">
                <label for="special">Специализация:</label>
                <select name="specialty" id="special">
                    <option value="eco">Экология</option>
                    <option value="animal">Помощь животным</option>
                    <option value="social">Социальная работа</option>
                </select>
            </div>
            <div class="field radio-group">
                <p style="font-size: 14px; margin-bottom: 5px;">Есть ли опыт волонтерства?</p>
                <label><input type="radio" name="exp" value="yes"> Да</label>
                <label><input type="radio" name="exp" value="no" checked> Нет</label>
            </div>
        </fieldset>

        <!-- Блок 3 (Допустим, он отключен до выбора специализации) -->
        <fieldset disabled>
            <legend>Дополнительно (только для профи)</legend>
            <div class="field">
                <label for="cert">Номер сертификата:</label>
                <input type="text" id="cert" name="certificate_id">
            </div>
        </fieldset>

        <button type="submit">Отправить анкету</button>
    </form>

</body>
</html>
```

## ⚠️ Типичные ошибки

1.  **Отсутствие `<legend>`**: `<fieldset>` без легенды теряет свой смысл для доступности. Если вам не нужен заголовок визуально, скройте его через CSS, но оставьте в коде.
2.  **Много вложенных групп**: Не стоит вкладывать один `<fieldset>` в другой слишком глубоко. Это запутывает и визуально, и программно.
3.  **Использование для дизайна**: Не используйте `<fieldset>` просто чтобы нарисовать рамку вокруг чего-то, что не является частью формы. Это семантический тег для элементов управления.
4.  **Забытый `form`**: Если форма очень сложная, помните, что все филдсеты должны быть внутри одного тега `<form>`.

---

## ❓ Часто задаваемые вопросы

### «Можно ли поставить две `<legend>` в один `<fieldset>`?»
Нет. По стандарту в одном филдсете может быть только одна легенда. Если вам нужно несколько подзаголовков, используйте внутри обычные теги `<h3>` или `<h4>`.

### «Как выровнять `<legend>` по центру или по правому краю?»
Через CSS свойство `text-align`. Хотя по умолчанию браузеры рисуют её слева, вы можете это изменить.
```css
legend { text-align: center; }
```

### «Будет ли работать валидация в `disabled` филдсете?»
Нет. Если группа полей отключена, браузер игнорирует все правила валидации (`required` и т.д.) внутри неё и не отправляет данные из этих полей на сервер.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **`<fieldset>`** — контейнер для логической группы полей.
> 2. **`<legend>`** — обязательный заголовок для этой группы.
> 3. Группировка критически важна для пользователей со скринридерами.
> 4. Атрибут **`disabled`** на филдсете выключает сразу все поля внутри него.
> 5. Визуальное разделение формы на блоки снижает когнитивную нагрузку на пользователя.
> 6. Филдсеты — идеальный способ оформления групп радиокнопок и чекбоксов.
