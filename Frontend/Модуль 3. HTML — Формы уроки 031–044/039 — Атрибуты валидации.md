## 📖 Что такое валидация и зачем она нужна?

Представьте, что вы создали форму регистрации, но пользователь в поле «Email» ввел «привет», а в поле «Возраст» — «минус десять». Если эти данные попадут в вашу базу данных, они её просто «замусорят» или вызовут сбои в работе программы.

**Валидация** — это процесс проверки данных на соответствие правилам перед их отправкой.
1. **Клиентская валидация** (в браузере): Происходит мгновенно, дает пользователю подсказки и экономит ресурсы сервера.
2. **Серверная валидация**: Происходит на стороне сервера. Она **обязательна**, так как браузерную валидацию можно легко обойти.

Сегодня мы сфокусируемся на том, как заставить браузер проверять данные за нас без написания сложного кода.

## 🛠️ Базовые атрибуты проверки

Эти атрибуты являются фундаментом надежной формы.

### 1. `required` (Обязательность)
Самый простой и важный атрибут. Он запрещает отправку формы, если поле пустое.
```html
<input type="text" name="username" required>
```
*Примечание: Для радиокнопок достаточно добавить `required` одной кнопке из группы, и браузер потребует выбрать хотя бы один вариант.*

### 2. `minlength` и `maxlength` (Длина строки)
Ограничивают количество символов, которые может ввести пользователь.
- `maxlength="10"`: Просто не даст напечатать больше 10 символов.
- `minlength="3"`: Позволит напечатать меньше, но при нажатии «Отправить» выдаст ошибку.

### 3. `min`, `max` и `step` (Числовые границы)
Работают для типов `number`, `range`, `date`, `time`.
```html
<input type="number" min="0" max="100" step="5">
```

## 🧩 Атрибут `pattern` — Регулярные выражения

Это «тяжелая артиллерия» валидации. Он позволяет задать точный формат данных с помощью регулярных выражений (RegEx).

### Как это работает:
Вы описываете шаблон, и если ввод ему не соответствует, браузер выдает ошибку.

**Популярные примеры шаблонов:**

| Задача | Код шаблона | Пояснение |
|---|---|---|
| Только латиница | `[A-Za-z]+` | Один или более символов A-Z |
| Номер телефона (формат 79...) | `7[0-9]{10}` | Цифра 7 и еще ровно 10 цифр |
| Почтовый индекс (6 цифр) | `[0-9]{6}` | Строго 6 цифр |
| Код товара (2 буквы, тире, 4 цифры) | `[A-Z]{2}-[0-9]{4}` | AB-1234 |

### Пример в коде:
```html
<label>Введите логин (только латиница, 5-12 символов):</label>
<input type="text" pattern="[A-Za-z]{5,12}" title="Используйте от 5 до 12 латинских букв">
```

> [!tip] Атрибут `title`
> Если вы используете `pattern`, обязательно добавляйте атрибут `title`. Большинство браузеров покажут текст из `title` в качестве подсказки, если пользователь ввел данные неверно.

## 🎨 Визуальная валидация через CSS

Браузер знает состояние каждого поля. Мы можем использовать специальные псевдоклассы в CSS, чтобы менять оформление полей «на лету».

### Основные псевдоклассы:
- **`:required`**: Стили для обязательных полей.
- **`:optional`**: Стили для необязательных полей.
- **`:valid`**: Стили для поля, которое заполнено правильно.
- **`:invalid`**: Стили для поля с ошибкой.

### Пример оформления:
```css
/* Подсветим неверно заполненные поля красной рамкой */
input:invalid {
    border: 2px solid #ff4d4d;
    background-color: #fff5f5;
}

/* Подсветим верно заполненные поля зеленой рамкой */
input:valid {
    border: 2px solid #2ecc71;
}

/* Добавим звездочку после обязательных полей */
label::after {
    content: " *";
    color: red;
}
```

## 🚫 Отключение валидации: `novalidate`

Иногда вам нужно отправить данные на сервер, даже если они неверны (например, чтобы сервер сам сохранил «черновик» и показал более сложные ошибки). Для этого у тега `<form>` есть атрибут **`novalidate`**.

```html
<form action="/save" novalidate>
  <!-- Браузер не будет проверять required и pattern -->
</form>
```

## 🧠 Психология валидации: Не бесите пользователя

Плохая валидация может убить конверсию вашего сайта. Вот несколько советов:

1.  **Не будьте агрессивными**: Не показывайте красные рамки сразу при загрузке пустой формы. Дайте пользователю шанс сначала что-то ввести.
2.  **Конкретика**: Сообщение «Данные неверны» бесполезно. Пишите: «Пароль должен содержать хотя бы одну цифру».
3.  **Помощь**: Используйте плейсхолдеры и подсказки (`help-text`) под полями, чтобы пользователь знал правила **до** того, как совершит ошибку.
4.  **Сохранение данных**: Если форма не прошла валидацию и страница перезагрузилась, убедитесь, что уже введенные данные не стерлись (это делается на стороне сервера или JS).

## 🧪 Практика: "Неприступная" форма

Давайте создадим форму с многоуровневой проверкой данных.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Валидация форм</title>
    <style>
        body { font-family: 'Inter', sans-serif; padding: 40px; background-color: #f3f4f6; }
        .form-box { max-width: 450px; margin: 0 auto; background: white; padding: 30px; border-radius: 12px; }

        .field { margin-bottom: 20px; }
        label { display: block; margin-bottom: 8px; font-weight: 600; }

        input {
            width: 100%;
            padding: 12px;
            border: 1px solid #d1d5db;
            border-radius: 6px;
            box-sizing: border-box;
            outline: none;
        }

        /* Улучшенная визуальная обратная связь */
        input:focus { border-color: #3b82f6; box-shadow: 0 0 0 3px rgba(59, 130, 246, 0.1); }

        /* Стили валидации сработают только после того, как пользователь начал ввод (псевдокласс :placeholder-shown) */
        input:not(:placeholder-shown):invalid {
            border-color: #ef4444;
            background-color: #fef2f2;
        }

        input:not(:placeholder-shown):valid {
            border-color: #10b981;
            background-color: #f0fdf4;
        }

        .error-msg {
            display: none;
            color: #ef4444;
            font-size: 12px;
            margin-top: 5px;
        }

        input:invalid + .error-msg {
            display: block; /* Показываем ошибку, если инпут невалиден */
        }

        button {
            width: 100%;
            padding: 12px;
            background: #1f2937;
            color: white;
            border: none;
            border-radius: 6px;
            font-weight: bold;
            cursor: pointer;
        }
    </style>
</head>
<body>

    <div class="form-box">
        <h2>Регистрация мастера</h2>
        <form action="/success" method="POST">

            <div class="field">
                <label for="nickname">Никнейм (3-10 букв):</label>
                <input
                    type="text"
                    id="nickname"
                    name="nick"
                    placeholder=" "
                    required
                    pattern="[A-Za-zА-Яа-я0-9]{3,10}"
                    title="От 3 до 10 букв или цифр"
                >
                <span class="error-msg">Никнейм должен быть от 3 до 10 символов.</span>
            </div>

            <div class="field">
                <label for="email">E-mail:</label>
                <input
                    type="email"
                    id="email"
                    name="email"
                    placeholder=" "
                    required
                >
                <span class="error-msg">Введите корректный адрес почты.</span>
            </div>

            <div class="field">
                <label for="code">Промокод (Формат: XXX-000):</label>
                <input
                    type="text"
                    id="code"
                    name="promo"
                    placeholder=" "
                    pattern="[A-Z]{3}-[0-9]{3}"
                    title="Три заглавные буквы, тире и три цифры"
                >
                <span class="error-msg">Формат промокода: ABC-123.</span>
            </div>

            <div class="field">
                <label for="exp">Стаж работы (лет):</label>
                <input
                    type="number"
                    id="exp"
                    name="years"
                    min="1"
                    max="50"
                    required
                    placeholder=" "
                >
                <span class="error-msg">Стаж должен быть от 1 до 50 лет.</span>
            </div>

            <button type="submit">Создать аккаунт</button>
        </form>
    </div>

</body>
</html>
```

## 🔍 Глубокое погружение: Constraint Validation API

За всеми этими атрибутами стоит мощный программный интерфейс (API). С помощью JavaScript вы можете:
- Проверить состояние поля: `input.validity.valid`.
- Узнать конкретную ошибку: `input.validity.valueMissing` (пустое) или `input.validity.patternMismatch` (не совпал шаблон).
- Установить свою ошибку: `input.setCustomValidity("Этот логин уже занят")`.
Это позволяет делать валидацию еще более гибкой и умной.

---

## ⚠️ Типичные ошибки

1.  **Доверие только клиентской валидации**: Запомните: **любую** браузерную проверку можно обойти (отключив JS или подменив запрос). Всегда дублируйте проверки на сервере.
2.  **Слишком сложные Regex**: Если ваш `pattern` настолько сложен, что вы сами его не понимаете — скорее всего, пользователи не смогут его заполнить.
3.  **Отсутствие подсказок**: Пользователь не должен гадать, почему его не пускает дальше.
4.  **Валидация того, что не нужно**: Не делайте `required` полями то, что необязательно (например, «Отчество» или «Дополнительный телефон»).

## ❓ Часто задаваемые вопросы

### «Можно ли изменить текст стандартного облачка с ошибкой?»
Напрямую через атрибуты — нет. Текст ошибки («Заполните это поле») генерируется браузером на языке системы пользователя. Чтобы изменить этот текст, нужно использовать JavaScript метод `setCustomValidity`.

### «Как проверить, что два пароля (основной и подтверждение) совпадают?»
В чистом HTML это сделать невозможно. Для сравнения значений двух разных полей всегда требуется JavaScript.

### «Почему `:invalid` срабатывает на пустых полях с `required`?»
Потому что пустое поле с `required` — это и есть «невалидное» состояние. Чтобы пустые поля не горели красным сразу, используйте трюк с `:not(:placeholder-shown)` из практической части этого урока.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **Валидация** экономит время и нервы пользователю и серверу.
> 2. **`required`**, **`minlength`**, **`maxlength`** — база для любого текстового поля.
> 3. **`pattern`** позволяет проверять данные по сложным шаблонам.
> 4. Всегда используйте **`title`** как текст-подсказку для ошибок в паттернах.
> 5. CSS псевдоклассы **`:valid`** и **`:invalid`** делают форму «живой».
> 6. Клиентская валидация — для удобства, серверная — для безопасности.
