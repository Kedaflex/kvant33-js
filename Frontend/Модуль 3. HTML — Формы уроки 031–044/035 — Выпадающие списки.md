## 🎯 Цели урока

- Изучить структуру выпадающих списков с помощью тегов `<select>` и `<option>`

- Изучить атрибуты `value`, `selected`, `disabled`
- Узнать о преимуществах и недостатках стандартных списков для мобильных устройств

## 📖 Когда вариантов слишком много

Представьте, что вам нужно предложить пользователю выбрать его страну. Стран в мире около 200. Если вы используете радиокнопки, страница превратится в бесконечный список.

Для таких случаев в HTML предусмотрен компактный элемент — **выпадающий список** (dropdown). Он занимает место одной строки, а полный список вариантов открывается только при клике.

---

## 🏗️ Базовая структура: `<select>` и `<option>`

Выпадающий список строится из двух тегов:
1.  **`<select>`** — контейнер для списка (аналог `<ul>` или `<form>`).
2.  **`<option>`** — отдельный пункт списка (аналог `<li>`).

### Базовый синтаксис:

```html
<label for="car-select">Выберите марку авто:</label>

<select name="cars" id="car-select">
  <option value="toyota">Тойота</option>
  <option value="bmw">БМВ</option>
  <option value="mercedes">Мерседес</option>
  <option value="audi">Ауди</option>
</select>
```

### Как это работает:
- Пользователь видит только один вариант (первый по списку или выбранный).
- При клике выпадает меню.
- На сервер уходит значение атрибута `value` выбранного тега `<option>`. Если `value` нет, отправится текст, который написан внутри тега.

---

## 📂 Группировка вариантов: `<optgroup>`

Если список длинный, его стоит разделить на логические части. Для этого используется тег **`<optgroup>`** (Option Group).

```html
<select name="city">
  <optgroup label="Россия">
    <option value="msk">Москва</option>
    <option value="spb">Санкт-Петербург</option>
  </optgroup>
  <optgroup label="Казахстан">
    <option value="ast">Астана</option>
    <option value="alm">Алматы</option>
  </optgroup>
</select>
```

**Особенности:**
- Атрибут `label` у `<optgroup>` обязателен — это название заголовка группы.
- Заголовок группы **нельзя выбрать**, он служит только для визуального разделения.

## ⚙️ Атрибуты элементов списка

### 1. `selected` — Выбор по умолчанию
По умолчанию выбран первый пункт списка. Если вы хотите выбрать другой вариант сразу при загрузке:
```html
<option value="urgent" selected>Срочно</option>
```

### 2. `disabled` — Запрет выбора
Можно отключить как отдельный пункт, так и всю группу сразу.
```html
<option value="old-model" disabled>Старая модель (снята с производства)</option>
```

### 3. `multiple` — Множественный выбор
Позволяет пользователю выбрать несколько вариантов (обычно через зажатый `Ctrl` на ПК).
```html
<select name="skills" multiple size="5">
  <option value="html">HTML</option>
  <option value="css">CSS</option>
  <!-- ... -->
</select>
```
*Примечание: Атрибут `size` определяет, сколько строк списка будет видно сразу без прокрутки.*

## 🧠 Психология выбора: Select vs Radio

Выбор между этими двумя элементами — классическая задача UX-дизайна.

| Используйте Радиокнопки, если... | Используйте Выпадающий список, если... |
|---|---|
| Вариантов мало (2–4) | Вариантов много (5 и более) |
| Важно видеть все варианты сразу для сравнения | Варианты хорошо известны (например, год рождения или месяц) |
| Есть место на экране | Место ограничено (мобильные интерфейсы) |
| Выбор критически важен (например, "Я согласен") | Выбор второстепенен |

> [!tip] Хорошая практика
> Если вы хотите заставить пользователя сделать осознанный выбор, первым пунктом сделайте "пустой" вариант с подсказкой:
> `<option value="" disabled selected>-- Выберите вариант --</option>`

## 📱 Особенности мобильных устройств

На смартфонах (iOS, Android) стандартный `<select>` выглядит не так, как в браузере на ПК. Операционная система обычно открывает свое собственное колесо прокрутки или полноэкранный список.
Это **очень удобно** для пользователей, поэтому не спешите заменять стандартные селекты на «красивые» самодельные библиотеки на JavaScript — они часто работают хуже на телефонах.

---

## 🛠️ Стилизация списков

Стандартный `<select>` довольно плохо поддается стилизации через CSS. Вы можете легко изменить:
- Цвет текста (`color`)
- Фон (`background-color`)
- Рамки (`border`)
- Шрифты (`font-family`)

Но вы **не можете** (стандартными средствами) изменить вид выпадающей части (самого меню со списком) или стрелочку справа. Для серьезного изменения дизайна используются либо сложные CSS-трюки, либо сторонние библиотеки.

## ⚠️ Типичные ошибки

1. **Забытый `value` у `<option>`**: Если вы напишете `<option>Москва</option>`, сервер получит "Москва". Если вы потом переведете сайт на английский (`<option>Moscow</option>`), ваш серверный код может сломаться. Всегда используйте неизменные `value` (например, `value="city_msk"`).
2. **Отсутствие `<label>`**: Ошибка та же, что и везде. Обязательно связывайте текст подписи с селектом.
3. **Слишком длинные группы**: Если в `<optgroup>` более 15 элементов, его становится трудно просматривать.
4. **Неудобство `multiple`**: Обычные пользователи редко знают, что для выбора нескольких пунктов нужно зажимать клавишу `Ctrl`. Если вам нужен множественный выбор, лучше используйте группу чекбоксов.
5. **Пустой селект**: Не оставляйте селект без выбранного варианта.

## 🔍 Глубокое погружение: Атрибут `size`

Атрибут `size` меняет поведение селекта: из выпадающего списка он превращается в список с прокруткой, где сразу видно несколько строк.
- `size="1"` (по умолчанию): Выпадающий список.
- `size="3"`: Видно 3 строки, остальное прокручивается.
Это полезно, если вы хотите показать пользователю, что вариантов много, не скрывая их полностью.

## ❓ Часто задаваемые вопросы

### «Как сделать поиск внутри выпадающего списка?»
Стандартный HTML-тег `<select>` не поддерживает поиск. Для этого используется либо тег `<datalist>` (мы изучим его позже), либо JavaScript-библиотеки (например, Select2 или Choices.js).

### «Можно ли добавить картинки (иконки флагов) внутрь `<option>`?»
Нет. Стандарт HTML запрещает вкладывать любые теги внутрь `<option>`. Там может быть только чистый текст. Чтобы сделать список с картинками, разработчики имитируют работу списка с помощью `<div>` и JavaScript.

### «Что отправится на сервер, если выбрано несколько пунктов?»
Если вы используете `multiple`, браузер отправит несколько пар с одинаковым именем: `skills=html&skills=css`. Как и в случае с чекбоксами, на стороне сервера (например, в PHP) стоит использовать имя `name="skills[]"`, чтобы получить массив.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **`<select>` + `<option>`** — компактный способ выбора из множества вариантов.
> 2. **`<optgroup>`** помогает структурировать длинные списки заголовками.
> 3. Атрибут **`value`** у опций — это то, что реально получит сервер.
> 4. Атрибут **`multiple`** позволяет выбирать несколько пунктов, но часто неудобен для простых пользователей.
> 5. На мобильных устройствах стандартные списки работают лучше всего.
> 6. Если вариантов меньше 5 — подумайте об использовании радиокнопок.
