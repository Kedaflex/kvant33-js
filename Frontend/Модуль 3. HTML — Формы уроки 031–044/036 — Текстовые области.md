## 📖 Когда одного инпута мало

До этого момента мы работали с однострочными полями ввода. Но что, если нам нужно, чтобы пользователь оставил подробный отзыв, написал статью или ввел текст сообщения? В стандартный инпут такая информация просто не поместится — он будет бесконечно прокручиваться вправо, что крайне неудобно для чтения и редактирования.

Для многострочного ввода в HTML используется специальный тег — **`<textarea>`**.

---

## 🏗️ Базовая структура тега `<textarea>`

В отличие от инпута, `<textarea>` — это **парный тег**. Это означает, что он имеет открывающую и закрывающую части.

### Базовый синтаксис:

```html
<label for="feedback">Ваш отзыв:</label>
<textarea name="user_feedback" id="feedback" placeholder="Напишите всё, что вы думаете о нашей пицце..."></textarea>
```

### Важнейшая особенность контента:
В текстовых инпутах мы использовали атрибут `value="текст"`. В `<textarea>` значение по умолчанию пишется **между** открывающим и закрывающим тегом.

```html
<!-- Правильно -->
<textarea name="comment">Это текст по умолчанию</textarea>

<!-- НЕПРАВИЛЬНО (работать не будет) -->
<textarea name="comment" value="Это текст"></textarea>
```

## ⚙️ Атрибуты управления размером

Хотя сегодня размеры элементов чаще задают через CSS, у `<textarea>` есть исторические атрибуты, которые всё еще полезны для задания базовой структуры.

### 1. `rows` (Строки)
Определяет видимую высоту текстовой области в количестве строк текста. Если вы укажете `rows="5"`, поле будет по высоте ровно в 5 строк шрифта по умолчанию.
```html
<textarea rows="5"></textarea>
```

### 2. `cols` (Колонки)
Определяет видимую ширину области в среднем количестве символов.
*Примечание: Этот атрибут работает не очень точно с пропорциональными шрифтами, поэтому CSS `width` предпочтительнее.*
```html
<textarea cols="40"></textarea>
```

> [!tip] Хорошая практика
> Используйте `rows` и `cols` для того, чтобы поле выглядело адекватно в случае, если CSS по какой-то причине не загрузится, но для реального дизайна используйте CSS `width: 100%` и `min-height`.

## 📐 Управление изменением размера (Resize)

Вы наверняка замечали маленький уголок в нижней правой части текстовых полей, за который их можно растягивать. Это поведение контролируется браузером, но мы можем им управлять через CSS.

Свойство **`resize`**:
- `none` — полностью запрещает менять размер.
- `both` — (по умолчанию) можно менять и высоту, и ширину.
- `vertical` — можно менять только высоту (часто самый лучший вариант для дизайна).
- `horizontal` — можно менять только ширину.

```css
textarea {
    resize: vertical; /* Пользователь сможет растянуть поле только вниз */
    min-height: 100px;
    max-height: 400px;
}
```

## 🔄 Атрибут `wrap` — Особенности переноса

Атрибут `wrap` определяет, как текст будет отправляться на сервер при достижении края поля.

1.  **`soft`** (по умолчанию): Текст переносится на экране для удобства пользователя, но на сервер отправляется одной длинной строкой без принудительных переносов.
2.  **`hard`**: Текст отправляется на сервер с реальными переносами строк в тех местах, где он переносился в поле.
    *Важно: При использовании `hard` необходимо обязательно указывать атрибут `cols`.*

## ⚠️ Ловушка с пробелами

Поскольку `<textarea>` — парный тег, браузер воспринимает **всё**, что находится между тегами, буквально. Включая пробелы и пустые строки.

```html
<!-- Ошибка: в поле появятся лишние пробелы и перенос строки -->
<textarea>
  Привет!
</textarea>

<!-- Правильно: текст начнется сразу -->
<textarea>Привет!</textarea>
```

Если вы сделаете перенос строки после открывающего тега и добавите отступы для красоты кода, всё это окажется внутри поля ввода пользователя. Это частая причина ошибок, когда пользователи жалуются, что в их комментариях в начале стоят странные пробелы.

## ♿ Доступность и UX для длинных текстов

1.  **Счетчик символов**: Если у вас есть ограничение `maxlength`, хорошим тоном считается показывать пользователю, сколько символов осталось (это делается через JavaScript).
2.  **Скролл**: Если текста очень много, браузер добавит полосу прокрутки. Убедитесь, что она контрастная и удобная.
3.  **Понятный Label**: Для текстовой области подпись должна четко говорить, чего мы ждем: "Краткая биография", "Текст сообщения" и т.д.

---

## 🔍 Глубокое погружение: Placeholder в Textarea

Использование `placeholder` в `<textarea>` имеет свои тонкости. Поскольку поле большое, вы можете использовать плейсхолдер для показа сложного примера формата. Например, если это ввод адреса:
```html
<textarea placeholder="Город, улица, дом&#10;Квартира/офис&#10;Почтовый индекс"></textarea>
```
*Интересный факт: Спецсимвол `&#10;` позволяет сделать перенос строки внутри самого плейсхолдера.*

## 🧪 Практика: Форма обратной связи

Давайте создадим профессионально оформленную текстовую область с ограничениями.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Текстовые области</title>
    <style>
        body { font-family: 'Inter', sans-serif; padding: 40px; background-color: #f3f4f6; }
        .container { max-width: 600px; background: white; padding: 30px; border-radius: 12px; }

        .field { margin-bottom: 20px; }

        label {
            display: block;
            margin-bottom: 10px;
            font-weight: 600;
            color: #1f2937;
        }

        textarea {
            width: 100%;
            padding: 15px;
            border: 1px solid #d1d5db;
            border-radius: 8px;
            font-size: 16px;
            font-family: inherit; /* Чтобы шрифт внутри не менялся на системный */
            color: #374151;
            background-color: #fff;

            /* Важные настройки */
            resize: vertical;
            min-height: 150px;
            box-sizing: border-box;
            transition: border-color 0.2s, ring 0.2s;
        }

        textarea:focus {
            outline: none;
            border-color: #3b82f6;
            box-shadow: 0 0 0 3px rgba(59, 130, 246, 0.1);
        }

        textarea::placeholder {
            color: #9ca3af;
        }

        .help-text {
            font-size: 13px;
            color: #6b7280;
            margin-top: 8px;
        }
    </style>
</head>
<body>

    <div class="container">
        <h2>Свяжитесь с нами</h2>
        <form>
            <div class="field">
                <label for="message">Ваше сообщение:</label>
                <textarea
                    name="message"
                    id="message"
                    placeholder="Опишите вашу проблему или предложение максимально подробно..."
                    maxlength="1000"
                    required
                ></textarea>
                <div class="help-text">Максимум 1000 символов.</div>
            </div>

            <button type="submit" style="padding: 12px 24px; background: #2563eb; color: white; border: none; border-radius: 6px; cursor: pointer; font-weight: 600;">
                Отправить
            </button>
        </form>
    </div>

</body>
</html>
```

## 🛡️ Безопасность и фильтрация

Когда данные из `<textarea>` попадают на сервер, они могут содержать опасный код (например, теги `<script>`). Это называется **XSS-атакой**.
Никогда не выводите текст из текстовых областей на страницу «как есть». Всегда используйте фильтрацию на сервере, которая превратит `<` в `&lt;`. На стороне HTML вы просто должны знать, что пользователь может ввести туда абсолютно любой текст.

## ❓ Часто задаваемые вопросы

### «Как запретить пользователю менять размер поля совсем?»
Просто добавьте в CSS для тега `textarea` свойство `resize: none;`. Это часто делается, чтобы не ломать жесткую сетку дизайна сайта.

### «Можно ли сделать так, чтобы высота поля подстраивалась под текст автоматически?»
К сожалению, стандартными средствами HTML и CSS это сделать невозможно. Тег `textarea` либо имеет фиксированную высоту, либо растягивается пользователем вручную. Для автоматического роста («автоскейла») при вводе используется JavaScript.

### «Почему внутри textarea не работает жирный шрифт через `<b>`?»
Потому что `<textarea>` — это поле для **чистого текста** (plain text). Она не понимает HTML-теги внутри себя и просто отобразит их как текст. Если вам нужно поле с форматированием (жирный, курсив, ссылки), используются специальные редакторы — WYSIWYG (типа TinyMCE или CKEditor), которые работают на базе JavaScript.

### «Как передать значение по умолчанию, если оно содержит HTML-код?»
Вы просто пишете код как текст. Браузер сам его экранирует.
```html
<textarea>Этот <b>тег</b> не станет жирным</textarea>
```

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **`<textarea>`** — для длинных текстов, отзывов и сообщений.
> 2. Это **парный тег**, значение по умолчанию пишется внутри него.
> 3. Любые пробелы и переносы внутри тега в коде станут реальными пробелами в поле.
> 4. Атрибуты **`rows`** и **`cols`** задают базовый размер, но CSS приоритетнее.
> 5. Свойство **`resize`** позволяет ограничить возможность пользователя растягивать поле.
> 6. Текстовая область работает только с чистым текстом, HTML внутри неё не рендерится.
> 7. Всегда используйте плейсхолдеры как подсказки по формату заполнения.
