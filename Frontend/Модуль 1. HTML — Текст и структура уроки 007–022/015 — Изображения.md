## 📖 Тег `<img>` — вставка изображения

**`<img>`** (image) — одиночный тег, который вставляет изображение на страницу.

### Базовый синтаксис

```html
<img src="images/photo.jpg" alt="Описание фотографии">
```
### Особенности тега `<img>`

|Свойство|Описание|
|---|---|
|**Одиночный тег**|Нет закрывающего тега `</img>`|
|**Строчный элемент**|По умолчанию располагается в строке с текстом|
|**Замещаемый элемент**|Браузер заменяет тег на реальную картинку|
|**Обязательные атрибуты**|`src` и `alt`|

---

## 📌 Атрибут `src` — источник изображения

`src` (source) указывает **путь к файлу** изображения. Без него браузеру нечего показывать.

### Варианты значений

|Тип|Пример|Когда использовать|
|---|---|---|
|Относительный путь|`src="images/photo.jpg"`|Картинка в твоём проекте|
|Относительный путь (из подпапки)|`src="../images/photo.jpg"`|Картинка на уровень выше|
|Абсолютный URL|`src="https://example.com/photo.jpg"`|Картинка с другого сайта|

```HTML

<!-- Картинка из папки images (твой проект) -->
<img src="images/cat.jpg" alt="Рыжий кот">

<!-- Картинка из интернета -->
<img src="https://via.placeholder.com/300x200" alt="Заглушка">
```

> [!warning] Картинки из интернета  
> Подгрузка картинок с чужих сайтов (`https://другой-сайт.com/image.jpg`) — **плохая практика** для реальных проектов. Владелец может удалить картинку, и на твоём сайте появится «битое» изображение. Для учебных целей — допустимо.

## 📌 Атрибут `alt` — альтернативный текст

`alt` (alternative text) — **обязательный** атрибут, который описывает содержимое картинки текстом.

```HTML

<img src="sunset.jpg" alt="Закат солнца над морем, оранжевое небо отражается в воде">
```

### Когда отображается `alt`?

|Ситуация|Что видит пользователь|
|---|---|
|Картинка загрузилась|Картинку (текст `alt` не виден)|
|Картинка **не** загрузилась|Текст из `alt` вместо картинки|
|Интернет медленный|Текст `alt`, пока картинка загружается|
|Экранная читалка|Произносит текст `alt` вслух|

### Как писать хороший `alt`

|Правило|Хорошо ✅|Плохо ❌|
|---|---|---|
|Описывай, что **на** картинке|`alt="Щенок лабрадора играет с мячом на лужайке"`|`alt="Картинка"`|
|Будь конкретным|`alt="График роста продаж за 2024 год"`|`alt="График"`|
|Не начинай с «Изображение...»|`alt="Логотип компании Apple"`|`alt="Изображение логотипа"`|
|Не дублируй подпись|Если рядом есть текстовая подпись — alt может быть короче|alt повторяет подпись слово в слово|
|Для декоративных — пустой alt|`alt=""` (именно пустые кавычки)|Пропустить alt совсем|
|Длина|1–2 предложения (до 125 символов)|Огромный абзац текста|

### Три типа изображений и их `alt`

|Тип изображения|Описание|Пример `alt`|
|---|---|---|
|**Информативное**|Несёт важную информацию|`alt="Схема подключения роутера к компьютеру"`|
|**Функциональное**|Кнопка, иконка, ссылка|`alt="Поиск"`, `alt="Закрыть меню"`|
|**Декоративное**|Украшение, не несёт информации|`alt=""` (пустой!)|

### Декоративные изображения — пустой `alt`

Если картинка — чисто **декоративная** (узор, фоновая текстура, разделитель), она не несёт информации. В этом случае пиши **пустой** `alt`:

```HTML

<!-- Декоративный разделитель — пустой alt -->
<img src="divider.png" alt="">

<!-- Иконка рядом с текстом, текст уже всё объясняет -->
<img src="icon-phone.png" alt=""> +7 (900) 123-45-67
```

> [!important] Пустой `alt=""` и отсутствующий `alt` — разные вещи!
> 
> |Код|Что делает экранная читалка|
> |---|---|
> |`<img src="x.jpg" alt="">`|**Пропускает** картинку (правильно для декоративных)|
> |`<img src="x.jpg">`|**Произносит имя файла** «икс точка джейпег» (ужасно)|

## 📐 Атрибуты `width` и `height` — размеры

Атрибуты `width` и `height` задают **ширину** и **высоту** изображения в пикселях.

```HTML

<img src="photo.jpg" alt="Фото" width="600" height="400">
```

### Зачем указывать размеры?

|Без width/height|С width/height|
|---|---|
|Пока картинка загружается, браузер **не знает** её размеры|Браузер **заранее** резервирует место для картинки|
|Содержимое страницы **прыгает** при загрузке (layout shift)|Содержимое **не сдвигается** — место уже зарезервировано|
|Плохой пользовательский опыт|Хороший пользовательский опыт|

> [!tip] Layout shift — «прыжки» контента  
> Ты наверняка бывал на сайтах, где текст «прыгает» вниз, когда загружается картинка. Это называется **Cumulative Layout Shift (CLS)** — один из показателей качества сайта. Указание `width` и `height` **полностью решает** эту проблему.

### Правила работы с размерами

|Правило|Пример|
|---|---|
|Значения указываются **без единиц измерения** (подразумеваются пиксели)|`width="600"`, не `width="600px"`|
|Указывай **оба** атрибута (width и height)|`width="600" height="400"`|
|Значения должны соответствовать **реальным пропорциям** картинки|Если картинка 1200×800, пиши `width="600" height="400"` (пропорция 3:2)|

### Что будет, если указать неправильные пропорции?

```HTML

<!-- Оригинал: 1200×800 (пропорция 3:2) -->

<!-- ✅ Правильные пропорции -->
<img src="photo.jpg" alt="Фото" width="600" height="400">

<!-- ❌ Неправильные пропорции — картинка растянется/сожмётся -->
<img src="photo.jpg" alt="Фото" width="600" height="600">
```

Неправильные пропорции приведут к **искажению** — картинка будет растянута или сжата.

### Размер через CSS vs атрибуты

|Способ|Для чего|Пример|
|---|---|---|
|Атрибуты `width`/`height`|**Резервирование места** (предотвращение прыжков)|`<img width="600" height="400">`|
|CSS|**Визуальный размер** на странице|`img { width: 100%; height: auto; }`|

> [!note] Лучшая практика  
> Указывай `width` и `height` в атрибутах (для резервирования места), а реальный визуальный размер контролируй через CSS. Мы разберём это подробнее в модуле CSS.

## 🔄 Адаптивные изображения — базовый приём

Чтобы картинка не вылезала за границы экрана на мобильных устройствах, добавь одну CSS-строку:

```HTML

<style>
  img {
    max-width: 100%;
    height: auto;
  }
</style>
```

|Свойство|Что делает|
|---|---|
|`max-width: 100%`|Картинка не будет шире своего контейнера|
|`height: auto`|Высота подстраивается автоматически, сохраняя пропорции|

Это **самый простой способ** сделать картинки адаптивными. Добавляй эти строки в каждый проект.

```HTML

<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Адаптивные картинки</title>
    <style>
      img {
        max-width: 100%;
        height: auto;
      }
    </style>
  </head>
  <body>
    <!-- Картинка не вылезет за границы экрана -->
    <img src="big-photo.jpg" alt="Большое фото" width="1200" height="800">
  </body>
</html>
```

> [!note] Подробнее об адаптивных изображениях  
> `<picture>`, `srcset` и продвинутые техники — в уроке [[151 — Адаптивные изображения]] модуля CSS.

## 🏷️ `<figure>` и `<figcaption>` — изображение с подписью

Часто к изображению нужна **подпись**. Для этого используется пара тегов `<figure>` и `<figcaption>`.

```HTML

<figure>
  <img src="graph.png" alt="График продаж: рост на 25% в 2024 году" width="600" height="400">
  <figcaption>Рис. 1 — Динамика продаж за 2024 год</figcaption>
</figure>
```

### Что делает каждый тег

|Тег|Роль|Обязательный?|
|---|---|---|
|`<figure>`|Контейнер для самостоятельного контента (изображение, диаграмма, код)|Нет, но рекомендуется|
|`<figcaption>`|Подпись к содержимому `<figure>`|Нет, но если есть `<figure>` — обычно нужна|

### Правила

|Правило|Подробности|
|---|---|
|`<figcaption>` — только внутри `<figure>`|Нельзя использовать `<figcaption>` отдельно|
|`<figcaption>` — первым или последним|Может стоять до или после `<img>` внутри `<figure>`|
|`<figure>` — для самостоятельного контента|Контент, который можно вынести из потока текста (как иллюстрация в книге)|

### Подпись сверху или снизу

```HTML

<!-- Подпись снизу (чаще всего) -->
<figure>
  <img src="chart.png" alt="Столбчатая диаграмма" width="500" height="300">
  <figcaption>Рис. 1 — Доходы компании по кварталам</figcaption>
</figure>

<!-- Подпись сверху (реже) -->
<figure>
  <figcaption>Фото 12 — Вид на Эльбрус с севера</figcaption>
  <img src="elbrus.jpg" alt="Гора Эльбрус, вид с северной стороны" width="800" height="500">
</figure>
```

### Несколько изображений в одном `<figure>`

```HTML

<figure>
  <img src="before.jpg" alt="Комната до ремонта" width="400" height="300">
  <img src="after.jpg" alt="Комната после ремонта" width="400" height="300">
  <figcaption>Комната до и после ремонта</figcaption>
</figure>
```

### `<figure>` — не только для картинок

`<figure>` можно использовать для любого **самостоятельного контента**:

```HTML

<!-- Код с подписью -->
<figure>
  <pre><code>console.log("Hello, World!");</code></pre>
  <figcaption>Пример вывода в консоль на JavaScript</figcaption>
</figure>

<!-- Цитата с подписью -->
<figure>
  <blockquote>
    <p>Единственный способ делать великую работу — любить то, что делаешь.</p>
  </blockquote>
  <figcaption>Стив Джобс</figcaption>
</figure>
```

## 🖼️ Изображение как ссылка

Картинка может быть **кликабельной ссылкой** — просто оберни `<img>` в `<a>`:

```HTML

<a href="https://example.com">
  <img src="banner.jpg" alt="Перейти на сайт Example.com" width="728" height="90">
</a>
```

### Важно: `alt` для картинки-ссылки

Когда картинка является ссылкой, `alt` должен описывать **куда ведёт ссылка**, а не что изображено:

|Ситуация|Хорошо ✅|Плохо ❌|
|---|---|---|
|Логотип-ссылка на главную|`alt="На главную страницу"`|`alt="Логотип"`|
|Баннер акции|`alt="Перейти к скидкам до 50%"`|`alt="Красный баннер"`|
|Иконка соцсети|`alt="Наша страница ВКонтакте"`|`alt="Иконка ВК"`|

## 📊 Атрибут `loading` — ленивая загрузка

Атрибут `loading` позволяет **отложить загрузку** картинок, которые находятся за пределами видимой области экрана.

```HTML

<!-- Картинки в начале страницы — загружаются сразу -->
<img src="hero.jpg" alt="Главный баннер" width="1200" height="600" loading="eager">

<!-- Картинки внизу страницы — загружаются когда пользователь до них доскроллит -->
<img src="photo-50.jpg" alt="Фото 50" width="400" height="300" loading="lazy">
```

|Значение|Поведение|Когда использовать|
|---|---|---|
|`eager`|Загружается **сразу** (по умолчанию)|Картинки в верхней части страницы|
|`lazy`|Загружается **при приближении** к видимой области|Картинки ниже первого экрана|

### Зачем нужна ленивая загрузка?

|Без `loading="lazy"`|С `loading="lazy"`|
|---|---|
|Все 50 картинок загружаются сразу|Загружаются только видимые (5–10)|
|Страница грузится **медленно**|Страница грузится **быстро**|
|Расходуется трафик на невидимые картинки|Трафик экономится|

```HTML

<!-- Первые картинки — eager (или просто не указывать loading) -->
<img src="hero.jpg" alt="Баннер" width="1200" height="600">

<!-- Остальные — lazy -->
<img src="photo1.jpg" alt="Фото 1" width="400" height="300" loading="lazy">
<img src="photo2.jpg" alt="Фото 2" width="400" height="300" loading="lazy">
<img src="photo3.jpg" alt="Фото 3" width="400" height="300" loading="lazy">
```

> [!tip] Простое правило  
> Добавляй `loading="lazy"` ко **всем картинкам, кроме первого экрана**. Это бесплатное ускорение сайта.

## 🧪 Практика

Создай файл и набери вручную:

```HTML

<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Изображения в HTML</title>
    <style>
      img {
        max-width: 100%;
        height: auto;
      }
    </style>
  </head>
  <body>

    <h1>Работа с изображениями</h1>

    <!-- Базовая картинка -->
    <h2>Простое изображение</h2>
    <img
      src="https://via.placeholder.com/600x400/3498db/ffffff?text=Hello+World"
      alt="Синий прямоугольник с надписью Hello World"
      width="600"
      height="400"
    >

    <!-- Картинка с подписью -->
    <h2>Изображение с подписью</h2>
    <figure>
      <img
        src="https://via.placeholder.com/500x300/2ecc71/ffffff?text=Nature"
        alt="Зелёный прямоугольник, символизирующий природу"
        width="500"
        height="300"
      >
      <figcaption>Рис. 1 — Пример изображения с подписью</figcaption>
    </figure>

    <!-- Картинка-ссылка -->
    <h2>Картинка-ссылка</h2>
    <p>Нажми на картинку, чтобы перейти на Google:</p>
    <a href="https://google.com" target="_blank" rel="noopener noreferrer">
      <img
        src="https://via.placeholder.com/200x80/e74c3c/ffffff?text=Click+Me"
        alt="Перейти на Google"
        width="200"
        height="80"
      >
    </a>

    <!-- Несколько изображений -->
    <h2>Галерея</h2>
    <figure>
      <img
        src="https://via.placeholder.com/250x250/9b59b6/ffffff?text=1"
        alt="Фото 1"
        width="250"
        height="250"
        loading="lazy"
      >
      <img
        src="https://via.placeholder.com/250x250/e67e22/ffffff?text=2"
        alt="Фото 2"
        width="250"
        height="250"
        loading="lazy"
      >
      <img
        src="https://via.placeholder.com/250x250/1abc9c/ffffff?text=3"
        alt="Фото 3"
        width="250"
        height="250"
        loading="lazy"
      >
      <figcaption>Мини-галерея из трёх изображений</figcaption>
    </figure>

    <!-- Демонстрация alt при битой картинке -->
    <h2>Что видно, когда картинка не загрузилась</h2>
    <img
      src="несуществующий-файл.jpg"
      alt="Эта картинка не загрузилась, и вы видите этот текст вместо неё"
      width="400"
      height="200"
    >

    <!-- Декоративное изображение -->
    <h2>Декоративное изображение</h2>
    <p>Разделитель ниже — декоративный, поэтому alt пустой:</p>
    <img
      src="https://via.placeholder.com/600x2/cccccc/cccccc"
      alt=""
      width="600"
      height="2"
    >
    <p>Текст после декоративного разделителя.</p>

  </body>
</html>
```

### Что проверить в браузере

|Что проверить|Ожидаемый результат|
|---|---|
|Все картинки-заглушки|Отображаются цветные прямоугольники с текстом|
|`<figcaption>`|Подписи под картинками|
|Картинка-ссылка|При клике открывает Google в новой вкладке|
|Битая картинка|Показывается текст alt и иконка «битой» картинки|
|Декоративный разделитель|Тонкая серая линия без alt-текста|
|Адаптивность|Сужай окно — картинки не вылезают за границы|

## 🔍 Отладка изображений через DevTools

|Проблема|Как найти причину|
|---|---|
|Картинка не показывается|DevTools → **Console** → ищи ошибку `404 Not Found`|
|Картинка не того размера|DevTools → **Elements** → кликни на `<img>` → справа увидишь computed размеры|
|Нет alt|DevTools → **Elements** → найди `<img>` → проверь атрибут alt|
|Медленная загрузка|DevTools → **Network** → столбец **Size** и **Time**|

### Быстрая проверка всех картинок без alt

Открой DevTools → Console и введи:

```JavaScript

document.querySelectorAll('img:not([alt])').forEach(img => console.warn('Нет alt:', img.src));
```

Команда выведет все картинки, у которых **отсутствует** атрибут `alt`.

## 📊 Все атрибуты `<img>` — сводка

|Атрибут|Обязательный|Назначение|Пример|
|---|---|---|---|
|`src`|✅|Путь к файлу изображения|`src="images/photo.jpg"`|
|`alt`|✅|Альтернативный текст|`alt="Описание картинки"`|
|`width`|Рекомендуется|Ширина в пикселях|`width="600"`|
|`height`|Рекомендуется|Высота в пикселях|`height="400"`|
|`loading`|Нет|Стратегия загрузки|`loading="lazy"`|
|`title`|Нет|Подсказка при наведении|`title="Кликни для увеличения"`|
|`class`|Нет|CSS-класс|`class="photo"`|
|`id`|Нет|Уникальный идентификатор|`id="main-photo"`|

## ⚠️ Типичные ошибки

### Ошибка 1: Нет атрибута `alt`

```HTML

<!-- ❌ Нет alt -->
<img src="photo.jpg">

<!-- ✅ Есть alt -->
<img src="photo.jpg" alt="Закат над морем">
```

---

### Ошибка 2: Бессмысленный `alt`

```HTML

<!-- ❌ Бессмысленно -->
<img src="photo.jpg" alt="Картинка">
<img src="photo.jpg" alt="Изображение фотографии">
<img src="photo.jpg" alt="photo.jpg">

<!-- ✅ Информативно -->
<img src="photo.jpg" alt="Команда разработчиков на конференции Google I/O 2024">
```

---

### Ошибка 3: Нет `width` и `height`

```HTML

<!-- ❌ Контент будет «прыгать» при загрузке -->
<img src="big-photo.jpg" alt="Фото">

<!-- ✅ Место зарезервировано заранее -->
<img src="big-photo.jpg" alt="Фото" width="800" height="600">
```

---

### Ошибка 4: Искажённые пропорции

```HTML

<!-- ❌ Оригинал 1200×800, но указано 600×600 — картинка сплющится -->
<img src="photo.jpg" alt="Фото" width="600" height="600">

<!-- ✅ Пропорции сохранены (3:2) -->
<img src="photo.jpg" alt="Фото" width="600" height="400">
```

---

### Ошибка 5: Огромный файл без оптимизации

|Размер файла|Время загрузки (3G)|Оценка|
|---|---|---|
|5 МБ|~15 секунд|❌ Ужасно|
|1 МБ|~3 секунды|⚠️ Многовато|
|200 КБ|~0.6 секунды|✅ Хорошо|
|50 КБ|~0.15 секунды|✅ Отлично|

> [!tip] Оптимизируй изображения  
> Перед добавлением на сайт сжимай картинки. Бесплатные сервисы: **tinypng.com**, **squoosh.app**. Подробнее о форматах и оптимизации — в следующем уроке [[016 — Форматы изображений]].

## ❓ Частые вопросы

### «Можно ли вставить GIF-анимацию?»

Да, тег `<img>` поддерживает GIF. Анимация воспроизводится автоматически:

```HTML

<img src="animation.gif" alt="Анимированная иконка загрузки" width="100" height="100">
```

### «Можно ли вставить SVG через `<img>`?»

Да:

```HTML

<img src="icon.svg" alt="Иконка" width="50" height="50">
```

SVG через `<img>` работает как обычная картинка. Есть и другие способы вставки SVG — разберём их в уроке [[016 — Форматы изображений]].

### «Как сделать картинку фоном?»

Через CSS-свойство `background-image`. Мы изучим это в уроке [[067 — Фон — цвет и изображение]]:

```CSS

.hero {
    background-image: url('hero.jpg');
}
```

### «Как узнать реальный размер картинки?»

Открой файл картинки в любой программе просмотра или через DevTools → Network → кликни на картинку → увидишь размеры. На Mac: правый клик → «Свойства». На Windows: правый клик → «Свойства» → «Подробно».

### «`<figcaption>` заменяет `alt`?»

Нет. `alt` — для экранных читалок и случаев, когда картинка не загрузилась. `<figcaption>` — видимая подпись на странице. Они **дополняют** друг друга, а не заменяют.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 
> 1. **`<img>`** — одиночный тег для вставки изображений
> 2. **`src`** — путь к файлу (обязательный)
> 3. **`alt`** — альтернативный текст (обязательный). Для декоративных — `alt=""`
> 4. **`width`** и **`height`** — указывай для предотвращения «прыжков» контента
> 5. **`loading="lazy"`** — откладывает загрузку невидимых картинок (ускоряет сайт)
> 6. **`<figure>` + `<figcaption>`** — картинка с подписью
> 7. `max-width: 100%; height: auto;` — базовая адаптивность картинок
> 8. Всегда **оптимизируй** размер файла перед добавлением на сайт
