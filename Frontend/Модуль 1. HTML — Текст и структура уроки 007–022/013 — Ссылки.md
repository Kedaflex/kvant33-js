## 📖 Почему ссылки так важны?

Слово **«веб»** (web) переводится как **«паутина»**. Интернет — это паутина, а **ссылки** — нити, которые соединяют страницы друг с другом.

Без ссылок каждая страница была бы изолированным островом. Именно ссылки делают интернет **интернетом** — позволяют переходить с одной страницы на другую одним кликом.

> [!tip] Факт
> Термин **гипертекст** (hypertext) в аббревиатуре HTML означает «текст со ссылками». Ссылки — это то, что отличает HTML от обычного текста.

## 🔗 Тег `<a>` — создание ссылки

**`<a>`** (anchor — якорь) — тег для создания ссылок.

### Базовый синтаксис

```html
<a href="https://google.com">Перейти в Google</a>
```

| Часть           | Код                             | Роль                                         |
| --------------- | ------------------------------- | -------------------------------------------- |
| Открывающий тег | `<a href="https://google.com">` | Начало ссылки, адрес                         |
| Содержимое      | `Перейти в Google`              | Текст, который увидит и кликнет пользователь |
| Закрывающий тег | `</a>`                          | Конец ссылки                                 |
### Как ссылка выглядит по умолчанию

| Состояние               | Внешний вид                         |
| ----------------------- | ----------------------------------- |
| Обычная (не посещённая) | Синий текст, подчёркнутый           |
| Посещённая              | Фиолетовый текст, подчёркнутый      |
| При наведении курсора   | Курсор меняется на «руку» (pointer) |
| При нажатии (активная)  | Красный текст                       |

> [!note] Стили по умолчанию можно изменить  
> Через CSS можно задать ссылкам **любой** вид: убрать подчёркивание, изменить цвет, добавить анимацию. Мы научимся этому в модуле CSS (уроки 109–111).

## 📌 Атрибут `href` — куда ведёт ссылка

`href` (hypertext reference) — **обязательный** атрибут тега `<a>`. Он указывает, куда перейдёт пользователь при клике.

### Типы значений href

|Тип|Пример|Куда ведёт|
|---|---|---|
|Внешняя ссылка (URL)|`href="https://google.com"`|На другой сайт|
|Внутренняя ссылка (файл)|`href="about.html"`|На другую страницу твоего сайта|
|Якорь|`href="#contacts"`|На секцию текущей страницы|
|Email|`href="mailto:info@example.com"`|Откроет почтовый клиент|
|Телефон|`href="tel:+79001234567"`|Откроет набор номера|
|Файл|`href="document.pdf"`|Скачает или откроет файл|
|Пустая ссылка|`href="#"`|Никуда (остаётся на месте, прокрутит наверх)|

## 🌐 Внешние ссылки

Ссылки на **другие сайты**. Указывается полный URL-адрес.

```HTML
<p>Поисковые системы:</p>
<ul>
  <li><a href="https://google.com">Google</a></li>
  <li><a href="https://yandex.ru">Яндекс</a></li>
  <li><a href="https://duckduckgo.com">DuckDuckGo</a></li>
</ul>
```

### Правила

|Правило|Пример|
|---|---|
|Всегда указывай протокол|`https://google.com`, не `google.com`|
|Предпочитай `https://`|Безопаснее, чем `http://`|

```HTML
<!-- ❌ Без протокола — может не работать -->
<a href="google.com">Google</a>

<!-- ✅ С протоколом — правильно -->
<a href="https://google.com">Google</a>
```

## 📄 Внутренние ссылки

Ссылки на **другие страницы твоего сайта**. Указывается путь к файлу.

```HTML
<!-- Ссылка на файл в той же папке -->
<a href="about.html">О нас</a>

<!-- Ссылка на файл в подпапке -->
<a href="pages/contacts.html">Контакты</a>

<!-- Ссылка на файл на уровень выше -->
<a href="../index.html">На главную</a>
```

> [!note] Подробнее о путях  
> Абсолютные и относительные пути к файлам — тема следующего урока [[014 — Абсолютные и относительные пути]]. Сейчас достаточно знать: если файл в той же папке — просто пишешь его имя.

## ⚓ Якорные ссылки (навигация внутри страницы)

Якорные ссылки позволяют **прокрутить страницу** к определённому элементу. Это полезно для длинных страниц.

### Как это работает

|Шаг|Что делать|Пример|
|---|---|---|
|1|Дай целевому элементу **id**|`<h2 id="contacts">Контакты</h2>`|
|2|Создай ссылку с `href="#id"`|`<a href="#contacts">Перейти к контактам</a>`|

При клике на ссылку страница **плавно прокрутится** (или мгновенно перепрыгнет) к элементу с указанным `id`.

### Полный пример

```HTML
<!-- Навигация (оглавление) -->
<h1>Руководство пользователя</h1>
<nav>
  <p>Содержание:</p>
  <ul>
    <li><a href="#installation">Установка</a></li>
    <li><a href="#settings">Настройка</a></li>
    <li><a href="#usage">Использование</a></li>
    <li><a href="#faq">Частые вопросы</a></li>
  </ul>
</nav>

<!-- Секции страницы -->
<h2 id="installation">Установка</h2>
<p>Скачайте программу с официального сайта...</p>
<p>Запустите установщик и следуйте инструкциям...</p>

<h2 id="settings">Настройка</h2>
<p>Откройте меню настроек...</p>
<p>Выберите язык интерфейса...</p>

<h2 id="usage">Использование</h2>
<p>Для начала работы нажмите кнопку «Старт»...</p>
<p>Результат появится через несколько секунд...</p>

<h2 id="faq">Частые вопросы</h2>
<p>Вопрос: Программа не запускается. Что делать?</p>
<p>Ответ: Попробуйте переустановить программу.</p>

<!-- Ссылка наверх -->
<p><a href="#top">⬆ Вернуться наверх</a></p>
```

> [!tip] Ссылка наверх  
> `href="#top"` — специальное значение, которое прокручивает страницу **в самое начало**, даже если элемента с `id="top"` не существует. Но лучше явно задать `id="top"` на `<h1>` или `<body>`.

---

### Якорь на элемент другой страницы

Можно перейти на **конкретную секцию** другой страницы:

```HTML
<!-- Перейти на страницу about.html и прокрутить до секции #team -->
<a href="about.html#team">Наша команда</a>

<!-- Перейти на секцию другого сайта -->
<a href="https://ru.wikipedia.org/wiki/HTML#История">История HTML (Wikipedia)</a>
```

## 🆕 Атрибут `target` — где открывать ссылку

По умолчанию ссылка открывается **в текущей вкладке**. Атрибут `target` меняет это поведение.

|Значение|Где откроется|Когда использовать|
|---|---|---|
|`_self`|В текущей вкладке|По умолчанию (можно не указывать)|
|`_blank`|В **новой** вкладке|Внешние ссылки, чтобы не терять пользователя|

```HTML
<!-- Откроется в текущей вкладке (по умолчанию) -->
<a href="about.html">О нас</a>

<!-- Откроется в новой вкладке -->
<a href="https://google.com" target="_blank">Google</a>
```

### Когда открывать в новой вкладке?

|Ситуация|target|Почему|
|---|---|---|
|Ссылка на **другой сайт**|`_blank`|Пользователь не покидает твой сайт|
|Ссылка на **свою** страницу|`_self`|Пользователь остаётся в рамках сайта|
|Ссылка на **документ** (PDF)|`_blank`|Документ откроется отдельно|
|Якорная ссылка|`_self`|Прокрутка на той же странице|

---

### Безопасность: `rel="noopener noreferrer"`

При использовании `target="_blank"` рекомендуется добавлять атрибут `rel`:

```HTML
<a href="https://google.com" target="_blank" rel="noopener noreferrer">Google</a>
```

|Значение|Зачем|
|---|---|
|`noopener`|Запрещает новой вкладке получить доступ к предыдущей (безопасность)|
|`noreferrer`|Не передаёт информацию о том, откуда пришёл пользователь|

> [!note] Современные браузеры  
> Современные браузеры (Chrome 88+, Firefox 79+) автоматически добавляют `noopener` для `target="_blank"`. Но для совместимости со старыми браузерами лучше указывать явно.

## 📧 Ссылка на email

```HTML
<a href="mailto:info@example.com">Написать нам</a>
```

При клике откроется **почтовый клиент** (Outlook, Thunderbird, Gmail) с заполненным полем «Кому».

### Дополнительные параметры

```HTML
<!-- С темой письма -->
<a href="mailto:info@example.com?subject=Вопрос с сайта">Написать нам</a>

<!-- С темой и текстом -->
<a href="mailto:info@example.com?subject=Вопрос&body=Здравствуйте!">Написать нам</a>

<!-- Несколько получателей -->
<a href="mailto:info@example.com,support@example.com">Написать нам</a>

<!-- С копией -->
<a href="mailto:info@example.com?cc=boss@example.com">Написать нам</a>
```

|Параметр|Назначение|Пример|
|---|---|---|
|`subject`|Тема письма|`?subject=Привет`|
|`body`|Текст письма|`?body=Здравствуйте!`|
|`cc`|Копия|`?cc=boss@example.com`|
|`bcc`|Скрытая копия|`?bcc=admin@example.com`|

## 📞 Ссылка на телефон

```HTML
<a href="tel:+79001234567">+7 (900) 123-45-67</a>
```

На **мобильном** устройстве при клике откроется **набор номера**. На компьютере может предложить выбрать приложение (Skype, FaceTime) или ничего не произойти.

### Правила

|Правило|Пример|
|---|---|
|В `href` — без пробелов, скобок и дефисов|`href="tel:+79001234567"`|
|В тексте ссылки — читаемый формат|`+7 (900) 123-45-67`|
|Международный формат с `+`|`tel:+79001234567`|

```HTML

<!-- ❌ Неправильно: пробелы и скобки в href -->
<a href="tel:+7 (900) 123-45-67">Позвонить</a>

<!-- ✅ Правильно -->
<a href="tel:+79001234567">+7 (900) 123-45-67</a>
```

## 📦 Что можно помещать внутрь `<a>`?

Содержимое ссылки — это **кликабельная область**. Внутрь можно помещать не только текст.

### Текст (самый частый случай)

```HTML

<a href="about.html">О компании</a>
```

### Текст с форматированием

```HTML

<a href="sale.html"><strong>Распродажа!</strong> Скидки до 70%</a>
```

### Изображение (картинка-ссылка)

```HTML

<a href="https://google.com">
  <img src="google-logo.png" alt="Логотип Google">
</a>
```

### Блок с несколькими элементами

В HTML5 ссылка может оборачивать **блочные элементы**:

```HTML
<a href="article.html">
  <h2>Заголовок статьи</h2>
  <p>Краткое описание статьи...</p>
</a>
```

### Что нельзя помещать внутрь `<a>`

|Нельзя ❌|Почему|
|---|---|
|Другую ссылку `<a>`|Ссылка внутри ссылки — запрещено|
|Кнопку `<button>`|Интерактивный элемент внутри интерактивного — запрещено|
|Форму `<form>`|Интерактивный элемент|

```HTML

<!-- ❌ Ссылка внутри ссылки — запрещено -->
<a href="page1.html">
  Текст <a href="page2.html">другая ссылка</a>
</a>

<!-- ❌ Кнопка внутри ссылки — запрещено -->
<a href="page.html">
  <button>Нажми</button>
</a>
```

## 📋 Атрибут `title` — подсказка

Атрибут `title` показывает **всплывающую подсказку** при наведении курсора на ссылку.

```HTML

<a href="https://developer.mozilla.org" title="Документация по веб-технологиям">
  MDN Web Docs
</a>
```

|С title|Без title|
|---|---|
|При наведении — подсказка|При наведении — ничего|
### Когда полезен title

|Ситуация|Пример|
|---|---|
|Ссылка-иконка (без текста)|`<a href="/" title="На главную">🏠</a>`|
|Уточнение, куда ведёт ссылка|`<a href="report.pdf" title="Скачать отчёт в формате PDF">Отчёт</a>`|
|Сокращённый текст ссылки|`<a href="..." title="Полное название статьи">Читать далее</a>`|

> [!warning] title не заменяет понятный текст ссылки  
> Не полагайся только на `title`. Мобильные устройства **не показывают** всплывающие подсказки (нет наведения курсора). Текст ссылки должен быть понятен сам по себе.

## 📥 Ссылка на скачивание — атрибут `download`

Атрибут `download` говорит браузеру **скачать** файл, а не открывать его.

```HTML

<!-- Без download — браузер откроет PDF -->
<a href="report.pdf">Открыть отчёт</a>

<!-- С download — браузер скачает файл -->
<a href="report.pdf" download>Скачать отчёт</a>

<!-- С download и именем файла -->
<a href="report.pdf" download="Годовой-отчёт-2024.pdf">Скачать отчёт</a>
```

|Без `download`|С `download`|
|---|---|
|Браузер пытается **открыть** файл|Браузер **скачивает** файл|

> [!note] Ограничения  
> Атрибут `download` работает только для файлов **с того же домена**. Для файлов с других сайтов браузер может проигнорировать этот атрибут.

## ✍️ Как писать хороший текст ссылки

Текст ссылки должен быть **понятным** и **информативным** — пользователь должен понимать, куда попадёт при клике.

### Правила

|Хорошо ✅|Плохо ❌|Почему|
|---|---|---|
|`<a href="...">Документация по CSS</a>`|`<a href="...">Нажмите сюда</a>`|«Нажмите сюда» ничего не говорит о содержимом|
|`<a href="...">Скачать прайс-лист (PDF, 2 МБ)</a>`|`<a href="...">Скачать</a>`|Не понятно, что именно скачается|
|`<a href="...">Читать статью «Основы HTML»</a>`|`<a href="...">тут</a>`|«Тут» — бессмысленное слово для экранной читалки|
|`<a href="...">Перейти на сайт Google</a>`|`<a href="...">https://google.com</a>`|Голый URL трудно читать|

### Почему «нажмите сюда» — плохо?

|Кто|Проблема|
|---|---|
|**Зрячий пользователь**|Сканирует страницу глазами и видит только ссылки. «Нажмите сюда» ничего не говорит|
|**Слабовидящий пользователь**|Экранная читалка озвучивает список ссылок: «Нажмите сюда, нажмите сюда, нажмите сюда» — бесполезно|
|**Поисковик**|Google использует текст ссылки для понимания содержимого целевой страницы. «Нажмите сюда» не помогает|

```HTML
          github.com
        </a>
      </li>
    </ul>

    <hr>

    <!-- Ссылка наверх -->
    <p><a href="#top">⬆ Вернуться к началу страницы</a></p>

  </body>
</html>
```
### Что проверить в браузере

|Что проверить|Ожидаемый результат|
|---|---|
|Ссылки содержания|Клик прокручивает к соответствующей секции|
|Внешние ссылки|Открываются в **новой вкладке**|
|Внутренние ссылки|Открываются в **текущей вкладке** (будет 404, и это нормально)|
|Email-ссылка|Открывает почтовый клиент|
|Телефон-ссылка|На мобильном — набор номера, на ПК — зависит от системы|
|Картинка-ссылка|Кликабельная картинка, ведёт на Google|
|«Вернуться наверх»|Прокручивает в начало страницы|
|Подсказка (title)|Наведи на CSS-Tricks — появится подсказка|

## ⚠️ Типичные ошибки

### Ошибка 1: Ссылка без `href`

```HTML

<!-- ❌ Бессмысленно: ссылка без адреса -->
<a>Текст ссылки</a>

<!-- ✅ Правильно -->
<a href="page.html">Текст ссылки</a>
```

Без `href` тег `<a>` не является кликабельной ссылкой.

---

### Ошибка 2: Внешняя ссылка без протокола

```HTML

<!-- ❌ Браузер воспримет как путь к локальному файлу -->
<a href="google.com">Google</a>

<!-- ✅ Правильно -->
<a href="https://google.com">Google</a>
```

---

### Ошибка 3: «Нажмите сюда»

```HTML

<!-- ❌ Бессмысленный текст ссылки -->
<p>Чтобы узнать подробности, <a href="details.html">нажмите сюда</a>.</p>

<!-- ✅ Информативный текст -->
<p><a href="details.html">Подробности о программе курса</a>.</p>
```

---

### Ошибка 4: Пробелы в `tel:`

```HTML

<!-- ❌ Пробелы и скобки в href -->
<a href="tel:+7 (900) 123-45-67">Позвонить</a>

<!-- ✅ Чистый номер в href -->
<a href="tel:+79001234567">+7 (900) 123-45-67</a>
```

---

### Ошибка 5: Забытый `alt` у картинки внутри ссылки

```HTML

<!-- ❌ Нет alt — плохо для доступности -->
<a href="page.html"><img src="banner.jpg"></a>

<!-- ✅ Есть alt — экранная читалка озвучит -->
<a href="page.html"><img src="banner.jpg" alt="Перейти на страницу акции"></a>
```

## ❓ Частые вопросы

### «Что такое `href="#"`?»

Это **заглушка** — ссылка, которая никуда не ведёт. Она прокручивает страницу наверх. Используется, когда реальный адрес ещё не определён или поведение ссылки управляется JavaScript.

### «Можно ли сделать ссылку некликабельной?»

Да, через CSS можно убрать реакцию на клик:

```CSS

a.disabled {
    pointer-events: none;
    color: gray;
}
```

Или через JavaScript — `event.preventDefault()`. Мы изучим это в модуле JavaScript.

### «Как открыть ссылку в новом окне, а не во вкладке?»

`target="_blank"` открывает в новой вкладке (в большинстве браузеров). Поведение зависит от настроек браузера пользователя. Разработчик **не может** принудительно открыть именно окно.

### «Внешние ссылки — всегда в новой вкладке?»

Это **рекомендация**, а не правило. Если пользователь кликает на внешнюю ссылку — он может не вернуться на твой сайт. `target="_blank"` решает эту проблему. Но для внутренних ссылок новая вкладка обычно не нужна.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 
> 1. **`<a href="...">`** — основной тег для создания ссылок
> 2. Внешние ссылки: `href="https://..."` + `target="_blank"` + `rel="noopener noreferrer"`
> 3. Внутренние ссылки: `href="page.html"` — на другую страницу сайта

> 6. Внутрь ссылки можно помещать текст, картинки, блоки — но **не другие ссылки**
> 7. Текст ссылки должен быть **информативным** — никаких «нажмите сюда»
