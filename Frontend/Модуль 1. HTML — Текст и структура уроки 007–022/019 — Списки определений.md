## 📖 Что такое список определений?

Мы уже знаем два типа списков:
- `<ul>` — маркированный (порядок не важен)
- `<ol>` — нумерованный (порядок важен)

Есть третий тип — **список определений**. Он предназначен для пар **«термин — описание»**.

| Тип списка | Теги | Для чего |
|---|---|---|
| Маркированный | `<ul>` + `<li>` | Перечисление равноправных пунктов |
| Нумерованный | `<ol>` + `<li>` | Последовательность шагов, рейтинг |
| **Определений** | `<dl>` + `<dt>` + `<dd>` | **Пары «термин — описание»** |

---

## 🔧 Три тега: `<dl>`, `<dt>`, `<dd>`

| Тег | Название | Роль |
|---|---|---|
| `<dl>` | Definition List | **Контейнер** для всего списка |
| `<dt>` | Definition Term | **Термин** (что определяем) |
| `<dd>` | Definition Description | **Описание** (определение термина) |

### Базовый синтаксис

```html
<dl>
  <dt>HTML</dt>
  <dd>Язык разметки для создания веб-страниц.</dd>

  <dt>CSS</dt>
  <dd>Язык стилей для оформления веб-страниц.</dd>

  <dt>JavaScript</dt>
  <dd>Язык программирования для создания интерактивных веб-страниц.</dd>
</dl>
````
### Как выглядит по умолчанию

|Элемент|Отображение|
|---|---|
|`<dt>` (термин)|Обычный текст, **без отступа**|
|`<dd>` (описание)|Текст с **отступом слева** (~40px)|

**Результат в браузере:**

HTML  
　　Язык разметки для создания веб-страниц.

CSS  
　　Язык стилей для оформления веб-страниц.

JavaScript  
　　Язык программирования для создания интерактивных веб-страниц.

---

## 📏 Правила использования

### Правило 1: Внутри `<dl>` — только `<dt>` и `<dd>`

```HTML

<!-- ❌ Неправильно: <p> напрямую внутри <dl> -->
<dl>
  <p>Это абзац</p>
  <dt>Термин</dt>
  <dd>Описание</dd>
</dl>

<!-- ✅ Правильно -->
<dl>
  <dt>Термин</dt>
  <dd>Описание</dd>
</dl>
```

> [!note] Исключение: `<div>` как обёртка  
> HTML5 допускает оборачивание пар `<dt>` + `<dd>` в `<div>` для стилизации:
> 
> 
> 
> ```HTML
> <dl>
>   <div>
>     <dt>Термин</dt>
>     <dd>Описание</dd>
>   </div>
> </dl>
> ```
> 
> Это единственный тег, допустимый внутри `<dl>` помимо `<dt>` и `<dd>`.

---

### Правило 2: Сначала `<dt>`, потом `<dd>`

Термин (`<dt>`) всегда идёт **перед** описанием (`<dd>`).

```HTML

<!-- ❌ Неправильно: описание перед термином -->
<dl>
  <dd>Описание</dd>
  <dt>Термин</dt>
</dl>

<!-- ✅ Правильно -->
<dl>
  <dt>Термин</dt>
  <dd>Описание</dd>
</dl>
```

---

### Правило 3: Один термин — несколько описаний

У одного термина может быть **несколько описаний**:

```HTML

<dl>
  <dt>JavaScript</dt>
  <dd>Язык программирования для веба.</dd>
  <dd>Создан Бренданом Айком в 1995 году.</dd>
  <dd>Используется как на клиенте (браузер), так и на сервере (Node.js).</dd>
</dl>
```

**Результат:**

JavaScript  
　　Язык программирования для веба.  
　　Создан Бренданом Айком в 1995 году.  
　　Используется как на клиенте (браузер), так и на сервере (Node.js).

---

### Правило 4: Несколько терминов — одно описание

Несколько терминов могут **разделять** одно описание:

```HTML

<dl>
  <dt>Frontend</dt>
  <dt>Фронтенд</dt>
  <dt>Клиентская часть</dt>
  <dd>Всё, что пользователь видит и с чем взаимодействует в браузере.</dd>
</dl>
```

**Результат:**

Frontend  
Фронтенд  
Клиентская часть  
　　Всё, что пользователь видит и с чем взаимодействует в браузере.

Это полезно для синонимов и альтернативных названий.

---

### Правило 5: Внутри `<dt>` и `<dd>` — разный контент

|Элемент|Что можно внутри|
|---|---|
|`<dt>`|Текст, строчные элементы (`<strong>`, `<em>`, `<a>`, `<abbr>`)|
|`<dd>`|Что угодно: текст, абзацы, списки, картинки, ссылки|

```HTML

<dl>
  <dt><strong>Git</strong></dt>
  <dd>
    <p>Система контроля версий, позволяющая отслеживать изменения в коде.</p>
    <p>Основные команды:</p>
    <ul>
      <li><code>git init</code> — инициализация репозитория</li>
      <li><code>git add</code> — добавление файлов</li>
      <li><code>git commit</code> — сохранение изменений</li>
    </ul>
  </dd>
</dl>
```

## 📋 Когда использовать `<dl>`

### Подходит `<dl>`

|Ситуация|Пример|
|---|---|
|**Глоссарий / Словарь**|Термин → определение|
|**FAQ**|Вопрос → ответ|
|**Метаданные**|Характеристика → значение|
|**Контактная информация**|Тип контакта → данные|
|**Характеристики товара**|Параметр → значение|
|**Ключ-значение**|Любые пары «название — описание»|

### Не подходит `<dl>`

|Ситуация|Что использовать|
|---|---|
|Простое перечисление|`<ul>`|
|Пошаговая инструкция|`<ol>`|
|Диалог / чат|`<p>` или другая разметка|
|Табличные данные|`<table>`|

> [!tip] Простое правило  
> Если данные можно описать как **«ключ: значение»** или **«термин: определение»** — это `<dl>`.

### 6. Оборачивание пар в `<div>`

Для удобства стилизации пары `<dt>` + `<dd>` можно оборачивать в `<div>`:

```HTML

<dl>
  <div>
    <dt>Имя</dt>
    <dd>Алексей</dd>
  </div>
  <div>
    <dt>Возраст</dt>
    <dd>28 лет</dd>
  </div>
  <div>
    <dt>Город</dt>
    <dd>Москва</dd>
  </div>
</dl>
```

Это позволяет легко стилизовать каждую пару как блок (например, `display: flex`):

```CSS

dl div {
  display: flex;
  gap: 10px;
  border-bottom: 1px solid #eee;
  padding: 8px 0;
}

dt {
  font-weight: bold;
  min-width: 150px;
}
```

> [!note] Стилизацию подробно разберём в модуле CSS. Пока важно знать, что `<div>` внутри `<dl>` допустим.

## ⚠️ Типичные ошибки

### Ошибка 1: `<dl>` для обычного перечисления

```HTML

<!-- ❌ Неправильно: нет пар «термин — описание» -->
<dl>
  <dt>Яблоки</dt>
  <dt>Бананы</dt>
  <dt>Апельсины</dt>
</dl>

<!-- ✅ Правильно: это обычный список -->
<ul>
  <li>Яблоки</li>
  <li>Бананы</li>
  <li>Апельсины</li>
</ul>
```

Если нет **описаний** — это не список определений.

---

### Ошибка 2: `<dd>` без `<dt>`

```HTML

<!-- ❌ Неправильно: описание без термина -->
<dl>
  <dd>Какое-то описание без термина.</dd>
</dl>

<!-- ✅ Правильно -->
<dl>
  <dt>Термин</dt>
  <dd>Описание термина.</dd>
</dl>
```

---

### Ошибка 3: `<dt>` и `<dd>` вне `<dl>`

```HTML

<!-- ❌ Неправильно: dt и dd без контейнера -->
<dt>Термин</dt>
<dd>Описание</dd>

<!-- ✅ Правильно: внутри <dl> -->
<dl>
  <dt>Термин</dt>
  <dd>Описание</dd>
</dl>
```

---

### Ошибка 4: Использование `<dl>` для макета страницы

```HTML

<!-- ❌ Неправильно: dl для двухколоночного макета -->
<dl>
  <dt>Логотип</dt>
  <dd>Навигация</dd>
</dl>

<!-- Для макетов используй семантические теги и CSS -->
```

`<dl>` — для **контента**, а не для вёрстки макета.

## 📊 Сравнение трёх типов списков

|Характеристика|`<ul>`|`<ol>`|`<dl>`|
|---|---|---|---|
|Назначение|Перечисление|Последовательность|Пары «термин — описание»|
|Порядок важен?|❌|✅|Зависит от контекста|
|Маркер/нумерация|● ○ ■|1. 2. 3.|Нет|
|Дочерние теги|`<li>`|`<li>`|`<dt>` + `<dd>`|
|Несколько описаний|—|—|✅ (несколько `<dd>`)|
|Несколько терминов|—|—|✅ (несколько `<dt>`)|
|Пример|Список покупок|Рецепт|Глоссарий|

## ❓ Частые вопросы

### «Можно ли вложить `<dl>` внутрь `<dd>`?»

Да, это допустимо. Внутри `<dd>` может быть любой контент, включая другой `<dl>`:

```HTML

<dl>
  <dt>Веб-разработка</dt>
  <dd>
    <dl>
      <dt>Frontend</dt>
      <dd>HTML, CSS, JavaScript</dd>

      <dt>Backend</dt>
      <dd>Python, PHP, Node.js</dd>
    </dl>
  </dd>
</dl>
```

Но не злоупотребляй — это ухудшает читаемость.

### «`<dl>` — блочный или строчный элемент?»

**Блочный.** `<dl>`, `<dt>` и `<dd>` — все блочные элементы.

### «Как стилизовать `<dl>` горизонтально (термин слева, описание справа)?»

Через CSS Grid или Flexbox:

```CSS

dl {
  display: grid;
  grid-template-columns: 200px 1fr;
  gap: 8px 16px;
}

dt {
  font-weight: bold;
}
```

Мы изучим это в модуле CSS.

### «Можно ли использовать `<dl>` для навигации?»

Нет. Навигация — это `<nav>` + `<ul>` + `<li>`. Список определений не предназначен для навигации.

### «Экранные читалки понимают `<dl>`?»

Да. Экранная читалка объявляет: «Список определений, 5 элементов». Для каждого `<dt>` скажет «термин», для `<dd>` — «определение». Это помогает слабовидящим пользователям понять структуру контента.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 
> 1. **`<dl>`** — контейнер списка определений (definition list)
> 2. **`<dt>`** — термин (definition term)
> 3. **`<dd>`** — описание термина (definition description)
> 4. У одного `<dt>` может быть **несколько `<dd>`** (несколько описаний)
> 5. Несколько `<dt>` могут иметь **одно `<dd>`** (синонимы)
> 6. Внутри `<dd>` может быть **любой контент**: абзацы, списки, ссылки
> 7. Используй `<dl>` для: **глоссариев, FAQ, характеристик, метаданных, контактов**
> 8. Если нет пар «ключ — значение» — это **не** список определений
