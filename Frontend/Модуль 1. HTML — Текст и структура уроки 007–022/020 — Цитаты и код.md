## 📖 Зачем нужны специальные теги?

Можно ли написать цитату просто текстом в `<p>`? Технически — да. Но тогда:
- Браузер не будет знать, что это цитата
- Экранная читалка не скажет «цитата»
- Поисковик не поймёт, что это чужой текст
- Другой разработчик не увидит намерение автора

То же самое с кодом: обычный текст и программный код — **разные вещи** семантически. Для каждого случая в HTML есть свой тег.

## 💬 Цитаты

### 📌 `<blockquote>` — блочная цитата

**`<blockquote>`** используется для **длинных цитат**, которые выделяются в **отдельный блок**.

```html
<blockquote>
  <p>Единственный способ делать великую работу — любить то, что делаешь. Если вы ещё не нашли своё дело — продолжайте искать. Не останавливайтесь.</p>
</blockquote>
```
#### Как выглядит по умолчанию

|Свойство|Значение|
|---|---|
|Тип элемента|Блочный|
|Отступы|Слева и справа (~40px margin)|
|Шрифт|Обычный (не курсив, не жирный)|
|Визуальное выделение|Только отступами|

#### Атрибут `cite`

Атрибут `cite` указывает **URL источника** цитаты. Он не отображается на странице, но доступен для поисковиков и программ.

```HTML

<blockquote cite="https://ru.wikipedia.org/wiki/Стив_Джобс">
  <p>Оставайтесь голодными. Оставайтесь безрассудными.</p>
</blockquote>
```

|Свойство атрибута `cite`|Описание|
|---|---|
|Виден пользователю?|❌ Нет|
|Виден поисковику?|✅ Да|
|Виден в DevTools?|✅ Да|
|Обязателен?|❌ Нет, но рекомендуется|

#### Цитата с указанием автора

HTML не имеет специального атрибута для **имени автора**. Используется тег `<cite>` или элемент `<footer>` внутри `<blockquote>`:

```HTML

<blockquote cite="https://ru.wikiquote.org/wiki/Альберт_Эйнштейн">
  <p>Воображение важнее знания. Знания ограничены, тогда как воображение охватывает целый мир.</p>
  <footer>— <cite>Альберт Эйнштейн</cite></footer>
</blockquote>
```

#### Несколько абзацев в цитате

```HTML

<blockquote>
  <p>Первый абзац цитаты. Длинный текст, который занимает несколько строк.</p>
  <p>Второй абзац цитаты. Продолжение мысли автора.</p>
  <p>Третий абзац. Завершение цитаты.</p>
  <footer>— <cite>Имя автора</cite>, «Название произведения»</footer>
</blockquote>
```

#### Когда использовать `<blockquote>`

|Подходит ✅|Не подходит ❌|
|---|---|
|Цитата из книги, статьи, речи|Выделение собственного текста (используй `<aside>` или `<p>`)|
|Отзыв клиента|Эпиграф, если он ваш собственный|
|Выдержка из документа|Просто визуальный отступ (используй CSS `margin`)|
|Цитирование другого автора|Пересказ своими словами (это не цитата)|

### 📌 `<q>` — строчная цитата

**`<q>`** (quote) используется для **коротких цитат** внутри текста. Браузер **автоматически** добавляет кавычки.

```HTML

<p>Как говорил Стив Джобс: <q>Оставайтесь голодными. Оставайтесь безрассудными.</q></p>
```

**Результат в браузере:**

Как говорил Стив Джобс: «Оставайтесь голодными. Оставайтесь безрассудными.»

#### Как выглядит по умолчанию

|Свойство|Значение|
|---|---|
|Тип элемента|Строчный (inline)|
|Кавычки|Автоматические (зависят от `lang`)|
|Шрифт|Обычный|

#### Кавычки зависят от языка

|Язык|Атрибут|Кавычки|
|---|---|---|
|Русский|`lang="ru"`|«...»|
|Английский|`lang="en"`|"..."|
|Немецкий|`lang="de"`|„..."|
|Французский|`lang="fr"`|«...» (с пробелами)|

Браузер подставляет кавычки **автоматически** на основе атрибута `lang` у `<html>` или у самого `<q>`.

#### Атрибут `cite`

Как и у `<blockquote>`, можно указать URL источника:

```HTML

<p>Аристотель утверждал, что <q cite="https://example.com/aristotle">целое больше суммы его частей</q>.</p>
```

#### Вложенные цитаты

Цитата внутри цитаты — браузер использует **другой тип кавычек**:

```HTML

<p>Учитель сказал: <q>Помните слова Пушкина: <q>Я помню чудное мгновенье</q> — это шедевр.</q></p>
```

**Результат (русский):** Учитель сказал: «Помните слова Пушкина: „Я помню чудное мгновенье" — это шедевр.»

---

### 📌 `<cite>` — название произведения

**`<cite>`** обозначает **название произведения**: книги, фильма, песни, картины, статьи, сайта.

```HTML

<p>Роман <cite>Война и мир</cite> написал Лев Толстой.</p>
<p>Фильм <cite>Начало</cite> снял Кристофер Нолан.</p>
<p>Песня <cite>Bohemian Rhapsody</cite> группы Queen — шедевр рок-музыки.</p>
```

#### Как выглядит по умолчанию

|Свойство|Значение|
|---|---|
|Тип элемента|Строчный (inline)|
|Шрифт|_Курсивный_|

#### `<cite>` — для названий, не для имён

|Правильно ✅|Неправильно ❌|
|---|---|
|`<cite>Война и мир</cite>` (название книги)|`<cite>Лев Толстой</cite>` (имя автора)|
|`<cite>Мона Лиза</cite>` (название картины)|`<cite>Леонардо да Винчи</cite>` (имя художника)|
|`<cite>MDN Web Docs</cite>` (название ресурса)|`<cite>Mozilla</cite>` (название компании)|

> [!note] Распространённая путаница  
> Многие разработчики используют `<cite>` для имён авторов. Спецификация HTML5 говорит, что `<cite>` — **только для названий произведений**. Для имён автора используй обычный текст или `<span>`.

---

### Разница между `<blockquote>`, `<q>` и `<cite>`

|Тег|Тип|Для чего|Пример|
|---|---|---|---|
|`<blockquote>`|Блочный|Длинная цитата (отдельный блок)|Абзац из книги|
|`<q>`|Строчный|Короткая цитата (внутри текста)|Фраза в предложении|
|`<cite>`|Строчный|Название произведения|Название книги, фильма|

## 💻 Код

### 📌 `<code>` — строчный код

**`<code>`** обозначает **фрагмент программного кода** внутри текста.

```HTML

<p>Для вывода текста в консоль используйте <code>console.log()</code>.</p>
<p>Тег <code>&lt;img&gt;</code> вставляет изображение на страницу.</p>
<p>В CSS свойство <code>display: flex</code> включает Flexbox.</p>
```

#### Как выглядит по умолчанию

|Свойство|Значение|
|---|---|
|Тип элемента|Строчный (inline)|
|Шрифт|Моноширинный (`monospace`)|
|Фон|Без фона (можно добавить через CSS)|

#### Стилизация `<code>` (распространённая)

Часто `<code>` стилизуют с фоном для лучшей видимости:

```CSS

code {
  background-color: #f4f4f4;
  padding: 2px 6px;
  border-radius: 4px;
  font-size: 0.9em;
}
```

### 📌 `<pre>` + `<code>` — блок кода

Мы уже знакомились с `<pre>` в уроке [[010 — Абзацы и переносы]]. Для отображения **многострочного программного кода** используется комбинация `<pre>` + `<code>`.

```HTML

<pre><code>function greet(name) {
  if (name) {
    console.log("Привет, " + name + "!");
  } else {
    console.log("Привет, мир!");
  }
}

greet("Алексей");</code></pre>
```

#### Зачем два тега?

|Тег|Роль|
|---|---|
|`<pre>`|Сохраняет **форматирование** (пробелы, отступы, переносы)|
|`<code>`|Семантически обозначает, что это **программный код**|

Использовать `<pre>` без `<code>` допустимо, но теряется семантика. Использовать `<code>` без `<pre>` — для коротких фрагментов кода внутри текста.

|Вариант|Когда использовать|
|---|---|
|`<code>` (без `<pre>`)|Короткий фрагмент кода **внутри текста**|
|`<pre><code>`|Многострочный **блок кода**|
|`<pre>` (без `<code>`)|Предварительно отформатированный текст, который **не является кодом**|

#### Важный нюанс: HTML-теги внутри `<code>`

Если ты хочешь показать **HTML-теги как текст** (а не как разметку), нужно заменить угловые скобки на **спецсимволы**:

|Символ|Спецсимвол|Пояснение|
|---|---|---|
|`<`|`&lt;`|Less than|
|`>`|`&gt;`|Greater than|

```HTML

<!-- ❌ Браузер интерпретирует <h1> как тег -->
<code><h1>Заголовок</h1></code>

<!-- ✅ Браузер покажет текст <h1>Заголовок</h1> -->
<code>&lt;h1&gt;Заголовок&lt;/h1&gt;</code>
```

Для многострочного HTML-кода:

```HTML

<pre><code>&lt;!DOCTYPE html&gt;
&lt;html lang="ru"&gt;
  &lt;head&gt;
    &lt;meta charset="UTF-8"&gt;
    &lt;title&gt;Пример&lt;/title&gt;
  &lt;/head&gt;
  &lt;body&gt;
    &lt;h1&gt;Привет!&lt;/h1&gt;
  &lt;/body&gt;
&lt;/html&gt;</code></pre>
```

> [!note] Подробнее о спецсимволах — в следующем уроке [[021 — Спецсимволы и комментарии]]

### 📌 `<kbd>` — клавиатурный ввод

**`<kbd>`** (keyboard) обозначает текст, который пользователь должен **ввести с клавиатуры** или **нажать на клавишу**.

```HTML

<p>Для сохранения файла нажмите <kbd>Ctrl</kbd> + <kbd>S</kbd>.</p>
<p>Чтобы открыть DevTools, нажмите <kbd>F12</kbd>.</p>
<p>Введите команду <kbd>git status</kbd> в терминале.</p>
```

#### Как выглядит по умолчанию

|Свойство|Значение|
|---|---|
|Тип элемента|Строчный|
|Шрифт|Моноширинный|

#### Стилизация `<kbd>` (распространённая)

Часто `<kbd>` стилизуют как «кнопку клавиатуры»:

```CSS

kbd {
  background-color: #eee;
  border: 1px solid #ccc;
  border-radius: 4px;
  padding: 2px 6px;
  font-size: 0.85em;
  box-shadow: 0 1px 0 #999;
}
```

#### Вложенный `<kbd>` — комбинация клавиш

Для комбинаций клавиш можно вкладывать `<kbd>` друг в друга:

```HTML

<p>Скопировать: <kbd><kbd>Ctrl</kbd> + <kbd>C</kbd></kbd></p>
<p>Вставить: <kbd><kbd>Ctrl</kbd> + <kbd>V</kbd></kbd></p>
```

Внешний `<kbd>` — комбинация, внутренние — отдельные клавиши.

---

### 📌 `<samp>` — вывод программы

**`<samp>`** (sample output) обозначает текст, который является **результатом работы программы** — вывод в консоль, сообщение об ошибке.

```HTML

<p>После выполнения команды вы увидите:</p>
<samp>Hello, World!</samp>

<p>Если файл не найден, система покажет:</p>
<samp>Error 404: File not found</samp>
```

#### Как выглядит по умолчанию

|Свойство|Значение|
|---|---|
|Тип элемента|Строчный|
|Шрифт|Моноширинный|

#### Блок вывода — `<pre>` + `<samp>`

```HTML

<p>Результат выполнения скрипта:</p>
<pre><samp>$ node app.js
Server started on port 3000
Connected to database
Ready to accept connections</samp></pre>
```

---

### 📌 `<var>` — переменная

**`<var>`** обозначает **переменную** — в математике, программировании или просто как заполнитель.

```HTML
<p>Формула площади прямоугольника: <var>S</var> = <var>a</var> × <var>b</var>.</p>
<p>Скопируйте файл <var>имя_файла</var> в папку <var>путь_назначения</var>.</p>
<p>В JavaScript: <code>let <var>x</var> = 10;</code></p>
```

#### Как выглядит по умолчанию

|Свойство|Значение|
|---|---|
|Тип элемента|Строчный|
|Шрифт|_Курсивный_|

## 🧪 Практика

Создай файл и набери вручную:

```HTML

<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Цитаты и код</title>
    <style>
      body { font-family: sans-serif; max-width: 800px; margin: 0 auto; padding: 20px; }
      code { background-color: #f4f4f4; padding: 2px 6px; border-radius: 4px; }
      pre { background-color: #282c34; color: #abb2bf; padding: 16px; border-radius: 8px; overflow-x: auto; }
      pre code { background: none; padding: 0; }
      kbd { background-color: #eee; border: 1px solid #ccc; border-radius: 4px; padding: 2px 6px; font-size: 0.85em; }
      blockquote { border-left: 4px solid #3498db; margin-left: 0; padding: 12px 20px; background-color: #f8f9fa; }
      blockquote footer { font-size: 0.9em; color: #666; margin-top: 8px; }
    </style>
  </head>
  <body>

    <h1>Цитаты и код в HTML</h1>

    <!-- Блочная цитата -->
    <h2>Блочная цитата (blockquote)</h2>
    <blockquote cite="https://ru.wikiquote.org/wiki/Стив_Джобс">
      <p>Ваше время ограничено, поэтому не тратьте его, проживая чужую жизнь. Не попадайтесь в ловушку догмы, которая говорит жить мыслями других людей.</p>
      <footer>— <cite>Стив Джобс</cite>, речь в Стэнфорде, 2005</footer>
    </blockquote>

    <blockquote>
      <p>Программирование — это не о том, что вы знаете. Это о том, что вы можете выяснить.</p>
      <footer>— <cite>Крис Пайн</cite></footer>
    </blockquote>

    <!-- Строчная цитата -->
    <h2>Строчная цитата (q)</h2>
    <p>Альберт Эйнштейн говорил: <q>Воображение важнее знания</q>.</p>
    <p>Как написано в документации: <q>Атрибут alt является обязательным для тега img</q>.</p>
    <p>Учитель сказал: <q>Помните слова Конфуция: <q>Выбери себе работу по душе, и тебе не придётся работать ни одного дня</q></q>.</p>

    <!-- Названия произведений -->
    <h2>Названия произведений (cite)</h2>
    <p>Роман <cite>Мастер и Маргарита</cite> — одно из величайших произведений русской литературы.</p>
    <p>В фильме <cite>Матрица</cite> поднимается вопрос о природе реальности.</p>
    <p>Справочник <cite>MDN Web Docs</cite> — лучший ресурс для веб-разработчика.</p>

    <hr>

    <!-- Строчный код -->
    <h2>Строчный код (code)</h2>
    <p>Для создания переменной используйте <code>let</code> или <code>const</code>.</p>
    <p>Тег <code>&lt;a href="..."&gt;</code> создаёт ссылку.</p>
    <p>Метод <code>document.querySelector()</code> находит элемент по CSS-селектору.</p>
    <p>CSS-свойство <code>display: flex</code> включает Flexbox-раскладку.</p>

    <!-- Блок кода -->
    <h2>Блок кода (pre + code)</h2>
    <p>Пример функции на JavaScript:</p>
    <pre><code>function fibonacci(n) {
  if (n &lt;= 1) return n;
  return fibonacci(n - 1) + fibonacci(n - 2);
}

console.log(fibonacci(10)); // 55</code></pre>

    <p>Пример HTML-разметки:</p>
    <pre><code>&lt;nav&gt;
  &lt;ul&gt;
    &lt;li&gt;&lt;a href="/"&gt;Главная&lt;/a&gt;&lt;/li&gt;
    &lt;li&gt;&lt;a href="/about"&gt;О нас&lt;/a&gt;&lt;/li&gt;
  &lt;/ul&gt;
&lt;/nav&gt;</code></pre>

    <!-- Клавиатурный ввод -->
    <h2>Клавиатурный ввод (kbd)</h2>
    <p>Сохранить файл: <kbd>Ctrl</kbd> + <kbd>S</kbd></p>
    <p>Отменить действие: <kbd>Ctrl</kbd> + <kbd>Z</kbd></p>
    <p>Открыть DevTools: <kbd>F12</kbd></p>
    <p>Копировать: <kbd>Ctrl</kbd> + <kbd>C</kbd>, вставить: <kbd>Ctrl</kbd> + <kbd>V</kbd></p>
    <p>Открыть терминал в VS Code: <kbd>Ctrl</kbd> + <kbd>`</kbd></p>

    <!-- Вывод программы -->
    <h2>Вывод программы (samp)</h2>
    <p>После запуска скрипта вы увидите: <samp>Hello, World!</samp></p>
    <p>Результат работы сервера:</p>
    <pre><samp>$ node server.js
Listening on port 3000
Database connected
Ready</samp></pre>

    <!-- Переменные -->
    <h2>Переменные (var)</h2>
    <p>Площадь круга: <var>S</var> = π × <var>r</var><sup>2</sup></p>
    <p>Скорость: <var>v</var> = <var>s</var> / <var>t</var></p>
    <p>Замените <var>username</var> на ваше имя пользователя.</p>

  </body>
</html>
```

### Что проверить в браузере

|Что проверить|Ожидаемый результат|
|---|---|
|`<blockquote>`|Цитаты с синей полоской слева (стили из CSS) и указанием автора|
|`<q>`|Автоматические кавычки «...»|
|Вложенные `<q>`|Вложенные кавычки „..."|
|`<cite>`|Названия произведений курсивом|
|`<code>` (строчный)|Моноширинный шрифт с серым фоном|
|`<pre><code>`|Блок кода с тёмным фоном и сохранёнными отступами|
|HTML в `<code>`|Теги отображаются как текст, не интерпретируются|
|`<kbd>`|Стилизация под клавиши клавиатуры|
|`<samp>`|Моноширинный текст|
|`<var>`|Курсивный текст для переменных|

## ⚠️ Типичные ошибки

### Ошибка 1: `<blockquote>` для визуального отступа

```HTML

<!-- ❌ Не цитата, просто хочу отступ -->
<blockquote>
  <p>Этот текст я написал сам, но хочу отступ слева.</p>
</blockquote>

<!-- ✅ Правильно: CSS для отступа -->
<p style="margin-left: 40px;">Этот текст с отступом через CSS.</p>
```

---

### Ошибка 2: Ручные кавычки вместо `<q>`

```HTML

<!-- ❌ Ручные кавычки — теряется семантика -->
<p>Как сказал Джобс: "Оставайтесь голодными".</p>

<!-- ✅ Автоматические кавычки через <q> -->
<p>Как сказал Джобс: <q>Оставайтесь голодными</q>.</p>
```

---

### Ошибка 3: `<cite>` для имени автора

```HTML

<!-- ❌ cite — для названий, не для имён -->
<blockquote>
  <p>Текст цитаты...</p>
  <cite>Стив Джобс</cite>
</blockquote>

<!-- ✅ Правильно -->
<blockquote>
  <p>Текст цитаты...</p>
  <footer>— <cite>Речь в Стэнфорде</cite>, Стив Джобс</footer>
</blockquote>
```

---

### Ошибка 4: HTML-теги в `<code>` без экранирования

```HTML

<!-- ❌ Браузер интерпретирует <p> как тег -->
<code><p>Текст</p></code>

<!-- ✅ Экранировано через спецсимволы -->
<code>&lt;p&gt;Текст&lt;/p&gt;</code>
```

---

### Ошибка 5: `<pre><code>` с лишними пробелами

```HTML

<!-- ❌ Лишний перенос и пробелы в начале — будут видны в браузере -->
<pre><code>
    function hello() {
      console.log("Hi");
    }
</code></pre>

<!-- ✅ Код начинается сразу после <code> -->
<pre><code>function hello() {
  console.log("Hi");
}</code></pre>
```

Внутри `<pre>` **каждый пробел и перенос** имеет значение. Не добавляй лишние.

## ❓ Частые вопросы

### «Чем `<code>` отличается от `<pre>`?»

|Тег|Назначение|Сохраняет пробелы?|Тип|
|---|---|---|---|
|`<code>`|Семантика: «это код»|❌|Строчный|
|`<pre>`|Формат: «сохрани пробелы»|✅|Блочный|

Для блока кода нужны **оба**: `<pre>` для формата + `<code>` для семантики.

### «Чем `<kbd>` отличается от `<code>`?»

|Тег|Для чего|Пример|
|---|---|---|
|`<code>`|Программный код|`<code>console.log()</code>`|
|`<kbd>`|Что пользователь **вводит с клавиатуры**|`<kbd>Ctrl</kbd> + <kbd>S</kbd>`|

### «Как добавить подсветку синтаксиса в блок кода?»

HTML сам по себе не умеет подсвечивать синтаксис. Для этого используются JavaScript-библиотеки:

|Библиотека|URL|
|---|---|
|**Prism.js**|prismjs.com|
|**Highlight.js**|highlightjs.org|

Мы не будем изучать их в этом курсе, но знай, что они существуют.

### «Нужно ли экранировать CSS и JS код внутри `<pre><code>`?»

Только если код содержит символы `<` или `>` или `&`. В CSS и JS они встречаются редко, поэтому обычно экранирование не требуется. В HTML-коде — **всегда** нужно экранировать.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 
> 1. **`<blockquote>`** — блочная цитата (длинная, отдельным блоком)
> 2. **`<q>`** — строчная цитата (короткая, внутри текста, с автоматическими кавычками)
> 3. **`<cite>`** — название произведения (книга, фильм, сайт), **не** имя автора
> 4. **`<code>`** — фрагмент кода внутри текста (строчный, моноширинный)
> 5. **`<pre><code>`** — многострочный блок кода (сохраняет форматирование)
> 6. **`<kbd>`** — клавиатурный ввод (клавиши, команды)
> 7. **`<samp>`** — вывод программы (результат выполнения)
> 8. **`<var>`** — переменная (математика, программирование)
> 9. Для HTML-тегов в `<code>` заменяй `<` на **`&lt;`** и `>` на **`&gt;`**
