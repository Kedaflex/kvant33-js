## 📖 Тег `<p>` — абзац

**`<p>`** (paragraph) — самый используемый тег для текста. Каждый блок текста, который является отдельной мыслью, оборачивается в `<p>`.

```html
<p>Это первый абзац. Он содержит одну или несколько мыслей.</p>
<p>Это второй абзац. Он начинается с новой строки и отделён отступом.</p>
<p>Это третий абзац.</p>
```
### Как браузер отображает `<p>`

|Свойство|Поведение|
|---|---|
|Начинается с **новой строки**|Каждый `<p>` — отдельный блок|
|Занимает **всю ширину**|Растягивается от края до края|
|**Отступы сверху и снизу**|Браузер автоматически добавляет вертикальные отступы (~16px)|
|Текст внутри идёт **в одну строку**|Переносится на новую строку только когда не помещается|

---

### Правила использования `<p>`

#### Что можно помещать внутрь `<p>`

|Можно ✅|Пример|
|---|---|
|Обычный текст|`<p>Просто текст.</p>`|
|Строчные теги|`<p>Текст <strong>жирный</strong> и <em>курсив</em>.</p>`|
|Ссылки|`<p>Перейти на <a href="#">сайт</a>.</p>`|
|Картинки|`<p><img src="icon.png" alt="Иконка"> Подпись.</p>`|
|`<br>`|`<p>Строка 1<br>Строка 2</p>`|
|`<span>`|`<p>Текст <span class="red">красный</span>.</p>`|

#### Что нельзя помещать внутрь `<p>`

|Нельзя ❌|Почему|
|---|---|
|Другой `<p>`|`<p>` не может содержать блочные элементы|
|`<div>`|Блочный элемент внутри `<p>`|
|`<h1>` – `<h6>`|Заголовки — блочные элементы|
|`<ul>`, `<ol>`|Списки — блочные элементы|
|`<table>`|Таблица — блочный элемент|

```HTML
<!-- ❌ Неправильно: div внутри p -->
<p>Текст <div>блок</div> ещё текст</p>

<!-- ✅ Правильно -->
<p>Текст перед блоком.</p>
<div>Блок</div>
<p>Текст после блока.</p>
```

> [!warning] Что сделает браузер с неправильным кодом?  
> Если поместить блочный элемент внутрь `<p>`, браузер **принудительно закроет** `<p>` перед блочным элементом. Результат будет непредсказуемым. Проверь в DevTools — увидишь, что браузер «переписал» твой код.

---

### Пустой абзац

```HTML
<!-- Пустой абзац — бессмысленно -->
<p></p>

<!-- Абзац с пробелами — тоже бессмысленно -->
<p>    </p>
```

Пустые абзацы не имеют смысла. Браузер их отобразит, но это **плохая практика**. Если нужен отступ — используй CSS (свойство `margin`).

---

### Абзац с одним предложением vs абзац с несколькими

Оба варианта допустимы. Зависит от контента:

```HTML
<!-- Короткие абзацы (блог, новости) -->
<p>Компания Apple представила новый iPhone.</p>
<p>Устройство получило улучшенную камеру.</p>
<p>Продажи начнутся в сентябре.</p>

<!-- Длинный абзац (статья, учебник) -->
<p>
  Компания Apple представила новый iPhone. Устройство получило
  улучшенную камеру с разрешением 48 мегапикселей. Также был
  обновлён процессор, который стал на 20% быстрее предыдущего
  поколения. Продажи начнутся в сентябре этого года.
</p>
```

> [!tip] Совет  
> Не делай абзацы слишком длинными. Стена текста без разбивки на абзацы **отпугивает** читателя. Оптимальный абзац — 3–5 предложений.

## 🔄 Как браузер обрабатывает пробелы и переносы

Это мы затрагивали в уроке [[007 — Теги и элементы]], но сейчас разберём подробнее, потому что это **напрямую связано** с абзацами и переносами.

### Правило схлопывания пробелов

Браузер преобразует **любое количество** пробелов, табуляций и переносов строк в **один пробел**.

|Что написано в коде|Что покажет браузер|
|---|---|
|`Слово слово`|Слово слово|
|`Слово[Tab]слово`|Слово слово|
|`Слово[Enter]слово`|Слово слово|
|`Слово[Enter][Enter][Enter]слово`|Слово слово|
|`Слово`|Слово|

### Наглядный пример

```HTML
<p>Этот     текст
имеет      много
    пробелов    и

переносов
    строк.</p>
```

**Результат в браузере:**

Этот текст имеет много пробелов и переносов строк.

Всё слилось в одну строку. Все пробелы, табуляции и переносы **схлопнулись**.

### Почему так?

HTML был создан для **разметки текста**, а не для его форматирования. Визуальное оформление (отступы, переносы) — задача CSS. HTML просто определяет, что это абзац, а как он выглядит — решает CSS.

Это также позволяет разработчику **красиво форматировать код** (с отступами), не влияя на отображение в браузере.

---

## ↩️ Тег `<br>` — перенос строки

**`<br>`** (line break) — принудительный перенос строки. Текст после `<br>` начинается с новой строки, но **без отступа** — это не новый абзац.

```HTML
<p>
  Розы красные,<br>
  Фиалки синие,<br>
  Сахар сладкий,<br>
  И ты тоже.
</p>
```

**Результат:**

Розы красные,  
Фиалки синие,  
Сахар сладкий,  
И ты тоже.

---

### Разница между `<br>` и новым `<p>`

|Характеристика|`<br>`|Новый `<p>`|
|---|---|---|
|Визуально|Текст с новой строки, **без отступа**|Текст с новой строки, **с отступом** сверху и снизу|
|Семантически|Часть **того же** абзаца|**Новый** абзац (новая мысль)|
|Когда использовать|Стихи, адреса, строки внутри одного блока|Разные мысли, разные смысловые блоки|

```HTML
<!-- <br> — строки в одном блоке -->
<p>
  ООО «Ромашка»<br>
  ул. Пушкина, д. 10<br>
  Москва, 123456<br>
  Россия
</p>

<!-- <p> — разные мысли -->
<p>HTML — это язык разметки.</p>
<p>CSS — это язык стилей.</p>
<p>JavaScript — это язык программирования.</p>
```

---

### Когда использовать `<br>`

|Уместно ✅|Пример|
|---|---|
|Стихи и песни|Каждая строка стихотворения|
|Почтовые адреса|Улица, город, страна|
|Короткие строки, связанные по смыслу|Подпись в письме|

|Неуместно ❌|Что делать вместо этого|
|---|---|
|Создание отступа между блоками текста|Используй разные `<p>`|
|Вертикальный пробел|Используй CSS `margin`|
|Разделение несвязанных мыслей|Используй новый `<p>`|

---

### Типичная ошибка: несколько `<br>` подряд

```HTML
<!-- ❌ Неправильно: br вместо отступов -->
<p>Первый абзац.</p>
<br>
<br>
<br>
<p>Второй абзац с огромным отступом.</p>
```

Это **плохая практика**. Для отступов используй CSS:

```CSS
/* Правильный способ — задать отступ через CSS */
p {
    margin-bottom: 40px;
}
```

> [!warning] Золотое правило  
> Если ты используешь `<br>` больше двух раз подряд — скорее всего, ты делаешь что-то не так. Скорее всего, тебе нужен CSS `margin` или новый `<p>`.

---

### Особенности тега `<br>`

|Свойство|Описание|
|---|---|
|Одиночный тег|Нет закрывающего тега|
|Запись|`<br>` (HTML5) или `<br />` (XHTML)|
|Блочный или строчный?|Строчный (inline)|
|Можно ли стилизовать?|Практически нет (нет содержимого)|

## ➖ Тег `<hr>` — тематический разделитель

**`<hr>`** (horizontal rule) — горизонтальная линия, которая разделяет секции контента. Это **тематический разделитель**, а не просто декоративная линия.

```HTML
<h2>Глава 1</h2>
<p>Содержимое первой главы...</p>

<hr>

<h2>Глава 2</h2>
<p>Содержимое второй главы...</p>
```

---

### Как выглядит `<hr>` по умолчанию

Браузер отображает `<hr>` как тонкую горизонтальную линию с отступами сверху и снизу.

|Свойство|Значение по умолчанию|
|---|---|
|Ширина|100% (вся ширина контейнера)|
|Цвет|Серый|
|Стиль|Вдавленная линия (inset border)|
|Отступы|~8px сверху и снизу|

---

### Когда использовать `<hr>`

|Уместно ✅|Пример|
|---|---|
|Смена темы|Между двумя разными темами на странице|
|Разделение глав|Между главами рассказа|
|Перед подвалом|Отделение основного контента от подвала|
|Между группами контента|Разделение блоков, не связанных по смыслу|

|Неуместно ❌|Что делать вместо этого|
|---|---|
|Чисто декоративная линия|CSS `border-bottom`|
|Между каждым абзацем|Убрать, абзацы и так разделены отступами|
|Внутри абзаца|`<hr>` — блочный элемент, ему не место внутри `<p>`|

---

### Семантика `<hr>`

В HTML5 тег `<hr>` имеет **семантическое значение** — это не просто «горизонтальная линия», а **смена темы** или **разрыв сюжета**. В книгах это обозначается символами `* * *` или пустым пространством между сценами.

```HTML
<!-- Рассказ со сменой сцены -->
<p>Герой вошёл в тёмную комнату. Дверь за ним захлопнулась.</p>

<hr>

<p>Прошло три часа. Солнце уже встало над городом.</p>
```

---

### Особенности тега `<hr>`

|Свойство|Описание|
|---|---|
|Одиночный тег|Нет закрывающего тега|
|Запись|`<hr>` (HTML5)|
|Блочный элемент|Занимает всю ширину, начинается с новой строки|
|Стилизация|Можно менять через CSS (border, color, width, height)|

---

## 📜 Тег `<pre>` — предварительно отформатированный текст

**`<pre>`** (preformatted) — тег, который **сохраняет** все пробелы, табуляции и переносы строк в коде. Браузер отображает текст **точно так**, как он написан.

```HTML
<pre>
Это    текст    с    пробелами.
    Этот текст с отступом.
        А этот — с двойным отступом.

Между этими строками — пустая строка.
</pre>
```

**Результат в браузере:**

```text
Это    текст    с    пробелами.
    Этот текст с отступом.
        А этот — с двойным отступом.

Между этими строками — пустая строка.
```

Все пробелы и переносы **сохранены**!

---

### Как `<pre>` отличается от `<p>`

|Свойство|`<p>`|`<pre>`|
|---|---|---|
|Пробелы|Схлопываются в один|**Сохраняются** все|
|Переносы строк|Игнорируются|**Сохраняются**|
|Шрифт|Пропорциональный (обычный)|**Моноширинный** (как в консоли)|
|Перенос длинных строк|Автоматический|Не переносятся (горизонтальный скролл)|

---

### Когда использовать `<pre>`

|Уместно ✅|Пример|
|---|---|
|Код программы|Фрагменты кода с отступами|
|ASCII-арт|Рисунки из символов|
|Табличные данные без `<table>`|Данные, выровненные пробелами|
|Стихи с особым форматированием|Когда `<br>` недостаточно|

```HTML
<!-- Пример: код программы -->
<pre>
function greet(name) {
    if (name) {
        console.log("Привет, " + name + "!");
    } else {
        console.log("Привет, мир!");
    }
}
</pre>
```

---

### `<pre>` + `<code>` — лучший способ показать код

Для отображения программного кода принято комбинировать `<pre>` и `<code>`:

```HTML
<pre><code>
let x = 10;
let y = 20;
console.log(x + y); // 30
</code></pre>
```

|Тег|Роль|
|---|---|
|`<pre>`|Сохраняет форматирование (пробелы, отступы)|
|`<code>`|Семантически обозначает, что это **программный код**|

> [!note] Подробнее о `<code>` — в уроке [[020 — Цитаты и код]]

## 🔹 Неразрывный пробел `&nbsp;`

Иногда нужно вставить **дополнительный пробел**, который браузер не схлопнет. Для этого используется **спецсимвол** `&nbsp;` (non-breaking space).

```HTML
<p>Обычные    пробелы    схлопываются.</p>
<p>А&nbsp;&nbsp;&nbsp;&nbsp;эти&nbsp;&nbsp;&nbsp;&nbsp;нет.</p>
```

**Результат:**

|Строка|Отображение|
|---|---|
|Обычные пробелы схлопываются.|Обычные пробелы схлопываются.|
|А&nbsp;&nbsp;&nbsp;&nbsp;эти&nbsp;&nbsp;&nbsp;&nbsp;нет.|А эти нет.|

---

### Что значит «неразрывный»?

`&nbsp;` — это пробел, на котором браузер **не разрывает** строку. Слова, соединённые `&nbsp;`, всегда будут на **одной строке**.

```HTML
<!-- Обычный пробел — браузер может разорвать строку -->
<p>10 кг</p>

<!-- Неразрывный пробел — "10" и "кг" всегда вместе -->
<p>10&nbsp;кг</p>
```

---

### Когда использовать `&nbsp;`

|Уместно ✅|Пример|
|---|---|
|Число + единица измерения|`10&nbsp;кг`, `100&nbsp;м`|
|Инициалы + фамилия|`А.&nbsp;С.&nbsp;Пушкин`|
|Короткие предлоги в конце строки|`в&nbsp;Москве` (чтобы «в» не осталось одно)|

|Неуместно ❌|Что делать вместо этого|
|---|---|
|Создание отступов|CSS `margin`, `padding` или `text-indent`|
|Выравнивание элементов|CSS Flexbox или Grid|
|Множество пробелов подряд|`<pre>` или CSS `white-space`|

> [!warning] Не злоупотребляй `&nbsp;`  
> Использование `&nbsp;` для отступов и выравнивания — признак плохого кода. Для этого существует CSS.

> [!note] Подробнее о спецсимволах — в уроке [[021 — Спецсимволы и комментарии]]

## 🧪 Практика: страница со всеми изученными тегами

Создай файл и набери вручную:

```HTML
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Абзацы и переносы</title>
  </head>
  <body>

    <h1>Изучаем абзацы и переносы</h1>

    <!-- Обычные абзацы -->
    <h2>Абзацы</h2>
    <p>Это первый абзац. Он отделён от следующего вертикальным отступом.</p>
    <p>Это второй абзац. Каждый абзац — отдельная мысль.</p>
    <p>Это третий абзац. Обрати внимание на автоматические отступы между ними.</p>

    <!-- Перенос строки -->
    <h2>Перенос строки</h2>
    <p>
      Александр Сергеевич Пушкин<br>
      «Я помню чудное мгновенье»<br>
      1825 год
    </p>

    <p>
      Я помню чудное мгновенье:<br>
      Передо мной явилась ты,<br>
      Как мимолетное виденье,<br>
      Как гений чистой красоты.
    </p>

    <!-- Адрес с <br> -->
    <h2>Адрес компании</h2>
    <p>
      ООО «Веб-студия»<br>
      ул. Программистов, д. 42<br>
      Москва, 101000<br>
      Телефон: +7&nbsp;(900)&nbsp;123-45-67
    </p>

    <hr>

    <!-- Разделитель тем -->
    <h2>Тематический разделитель</h2>
    <p>Текст до разделителя. Он относится к одной теме.</p>
    <hr>
    <p>Текст после разделителя. Это уже другая тема.</p>

    <!-- Предварительно отформатированный текст -->
    <h2>Код программы</h2>
    <pre><code>function sum(a, b) {
    return a + b;
}

let result = sum(3, 5);
console.log(result); // 8</code></pre>

    <!-- Демонстрация схлопывания пробелов -->
    <h2>Эксперимент с пробелами</h2>
    <p>Этот      текст      имеет      много      пробелов,      но      они      схлопнутся.</p>

    <pre>А   этот   текст   сохранит   все   пробелы.</pre>

    <!-- Неразрывный пробел -->
    <h2>Неразрывный пробел</h2>
    <p>Масса Земли — 5,97&nbsp;×&nbsp;10²⁴&nbsp;кг.</p>
    <p>Автор: А.&nbsp;С.&nbsp;Пушкин.</p>

  </body>
</html>
```

### Что проверить в браузере

|Что проверить|Ожидаемый результат|
|---|---|
|Абзацы|Три абзаца с вертикальными отступами между ними|
|Стихотворение|Каждая строка — с новой строки, но без отступов между ними|
|Адрес|Каждая строка адреса — на отдельной строке|
|`<hr>`|Горизонтальная линия между блоками|
|Код в `<pre>`|Моноширинный шрифт, отступы сохранены|
|Пробелы в `<p>`|Схлопнулись в один|
|Пробелы в `<pre>`|Сохранились все|
|Неразрывный пробел|Число и единица измерения не разрываются|

## ❓ Частые вопросы

### «Как сделать отступ в первой строке абзаца (красная строка)?»

Через CSS-свойство `text-indent`:

```CSS
p {
    text-indent: 30px;
}
```

**Не используй** `&nbsp;&nbsp;&nbsp;` для этого.

### «Как сделать большой отступ между абзацами?»

Через CSS-свойство `margin`:

```CSS
p {
    margin-bottom: 30px;
}
```

**Не используй** несколько `<br>` для этого.

### «Можно ли использовать `<p>` без текста, просто для отступа?»

Технически можно, но это **бессмысленно и неправильно**. Пустой `<p></p>` не несёт семантической нагрузки. Используй CSS `margin` для отступов.

### «`<br>` или `<br />`?»

В HTML5 правильно `<br>`. Вариант `<br />` — наследие XHTML. Оба работают, но `<br>` — современный стандарт.

### «Можно ли вложить `<pre>` в `<p>`?»

Нет. `<pre>` — блочный элемент, `<p>` не может содержать блочные элементы. Используй `<pre>` отдельно.

### «Почему `<pre>` показывает моноширинный шрифт?»

По умолчанию браузер задаёт для `<pre>` шрифт `monospace` (все символы одинаковой ширины). Это удобно для кода и таблиц. Шрифт можно изменить через CSS.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 
> 1. **`<p>`** — абзац текста. Каждая мысль — отдельный `<p>`
> 2. Браузер **схлопывает** лишние пробелы и переносы строк в один пробел
> 3. **`<br>`** — принудительный перенос строки. Уместен в стихах и адресах
> 4. **Не используй** несколько `<br>` для создания отступов — для этого есть CSS
> 5. **`<hr>`** — тематический разделитель (смена темы), не просто линия
> 6. **`<pre>`** — сохраняет все пробелы и переносы, моноширинный шрифт
> 7. **`&nbsp;`** — неразрывный пробел, не схлопывается и не разрывает строку
