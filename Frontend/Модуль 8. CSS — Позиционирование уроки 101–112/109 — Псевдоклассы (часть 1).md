## 📖 Введение: Элементы с характером

До сих пор мы писали CSS-правила, которые применялись к элементам один раз и навсегда. Кнопка была синей, ссылка — подчеркнутой. Но веб-интерфейс — это живая среда. Пользователь водит мышкой, нажимает кнопки, переключается между полями ввода.

Как сказать браузеру: «Покрась кнопку в темно-синий, когда пользователь наведет на неё курсор»? Или «Сделай рамку поля ввода красной, если в него сейчас печатают текст»?

В HTML нет специальных тегов для этих ситуаций. Вместо этого в CSS есть **псевдоклассы**. Это специальные ключевые слова, которые добавляются к селектору и описывают **состояние** элемента. Они пишутся через двоеточие, например: `a:hover`.

---

## 🖐️ Динамические псевдоклассы

Эти псевдоклассы срабатывают в ответ на действия пользователя.

### 1. :hover (Наведение)
Самый популярный псевдокласс. Срабатывает, когда курсор мыши находится над элементом.
- **Где использовать**: Кнопки, ссылки, карточки товаров, пункты меню.
```css
.button {
  background-color: blue;
  transition: 0.3s; /* Плавный переход */
}
.button:hover {
  background-color: darkblue;
}
```

### 2. :active (Нажатие)
Срабатывает в момент клика — от нажатия кнопки мыши до момента, когда вы её отпустите.
- **Эффект**: Создает ощущение физического отклика («кнопка нажалась»).
```css
.button:active {
  transform: scale(0.95); /* Эффект уменьшения при нажатии */
}
```

### 3. :focus (Фокус)
Срабатывает, когда элемент «готов к вводу». Это происходит при клике в текстовое поле или при переходе по элементам с помощью клавиши **Tab**.
- **Важно**: Это критически важно для доступности!
```css
input:focus {
  outline: 2px solid orange;
  background-color: #fffef0;
}
```

## 🔗 Состояния ссылок (LVHA)

У ссылок есть свои уникальные псевдоклассы, которые браузер использует для управления историей посещений.

- **:link** — обычная, еще не посещенная ссылка.
- **:visited** — ссылка, по которой пользователь уже кликал.
- **:any-link** — любая ссылка (и новая, и старая).

> [!danger] Правило порядка (LVHA)
> Если вы стилизуете все состояния ссылок, вы **обязаны** писать их в строгом порядке, иначе они будут перебивать друг друга:
> 1. **L**ink
> 2. **V**isited
> 3. **H**over
> 4. **A**ctive
> *(Запомните фразу: **L**o**V**e **H****A**te — Любовь и Ненависть)*.

---

## 🔍 Тонкости использования

1. **Не только для ссылок**: Псевдокласс `:hover` можно применять к любому элементу: `div`, `li`, `p` и даже `body`. Раньше (в IE6) он работал только для ссылок, но те времена давно прошли.
2. **Мобильные устройства**: На смартфонах нет «наведения» (курсора). Поэтому `:hover` там срабатывает только в момент касания, что иногда выглядит странно (кнопка залипает в состоянии ховера). Профессионалы используют специальные медиа-запросы, чтобы отключать ховер на тач-скринах.
3. **Безопасность :visited**: Из соображений приватности (чтобы сайты не могли шпионить, на каких порносайтах или банковских сайтах вы были), возможности стилизации `:visited` сильно ограничены. Вы можете поменять только цвет текста, фон и цвет рамки. Никаких изменений размеров или теней.

## 🧪 Практика: Интерактивная кнопка

Давайте создадим кнопку, которая ведет себя как настоящая.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <style>
    .btn {
      display: inline-block;
      padding: 15px 30px;
      font-size: 18px;
      color: white;
      background-color: #3498db;
      border: none;
      border-radius: 8px;
      cursor: pointer;
      text-decoration: none;
      transition: all 0.2s ease; /* Плавность для всех изменений */
      outline: none;
    }

    /* Состояние: Наведение */
    .btn:hover {
      background-color: #2980b9;
      box-shadow: 0 5px 15px rgba(0,0,0,0.2);
    }

    /* Состояние: Фокус (Tab) */
    .btn:focus {
      box-shadow: 0 0 0 4px rgba(52, 152, 219, 0.4);
    }

    /* Состояние: Клик */
    .btn:active {
      background-color: #1f6391;
      transform: translateY(2px); /* Смещение вниз */
      box-shadow: 0 2px 5px rgba(0,0,0,0.2);
    }
  </style>
</head>
<body>

  <a href="#" class="btn">Нажми на меня</a>
  <button class="btn">Я тоже кнопка</button>

</body>
</html>
```

## ❓ Часто задаваемые вопросы

### «В чем разница между `:focus` и `:active`?»
- `:focus` — это когда элемент «выбран» (вы кликнули в поле и оно ждет ввода). Состояние длится, пока вы не кликните в другое место.
- `:active` — это очень короткий момент, пока кнопка мыши зажата физически.

### «Можно ли изменить стиль родителя при наведении на ребенка?»
В современном CSS появился селектор `:has()`, который это позволяет (`.parent:has(.child:hover)`). Но это новая технология, раньше это было невозможно без JavaScript.

### «Зачем нужно :link, если можно просто написать селектор `a`?»
Селектор `a` выберет все ссылки, включая те, у которых нет атрибута `href` (якоря). Псевдокласс `:link` выберет только настоящие работающие ссылки.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **Псевдоклассы** описывают состояние элемента (что с ним происходит сейчас).
> 2. `:hover` — мышь сверху, `:active` — кнопка нажата, `:focus` — элемент активен для ввода.
> 3. Порядок для ссылок важен: **Link → Visited → Hover → Active**.
> 4. Псевдоклассы делают интерфейс понятным и «отзывчивым».
> 5. Никогда не убирайте индикацию `:focus` (доступность!).

## 🏁 Итог

Мы научили элементы реагировать на пользователя. Это основа пользовательского опыта (UX). Теперь ваши кнопки — это не просто картинки, а интерактивные объекты.

Но псевдоклассы умеют не только следить за мышкой. Они могут выбирать элементы по их «номеру в очереди» или по тому, являются ли они единственными детьми.
