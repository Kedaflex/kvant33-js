# ⛵ Обтекание: Свойства float и clear

> - Понять принцип работы свойства `float`.
> - Научиться решать проблему «схлопывания» родителя с помощью `clear` и «clearfix».

## 📖 Введение: Газетная верстка

Вы наверняка видели в газетах или журналах, как картинка стоит сбоку, а текст аккуратно обтекает её со всех сторон. В ранние годы интернета это было практически невозможно сделать в вебе — всё шло строго друг под другом.

Для решения этой задачи в CSS было введено свойство `float` (плавать/плыть). Оно позволяло «прижать» элемент к левому или правому краю контейнера, заставляя весь остальной контент течь вокруг него.

Хотя сегодня для создания сложных сеток (колонок сайта) используют более современные Flexbox и Grid, свойство `float` всё еще незаменимо для своей прямой задачи — обтекания текстом изображений.

---

## 🏗️ Как работает float?

Когда вы задаете элементу `float: left` или `float: right`, происходят две важные вещи:
1. Элемент прижимается к указанному краю родителя.
2. Элемент **частично вырывается** из нормального потока.

Что значит «частично»? Блочные элементы за ним перестают его «видеть» и встают так, будто его нет (занимают его место). Но **контент** (текст, строки) внутри этих блоков видит препятствие и обтекает его.

```css
.image-aside {
  float: left;
  margin-right: 20px;
  margin-bottom: 10px;
}
```

### Доступные значения:
- `left` — прижать влево.
- `right` — прижать вправо.
- `none` — (по умолчанию) элемент остается в обычном месте.

---

## 🧹 Проблема «схлопывания» родителя

Это самый коварный эффект `float`. Поскольку «плавающий» элемент вырван из потока, его родитель (контейнер) перестает учитывать его высоту. Если внутри родителя **только** элементы с `float`, высота родителя станет равной **нулю**.

**Результат**: Фон родителя исчезнет, рамка схлопнется, а следующие за ним элементы наедут на ваши «плавающие» блоки. Это выглядит как катастрофа.

---

## 🛡️ Свойство clear (Очистка)

Чтобы остановить обтекание и сказать следующему элементу: «Стой! Не лезь наверх, подожди, пока закончатся все плавающие блоки», используется свойство `clear`.

### Значения clear:
- `left` — не обтекать элементы с `float: left`.
- `right` — не обтекать элементы с `float: right`.
- `both` — (самое частое) очистить обе стороны.

```css
.footer {
  clear: both; /* Футер всегда будет под всеми колонками, даже если они float */
}
```

## 🧙 Магия Clearfix

А что если нам нужно, чтобы сам родитель «увидел» свои плавающие элементы и растянулся? Раньше в конец родителя добавляли пустой `<div style="clear:both"></div>`, но это «грязный» HTML.

Профессиональное решение — использовать CSS-псевдоэлемент `::after`. Этот прием называют **Clearfix**.

```css
.container::after {
  content: "";       /* Пустое содержимое */
  display: table;    /* Или block */
  clear: both;       /* Очистка */
}
```
Теперь любой блок с классом `.container` будет автоматически растягиваться, подстраиваясь под высоту своих «плавающих» детей.

---

## 🔍 Важные нюансы float

1. **Тип отображения**: Элемент с `float` автоматически становится блочным (`display: block`). Даже если вы примените его к `<span>`, ему можно будет задать ширину и высоту.
2. **Ширина по контенту**: Если вы не зададите ширину элементу с `float`, он сожмется до размера своего содержимого.
3. **Порядок**: Если места по ширине не хватает, плавающий элемент упадет на следующую строку.
4. **Центрирование**: `margin: auto` **не работает** для элементов с `float`.

## ❓ Часто задаваемые вопросы

### «Стоит ли использовать float для создания колонок сайта?»
В 2024 году — **нет**. Для этого есть Flexbox и Grid. Верстка на флоатах сложна в поддержке, требует постоянной «очистки» и плохо подходит для адаптивности. Используйте `float` только для того, для чего он был создан — для обтекания контента.

### «Почему мой текст прилип к картинке с float?»
Потому что `float` не создает отступов автоматически. Вам нужно вручную добавить `margin-right` (если картинка слева) или `margin-left` (если справа).

### «Влияет ли float на z-index?»
Плавающие элементы имеют чуть более высокий приоритет отрисовки, чем обычные блоки в потоке, но они всё равно подчиняются правилам наложения, которые мы изучим позже.

## 🧪 Практика: Статья с обтеканием

Создадим классический газетный макет для одного абзаца.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <style>
    .article {
      max-width: 600px;
      margin: 50px auto;
      border: 1px solid #ccc;
      padding: 20px;
      /* Применяем clearfix через современный способ */
      display: flow-root;
    }

    .photo {
      float: left;
      width: 200px;
      margin-right: 20px;
      margin-bottom: 10px;
      border: 5px solid #eee;
    }

    .photo-caption {
      font-size: 12px;
      color: #666;
      text-align: center;
      display: block;
    }

    p {
      line-height: 1.6;
      text-align: justify;
    }
  </style>
</head>
<body>

  <div class="article">
    <div class="photo">
      <img src="https://via.placeholder.com/200x150" alt="Природа" style="width: 100%;">
      <span class="photo-caption">Подпись к фото</span>
    </div>
    <p>
      Этот текст будет аккуратно обтекать изображение слева.
      Свойство float позволяет нам создавать макеты, похожие на печатные издания.
      Обратите внимание на отступ справа у картинки — без него буквы бы прилипали к рамке.
      Даже если текста будет очень много, он заполнит всё свободное пространство справа,
      а когда картинка закончится — текст продолжит идти на всю ширину контейнера.
    </p>
  </div>

</body>
</html>
```

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **float** создавался для обтекания картинок текстом.
> 2. Плавающий элемент прижимается к краю и заставляет строки течь вокруг себя.
> 3. Плавающие элементы «выпадают» из высоты родителя (схлопывание).
> 4. **clear: both** останавливает обтекание и возвращает поток в норму.
> 5. **display: flow-root** — самый современный способ решить проблему схлопывания родителя.
> 6. Не используйте `float` для общей сетки сайта, если есть выбор.
