# 🚀 Абсолютное позиционирование: position absolute

← [[103 — position static и relative]] | [[105 — position fixed]] →

> [!info] Цели урока
> - Узнать, как полностью вырвать элемент из потока документа.
> - Понять связь между `absolute` у ребенка и `relative` у родителя.
> - Научиться точно размещать элементы в любой точке контейнера.

---

## 📖 Введение: Полная свобода

Представьте, что нормальный поток — это текст на листе бумаги. Вы не можете просто взять слово и перенести его в правый верхний угол страницы, не перепечатывая всё остальное.

Но что если бы у вас была наклейка? Вы можете приклеить её куда угодно: поверх текста, на полях или вообще в углу листа. При этом текст под наклейкой никак не изменится — он просто будет перекрыт.

В CSS такой «наклейкой» является `position: absolute`. Это один из самых мощных и часто используемых инструментов верстки.

---

## 🏗️ Как работает absolute?

Когда вы задаете элементу `position: absolute`, происходят радикальные изменения:

1. **Выход из потока**: Элемент перестает существовать для своих соседей. Они смыкаются, как будто его и не было. Он больше не занимает места.
2. **Ширина по контенту**: Если не задать ширину, элемент «схлопнется» до размеров своего содержимого.
3. **Система координат**: Элемент теперь перемещается с помощью `top`, `bottom`, `left`, `right`.

**Но главный вопрос: относительно ЧЕГО он двигается?**

---

## ⚓ Поиск ориентира (Ближайший родитель)

Многие новички думают, что «абсолютный» элемент позиционируется относительно окна браузера. Это правда, но лишь отчасти.

**Правило поиска родителя:**
Элемент с `position: absolute` ищет ближайшего предка, у которого свойство `position` **НЕ** равно `static` (то есть `relative`, `absolute`, `fixed` или `sticky`).

- Если такой родитель найден — координаты отсчитываются от его границ.
- Если до самого верха (`<html>`) ничего не найдено — элемент позиционируется относительно **окна браузера** (точнее, области просмотра).

> [!tip] Связка-стандарт
> В 99% случаев используется связка: **Родителю — `position: relative`**, а **Ребенку — `position: absolute`**. Это позволяет «запереть» летучий элемент внутри конкретного блока.

---

## 🧭 Координаты в абсолютном мире

Координаты работают как «притяжение» к краям родителя:
- `top: 0; left: 0;` — прижать к верхнему левому углу.
- `bottom: 10px; right: 10px;` — прижать к нижнему правому углу с отступом 10px.
- `top: 50%; left: 50%;` — поместить начало элемента в центр родителя.

### Растягивание элемента
Интересный трюк: если вы зададите одновременно `left: 0; right: 0;`, элемент растянется на всю ширину родителя (аналогично `width: 100%`). Если добавить и `top: 0; bottom: 0;` — он полностью перекроет родителя.

---

## 🎨 Примеры использования

1. **Иконки на картинках**: Бейдж «Sale» в углу фото товара.
2. **Кастомные элементы форм**: Красивые галочки вместо стандартных чекбоксов.
3. **Модальные окна**: Кнопка «Закрыть» (крестик) в углу окна.
4. **Сложный декор**: Круги, полоски и другие элементы дизайна, которые «наползают» на контент.

---

## 🔍 Важные нюансы

1. **Перекрытие**: Абсолютные элементы всегда рисуются **поверх** обычного контента. Если их несколько, побеждает тот, кто ниже в HTML-коде (или у кого больше `z-index`).
2. **Margin**: Вы всё еще можете использовать `margin` вместе с `absolute`. Он будет добавляться к значениям координат.
3. **Прокрутка**: Элемент с `absolute` прокручивается вместе со своим родителем. Он не «прилипает» к экрану (для этого есть `fixed`).

## ❓ Часто задаваемые вопросы

### «Почему мой элемент улетел в начало страницы?»
Проверьте, задали ли вы `position: relative` родителю. Если нет — он улетел к краям браузера.

### «Можно ли сделать абсолютный элемент внутри другого абсолютного?»
Да, без проблем. Внутренний будет искать координаты относительно внешнего.

### «Как центрировать абсолютный элемент?»
Если вы задали `left: 50%`, элемент начнется с центра, но сам он не будет по центру. Чтобы это исправить, часто используют комбинацию с `transform: translate(-50%, -50%)`, но об этом мы поговорим в модуле про трансформации.

## 🧪 Практика: Бейдж на карточке

Создадим карточку товара и приклеим в её угол бейдж со скидкой.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <style>
    .card {
      width: 250px;
      height: 300px;
      background-color: #f9f9f9;
      border: 1px solid #ddd;
      margin: 50px auto;

      /* ЯКОРЬ ДЛЯ БЕЙДЖА */
      position: relative;

      display: flex;
      align-items: center;
      justify-content: center;
      font-family: sans-serif;
    }

    .badge {
      /* ВЫРЫВАЕМ ИЗ ПОТОКА */
      position: absolute;

      /* КООРДИНАТЫ */
      top: 10px;
      right: -10px; /* Выходит за край! */

      background-color: #e74c3c;
      color: white;
      padding: 5px 15px;
      font-size: 12px;
      font-weight: bold;
      box-shadow: 2px 2px 5px rgba(0,0,0,0.2);
    }
  </style>
</head>
<body>

  <div class="card">
    <div class="badge">SALE -50%</div>
    Здесь фото товара
  </div>

</body>
</html>
```

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **absolute** полностью вырывает элемент из потока.
> 2. Элемент ищет ближайшего «не-static» родителя.
> 3. Стандартная связка: родитель `relative`, ребенок `absolute`.
> 4. Координаты `top/bottom/left/right` управляют положением.
> 5. Элемент не занимает места в макете — соседи его не видят.
