# ⚓ Фиксация на экране: position fixed

## 📖 Введение: Вне прокрутки

Вы наверняка замечали на сайтах элементы, которые остаются на месте, даже когда вы крутите колесико мыши. Это может быть навигационное меню сверху, рекламный баннер сбоку или кнопка чата в углу.

В реальной жизни это похоже на наклейку на стекле ваших очков. Что бы вы ни делали, куда бы ни смотрели — наклейка всегда перед глазами в одной и той же точке.

В CSS для этого используется `position: fixed`. Это значение вырывает элемент не просто из потока, а из самой логики страницы, привязывая его напрямую к «иллюминатору» браузера (Viewport).

---

## 🏗️ Как работает fixed?

`position: fixed` ведет себя почти так же, как `absolute`, но с двумя принципиальными отличиями:

1. **Ориентир — Окно (Viewport)**: Элемент всегда позиционируется относительно видимой области браузера. Ему плевать, что у его родителей написано `position: relative`. Он всегда ищет только границы окна.
2. **Неподвижность**: Элемент не прокручивается вместе со страницей. Он буквально замирает на экране.

### Основные свойства:
- **Выход из потока**: Место элемента в коде не учитывается, соседи смыкаются.
- **Ширина**: По умолчанию сжимается по контенту. Чтобы сделать меню на всю ширину, нужно задать `width: 100%` или `left: 0; right: 0;`.

---

## 🧭 Координаты для fixed

Координаты работают от края экрана:
- `top: 0` — приклеить к верхнему краю экрана.
- `bottom: 20px; right: 20px` — приклеить в правый нижний угол.

```css
.fixed-header {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  height: 60px;
  background: white;
  z-index: 1000; /* Чтобы быть поверх всего контента */
}
```

---

## 🎨 Где это применяется?

1. **Верхняя навигация (Sticky Header)**: Чтобы меню всегда было под рукой.
2. **Кнопки мессенджеров**: Значок WhatsApp или Telegram в углу.
3. **Модальные окна (Overlay)**: Темный фон, перекрывающий всю страницу.
4. **Панели инструментов**: Боковые панели в личных кабинетах.

## 🔍 Подводный камень: «Исчезновение» контента

Это классическая ошибка при использовании `position: fixed`.
Так как фиксированный элемент (например, шапка) вырван из потока, он больше не занимает места. Элемент, который идет в коде после него (например, основной текст), поднимется в самый верх страницы и **окажется спрятанным под шапкой**.

**Решение**: Нужно добавить верхний отступ (`margin-top` или `padding-top`) следующему элементу (обычно `body` или `main`), равный высоте вашей фиксированной шапки.

```css
body {
  padding-top: 60px; /* Если высота шапки 60px */
}
```

## ❓ Часто задаваемые вопросы

### «Почему `fixed` иногда ломается на мобильных телефонах?»
В мобильных браузерах (особенно в Safari на iPhone) клавиатура или адресная строка могут менять размер окна. Это иногда приводит к «прыжкам» фиксированных элементов. Сейчас это работает лучше, чем раньше, но тестировать на реальных устройствах всё равно нужно.

### «Можно ли сделать fixed внутри маленького блока?»
Нет. `fixed` всегда привязывается к окну браузера. Если вам нужно, чтобы элемент «залипал» только внутри родителя — вам нужно свойство `sticky`, которое мы изучим в следующем уроке.

### «Влияет ли fixed на производительность?»
Да, если фиксированный элемент очень сложный (много теней, прозрачности, градиентов). Браузеру приходится перерисовывать его в новой позиции при каждом пикселе прокрутки.

## 🧪 Практика: Кнопка «Наверх»

Создадим простую кнопку, которая всегда висит в углу.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <style>
    body {
      height: 2000px; /* Чтобы была прокрутка */
      background: linear-gradient(to bottom, #eee, #333);
      font-family: sans-serif;
      padding: 20px;
    }

    .back-to-top {
      position: fixed;
      bottom: 30px;
      right: 30px;

      width: 50px;
      height: 50px;
      background-color: #3498db;
      color: white;
      text-decoration: none;
      border-radius: 50%;

      display: flex;
      align-items: center;
      justify-content: center;
      font-size: 24px;
      box-shadow: 0 4px 10px rgba(0,0,0,0.3);
      transition: transform 0.3s;
    }

    .back-to-top:hover {
      transform: scale(1.1);
      background-color: #2980b9;
    }
  </style>
</head>
<body>

  <h1>Прокрутите страницу вниз...</h1>
  <p>Кнопка всегда будет в правом нижнем углу, независимо от скролла.</p>

  <a href="#" class="back-to-top">↑</a>

</body>
</html>
```

> [!abstract] Запомни

> 3. Игнорирует `position: relative` у родителей.
