## 📖 Введение: Бутерброд из пикселей

До сих пор мы работали в двух измерениях: ширина (X) и высота (Y). Но как только мы начали использовать позиционирование, элементы стали накладываться друг на друга. Веб-страница превратилась в «стопку» прозрачных листов.

Казалось бы, всё просто: кто в коде ниже, тот и сверху. Или задай `z-index: 9999` и будь королем горы. Но почему тогда иногда элемент с `z-index: 10` оказывается выше, чем элемент с `z-index: 999999`? Почему модальное окно иногда прячется под шапкой, хотя вы всё сделали «правильно»?

Сегодня мы разберем механику «оси Z» и узнаем, почему `z-index` — это не просто число, а сложная иерархия.

---

## 🏗️ Порядок наложения по умолчанию (Natural Stacking Order)

Если вы не задали ни одного `z-index`, браузер рисует элементы в следующем порядке (от нижнего к верхнему):

1. **Фон и границы родителя**.
2. **Блоки в нормальном потоке** (в порядке их появления в HTML).
3. **Плавающие элементы** (`float`).
4. **Строчные элементы** (текст).
5. **Позиционированные элементы** (те, у которых `position` не `static`).

> [!tip] Факт
> Даже простая смена `position: static` на `position: relative` (без координат!) мгновенно поднимает элемент выше всех его неподвижных соседей.

---

## 🔢 Свойство z-index

Это свойство позволяет нам вручную изменить порядок в стопке.

- Значение — любое целое число (положительное, отрицательное или ноль).
- **Важно**: `z-index` работает **только** для позиционированных элементов (relative, absolute, fixed, sticky). Для обычных `static` элементов оно просто игнорируется.

```css
.modal {
  position: fixed;
  z-index: 100; /* Очень высоко */
}
.overlay {
  position: fixed;
  z-index: 50; /* Под модальным окном */
}
```

## 🛑 Главная тайна: Контекст наложения (Stacking Context)

Представьте, что `z-index` — это нумерация страниц в книгах.
У вас есть две книги (два родительских контейнера). В первой книге страница №10. Во второй книге страница №1.

Можете ли вы сказать, что страница №10 из первой книги «выше» или «важнее», чем страница №1 из второй? Нет, это зависит от того, какая книга лежит сверху в стопке на столе.

**Контекст наложения** — это и есть такая «книга». Если родительский элемент создал свой контекст наложения, то все его дети «запираются» внутри него. Их `z-index` теперь сравнивается только между собой, но они никогда не смогут вырваться выше или упасть ниже, чем позволяет позиция их родителя.

### Что создает контекст наложения?
Вы создаете новый контекст (новую «книгу»), если:
1. Элемент является корнем документа (`<html>`).
2. Элемент позиционирован (`absolute` или `relative`) и у него есть `z-index` отличный от `auto`.
3. Элемент имеет `position: fixed` или `sticky`.
4. Элемент имеет прозрачность (`opacity`) меньше 1.
5. Элементу заданы современные свойства: `transform`, `filter`, `perspective`, `clip-path` и др.
6. Элемент является Flex или Grid ребенком с заданным `z-index`.

## 🔍 Разбор классической ошибки

```html
<div class="header" style="position: relative; z-index: 10;">
   <div class="menu" style="position: absolute; z-index: 9999;">Меню</div>
</div>

<div class="content" style="position: relative; z-index: 20;">
   Контент сайта
</div>
```

**Вопрос**: Будет ли меню (z-index: 9999) поверх контента (z-index: 20)?
**Ответ**: **НЕТ**.
Потому что родитель меню (`.header`) имеет `z-index: 10`, а соседний блок `.content` имеет `z-index: 20`. Поскольку 20 больше 10, весь блок контента (со всеми своими внутренностями) будет лежать выше всего блока хедера. Меню «заперто» внутри хедера и не может пробиться сквозь «потолок», установленный его родителем.

---

## 🛠️ Как отлаживать проблемы с z-index?

1. **Не используйте огромные числа**: Если вам нужны значения вроде `9999999`, значит, ваша структура слоев сломана. Старайтесь использовать шаг 10 (10, 20, 30...).
2. **Проверьте родителей**: Если элемент не всплывает, посмотрите, нет ли у его родителя `z-index`, `opacity` или `transform`. Скорее всего, создался новый контекст.
3. **Инструменты разработчика**: В Chrome DevTools есть вкладка **Layers** (Слои), которая позволяет крутить сайт в 3D и видеть, кто на каком уровне находится.
4. **Правило иерархии**: Если вам нужно, чтобы элемент был поверх всего (например, модальное окно), лучше размещайте его в самом конце HTML-кода, прямо перед закрывающим тегом `</body>`. Так меньше шансов попасть в ловушку чужого контекста наложения.

## ❓ Часто задаваемые вопросы

### «Зачем нужен отрицательный z-index?»
Чтобы спрятать элемент за фон родителя. Например, декоративный узор, который должен быть под текстом. Но помните: если у родителя не создан свой контекст наложения, элемент с `z-index: -1` может улететь даже за фон самого `body` и исчезнуть.

### «Почему `opacity: 0.99` меняет слои?»
Это одна из странностей спецификации CSS. Как только вы меняете прозрачность (даже на 0.01), браузер вынужден отрисовывать этот элемент в отдельном слое, чтобы правильно смешать цвета. А создание отдельного слоя автоматически порождает новый контекст наложения.

### «Работает ли z-index без position?»
Нет. Если вы напишете `div { z-index: 10; }` для обычного статического блока, ничего не изменится.

## 🧪 Практика: Эксперимент с книгами

Создадим две «книги» и увидим магию контекста наложения.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <style>
    .book {
      width: 200px;
      height: 200px;
      padding: 20px;
      position: relative;
      border: 2px solid #333;
    }

    .book-1 {
      background: #e74c3c;
      z-index: 10; /* Книга 1 */
    }

    .book-2 {
      background: #2ecc71;
      z-index: 5;  /* Книга 2 (ниже первой) */
      margin-top: -100px;
      margin-left: 50px;
    }

    .page {
      width: 100px;
      height: 100px;
      position: absolute;
      background: white;
      border: 1px solid black;
      display: flex;
      align-items: center;
      justify-content: center;
      font-weight: bold;
    }

    .page-in-book-2 {
      top: -30px;
      left: 30px;
      z-index: 9999; /* ОГРОМНЫЙ индекс! */
    }
  </style>
</head>
<body>

  <div class="book book-1">
    Книга 1 (z-index: 10)
  </div>

  <div class="book book-2">
    Книга 2 (z-index: 5)
    <div class="page page-in-book-2">
      Стр 9999
    </div>
  </div>

  <p style="margin-top: 50px;">
    Почему "Стр 9999" оказалась ПОД красной Книгой 1? <br>
    Потому что Книга 2 имеет z-index: 5, а Книга 1 имеет z-index: 10. <br>
    Весь контент Книги 2 никогда не будет выше Книги 1.
  </p>

</body>
</html>
```

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **z-index** работает только для позиционированных элементов.
> 2. Слой элемента определяется его **контекстом наложения**.
> 3. Внутри контекста `z-index` работает локально («внутри книги»).
> 4. Родитель с `z-index` создает «потолок» для своих детей.
> 5. Прозрачность, трансформации и фиксированное положение создают новые контексты.
> 6. Чем проще структура слоев, тем меньше багов.
