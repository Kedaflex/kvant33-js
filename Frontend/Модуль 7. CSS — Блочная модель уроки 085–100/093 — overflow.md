# 🌊 Укрощение потока: Свойство overflow

← [[092 — box-sizing]] | [[094 — outline и box-shadow]] →

> [!info] Цели урока
> - Узнать, что происходит, когда контент не помещается в границы блока.
> - Изучить значения `visible`, `hidden`, `scroll` и `auto`.
> - Научиться управлять переполнением по отдельным осям (X и Y).

---

## 📖 Введение: Когда места слишком мало

В идеальном мире дизайна текст всегда идеально вписывается в отведенное ему пространство. В реальном мире веба контент непредсказуем. Пользователь может увеличить шрифт в браузере, прийти длинное сообщение из базы данных или вы загрузите картинку, которая шире, чем экран смартфона.

Что должен делать браузер, если элемент размером 200x200 пикселей содержит в себе текст на 500 пикселей высоты?

По умолчанию CSS позволяет контенту «выливаться» наружу. Это выглядит неопрятно и часто ломает верстку. Свойство `overflow` (переполнение) позволяет нам четко сказать браузеру, как поступить с «лишним» контентом.

---

## 🛠️ Основные значения overflow

Свойство `overflow` применяется к контейнеру, в котором происходит переполнение.

### 1. visible (По умолчанию)
Контент не обрезается и отрисовывается за пределами границ блока. Он может накладываться на соседние элементы. Сам блок при этом не растягивается.
```css
.box { overflow: visible; }
```

### 2. hidden (Скрыть)
Всё, что выходит за границы блока, становится невидимым. Никаких полос прокрутки не появляется. Это отличный способ «отрезать» лишнее, но будьте осторожны: пользователь может никогда не увидеть скрытую часть текста.
```css
.box { overflow: hidden; }
```

### 3. scroll (Прокрутка)
Браузер добавляет полосы прокрутки в любом случае — даже если контента мало и он помещается. Это гарантирует, что дизайн не «прыгнет», когда прокрутка внезапно появится, но пустые полосы прокрутки часто выглядят некрасиво.
```css
.box { overflow: scroll; }
```

### 4. auto (Умная прокрутка)
Самое популярное значение. Полосы прокрутки появляются **только тогда**, когда контент действительно не помещается. Если всё в порядке — блок выглядит как обычно.
```css
.box { overflow: auto; }
```

## 📐 Разделение осей: overflow-x и overflow-y

Иногда нам нужно скрыть переполнение по горизонтали, но разрешить скролл по вертикали (или наоборот). Для этого есть два отдельных свойства:

- `overflow-x` — управляет горизонтальным переполнением (влево-вправо).
- `overflow-y` — управляет вертикальным переполнением (вверх-вниз).

*Пример: Длинная таблица на мобильном телефоне.*
```css
.table-container {
  overflow-x: auto; /* Появится горизонтальный скролл */
  overflow-y: hidden; /* Вертикальный скролл не нужен */
}
```

## 🔍 Важные нюансы

1. **Контекст наложения**: Как только вы устанавливаете любое значение `overflow`, кроме `visible`, элемент создает новый контекст форматирования. Это влияет на то, как внутри него будут работать некоторые сложные вещи вроде `z-index` или схлопывания маржинов (об этом в уроке 095).
2. **Скроллбары и дизайн**: В разных операционных системах полосы прокрутки выглядят по-разному. В macOS они часто скрыты и появляются только при движении, а в Windows это серые полосы, которые занимают место (около 15-17px), уменьшая доступную ширину для вашего контента.
3. **Обрезание теней**: Если у вашего элемента есть тень (`box-shadow`), выходящая за его пределы, `overflow: hidden` её обрежет! Это частая ошибка новичков.

## ❓ Часто задаваемые вопросы

### «Почему `overflow: hidden` иногда помогает 'починить' верстку?»
Раньше это был стандартный способ решения проблемы с «плавающими» элементами (`float`). Также это «лечит» вылетание дочерних маржинов из родителя. Сегодня для этого есть более современные способы, но `overflow: hidden` всё еще остается «швейцарским ножом» верстальщика.

### «Можно ли изменить дизайн полосы прокрутки?»
Да, с помощью псевдоэлементов `::-webkit-scrollbar` (для Chrome/Safari/Edge) или свойства `scrollbar-color` (для Firefox). Но это довольно кропотливая работа, так как нет единого стандарта для всех браузеров.

### «Что делать, если картинка вылезает за край экрана на телефоне?»
Пропишите для `body` или основного контейнера `overflow-x: hidden;`. Это уберет горизонтальную тряску сайта на мобильных устройствах (хотя лучше найти и исправить саму причину вылета элемента).

## 🧪 Практика: Окно с текстом

Создадим блок с ограниченной высотой и «умной» прокруткой.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <style>
    .scroll-box {
      width: 300px;
      height: 200px;
      border: 2px solid #3498db;
      padding: 15px;
      margin: 50px auto;

      /* Магия переполнения */
      overflow-y: auto;
      overflow-x: hidden;

      background-color: #f9f9f9;
      font-family: sans-serif;
      line-height: 1.6;
    }

    .scroll-box h2 {
      margin-top: 0;
      color: #2980b9;
    }
  </style>
</head>
<body>

  <div class="scroll-box">
    <h2>Условия соглашения</h2>
    <p>Здесь очень много текста...</p>
    <p>Этот текст не помещается в блок высотой 200 пикселей.</p>
    <p>Благодаря overflow-y: auto, вы можете прокрутить его вниз и прочитать полностью.</p>
    <p>А overflow-x: hidden гарантирует, что у нас не появится случайная горизонтальная прокрутка.</p>
    <p>Конец текста.</p>
  </div>

</body>
</html>
```

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **Overflow** управляет контентом, который больше своего контейнера.
> 2. `visible` — всё видно (по умолчанию).
> 3. `hidden` — всё лишнее обрезается.
> 4. `auto` — прокрутка появляется только при необходимости (лучший выбор).
> 5. Можно разделять управление по горизонтали (`overflow-x`) и вертикали (`overflow-y`).
> 6. Осторожно: `overflow: hidden` может обрезать нужные тени или выпадающие меню.
