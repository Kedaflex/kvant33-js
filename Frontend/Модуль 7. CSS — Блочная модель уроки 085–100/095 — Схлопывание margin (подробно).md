# 🛸 Схлопывание margin: Великая тайна CSS

> - Изучить три сценария, в которых происходит схлопывание.

## 📖 Введение: Куда делись мои пиксели?

Представьте ситуацию: вы создали два блока. Первому вы задали `margin-bottom: 50px`, а второму — `margin-top: 30px`. Логично ожидать, что расстояние между ними будет суммой этих значений — 80 пикселей.

Но когда вы открываете браузер, вы видите зазор ровно в 50 пикселей. 30 пикселей от второго блока просто... исчезли. Они не прибавились, они «схлопнулись» с большим отступом соседа.

Это поведение называется **Margin Collapsing** (схлопывание внешних отступов). Оно считается одной из самых запутанных тем в CSS для новичков, хотя у него есть своя четкая логика, идущая еще из времен книгопечатания. Сегодня мы разберем это явление на атомы.

---

## 🏛️ Почему это происходит? (Исторический контекст)

Схлопывание было придумано для типографики. Когда в тексте идут друг за другом два абзаца, у каждого из них есть отступы сверху и снизу. Если бы отступы суммировались, расстояние между абзацами было бы в два раза больше, чем расстояние от верхнего края страницы. Это выглядело бы некрасиво и неритмично.

Схлопывание позволяет поддерживать единый ритм пустых пространств на странице, выбирая «сильнейший» отступ из имеющихся.

---

## 🔍 Три сценария схлопывания

Схлопывание происходит **только для вертикальных отступов** (top и bottom). Горизонтальные отступы (left и right) никогда не схлопываются.

### 1. Соседние элементы (Adjacent Siblings)
Это самый простой случай. Когда один блок стоит под другим, их соприкасающиеся отступы объединяются в один.
- Если оба положительные: выбирается наибольший.
- Если один положительный, а другой отрицательный: они суммируются (например, 50px + (-20px) = 30px).
- Если оба отрицательные: выбирается самый «отрицательный» (например, -50px и -20px дадут зазор -50px).

### 2. Родитель и первый/последний потомок
Это тот самый случай, который сводит с ума. Если у родителя нет `border`, `padding` или другого содержимого сверху, то `margin-top` его первого ребенка «вылетает» наружу и становится маржином самого родителя.

**Результат**: Ребенок не отодвигается от верхнего края родителя, вместо этого весь родитель вместе с ребенком съезжает вниз от своего соседа.

### 3. Пустые элементы
Если у элемента нет высоты, рамок, внутренних отступов и контента, то его собственный `margin-top` и `margin-bottom` соприкасаются и схлопываются между собой в один.

---

## 🛠️ Как «вылечить» схлопывание?

Иногда схлопывание полезно, но чаще всего оно мешает. Вот проверенные способы его остановить:

### Для случая с родителем и ребенком:
1. **Добавьте padding**: Даже `padding-top: 0.1px` создаст преграду, и маржины перестанут соприкасаться.
2. **Добавьте border**: Тонкая прозрачная рамка также разделяет маржины.
3. **Используйте overflow: hidden**: Любое значение `overflow`, кроме `visible`, создает «контекст форматирования», который запирает маржины ребенка внутри.
4. **display: flow-root**: Современный и самый «чистый» способ. Он специально создан для того, чтобы контейнер вел себя предсказуемо и удерживал в себе все отступы.

### Для соседних элементов:
Если вам нужно, чтобы отступы между блоками суммировались, просто замените один из `margin` на `padding`. Внутренние отступы никогда не схлопываются.

## 🧪 Практика: Наглядные опыты

Давайте создадим пример, где мы увидим «вылетающий» маржин и научимся его ловить.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <style>
    .section {
      background-color: #f1c40f;
      /* Сейчас у родителя нет ни рамок, ни паддингов */
    }

    .child {
      background-color: #e67e22;
      margin-top: 50px; /* Этот отступ "вылетит" наружу! */
      padding: 20px;
      color: white;
    }

    .fix-container {
      margin-top: 50px;
      background-color: #2ecc71;

      /* РЕШЕНИЕ: */
      display: flow-root;
    }

    .fix-child {
      margin-top: 50px; /* Теперь этот отступ останется ВНУТРИ */
      background-color: #27ae60;
      padding: 20px;
    }
  </style>
</head>
<body>

  <h3>Сценарий: Маржин "вылетел"</h3>
  <div class="section">
    <div class="child">
      Мой margin-top: 50px. Но я прилип к верху родителя,
      а весь желтый блок уехал вниз.
    </div>
  </div>

  <hr>

  <h3>Сценарий: Исправлено через flow-root</h3>
  <div class="fix-container">
    <div class="fix-child">
      Теперь мой margin-top честно отодвинул меня от края зеленого родителя.
    </div>
  </div>

</body>
</html>
```

## 🔍 Где НЕ происходит схлопывание

Есть ситуации, когда правила CSS отключают схлопывание автоматически. Об этом полезно знать, чтобы не искать проблему там, где её нет:
- У элементов с `display: flex` и `display: grid` (их дочерние элементы никогда не схлопывают маржины).
- У абсолютно позиционированных элементов (`position: absolute`).
- У элементов с `float` (хотя флоаты сейчас используются редко).
- Если между элементами есть что-то еще (даже пустая строка текста или `padding`).

## ❓ Часто задаваемые вопросы

### «Почему горизонтальные маржины не схлопываются?»
Потому что в тексте слова должны стоять рядом, и расстояние между ними — это сумма пробелов. Схлопывание по горизонтали сделало бы чтение текста невозможным. Веб изначально строился как «текстовая среда», отсюда и такие правила.

### «Что будет, если один маржин 50px, а другой -100px?»
Они суммируются. `50 + (-100) = -50`. В итоге второй блок наедет на первый на 50 пикселей.

### «Нужно ли мне всегда использовать `display: flow-root`?»
Не обязательно всегда, но это хороший тон для крупных контейнеров (секций сайта, карточек), чтобы их внутренние отступы не влияли на внешнюю верстку неожиданным образом.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. Схлопываются **только вертикальные** отступы.
> 2. При схлопывании двух положительных маржинов **выбирается самый большой**.
> 3. Маржины детей могут «вылетать» за границы родителя, если их ничего не разделяет.
> 4. Разделители: `padding`, `border`, `overflow` (не visible) или любое содержимое.
> 5. Современное решение для изоляции — `display: flow-root`.
> 6. В Flexbox и Grid схлопывания **не существует**.
