# 🛸 Внешние отступы: Свойство margin

← [[089 — padding]] | [[091 — border]] →

> [!info] Цели урока
> - Узнать, как создавать расстояние между соседними элементами.
> - Понять магию `margin: auto` для центрирования блоков.
> - Разобраться в отличиях `margin` от `padding`.

---

## 📖 Введение: Личное пространство

Если `padding` — это «пупырчатая пленка» внутри коробки, то `margin` — это «зона отчуждения» вокруг неё. Представьте себе людей, стоящих в очереди. Расстояние между ними — это `margin`. Если вы хотите, чтобы кто-то отошел от вас подальше, вы увеличиваете свой внешний отступ.

Внешние отступы — это главный инструмент для создания структуры и ритма на странице. Именно они определяют, насколько далеко друг от друга будут находиться абзацы, картинки и кнопки.

---

## 🛠️ Синтаксис и сокращенная запись

Синтаксис `margin` полностью идентичен `padding`. У нас есть те же четыре стороны и та же логика сокращений.

- `margin-top`, `margin-right`, `margin-bottom`, `margin-left`
- Сокращение: `margin: [верх] [право] [низ] [лево];` (по часовой стрелке).

```css
.card {
  margin: 20px; /* Со всех сторон */
  margin: 10px 0; /* 10px сверху и снизу, 0 слева и справа */
  margin: 0 auto 20px; /* 0 сверху, авто по бокам, 20px снизу */
}
```

### Важнейшие отличия от padding:
1. **Прозрачность**: `margin` всегда прозрачен. Через него виден фон родительского элемента.
2. **Отрицательные значения**: В отличие от padding, `margin` может быть отрицательным (например, `margin-top: -20px;`). Это заставляет элемент «наезжать» на соседа или выходить за границы родителя.
3. **Схлопывание (Collapsing)**: Это уникальная и сложная особенность вертикальных отступов, которую мы подробно разберем в уроке 095. Если коротко: вертикальные отступы двух соседних блоков не суммируются, а выбирается самый большой из них.

---

## 🎯 Магия margin: auto

Это один из самых частых приемов в верстке. Если вы зададите блочному элементу фиксированную ширину и напишете `margin-left: auto; margin-right: auto;`, блок встанет ровно по центру своего родителя.

```css
.container {
  width: 80%;
  max-width: 1000px;
  margin-left: auto;
  margin-right: auto;
  /* Или кратко: margin: 0 auto; */
}
```

**Как это работает?**
Ключевое слово `auto` говорит браузеру: «Возьми всё свободное место в родительском блоке и раздели его поровну между левым и правым отступом».

> [!warning] Важное условие
> Центрирование через `auto` работает только если у элемента **задана ширина** (`width` или `max-width`) и она меньше, чем ширина родителя. Иначе свободного места просто не останется.

---

## 🧬 Отрицательный margin: Когда всё идет не по плану

Отрицательный `margin` — это мощный, но опасный инструмент.
- `margin-left: -20px;` — сдвинет элемент влево, за его нормальную границу.
- `margin-bottom: -50px;` — подтянет следующий за ним элемент на 50 пикселей вверх, заставляя их перекрываться.

Это часто используется для создания сложных дизайнерских эффектов, когда один блок должен слегка «наплывать» на другой.

---

## 🔍 Margin у строчных элементов

Как и в случае с `padding`, с `margin` у строчных (`inline`) элементов есть свои особенности:
- **Горизонтальные отступы** (left/right) работают отлично.
- **Вертикальные отступы** (top/bottom) — **не работают**. Совсем. Вы можете задать `margin-top: 100px;` для `<span>`, но он не сдвинется ни на пиксель относительно соседних строк.

---

## ⚖️ Padding vs Margin: Что выбрать?

Это классический вопрос новичка.

| Ситуация | Используйте... |
| :--- | :--- |
| Нужно отодвинуть текст от границы рамки | `padding` |
| Нужно сделать кнопку больше (увеличить кликабельную зону) | `padding` |
| Нужно отодвинуть один блок от другого | `margin` |
| Нужно центрировать блок | `margin: auto` |
| Нужен отступ, который закрашивается фоном | `padding` |

> [!tip] Золотое правило
> Если вы хотите отодвинуть **соседей** — используйте `margin`. Если хотите отодвинуть **содержимое** внутри себя — используйте `padding`.

## 🧪 Практика: Управляем пространством

Давайте создадим два блока и поиграем с их отступами.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <style>
    .parent {
      background-color: #ecf0f1;
      padding: 30px; /* Воздух внутри родителя */
      border: 2px solid #bdc3c7;
    }

    .child {
      width: 200px;
      padding: 20px;
      background-color: #3498db;
      color: white;

      /* Центрируем блок и отодвигаем от соседа снизу */
      margin: 0 auto 20px;
    }

    .child-2 {
      width: 150px;
      padding: 10px;
      background-color: #e67e22;
      color: white;

      /* Наезжаем на верхний блок */
      margin: -30px auto 0;
      border: 2px solid white;
    }
  </style>
</head>
<body>

  <div class="parent">
    <div class="child">Блок 1 (Центрирован)</div>
    <div class="child-2">Блок 2 (Наехал на Блок 1)</div>
  </div>

</body>
</html>
```

## ❓ Часто задаваемые вопросы

### «Почему `margin-top` у дочернего элемента двигает весь родительский блок вниз?»
Это классическое проявление «схлопывания маржинов». Браузер считает, что верхний отступ ребенка — это на самом деле верхний отступ родителя. Лечится добавлением `padding` или `border` к родительскому элементу. Подробнее об этом — в уроке 095.

### «Работает ли `margin: auto` для выравнивания по вертикали?»
В обычном потоке (block) — **нет**. Браузер не знает, какую высоту считать за основу для деления свободного места. Но `margin: auto` волшебным образом работает для вертикального центрирования внутри **Flexbox** и **Grid**.

### «Стоит ли использовать `margin-bottom` или `margin-top` для разделения абзацев?»
Профессионалы рекомендуют выбрать одну стратегию. Чаще всего используют только `margin-bottom`. Так проще предсказать расстояние между элементами и избежать лишнего суммирования отступов.
