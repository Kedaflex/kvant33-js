# 🏘️ Ближние и дальние соседи: Селекторы связей

## 📖 Введение: Жизнь в одном контейнере

В прошлом уроке мы разобрались, как выбирать элементы «вглубь» (родитель → ребенок). Но HTML-элементы часто живут бок о бок, как квартиры на одной лестничной клетке. У них один и тот же родитель, и они следуют друг за другом в коде.

Иногда нам нужно применить стиль к элементу, основываясь на том, кто стоит **перед ним**. Например: «сделай отступ сверху у каждого параграфа, но только если перед ним идет заголовок». Или: «подсвети все картинки, которые идут после этой статьи».

В CSS для этого есть два «соседских» селектора: смежный и общий.

---

## 🤝 Смежный соседний селектор (Adjacent Sibling)

Этот селектор выбирает элемент, который идет **сразу же** после указанного и имеет того же родителя.

**Синтаксис**: `Элемент1 + Элемент2`

```css
/* Выбрать <p>, который идет сразу после <h2> */
h2 + p {
  font-weight: bold;
  margin-top: 0;
}
```

### Важные условия:
1. **Строгий порядок**: Элемент2 должен быть именно следующим по счету. Если между `h2` и `p` вставить `div`, селектор перестанет работать.
2. **Один родитель**: Они должны быть «братьями».
3. **Только вперед**: CSS не умеет смотреть назад. Вы не можете выбрать `h2`, перед которым стоит `p`. Только наоборот.

### Зачем это нужно?
Классический пример: убрать лишний отступ. Обычно у заголовков и абзацев есть свои `margin`. Когда они встречаются, отступ получается слишком большим. С помощью `h2 + p` вы можете точечно его уменьшить.

---

## 👨‍👨‍👦 Общий соседний селектор (General Sibling)

Этот селектор менее строгий. Он выбирает **все** подходящие элементы, которые идут после указанного, даже если они не стоят вплотную.

**Синтаксис**: `Элемент1 ~ Элемент2`

```css
/* Выбрать ВСЕ изображения, которые стоят после заголовка <h1> */
h1 ~ img {
  border: 2px solid blue;
}
```

### Пример из жизни
Представьте ситуацию: у вас есть заголовок раздела, а за ним следует текст, картинки, списки и снова картинки. Если вы используете `h1 ~ img`, то **все** картинки в этом разделе (после h1) получат рамку, независимо от того, сколько текста лежит между ними.

---

## ⚖️ Сравнение: «+» vs «~»

| Селектор | Что выбирает? | Пример |
| :--- | :--- | :--- |
| `A + B` | Только **один** элемент B, если он идет **сразу** за A. | `h1 + p` (первый абзац после h1) |
| `A ~ B` | **Все** элементы B, которые идут где-то **после** A. | `h1 ~ p` (все абзацы после h1) |

## 🛠️ Практический прием: Лобовое столкновение

Один из самых популярных приемов в современной верстке — это добавление отступа между одинаковыми элементами, чтобы не задавать его первому или последнему.

```css
/* Добавить отступ сверху всем элементам списка, кроме самого первого */
li + li {
  margin-top: 10px;
}
```
*Как это работает?* Самый первый `li` не имеет перед собой соседа `li`, поэтому к нему правило не применится. А ко второму, третьему и десятому — применится. Это гораздо чище, чем писать `li { margin-top: 10px; }` и потом обнулять его для первого элемента через `:first-child`.

## ❓ Часто задаваемые вопросы

### «Почему нет селектора для предыдущего соседа (назад)?»
Это связано с тем, как браузеры отрисовывают страницу. Они читают HTML сверху вниз и сразу применяют стили. Если бы CSS мог смотреть «назад», браузеру пришлось бы постоянно перерисовывать уже готовые части страницы, когда он находит новые правила внизу. Это бы сильно замедлило интернет.

### «Можно ли комбинировать классы и соседские селекторы?»
Да, и это очень мощно!
`.active + .tab-content { display: block; }` — показать контент, который идет сразу за активной вкладкой.

### «Работает ли это для элементов разного типа?»
Конечно. Вы можете написать `img + span`, `div ~ p` или даже `.my-class + #my-id`. Главное — их взаимное расположение в HTML-коде.

## 🧪 Практика: Умные отступы

Давайте создадим форму, где сообщения об ошибках появляются только после неправильно заполненных полей.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <style>
    .input-group {
      margin-bottom: 20px;
    }

    input {
      display: block;
      padding: 10px;
      border: 1px solid #ccc;
    }

    /* Сообщение об ошибке по умолчанию скрыто */
    .error-msg {
      display: none;
      color: red;
      font-size: 12px;
      margin-top: 5px;
    }

    /* Если у инпута есть класс .invalid, показываем ошибку за ним */
    input.invalid + .error-msg {
      display: block;
    }

    /* Стилизуем идущие подряд кнопки */
    button + button {
      margin-left: 10px;
    }
  </style>
</head>
<body>

  <div class="input-group">
    <label>Email:</label>
    <input type="email" class="invalid" value="ошибка@почта">
    <div class="error-msg">Пожалуйста, введите корректный адрес.</div>
  </div>

  <div class="input-group">
    <label>Пароль:</label>
    <input type="password">
    <div class="error-msg">Пароль слишком короткий.</div>
  </div>

  <button>Сохранить</button>
  <button>Отмена</button>

</body>
</html>
```

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. Соседские селекторы работают только с элементами **одного родителя**.
> 2. **Знак «+»** — это строгий сосед (следующий сразу за).
> 3. **Знак «~»** — это общий сосед (любой из следующих после).
> 4. CSS смотрит только **вперед** (вниз по коду).
> 5. `li + li` — отличный способ задать отступы между элементами списка.
> 6. Соседские селекторы помогают избежать «лишних» классов в HTML.
