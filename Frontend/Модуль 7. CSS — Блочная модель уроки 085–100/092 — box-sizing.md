## 📖 Введение: Математический кошмар

В предыдущих уроках мы постоянно сталкивались с одной и той же проблемой:
«Если я задал ширину 100px, добавил padding 20px и border 5px, то блок внезапно стал 150px».

Представьте, что вы дизайнер интерьеров. Вы заказали шкаф шириной ровно 1 метр, чтобы он влез в нишу. Но когда шкаф привезли, оказалось, что мастер добавил к метру еще 10 сантиметров толщины стенок и 5 сантиметров декоративных ручек. Шкаф не влезает! Вы в ярости.

Именно так работал CSS по умолчанию долгие годы. Это называлось **Content-box model**. Разработчикам приходилось постоянно вычитать отступы из ширины в уме: «Так, мне нужен блок 300px, у него отступы по 20px, значит в CSS я пишу `width: 260px`». Это было неудобно, приводило к ошибкам и сводило с ума.

К счастью, сегодня у нас есть решение — свойство `box-sizing`.

---

## ⚖️ Два режима расчета размеров

Свойство `box-sizing` имеет два основных значения, которые полностью меняют правила игры.

### 1. content-box (По умолчанию)
В этом режиме свойство `width` задает ширину **только контента**. Всё остальное (padding и border) «нарастает» сверху, увеличивая физический размер блока.
- **Формула ширины**: `width + padding + border = Реальный размер`.

### 2. border-box (Спасение верстальщика)
В этом режиме свойство `width` задает **полную физическую ширину** блока. Если вы добавляете padding или border, они не расширяют блок вовне, а «вдавливаются» внутрь, сжимая контентную область.
- **Формула ширины**: `width = Реальный размер`. (Padding и border уже включены в это число).

```css
/* Сравнение */
.box-1 {
  box-sizing: content-box;
  width: 200px;
  padding: 20px;
  border: 10px solid black;
  /* Итог: 260px на экране */
}

.box-2 {
  box-sizing: border-box;
  width: 200px;
  padding: 20px;
  border: 10px solid black;
  /* Итог: РОВНО 200px на экране. Контент сожмется до 140px. */
}
```

---

## 🏆 Почему border-box победил?

Использование `border-box` делает верстку предсказуемой.
- Если вы сказали, что колонка занимает 50% ширины экрана, она **всегда** будет занимать 50%, сколько бы отступов и рамок вы ей ни добавили.
- Вам больше не нужно брать калькулятор при каждом изменении `padding`.
- Это особенно критично в адаптивной верстке, где мы смешиваем проценты и пиксели (например, ширина 100% минус отступ 20px).

---

## 🛠️ Глобальный сброс (Best Practice)

Профессиональные разработчики не прописывают `box-sizing` для каждого элемента отдельно. Они применяют его ко всем элементам на странице сразу в самом начале CSS-файла. Это называется «Глобальный сброс коробочной модели».

Существует «золотой стандарт» такой записи:

```css
/* 1. Применяем ко всем элементам, включая псевдоэлементы */
*, *::before, *::after {
  box-sizing: border-box;
}
```

С этой магической строчкой в начале вашего CSS ваша жизнь станет в десять раз проще. Теперь `width` — это действительно ширина всего блока.

---

## 🔍 Интрига: Что насчет margin?

Важно понимать: свойство `box-sizing: border-box` включает в ширину только **padding** и **border**.
**Margin (внешний отступ) всё еще остается снаружи!**

Даже с `border-box`, если вы зададите блоку `width: 100%` и `margin: 20px`, он всё равно «вылетит» за границы родителя, потому что `margin` всегда добавляется сверху к итоговому размеру.

---

## ❓ Часто задаваемые вопросы

### «Почему `content-box` всё еще является значением по умолчанию?»
Потому что CSS очень дорожит обратной совместимостью. Если бы консорциум W3C внезапно изменил поведение по умолчанию, миллионы старых сайтов, созданных в 90-х и 2000-х, мгновенно бы «развалились».

### «Есть ли случаи, когда `content-box` лучше?»
Очень редко. Иногда это полезно для очень специфических математических расчетов в сложной графике на CSS, но в 99% случаев обычной верстки сайтов `border-box` — лучший выбор.

### «Влияет ли это на скорость загрузки?»
Нет, это свойство обрабатывается браузером мгновенно и никак не влияет на производительность.

## 🧪 Практика: Наглядное сравнение

Давайте создадим два одинаковых на первый взгляд блока и увидим разницу.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <style>
    .demo-box {
      width: 300px;
      height: 100px;
      padding: 30px;
      border: 10px solid #34495e;
      margin-bottom: 20px;
      color: white;
      text-align: center;
      line-height: 1.2;
    }

    .classic {
      box-sizing: content-box;
      background-color: #e74c3c;
    }

    .modern {
      box-sizing: border-box;
      background-color: #2ecc71;
    }
  </style>
</head>
<body>

  <div class="demo-box classic">
    Я <b>content-box</b>.<br>
    Мой реальный размер: 380px.<br>
    (300 + 30+30 + 10+10)
  </div>

  <div class="demo-box modern">
    Я <b>border-box</b>.<br>
    Мой реальный размер: 300px.<br>
    (ровно столько, сколько в width)
  </div>

</body>
</html>
```

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. По умолчанию (`content-box`) отступы и рамки **увеличивают** ширину блока.
> 2. С `border-box` ширина блока **остается неизменной**, а контент сжимается внутри.
> 3. `border-box` делает расчеты размеров интуитивно понятными.
> 4. Всегда используйте глобальный сброс через селектор `*` в начале проекта.
> 5. **Margin** никогда не входит в расчет `width`, даже с `border-box`.
