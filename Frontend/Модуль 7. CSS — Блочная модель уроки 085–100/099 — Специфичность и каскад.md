> [!info] Цели урока
> - Понять, как браузер разрешает конфликты между CSS-правилами.
> - Научиться рассчитывать «вес» (специфичность) любого селектора.
> - Узнать, что такое каскад и почему порядок правил в файле имеет значение.

---

## 📖 Введение: Кто здесь главный?

Вы написали стиль для кнопки: `button { color: red; }`. Но кнопка почему-то синяя. Вы проверяете код и видите другое правило: `.nav-button { color: blue; }`. Почему браузер выбрал именно его?

CSS расшифровывается как **Cascading Style Sheets** (Каскадные таблицы стилей). Слово «Каскад» здесь ключевое. Это набор правил, по которым браузер решает, какое свойство применить к элементу, если на него претендуют сразу несколько противоречивых инструкций.

Это похоже на судебное разбирательство, где у каждой стороны есть свой «вес» и свои аргументы. Чтобы не гадать, почему стили не применяются, нам нужно изучить законы этого «суда».

---

## 🌊 1. Каскад (Порядок появления)

Самое простое правило: если два селектора имеют **абсолютно одинаковый вес**, побеждает тот, который написан **ниже (позже)** в коде.

```css
p { color: red; }
p { color: blue; } /* Победит синий */
```
Это касается как одного файла, так и нескольких подключенных файлов. Тот CSS-файл, который подключен в HTML позже всех, имеет «последнее слово».

---

## ⚖️ 2. Специфичность (Вес селектора)

Если селекторы разные, браузер больше не смотрит на порядок. Он начинает взвешивать их. У каждого типа селектора есть свои «баллы».

Давайте представим вес как четырехзначное число `(0, 0, 0, 0)`:

1. **Inline-стили (в теге)**: `style="..."`. Дают **1000** баллов. Почти непобедимы.
2. **ID селекторы**: `#header`. Дают **100** баллов.
3. **Классы, Атрибуты, Псевдоклассы**: `.btn`, `[type]`, `:hover`. Дают **10** баллов.
4. **Теги и Псевдоэлементы**: `div`, `p`, `::before`. Дают **1** балл.

> [!tip] Универсальный селектор (`*`)
> Он имеет нулевую специфичность — (0, 0, 0, 0). Любое другое правило его перебьет.

### Примеры расчетов:
- `p` = 1 балл (0, 0, 0, 1)
- `.text` = 10 баллов (0, 0, 1, 0)
- `div .text` = 1 + 10 = 11 баллов (0, 0, 1, 1)
- `#main .sidebar p` = 100 + 10 + 1 = 111 баллов (0, 1, 1, 1)
- `body #main .sidebar p` = 1 + 100 + 10 + 1 = 112 баллов (0, 1, 1, 2)

**Важно**: 10 тегов никогда не перебьют 1 класс. Это не десятичная система, где 10 единиц превращаются в десяток. Это скорее категории «тяжеловесности».

---

## 🔨 3. !important (Ядерный вариант)

Если вы добавите к значению свойства ключевое слово `!important`, оно перебьет **любую** специфичность и **любой** порядок.

```css
p { color: red !important; }
#main .text { color: blue; } /* Красный всё равно победит */
```

> [!danger] Правило хорошего тона
> Использование `!important` считается признаком «плохого кода». Это костыль, который ломает естественную работу каскада. Используйте его только в крайних случаях (например, когда нужно перебить стили стороннего плагина, к коду которого у вас нет доступа).

---

## 👨‍👩‍👧 4. Наследование

Некоторые свойства (например, `color`, `font-family`, `line-height`) передаются от родителя к детям «по наследству». Если вы задали цвет текста для `<body>`, все абзацы на странице станут такими же, пока вы не зададите им другой цвет явно.

Но помните: **любой прямой селектор всегда сильнее наследования.**
Если у родителя `.parent { color: red; }`, а у ребенка `p { color: blue; }`, абзац будет синим, даже если у класса специфичность выше. Почему? Потому что для абзаца правило `p` — это «личное» указание, а `red` — всего лишь «наследство от папы».

---

## 🔍 Как браузер ищет правду? (Алгоритм)

1. Посмотреть, есть ли `!important`. Если да — это финал.
2. Посмотреть на специфичность. У кого больше баллов, тот и главный.
3. Если баллы равны — применить то, что написано ниже в коде.
4. Если правил нет — посмотреть, не наследуется ли свойство от родителя.
5. Если и наследства нет — применить значение по умолчанию (User Agent Styles).

## ❓ Часто задаваемые вопросы

### «Почему мои стили не работают, хотя я всё написал правильно?»
Откройте инструменты разработчика (F12) и посмотрите на вкладку **Computed** или **Styles**. Браузер зачеркивает те свойства, которые были перебиты более сильными селекторами. Это лучший способ найти «виновника».

### «Как мне перебить стиль ID, если я не могу менять HTML?»
Вам придется написать селектор с еще большим весом. Например, добавить к ID селектор тега или класса родителя: `body #my-id`.

### «Влияет ли вложенность на специфичность?»
Да. `.container .content p` (21 балл) сильнее, чем `.content p` (11 баллов). Чем точнее вы описываете путь к элементу, тем он «тяжелее».

## 🧪 Практика: Лаборатория приоритетов

Попробуйте угадать, какого цвета будет текст в каждом случае, а потом проверьте в браузере.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <style>
    /* 1 балл */
    div { color: black; }

    /* 10 баллов */
    .red-text { color: red; }

    /* 11 баллов */
    div .red-text { color: blue; }

    /* 100 баллов */
    #unique { color: green; }

    /* 1000 баллов (условно, если бы это было в style) */
    /* Но здесь мы добавим !important */
    .red-text { color: orange !important; }
  </style>
</head>
<body>

  <div id="unique" class="red-text" style="color: purple;">
    Какого я буду цвета?
  </div>

</body>
</html>
```

*Ответ*: Текст будет оранжевым из-за `!important`. Если убрать `!important`, победит `purple` (inline-стиль). Если убрать и его — победит `green` (ID).

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **Каскад** — это порядок и приоритеты.
> 2. **Специфичность** — это вес селектора.
> 3. Порядок веса: `!important` > `inline` > `ID` > `Class/Attr` > `Tag`.
> 4. При равном весе побеждает тот, кто **ниже в коде**.
> 5. Любой селектор сильнее наследования.
> 6. Не злоупотребляйте `!important` и `ID` в стилях — это усложняет жизнь в будущем.
