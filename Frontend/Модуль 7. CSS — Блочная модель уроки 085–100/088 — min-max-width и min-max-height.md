# ⚓ Ограничители размеров: Минимумы и Максимумы

← [[087 — width и height]] | [[089 — padding]] →

> [!info] Цели урока
> - Узнать, как создавать гибкие, но контролируемые макеты.
> - Понять разницу между «жесткой» шириной и «умными» ограничениями.
> - Научиться использовать `max-width` для адаптации сайтов под мобильные устройства.

---

## 📖 Введение: Золотая середина

Представьте, что вы строите дом. Вы можете сделать стены из бетона (фиксированная ширина), а можете сделать их из резины (процентная ширина). Но в вебе нам часто нужно что-то среднее.

Например, мы хотим, чтобы основной текст статьи растягивался на весь экран смартфона, но на огромном 27-дюймовом мониторе он не превращался в одну бесконечную строку, которую невозможно читать. Или мы хотим, чтобы кнопка была гибкой, но не сжималась настолько, что текст в ней превратится в кашу.

Для решения этих задач в CSS есть свойства-ограничители: `min-width`, `max-width`, `min-height` и `max-height`. Они — настоящие герои адаптивного дизайна.

---

## 🛡️ Свойство max-width (Максимальная ширина)

Это, пожалуй, самое важное свойство из всей четверки. Оно говорит браузеру: «Ты можешь быть любым, но не шире, чем вот столько».

### Как это работает на практике
Обычно мы задаем блоку `width: 100%` (чтобы он был гибким) и `max-width: 1200px` (чтобы он не стал слишком широким на больших экранах).

```css
.container {
  width: 100%;       /* Растянись на всё доступное место... */
  max-width: 1200px; /* ...но остановись на 1200 пикселях! */
  margin: 0 auto;    /* Центрирование (подробнее в уроке 090) */
}
```

**Почему это круто?**
1. На маленьком телефоне (ширина 375px) блок займет 100% ширины.
2. На планшете (768px) блок займет 100% ширины.
3. На мониторе (1920px) блок замрет на отметке 1200px и встанет по центру. Контент защищен!

---

## 🧱 Свойство min-width (Минимальная ширина)

Это свойство гарантирует, что элемент не станет меньше указанного размера.

### Когда это полезно?
- **Кнопки**: Мы хотим, чтобы кнопка всегда была удобной для нажатия, даже если на ней написано всего одна буква.
- **Сайдбары**: Боковая панель не должна превращаться в узкую полоску, где не помещаются пункты меню.

```css
.button {
  min-width: 120px; /* Даже если текст "ОК", кнопка будет 120px */
  padding: 10px;
}
```

> [!warning] Осторожно!
> Если вы зададите `min-width` больше, чем ширина экрана смартфона, у пользователя появится горизонтальная полоса прокрутки. Это считается плохим тоном в мобильной верстке.

## 📏 Ограничители высоты: min-height и max-height

Они работают точно так же, но по вертикали.

### min-height: Страховка для пустоты
Это свойство используется очень часто. Например, у вас есть блок для карточки товара. Если описания товара нет, блок может «схлопнуться» и стать некрасивым. `min-height` спасет ситуацию.

```css
.hero-section {
  min-height: 400px; /* Секция всегда будет высокой, даже если в ней нет текста */
  display: flex;
  align-items: center;
}
```

### max-height: Контроль переполнения
Используется реже, обычно для выпадающих списков или модальных окон, которые не должны уходить за нижний край экрана. Если контента внутри станет больше, чем `max-height`, он либо обрежется, либо появится внутренний скролл (зависит от свойства `overflow`).

---

## ⚖️ Приоритеты: Кто главнее?

Что произойдет, если вы зададите одному элементу сразу все свойства с противоречивыми значениями?

```css
.box {
  width: 500px;
  max-width: 300px;
}
```
**Результат**: Ширина будет 300px. Ограничитель `max-width` сильнее, чем обычный `width`.

Но есть одно железное правило: **min-width всегда побеждает max-width.**
```css
.box {
  min-width: 600px;
  max-width: 400px;
}
```
**Результат**: Ширина будет 600px. Безопасность (минимум) важнее ограничений (максимум).

---

## 🔍 Практический прием: Резиновые изображения

Это стандарт индустрии. Чтобы картинки никогда не «ломали» ваш макет и не вылезали за края контейнера, им задают следующие свойства:

```css
img {
  max-width: 100%; /* Не становись шире своего родителя */
  height: auto;    /* Сохраняй пропорции при изменении ширины */
}
```
С этим кодом любая огромная фотография аккуратно впишется даже в маленькую колонку.

## ❓ Часто задаваемые вопросы

### «Зачем использовать max-width: 1000px, если можно просто задать width: 1000px?»
Потому что `width: 1000px` — это «приговор». Элемент всегда будет 1000px, и на телефоне он просто уедет за край. `max-width: 1000px` говорит: «Будь 1000px на компьютере, но сжимайся на телефоне».

### «Можно ли использовать проценты в min-width?»
Да, можно. Например, `min-width: 50%`. Элемент никогда не станет меньше половины ширины своего родителя.

### «Как сделать блок, который занимает всю высоту экрана, но не меньше 500px?»
Комбинируйте!
`height: 100vh; min-height: 500px;`. Теперь на больших экранах он на всю высоту, а на очень маленьких (если телефон в горизонтальном режиме) он не превратится в «щель», а останется высотой 500px.

## 🧪 Практика: Гибкая карточка

Давайте создадим блок, который ведет себя по-разному в зависимости от количества текста и размера окна.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <style>
    .card {
      background: #f9f9f9;
      border: 1px solid #ddd;
      padding: 20px;
      margin: 20px auto;

      /* Магия здесь */
      width: 80%;
      max-width: 600px;
      min-width: 280px;
      min-height: 200px;
    }

    .content {
      background: white;
      padding: 10px;
      border-left: 5px solid #3498db;
    }
  </style>
</head>
<body>

  <div class="card">
    <h2>Гибкая карточка</h2>
    <div class="content">
      Попробуйте изменить размер окна браузера. <br>
      1. На узком экране я не стану меньше 280px. <br>
      2. На широком экране я не вырасту больше 600px. <br>
      3. Я всегда буду высотой минимум 200px, даже если тут мало текста.
    </div>
  </div>

</body>
</html>
```

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **max-width** — главный инструмент адаптивности (защищает от слишком большой ширины).
> 2. **min-width** — гарантирует наличие минимального места для контента.
> 3. **min-height** — часто используется для секций и карточек, чтобы они не «схлопывались».
> 4. Комбинация **width: 100%** + **max-width** — база современной верстки.
> 5. Ограничители всегда имеют приоритет над обычными `width` и `height`.
