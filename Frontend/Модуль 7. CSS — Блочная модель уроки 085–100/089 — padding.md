# 💨 Внутренние отступы: Свойство padding

## 📖 Введение: Зачем нужен воздух?

Вы когда-нибудь видели текст, который прилипает вплотную к границам рамки или края экрана? Читать такой текст неприятно, он кажется «зажатым». В дизайне интерьеров это называется теснотой, в веб-дизайне — отсутствием воздуха.

Чтобы контент внутри блока «дышал», используются внутренние отступы — `padding`. Это расстояние от края контента до внутренней границы рамки (`border`).

Представьте себе посылку: `padding` — это слой пупырчатой пленки между самой вещью и стенками коробки. Чем толще этот слой, тем больше коробка, но вещь внутри остается в безопасности и не касается стенок.

---

## 🛠️ Синтаксис и стороны

Мы можем задавать отступ для каждой стороны отдельно:

- `padding-top` — сверху.
- `padding-right` — справа.
- `padding-bottom` — снизу.
- `padding-left` — слева.

```css
.box {
  padding-top: 10px;
  padding-right: 20px;
  padding-bottom: 15px;
  padding-left: 5px;
}
```

Но писать четыре строки долго. В CSS есть удобная сокращенная запись.

### Сокращенная запись (Shortand)

Порядок значений всегда идет **по часовой стрелке**, начиная с 12 часов (верх).

1. **Одно значение**: `padding: 20px;` (все стороны по 20px).
2. **Два значения**: `padding: 10px 20px;` (первое — верх/низ, второе — лево/право).
3. **Три значения**: `padding: 10px 20px 30px;` (верх, лево/право, низ).
4. **Четыре значения**: `padding: 10px 20px 30px 40px;` (верх, право, низ, лево).

> [!tip] Как запомнить?
> Представьте циферблат часов: 12 (Top) → 3 (Right) → 6 (Bottom) → 9 (Left). TRBL (произносится как «трабл»).

## 📏 Padding и размер блока

Это самый важный технический момент урока. По умолчанию в CSS `padding` **добавляется** к ширине и высоте блока.

Если у вас есть блок:
```css
.box {
  width: 200px;
  height: 100px;
  padding: 20px;
  border: 5px solid black;
}
```
Его реальная ширина на экране будет **250px** (`200 + 20 + 20 + 5 + 5`).
Его реальная высота — **150px** (`100 + 20 + 20 + 5 + 5`).

Это часто приводит к тому, что верстка «разваливается», когда вы добавляете отступы. В уроке про `box-sizing` мы научимся это исправлять, но пока помните об этом эффекте «раздувания».

## 🎨 Особенности padding

1. **Цвет фона**: Внутренние отступы закрашиваются тем же фоновым цветом (`background-color`), что и сам контент. В этом их главное отличие от `margin` (внешних отступов), которые всегда прозрачны.
2. **Кликабельность**: Если блок является ссылкой, то область `padding` тоже будет кликабельной. Это очень важно для кнопок — мы увеличиваем `padding`, чтобы по кнопке было легче попасть пальцем на телефоне.
3. **Отрицательные значения**: В отличие от `margin`, `padding` **не может быть отрицательным**. Значение `-10px` просто проигнорируется браузером.
4. **Проценты**: Если вы задаете `padding: 10%`, этот процент рассчитывается от **ширины** родительского элемента (даже для верхнего и нижнего отступа!).

## 🔍 Padding у строчных элементов

В первом уроке модуля мы говорили, что вертикальные отступы у строчных элементов ведут себя странно.
Если вы зададите `<span>` отступ `padding: 20px`, визуально фон элемента расширится и может даже «наехать» на строки выше и ниже. Но при этом строки текста **не раздвинутся**.

> [!warning] Запомните
> Чтобы `padding` работал предсказуемо и раздвигал соседние элементы, элемент должен быть `block` или `inline-block`.

---

## ❓ Часто задаваемые вопросы

### «Что лучше использовать для кнопок: ширину или padding?»
Лучше использовать `padding`. Если вы зададите кнопке фиксированную ширину `width: 100px`, а текст внутри будет длинным (например, "Зарегистрироваться"), он вылезет за края. Если вы зададите `padding: 10px 20px`, кнопка всегда будет подстраиваться под размер текста, сохраняя красивые отступы.

### «Можно ли использовать разные единицы измерения?»
Конечно! Вы можете написать `padding: 1rem 10% 20px 5vw;`. Это будет работать, хотя такая смесь может усложнить поддержку кода.

### «Почему мой padding не работает?»
Проверьте, не является ли элемент строчным (`inline`). Если это ссылка или span, попробуйте добавить `display: inline-block`.

## 🧪 Практика: Создаем идеальную кнопку

Давайте создадим кнопку, используя только `padding` и фоновые свойства.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <style>
    .btn {
      /* Сброс стандартных стилей ссылки */
      text-decoration: none;
      display: inline-block;

      background-color: #3498db;
      color: white;
      font-family: sans-serif;
      border-radius: 5px;

      /* Магия отступов */
      padding: 12px 24px;

      transition: background 0.3s;
    }

    .btn:hover {
      background-color: #2980b9;
    }

    .container {
      padding: 50px;
      background-color: #f0f0f0;
    }
  </style>
</head>
<body>

  <div class="container">
    <a href="#" class="btn">Нажми на меня</a>
    <a href="#" class="btn">Короткая</a>
    <a href="#" class="btn">Очень-очень длинная кнопка для проверки</a>
  </div>

</body>
</html>
```

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **Padding** — это пространство ВНУТРИ элемента.
> 2. Он увеличивает размер блока (по умолчанию).
> 3. Порядок в сокращенной записи: **Верх → Право → Низ → Лево** (TRBL).
> 4. Padding закрашивается фоном элемента.
> 5. У строчных элементов вертикальный padding не раздвигает соседние строки.
