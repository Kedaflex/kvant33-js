## 📖 Введение: За пределами тегов и классов

Представьте, что у вас есть огромная форма с десятками полей `<input>`. Одни — для текста, другие — для паролей, третьи — для загрузки файлов. Все они называются одинаково — `input`. Как покрасить только поля для паролей, не добавляя каждому класс `.password-field`?

Или у вас есть список ссылок, и вы хотите автоматически добавлять маленькую иконку PDF рядом с теми, что ведут на документы.

В HTML у тегов есть атрибуты: `src`, `href`, `type`, `target`, `alt` и многие другие. CSS позволяет нам использовать значения этих атрибутов как условия для выбора. Это делает селекторы невероятно точными и «умными».

---

## 🛠️ Базовый синтаксис

Селектор атрибута всегда пишется в квадратных скобках: `[атрибут]`.

### 1. Наличие атрибута
Выбирает все элементы, у которых просто **есть** такой атрибут, неважно с каким значением.
```css
/* Выбрать все картинки, у которых есть alt (хорошо для доступности!) */
img[alt] {
  border: 2px solid green;
}
```

### 2. Точное совпадение
Выбирает элементы, значение атрибута которых **в точности** совпадает с указанным.
```css
/* Выбрать только поля для пароля */
input[type="password"] {
  background-color: #fff9c4;
}
```

---

## 🚀 Продвинутый поиск (Регулярные выражения)

Иногда нам не нужно точное совпадение, достаточно части слова. Здесь CSS предлагает магические символы:

### 1. Начинается с... (^)
Символ `^` ищет совпадение в **начале** строки. Идеально для поиска внешних ссылок.
```css
/* Все ссылки, начинающиеся на https */
a[href^="https"] {
  color: green;
}
```

### 2. Заканчивается на... ($)
Символ `$` ищет совпадение в **конце** строки. Незаменимо для стилизации файлов по расширению.
```css
/* Все ссылки на PDF-файлы */
a[href$=".pdf"] {
  padding-left: 20px;
  background: url('pdf-icon.png') no-repeat left center;
}
```

### 3. Содержит... (*)
Символ `*` ищет совпадение в **любом месте** строки.
```css
/* Выбрать все элементы, в ссылке которых есть слово 'google' */
a[href*="google"] {
  border-bottom: 2px solid #ea4335;
}
```

---

## 🔍 Тонкие настройки: Регистр и слова

- **Поиск слова в списке (~)**: Если атрибут содержит список слов через пробел (как `class`), `[attr~="word"]` найдет именно отдельное слово.
- **Игнорирование регистра (i)**: Если вы не уверены, написано расширение файла `.PDF` или `.pdf`, добавьте букву `i` перед закрывающей скобкой.
  `a[href$=".pdf" i]` — найдет оба варианта.

## 🎨 Практические примеры

### Стилизация иконок для соцсетей
Вместо того чтобы плодить классы вроде `.icon-fb`, `.icon-tw`, можно написать:
```css
a[href*="facebook.com"] { background: blue; }
a[href*="twitter.com"] { background: lightblue; }
```

### Скрытие «битых» картинок
```css
img:not([src]), img[src=""] {
  display: none;
}
```

### Стилизация обязательных полей
```css
input[required] {
  border-left: 5px solid red;
}
```

---

## ⚖️ Почему это лучше классов?

1. **Меньше кода**: Вы не дублируете в CSS то, что и так уже написано в HTML.
2. **Надежность**: Вы стилизуете функционал. Если поле `type="email"` изменится на `type="text"`, стиль применится автоматически (или перестанет применяться), что логично.
3. **Автоматизация**: Вы можете стилизовать контент, который генерируется автоматически (например, ссылки в статьях из CMS), где вы не можете вручную расставить классы.

## ❓ Часто задаваемые вопросы

### «Замедляют ли селекторы атрибутов сайт?»
Немного больше, чем классы, особенно если использовать поиск «содержит» (`*`). Но на 99% сайтов вы никогда не заметите разницы. Это профессиональный инструмент, и его можно смело использовать.

### «Можно ли комбинировать атрибуты?»
Да! `input[type="text"][name="username"]` выберет текстовое поле, у которого имя именно "username".

### «Работает ли это с data-атрибутами?»
Да, и это одно из главных применений. Верстальщики часто используют `data-` атрибуты для своих нужд, и стилизовать их через `[data-status="active"]` — отличная практика.

## 🧪 Практика: Умный список загрузок

Давайте создадим список файлов, который сам «понимает», какую иконку поставить рядом.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <style>
    body { font-family: sans-serif; padding: 20px; }

    ul { list-style: none; padding: 0; }

    li { margin-bottom: 10px; }

    a {
      text-decoration: none;
      color: #333;
      padding: 5px 5px 5px 30px;
      display: inline-block;
      border: 1px solid transparent;
    }

    /* Базовые иконки через эмодзи (для простоты примера) */
    a[href$=".pdf"]::before { content: "📕 "; position: absolute; margin-left: -25px; }
    a[href$=".doc"]::before { content: "📘 "; position: absolute; margin-left: -25px; }
    a[href$=".zip"]::before { content: "📦 "; position: absolute; margin-left: -25px; }

    /* Внешние ссылки */
    a[href^="http"] {
      color: #3498db;
      border-bottom: 1px dashed #3498db;
    }

    a[href^="http"]::after {
      content: " ↗";
      font-size: 0.8em;
    }

    /* Ссылки внутри сайта (начинаются с /) */
    a[href^="/"] {
      font-weight: bold;
    }
  </style>
</head>
<body>

  <h2>Ваши документы:</h2>
  <ul>
    <li><a href="manual.pdf">Инструкция (PDF)</a></li>
    <li><a href="report.doc">Отчет (DOC)</a></li>
    <li><a href="archive.zip">Все фото (ZIP)</a></li>
    <li><a href="/about">О компании (Внутренняя ссылка)</a></li>
    <li><a href="https://google.com">Поиск в Google (Внешняя ссылка)</a></li>
  </ul>

</body>
</html>
```

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. `[attr]` — просто наличие атрибута.
> 2. `[attr="val"]` — точное совпадение.
> 3. `^=` — начинается с.
> 4. `$=` — заканчивается на.
> 5. `*=` — содержит внутри.
> 6. Селекторы атрибутов позволяют стилизовать элементы на основе их смысла и функции, а не только внешнего вида.
