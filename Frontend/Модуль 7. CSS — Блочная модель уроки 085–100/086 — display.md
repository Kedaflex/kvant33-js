# 🪄 Свойство display: Магия трансформации

> - Научиться скрывать элементы так, чтобы они не занимали место на странице.

## 📖 Введение: Смена ролей

В прошлом уроке мы узнали, что у каждого HTML-элемента есть своя «природа»: `<div>` рождается блочным, а `<span>` — строчным. В эпоху раннего интернета это было законом. Если вам нужна была кнопка в ряд с текстом, вы использовали строчный элемент. Если нужен был заголовок — блочный.

Но современный веб-дизайн требует гибкости. Что если мы хотим, чтобы ссылка выглядела как большая кнопка? Или чтобы пункты меню (которые являются блочными `<li>`) выстроились в красивый горизонтальный ряд?

Для этого в CSS существует свойство `display`. Оно позволяет нам менять тип отображения элемента, не меняя его сути в HTML. Это одно из самых мощных свойств в арсенале верстальщика.

---

## 🏗️ display: block

Это значение превращает любой элемент в блочный. Даже если вы примените его к ссылке `<a>` или маленькому `<span>`, они начнут вести себя как гордый `<div>`.

### Когда это полезно?
- **Вертикальные меню**: Превращаем ссылки в блоки, чтобы они занимали всю ширину боковой панели и на них было легче нажать.
- **Карточки-ссылки**: Когда мы хотим, чтобы вся область карточки (с заголовком и текстом) была кликабельной.
- **Оформление**: Когда строчному элементу нужно задать четкую ширину или высоту.

```css
/* Теперь ссылка занимает всю строку */
a.button-large {
  display: block;
  background: #3498db;
  color: white;
  padding: 15px;
  text-align: center;
}
```

---

## 🧵 display: inline

Это значение делает элемент строчным. Он встраивается в поток текста и занимает ровно столько места, сколько нужно его содержимому.

### Когда это полезно?
- **Текстовые вставки**: Если вы по какой-то причине использовали `<div>` для выделения слова в тексте (что странно, но бывает), `display: inline` вернет его в строй.
- **Списки в строку**: Иногда (хотя сейчас для этого есть способы лучше) это используется для выравнивания элементов списка.

> [!warning] Напоминание
> Помните, что как только вы ставите `display: inline`, свойства `width`, `height` и вертикальные `margin` перестают работать в привычном понимании.

---

## 🧩 display: inline-block (Лучшее из двух миров)

Это, пожалуй, самое интересное значение для новичка. Оно объединяет в себе преимущества и блоков, и строк.

**Как он работает?**
- Снаружи он ведет себя как **строка**: элементы выстраиваются в ряд друг за другом.
- Внутри он ведет себя как **блок**: ему можно задавать `width`, `height`, `padding` и все виды `margin`.

### Когда это полезно?
- **Горизонтальные списки**: Идеально для навигационных меню.
- **Кнопки**: Когда нам нужны кнопки с четкими размерами, стоящие в один ряд.
- **Карточки товаров**: Если мы хотим расположить несколько плиток в ряд без использования сложных систем вроде Flexbox или Grid.

```css
/* Элементы меню стоят в ряд, но имеют размеры */
.menu-item {
  display: inline-block;
  width: 120px;
  padding: 10px;
  text-align: center;
  background: #f1f1f1;
}
```

## 👻 display: none (Исчезновение)

Это значение полностью убирает элемент с экрана. Элемент не просто становится невидимым — он перестает занимать какое-либо место, как будто его вообще нет в HTML-коде.

### Важные нюансы:
1. **Место освобождается**: Соседние элементы сдвигаются, заполняя пустоту.
2. **Доступность**: Элемент с `display: none` игнорируется программами-экранными дикторами (для слабовидящих).
3. **JS-взаимодействие**: Элемент всё еще существует в структуре документа (DOM). Вы можете «вернуть» его, изменив через JavaScript `display: none` на `display: block`.

### Отличие от visibility: hidden
Часто путают `display: none` и `visibility: hidden`.
- `visibility: hidden` делает элемент невидимым, но **оставляет пустое место**, которое он занимал.
- `display: none` «схлопывает» элемент в ноль.

## ⚖️ Сводная таблица display

| Значение | Поведение в ряду | Принимает width/height? | Занимает место? |
| :--- | :--- | :--- | :--- |
| `block` | С новой строки | Да | Да |
| `inline` | В ряд с текстом | Нет | Да |
| `inline-block`| В ряд | Да | Да |
| `none` | Исчезает | — | Нет |

## 🔍 Тонкости и продвинутые значения

Свойство `display` — это огромный мир. Кроме базовых значений, есть современные системы раскладки:
- `display: flex`: Включает режим Flexbox (Модуль 9).
- `display: grid`: Включает режим Grid (Модуль 10).
- `display: table`: Заставляет элементы вести себя как ячейки таблицы (устаревший способ верстки, но иногда полезен).

Также есть значение `display: contents`. Оно заставляет браузер игнорировать сам контейнер, но отображать его дочерние элементы так, как будто они являются прямыми потомками родителя. Это бывает полезно для исправления проблем с доступностью или версткой.

## ❓ Часто задаваемые вопросы

### «Почему `inline-block` элементы не стоят вплотную, между ними какой-то зазор?»
Это те самые 4-5 пикселей пробела, о которых мы говорили в прошлом уроке. Они возникают из-за переносов строк в вашем HTML-коде. Решение: либо писать код в одну строку, либо использовать Flexbox.

### «Я скрыл элемент через `display: none`, а он всё равно виден в поиске Google?»
Да, поисковые роботы видят HTML-код. `display: none` скрывает элемент только от глаз пользователя в браузере.

### «Можно ли анимировать переход от `display: none` к `display: block`?»
Напрямую — нет. Браузер не может плавно «рассчитать» переход от состояния «меня нет» к состоянию «я есть». Для плавного появления обычно используют прозрачность (`opacity`) вместе с `display`.

## 🧪 Практика: Переключатель вида

Давайте создадим простую навигацию и посмотрим, как свойство `display` меняет её облик.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <style>
    nav ul {
      list-style: none; /* Убираем точки списка */
      padding: 0;
      background: #eee;
    }

    /* По умолчанию делаем список горизонтальным */
    nav li {
      display: inline-block;
      margin-right: 20px;
      padding: 10px;
      border: 1px solid #ccc;
    }

    /* А теперь попробуйте в консоли браузера поменять
       inline-block на block для nav li */

    .secret-box {
      display: none;
      background: lightcoral;
      padding: 20px;
      margin-top: 20px;
    }

    /* При наведении на меню показываем секретный блок */
    nav:hover + .secret-box {
      display: block;
    }
  </style>
</head>
<body>

<nav>
  <ul>
    <li>Главная</li>
    <li>О нас</li>
    <li>Контакты</li>
  </ul>
</nav>

<div class="secret-box">
  Ух ты! Ты навел мышку на меню и я появился!
</div>

</body>
</html>
```

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **display** — это переключатель «характера» элемента.
> 2. `block` — растягивается на всю ширину, всё остальное — под ним.
> 3. `inline` — течет вместе с текстом, игнорирует размеры.
> 4. `inline-block` — стоит в ряду, но слушается команд по размеру.
> 5. `none` — полностью удаляет элемент из визуального потока.
> 6. Не путайте `display: none` (удаление) и `visibility: hidden` (прозрачность с сохранением места).
