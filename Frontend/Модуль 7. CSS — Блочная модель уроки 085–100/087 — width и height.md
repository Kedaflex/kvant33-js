# 📏 Ширина и высота: Укрощение размеров

> - Понять, как работают свойства `width` и `height`.
> - Узнать разницу между фиксированными, процентными и автоматическими размерами.
> - Разобраться, когда стоит задавать размеры, а когда — довериться контенту.

## 📖 Введение: Геометрия веба

В мире дизайна всё имеет свои границы. Лист бумаги имеет формат A4, экран смартфона — определенное разрешение. В CSS мы тоже должны уметь задавать размеры нашим «коробкам».

Казалось бы, что может быть проще? `width` — это ширина, `height` — это высота. Но в веб-разработке размеры работают немного иначе, чем в графических редакторах вроде Photoshop или Figma. Здесь элементы живут в «потоке», они могут растягиваться в зависимости от текста внутри или размера окна браузера.

Если вы будете задавать фиксированные размеры всем подряд элементам, ваш сайт «сломается», как только пользователь откроет его на телефоне. Поэтому наша задача сегодня — не просто научиться задавать цифры, а понять философию размеров в вебе.

---

## 📐 Свойство width (Ширина)

Свойство `width` определяет горизонтальный размер контентной области элемента.

### Основные значения:

1. **px (Пиксели)**: Фиксированный размер.
   ```css
   .box { width: 300px; }
   ```
   *Плюс*: Точный контроль. *Минус*: Не гибко, может «вылезти» за края маленького экрана.

2. **% (Проценты)**: Относительный размер. Рассчитывается от ширины **родительского** элемента.
   ```css
   .sidebar { width: 25%; } /* Занимает четверть родителя */
   ```
   *Плюс*: Адаптивность. Элемент сужается и расширяется вместе с окном.

3. **auto (Автоматически)**: Значение по умолчанию.
   - Для **блочных** элементов `width: auto` означает «занять всё доступное пространство по горизонтали» (учитывая отступы).
   - Для **строчных** и **inline-block** элементов — «занять столько места, сколько нужно контенту».

## 🔝 Свойство height (Высота)

Свойство `height` определяет вертикальный размер. И здесь новичков ждет самый большой сюрприз.

### «Проблема» процентов в высоте
Если вы зададите `width: 50%`, элемент послушно займет половину ширины. Но если вы зададите `height: 50%`, скорее всего, **ничего не произойдет**. Почему?

Потому что браузер знает ширину родителя (она ограничена окном браузера), но он «не знает» его высоту. Высота веб-страницы в теории бесконечна (скролл вниз).
> [!warning] Правило высоты
> Чтобы процентная высота сработала, у родительского элемента должна быть явно задана высота (в пикселях или других единицах).

### Почему height: auto — ваш лучший друг
В 90% случаев для блоков стоит оставлять `height: auto`.
- Контент (текст, картинки) сам растянет блок до нужной высоты.
- Если вы зададите фиксированную высоту (например, `height: 100px`), а текста будет много, он просто «вывалится» из блока, накладываясь на другие элементы. Это называется **переполнением (overflow)**.

## 🧬 Относительные единицы: vw и vh

В современном CSS часто используются единицы, привязанные к размеру экрана (viewport):
- **1vw** = 1% от ширины окна браузера.
- **1vh** = 1% от высоты окна браузера.

Хотите сделать блок на весь первый экран? Используйте `height: 100vh;`. Теперь, независимо от устройства, пользователь увидит этот блок растянутым на всю высоту своего дисплея.

## 🧬 Внутренние размеры: min-content, max-content и fit-content

Помимо привычных цифр и процентов, в современном CSS есть «умные» ключевые слова, которые позволяют элементу самому определять свой размер на основе содержимого.

1. **max-content**: Элемент растягивается настолько, насколько это нужно его самому длинному содержимому без переносов. Если это длинное предложение, блок станет очень длинным и выйдет за экран, но не перенесет текст.
2. **min-content**: Элемент сжимается до размера самого длинного слова или неразрывного элемента (например, картинки) внутри. Текст будет максимально переноситься по строкам.
3. **fit-content**: Самое полезное из трех. Элемент ведет себя как `max-content`, но не позволяет себе стать шире, чем доступное пространство (родительский блок).

```css
.badge {
  width: fit-content;
  background: gold;
  padding: 5px 10px;
}
```
Это идеальный способ сделать «плашку» или «тег», который будет ровно по ширине текста, но не растянется на всю строку как обычный `div`.

## 🔍 Интрига: На что влияет width?

Важно понимать, что по умолчанию `width` задает ширину только **внутренней части** (контента). Если вы добавите рамку (`border`) или внутренние отступы (`padding`), реальный размер блока на экране станет **больше**, чем вы указали в `width`.

*Пример:*
```css
.box {
  width: 200px;
  padding: 20px;
  border: 10px solid black;
}
```
Реальная ширина блока будет: `200 + 20(лев) + 20(прав) + 10(лев) + 10(прав) = 260px`.
Это часто сбивает с толку, и в уроке 092 мы узнаем, как это исправить с помощью свойства `box-sizing`.

## ❓ Часто задаваемые вопросы

### «Когда всё-таки можно использовать фиксированную высоту?»
Обычно фиксированную высоту задают маленьким элементам, размер которых не зависит от текста: иконкам, кнопкам (и то лучше через padding), шапкам сайта (header), если там строгий дизайн.

### «Чем отличается width: 100% от width: auto?»
Это отличный вопрос!
- `width: 100%`: Блок будет ровно такой же ширины, как родитель. Если у блока есть `margin` или `padding`, он может «вылезти» за границы родителя.
- `width: auto`: Блок «умный». Он займет всё место, но аккуратно впишется внутрь, учитывая свои отступы и рамки.

> [!tip] Совет
> Для обычных `div`-ов почти всегда лучше использовать `width: auto` (или просто ничего не писать, так как это значение по умолчанию).

## 🧪 Практика: Экспериментируем с гибкостью

Создайте два блока и попробуйте изменить размер окна браузера. Посмотрите, как они себя ведут.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <style>
    .fixed {
      width: 500px;
      height: 150px;
      background: #e74c3c;
      color: white;
      margin-bottom: 20px;
    }

    .fluid {
      width: 80%;
      height: auto; /* Высота подстроится под текст */
      background: #2ecc71;
      color: white;
      padding: 20px;
    }
  </style>
</head>
<body>

  <div class="fixed">
    Я фиксированный блок (500px).
    Если окно станет меньше 500px, я не сужусь, а появится полоса прокрутки.
  </div>

  <div class="fluid">
    Я гибкий блок (80%).
    Я всегда занимаю большую часть экрана.
    Моя высота меняется в зависимости от того, сколько слов во мне написано.
    Попробуйте сжать окно браузера!
  </div>

</body>
</html>
```

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **width: auto** — стандарт для блоков (занимает всё место).
> 2. **width: 100%** — может вызвать проблемы с выходом за границы из-за padding/margin.
> 3. **height: auto** — лучшее решение в большинстве случаев. Пусть контент решает, какой высоты будет блок.
> 4. **vh и vw** — единицы, зависящие от размера окна браузера (viewport).
> 5. Фиксированные размеры (px) — враг адаптивности, используйте их осторожно.
