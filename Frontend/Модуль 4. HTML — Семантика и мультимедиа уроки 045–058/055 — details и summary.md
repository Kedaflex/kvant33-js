## 📖 Введение: Интерактив без кода

Долгое время, чтобы сделать на сайте простую кнопку «Показать больше» или блок с вопросами и ответами (FAQ), разработчикам приходилось писать код на JavaScript. Нужно было отслеживать клик, менять стили, следить за анимацией.

Стандарт HTML5 принес нам удивительную пару тегов — **`<details>`** и **`<summary>`**. Они позволяют создавать полностью функциональные раскрывающиеся блоки, используя только HTML. Браузер сам берет на себя всю логику: нажатие, открытие, закрытие и даже смену иконки-стрелочки.

---

## 🏗️ Базовый синтаксис

Конструкция всегда состоит из двух частей:

```html
<details>
  <summary>Нажми меня, чтобы узнать секрет</summary>
  <p>А вот и секрет! Этот текст был скрыт, пока ты не кликнул на заголовок.</p>
</details>
```

### Как это работает?
- **`<details>`**: Тег-контейнер. Всё, что находится внутри него (кроме `summary`), будет скрыто по умолчанию.
- **`<summary>`**: Заголовок блока. Это то, что пользователь видит всегда. При клике на этот текст блок раскрывается.

---

## ⚙️ Атрибут `open`

У тега `<details>` есть один важный атрибут — **`open`**.

1. Если атрибута нет: блок закрыт (по умолчанию).
2. Если атрибут есть: блок будет открыт сразу при загрузке страницы.

```html
<details open>
  <summary>Этот блок уже открыт</summary>
  <p>Вы видите этот текст сразу.</p>
</details>
```

Этот атрибут полезен, если вы хотите выделить первый вопрос в списке FAQ или привлечь внимание к важной информации.

---

## 🎨 Стилизация: Как сделать красиво?

Хотя браузер дает стандартный вид (текст со стрелочкой слева), мы можем сильно изменить его через CSS.

### 1. Убираем или меняем стрелочку
Стрелочка (маркер) стилизуется через специальный псевдоэлемент:
```css
/* Для Chrome, Safari, Edge */
details summary::-webkit-details-marker {
  display: none;
}

/* Для Firefox и современных стандартов */
summary {
  list-style: none;
}
```

### 2. Стили при открытии
Мы можем менять оформление блока, когда он открыт, используя селектор атрибута:
```css
details[open] {
  background-color: #f9f9f9;
  border: 1px solid #ccc;
}
```

## 💡 Сценарии использования

Где эти теги пригодятся вам в реальных проектах?

1. **FAQ (Вопросы и ответы)**: Самый популярный кейс. Десятки вопросов не занимают много места, пока пользователь не захочет прочитать ответ.
2. **Спойлеры**: Скрыть концовку фильма или решение задачи в обучающей статье.
3. **Технические подробности**: В карточке товара можно скрыть детальные характеристики (вес, размеры, вольтаж), оставив только главное.
4. **Виджеты в сайдбаре**: Список архивов или категорий, который можно свернуть.

## ♿ Доступность (A11y)

Теги `<details>` и `<summary>` очень хороши для доступности, потому что:
- Они работают с клавиатуры (можно переключиться кнопкой **Tab** и нажать **Enter** или **Space** для открытия).
- Скринридеры понимают их роль и сообщают пользователю: «Кнопка, свернуто» или «Кнопка, развернуто».

> [!warning] Важно
> Внутри `<summary>` нельзя вкладывать другие интерактивные элементы, такие как ссылки (`<a>`) или кнопки (`<button>`). Это запутает и браузер, и пользователя. Внутри `summary` должен быть только текст или иконка.

## ⚔️ HTML vs JavaScript Аккордеоны

Стоит ли всегда использовать `<details>`?

| Характеристика | `<details>` (HTML) | Аккордеон на JS |
|---|---|---|
| **Сложность** | Очень низкая | Средняя |
| **Вес кода** | 0 КБ | Минимум 1-2 КБ скриптов |
| **Анимация** | Сложно анимировать плавно | Любая плавная анимация |
| **Поиск (SEO)** | Текст внутри индексируется | Зависит от реализации |
| **Взаимодействие** | Блоки открываются независимо | Можно сделать, чтобы при открытии одного закрывался другой |

Если вам нужна простая функциональность — выбирайте HTML. Если нужен сложный дизайн с плавным «выезжанием» контента и хитрой логикой — придется использовать JavaScript.

## 🧪 Практика: Создаем блок FAQ

Давайте сделаем профессиональный раздел вопросов и ответов для сайта.

```html
<section class="faq">
    <h2>Часто задаваемые вопросы</h2>

    <details>
        <summary>Сколько времени занимает обучение?</summary>
        <div class="answer">
            <p>Основы можно освоить за 1-2 месяца регулярных занятий. Полный путь до трудоустройства обычно занимает от 6 до 9 месяцев.</p>
        </div>
    </details>

    <details>
        <summary>Нужен ли мощный компьютер?</summary>
        <div class="answer">
            <p>Нет, для верстки сайтов подойдет любой компьютер, на котором запускается современный браузер и текстовый редактор VS Code.</p>
        </div>
    </details>

    <details>
        <summary>Вы выдаете сертификат?</summary>
        <div class="answer">
            <p>Да, после выполнения всех практических работ и финального проекта вы получите цифровой сертификат нашей школы.</p>
        </div>
    </details>
</section>
```

## ⚠️ Типичные ошибки

1. **Забытый тег `<summary>`**: Если вы его не напишете, браузер сам добавит заголовок «Подробнее» (или "Details" на английском). Это выглядит непрофессионально.
2. **Вложенность**: Пытаться вложить `<summary>` не первым элементом в `<details>`. По стандарту он должен быть первым.
3. **Отсутствие стилей**: Без CSS `details` выглядит очень скромно. Не забывайте добавлять хотя бы небольшие отступы и курсор `cursor: pointer` для заголовка.
4. **Использование для навигации**: Не стоит делать главное меню сайта через `details`. Это может быть неудобно для пользователей мобильных устройств.

## ❓ Часто задаваемые вопросы

### «Можно ли сделать так, чтобы при открытии одного блока закрывались остальные?»
Средствами чистого HTML — нет. Каждый `<details>` живет своей жизнью. Если вам нужно поведение «только один открытый блок», понадобится пара строк кода на JavaScript.

### «Индексируется ли скрытый текст поисковиками?»
Да! Google и Яндекс прекрасно видят текст внутри закрытого `<details>`. Это отличный способ добавить на страницу много полезного текста (для SEO), не загромождая интерфейс для пользователя.

### «Как добавить плавную анимацию открытия?»
В чистом CSS это сделать довольно сложно, так как высота блока меняется с 0 до `auto`. Самый простой способ — анимировать `opacity` (прозрачность) контента внутри, но само «раскрытие» в старых браузерах всё равно будет мгновенным. В современных браузерах (с 2023-2024 года) появляется поддержка `interpolate-size`, которая решит эту проблему.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **`<details>` и `<summary>`** — это нативный способ создать интерактивный раскрывающийся блок.
> 2. **`<summary>`** — это видимый заголовок, на который нужно кликать.
> 3. Атрибут **`open`** позволяет оставить блок открытым при загрузке.
> 4. Текст внутри блоков **индексируется поисковиками**.
> 5. Это идеальное решение для FAQ, спойлеров и дополнительных сносок.
> 6. Работает без JavaScript и отлично поддерживает доступность (клавиатура, скринридеры).
