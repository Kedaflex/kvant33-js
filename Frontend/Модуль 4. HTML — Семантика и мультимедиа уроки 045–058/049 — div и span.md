## 📖 Введение: Универсальные солдаты

На прошлых уроках мы много говорили о семантике: тегах, которые имеют смысл. Но что делать, если нам нужно просто обернуть три кнопки, чтобы подвинуть их вправо? Или выделить одно слово в предложении красным цветом?

Для таких случаев в HTML есть «пустые» теги: **`<div>`** и **`<span>`**. У них нет врожденного смысла, они не говорят поисковикам «это статья» или «это навигация». Они — просто контейнеры, чистые листы, на которых вы рисуете с помощью CSS.

---

## 📦 Тег `<div>` (Блочный контейнер)

Название `div` происходит от английского *division* (деление, раздел). Это самый популярный тег в истории веба.

### Характеристики `div`:
- **Блочный элемент**: Он всегда начинается с новой строки и стремится занять всю доступную ширину родителя.
- **Предназначение**: Группировка других элементов для верстки макетов, создания сеток и применения стилей к целым блокам.

### Когда использовать `div`?
1. **Для дизайна**: Если блок нужен только для того, чтобы задать фоновый цвет, рамку или отступы.
2. **Для сеток**: Когда вы используете Flexbox или CSS Grid, вам часто нужны технические обертки для элементов.
3. **Когда нет семантического аналога**: Если контент внутри не подходит ни под один из семантических тегов (article, section и т.д.).

```html
<!-- Контейнер для центрирования контента -->
<div class="container">
  <h1>Заголовок</h1>
  <p>Текст...</p>
</div>
```

## 🏷️ Тег `<span>` (Строчный контейнер)

Тег `span` — это «младший брат» тега `div`.

### Характеристики `span`:
- **Строчный элемент**: Он не начинает новую строку. Он живет внутри строки текста и занимает ровно столько места, сколько нужно его содержимому.
- **Предназначение**: Выделение части текста или небольших элементов внутри строки для стилизации через CSS или манипуляций через JavaScript.

### Когда использовать `span`?
1. **Смена цвета части текста**: Выделить одно слово другим цветом.
2. **Иконки в тексте**: Добавить маленькую иконку рядом со словом.
3. **Маркеры и счетчики**: Например, число непрочитанных сообщений в меню.

```html
<p>Ваш баланс: <span class="price">1500 рублей</span></p>
```

## ⚖️ Div vs Span: В чем разница?

Главное отличие — в их поведении в потоке документа.

| Характеристика | `<div>` | `<span>` |
|---|---|---|
| **Тип** | Блочный (Block) | Строчный (Inline) |
| **С новой строки?** | Да | Нет |
| **Ширина** | 100% родителя по умолчанию | Только по контенту |
| **Для чего?** | Для крупных блоков, макетов | Для текста, мелких деталей |
| **Вложенность** | Можно класть div, span, p, h1 и т.д. | Можно класть только строчные элементы (span, a, em) |

> [!warning] Важное правило вложенности
> Нельзя класть блочные элементы (`div`, `p`, `h1`) внутрь строчных элементов (`span`, `a`). Это нарушение стандартов HTML, которое может привести к ошибкам отображения.

## 🎭 Когда использовать их вместо семантики?

Хотя мы стремимся к семантичному коду, `div` и `span` всё еще незаменимы.

### Сценарий 1: Обертка для центрирования
Многие сайты имеют контент, который не растягивается на весь экран, а идет узкой полосой по центру. Для этого используют техническую обертку:
```html
<header>
  <!-- Это не навигация и не секция, это просто технический блок -->
  <div class="wrapper">
    <img src="logo.png">
    <nav>...</nav>
  </div>
</header>
```

### Сценарий 2: Декоративные элементы
Вам нужно добавить на фон страницы «красивые кружочки». Кружочки не несут смысла, они — часть дизайна.
```html
<div class="background-circle"></div>
```

## 🦠 Болезнь "Дивоз" (Divitis)

Новички часто впадают в крайность и оборачивают каждый чих в три слоя `div`. Это делает код «тяжелым» и непонятным.

### Плохой пример (Дивоз):
```html
<div class="header">
  <div class="nav-container">
    <div class="nav-item"><a href="#">Главная</a></div>
  </div>
</div>
```

### Хороший пример (Чистота):
```html
<header>
  <nav>
    <a href="#">Главная</a>
  </nav>
</header>
```

**Как бороться с дивозом?**
Задайте себе вопрос: «Могу ли я применить этот стиль напрямую к тегу `p` или `nav`?». Если да — лишний `div` не нужен.

---

## ♿ Доступность и универсальные теги

Для программ чтения с экрана (скринридеров) `div` и `span` — это «призраки». Они их не замечают.
- Если вы сделаете кнопку через `<div class="btn">`, скринридер не скажет пользователю, что это кнопка. Пользователь просто услышит текст внутри, но не поймет, что на него можно нажать.
- Всегда используйте интерактивные теги (`<a>`, `<button>`) вместо оберток `div`, если элемент должен что-то делать.

## 🧪 Практика: Группировка и стилизация

Давайте посмотрим, как `div` помогает организовать карточку товара, где нам нужно сгруппировать цену и кнопку.

```html
<article class="product-card">
    <img src="phone.jpg" alt="Смартфон X">
    <h2>Смартфон X</h2>
    <p>Мощный процессор и отличная камера.</p>

    <!-- Используем div для группировки нижнего ряда -->
    <div class="product-footer">
        <span class="old-price">50 000 руб.</span>
        <span class="current-price">45 000 руб.</span>
        <button type="button">В корзину</button>
    </div>
</article>
```

Здесь `div` с классом `product-footer` позволит нам в CSS выстроить цену и кнопку в одну линию (например, через Flexbox), а `span` позволит сделать старую цену зачеркнутой и серой.

## ❓ Часто задаваемые вопросы

### «Что лучше использовать для кнопок: div или button?»
Всегда `<button>`. Кнопка, сделанная через `div`, не работает с клавиатуры (клавиша Enter) и не видна скринридерам как интерактивный элемент.

### «Можно ли задать span ширину и высоту?»
По умолчанию — нет. Строчные элементы игнорируют `width` и `height`. Чтобы это заработало, в CSS нужно изменить тип отображения на `inline-block` или `block`.

### «Если я использую только семантические теги, мой код будет идеальным?»
Не совсем. Избыточная семантика (например, `<section>` внутри которого только одна строка текста) — это так же плохо, как и дивоз. Баланс — ключ к успеху.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **`<div>`** — блочный контейнер без смысла, используется для макетов и дизайна.
> 2. **`<span>`** — строчный контейнер без смысла, используется для оформления частей текста.
> 3. Эти теги **невидимы** для скринридеров и поисковиков.
> 4. Не используйте их там, где есть **семантический аналог** (например, `<header>` лучше, чем `<div class="header">`).
> 5. **Дивоз** — это плохо. Пишите чистый код, не плодите лишние обертки.
