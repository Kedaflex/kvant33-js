## 📖 Введение: Универсальные ключи

До этого момента мы изучали атрибуты, которые специфичны для конкретных тегов: `href` для ссылок, `src` для картинок, `controls` для аудио. Но в HTML существует группа атрибутов, которые можно добавить абсолютно к **любому** элементу: будь то заголовок, абзац или даже пустой `div`.

Эти атрибуты называются **глобальными**. Они являются «мостиками», которые позволяют нам обращаться к элементам из CSS (для красоты) или из JavaScript (для логики), а также добавлять элементам дополнительные свойства, понятные браузеру.

---

## 🏗️ Основная четверка

Эти четыре атрибута вы будете использовать в 99% своих проектов.

### 1. `class` (Классы)
Самый важный атрибут для стилизации. Он позволяет объединять элементы в группы.
- Одному элементу можно дать **несколько классов** (через пробел).
- Один и тот же класс можно использовать на странице **сколько угодно раз**.
```html
<p class="text-red bold centered">Этот текст красный, жирный и по центру.</p>
```

### 2. `id` (Идентификатор)
Это уникальное имя элемента.
- На одной странице значение `id` должно встречаться **только один раз**.
- Используется для якорных ссылок (переход к разделу) и для очень точной настройки в JavaScript.
```html
<section id="contact-form">...</section>
```

### 3. `title` (Всплывающая подсказка)
Добавляет поясняющий текст, который появляется, если пользователь задержит курсор мыши над элементом на пару секунд.
- Полезно для иконок без текста или сокращений.
```html
<abbr title="HyperText Markup Language">HTML</abbr>
```

### 4. `style` (Внутренние стили)
Позволяет писать CSS-код прямо внутри тега.
- **Плохая практика**: Старайтесь не использовать его для постоянного дизайна. Стили должны лежать в отдельном файле.
- **Когда можно?**: Для быстрых правок, тестов или если значение стиля вычисляется динамически на лету.
```html
<div style="color: blue; margin-top: 20px;">Текст</div>
```

## 🛠️ Атрибуты управления состоянием

### `hidden` (Скрыть элемент)
Если этот атрибут присутствует, элемент полностью исчезает со страницы.
- Это семантически правильнее, чем `display: none` в CSS, если вы хотите сказать: «Этот элемент сейчас не имеет значения».
```html
<p hidden>Вы не увидите этот текст.</p>
```

### `tabindex` (Порядок навигации)
Определяет, в каком порядке элементы будут получать фокус при нажатии клавиши **Tab**.
- `tabindex="0"`: Делает элемент (например, `div`) доступным для клавиатуры.
- `tabindex="-1"`: Исключает элемент из навигации кнопкой Tab.

### `contenteditable` (Редактирование)
Превращает любой элемент в поле ввода. Пользователь может кликнуть на текст и поменять его прямо в браузере.
- Часто используется в админ-панелях для быстрого редактирования контента.
```html
<h2 contenteditable="true">Кликни и поменяй меня!</h2>
```

## 📊 Пользовательские данные: `data-*` атрибуты

Это «секретный карман» разработчика. Вы можете придумать свой собственный атрибут, начав его название с префикса `data-`.

### Зачем это нужно?
В этих атрибутах хранят информацию, которая нужна скриптам (JavaScript), но не важна для обычного пользователя.
```html
<button class="buy-btn" data-product-id="452" data-price="1200">
  Купить
</button>
```
Здесь мы сохранили ID товара и его цену прямо в кнопке. Когда пользователь нажмет на неё, JavaScript легко достанет эти данные и добавит нужный товар в корзину.

## 🌍 Язык и перевод: `lang` и `translate`

### `lang`
Мы привыкли видеть его в теге `<html>`, но его можно вешать и на отдельные блоки, если внутри используется другой язык. Это помогает скринридерам переключить произношение, а браузерам — предложить перевод.
```html
<p>В этой книге есть фраза на английском: <span lang="en">"To be or not to be"</span>.</p>
```

### `translate`
Подсказывает автоматическим переводчикам (типа Google Translate), нужно ли переводить этот блок.
- Полезно для названий брендов или программного кода.
```html
<span translate="no">MyBrandName</span>
```

## ⚔️ ID против Class: Что выбрать?

Это классический вопрос новичка. Давайте проведем сравнение:

| Характеристика | Class (`.name`) | ID (`#name`) |
|---|---|---|
| **Уникальность** | Можно много на странице | Только один на страницу |
| **Количество на тег** | Сколько угодно | Только один |
| **Для чего?** | Для стилизации (CSS) | Для навигации (якоря) и JS |
| **Приоритет в CSS** | Средний | Высокий (перебивает классы) |

**Золотое правило**: Для дизайна всегда используйте **классы**. Оставляйте **ID** только для якорных ссылок и специфических задач в программировании.

## ♿ Доступность и глобальные атрибуты

Атрибут `title` часто ошибочно считают достаточным для доступности. На самом деле:
- Скринридеры не всегда зачитывают `title`.
- На мобильных устройствах «наведение мыши» невозможно, поэтому пользователь никогда не увидит эту подсказку.
- Для важной информации лучше использовать скрытый текст или специальные ARIA-атрибуты (которые мы изучим позже).

## ⚠️ Типичные ошибки

1. **Дублирование ID**: Если вы сделаете две кнопки с `id="btn"`, CSS может сработать на обеих, но JavaScript найдет только первую. Это вызовет трудноловимые баги.
2. **Классы-описания вместо классов-смыслов**: Называть класс `red-text-left`. Если завтра дизайнер скажет сделать текст синим и по центру, вам придется менять не только CSS, но и весь HTML. Лучше называть классы по смыслу: `warning-message`, `article-title`.
3. **Числа в начале**: Имена классов и ID не должны начинаться с цифр. Это запрещено стандартом.
4. **Злоупотребление `style`**: Если у вас 20 тегов с одинаковым `style="..."`, ваш код превращается в кашу. Вынесите это в класс.

## 🧪 Практика: Интерактивная карточка

Давайте создадим сложный элемент, используя максимум глобальных атрибутов.

```html
<article
    id="featured-article"
    class="card highlight"
    title="Нажмите, чтобы прочитать подробнее"
    data-category="programming"
    data-views="1500"
>
    <header>
        <h2 class="card-title">Изучаем HTML-атрибуты</h2>
    </header>

    <p class="card-text">
        В этом блоке мы используем <span class="term" title="Это атрибуты, доступные всем тегам">глобальные атрибуты</span>.
    </p>

    <!-- Элемент, который можно редактировать прямо на странице -->
    <div class="user-note" contenteditable="true">
        Кликните сюда, чтобы оставить свою заметку к этой статье.
    </div>

    <!-- Элемент скрыт до определенного действия (через JS в будущем) -->
    <p id="secret-message" hidden>Поздравляем! Вы нашли пасхалку.</p>

    <footer class="card-footer">
        <button class="btn-main" tabindex="0">Читать полностью</button>
    </footer>
</article>
```

## ❓ Часто задаваемые вопросы

### «Можно ли называть классы русскими буквами?»
Технически — да, современные браузеры это поймут. Но на практике — **категорически нет**. Это считается признаком крайне низкого уровня разработчика. Всегда используйте английский язык (латиницу).

### «Как data-атрибуты влияют на SEO?»
Никак. Поисковые роботы игнорируют всё, что начинается с `data-`. Это данные только для ваших внутренних нужд.

### «Что важнее: id или class?»
В иерархии CSS `id` всегда сильнее класса. Если у элемента есть класс `red` и id `blue`, и в обоих прописан цвет — элемент будет синим. Именно поэтому для дизайна лучше использовать только классы, чтобы не создавать войн приоритетов.

### «Зачем нужен атрибут accesskey?»
Он позволяет назначить горячую клавишу для элемента (например, `Alt + S` для кнопки «Сохранить»). На практике используется редко, так как часто конфликтует с горячими клавишами самого браузера.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **Глобальные атрибуты** подходят любому тегу.
> 2. **`class`** — для групп элементов и CSS.
> 3. **`id`** — для уникальных элементов и ссылок-якорей.
> 4. **`title`** — для всплывающих подсказок.
> 5. **`data-*`** — для хранения технической информации для скриптов.
> 6. **`hidden`** — скрывает элемент семантически правильно.
> 7. Помните об уникальности ID и используйте осмысленные имена для классов.
