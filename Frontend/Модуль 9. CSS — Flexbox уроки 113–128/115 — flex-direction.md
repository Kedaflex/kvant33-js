## 📖 Что такое flex-direction?

В предыдущем уроке мы узнали, что во Flexbox есть две оси. Свойство `flex-direction` — это главный «рубильник», который определяет, какая из осей будет **Главной (Main Axis)**.

Это свойство задается исключительно родительскому контейнеру. Оно диктует базовую логику расположения: «Я хочу, чтобы мои дети выстроились в горизонт или в вертикальную башню». Без этого свойства браузер всегда выбирает `row`, но в современном веб-дизайне, особенно мобильном, мы постоянно переключаем этот режим.

Существует четыре основных значения:
1. `row` (по умолчанию)
2. `column`
3. `row-reverse`
4. `column-reverse`

Давайте разберем каждое из них максимально подробно, чтобы понять, что происходит «под капотом» браузера.

---

## ⬅️➡️ Ряды: row и row-reverse

Работа с рядами — это классический подход для десктопных интерфейсов.

### flex-direction: row
Это значение установлено по умолчанию, поэтому его часто не пишут, но важно понимать его параметры:
- **Главная ось:** Горизонтальная, направлена слева направо (в LTR языках).
- **Поперечная ось:** Вертикальная, направлена сверху вниз.
- **Точки:** Main-start находится слева, Main-end — справа.
- **Порядок:** Элементы отображаются строго в том же порядке, в котором они идут в HTML-коде.

### flex-direction: row-reverse
Это значение не просто выравнивает элементы по правому краю, оно буквально разворачивает главную ось в обратную сторону.
- **Главная ось:** Справа налево.
- **Точки:** Теперь Main-start находится **справа**, а Main-end — **слева**.
- **Результат:** Элементы прижимаются к правому краю контейнера и меняют порядок: первый элемент в коде (`index 1`) окажется самым правым. Второй — слева от него, и так далее.
- **Зачем это нужно:** Часто используется для адаптации интерфейса под языки с письмом справа налево (RTL), или для быстрой смены визуального приоритета без изменения структуры HTML.

## ⬆️⬇️ Колонки: column и column-reverse

Как только мы переключаемся в режим колонок, мир Flexbox переворачивается на 90 градусов. Это фундаментальный сдвиг в логике.

### flex-direction: column
- **Главная ось:** Теперь она идет **вертикально**, сверху вниз.
- **Поперечная ось:** Стала **горизонтальной**, слева направо.
- **Поведение:** Элементы встают друг под другом.
- **Размеры:** Теперь флекс-элементы стремятся растянуться на всю ширину контейнера по умолчанию (так как поперечная ось теперь горизонтальна и свойство `align-items: stretch` работает по ней). Высота же элементов теперь определяется их контентом.

### flex-direction: column-reverse
- **Главная ось:** Вертикально, **снизу вверх**.
- **Точки:** Main-start находится внизу, Main-end — вверху.
- **Результат:** Первый элемент в коде окажется в самом низу контейнера.
- **Применение:** Идеально подходит для интерфейсов чатов, где последние сообщения должны быть внизу, а список растет вверх, или для специфических мобильных меню.

## 💡 Великая путаница: Смена ролей

Это самый важный момент для понимания Flexbox, на котором «валятся» 80% новичков. **Свойства выравнивания привязаны не к физическим координатам экрана, а к абстрактным осям.**

Представьте свойство `justify-content` (мы изучим его завтра), которое выравнивает элементы по **главной оси**.
1. Если у вас `flex-direction: row`, то `justify-content` будет двигать элементы **влево и вправо**.
2. Если у вас `flex-direction: column`, то то же самое свойство `justify-content` будет двигать элементы **вверх и вниз**.

Многие новички совершают ошибку, пытаясь выровнять элементы по вертикали в колонке через `align-items`. Запомните: в колонке за вертикаль отвечает свойство главной оси! Если вы это осознаете, вы станете мастером Flexbox.

## 🛠️ Когда и что выбирать?

- **`row`**: Стандарт для навигационных меню, списков карточек в десктопной версии, групп кнопок, иконок с подписями.
- **`column`**: Формы регистрации, боковые панели (сайбары), мобильные версии сайтов (стековая верстка), карточки с вертикальной структурой (картинка сверху, текст снизу).
- **`row-reverse`**: Зеркальное отображение блоков (например, в списке преимуществ: картинка слева - текст справа, в следующем блоке наоборот). Можно просто менять класс контейнера.
- **`column-reverse`**: Ленты новостей, где самый старый пост должен быть внизу, а новые добавляются сверху (хотя чаще это решается порядком в БД, в вебе это полезно для визуальных эффектов или специфических UI-паттернов).

## ❓ Часто задаваемые вопросы

### «Меняет ли reverse порядок табуляции и чтения?»
**Это критически важный вопрос.** Нет, не меняет! Если незрячий пользователь использует скринридер или обычный пользователь нажимает клавишу `Tab`, фокус будет переходить по элементам в том порядке, в котором они записаны в HTML.
Использование `reverse` создает «разрыв» между тем, что мы видим, и тем, как с этим взаимодействуем. Если вы визуально поставили кнопку "Ок" первой через `row-reverse`, но в коде она последняя — это плохой UX. Используйте `reverse` только для декоративных целей, а не для изменения смысла интерфейса.

### «Почему элементы в колонке не растягиваются на всю высоту контейнера?»
Потому что в режиме `column` высота контейнера обычно зависит от контента. Если вы хотите, чтобы элементы распределились по всей высоте, вам нужно явно задать контейнеру `height` или `min-height` (например, `100vh`), а затем использовать свойства распределения свободного места.

### «Можно ли комбинировать направления?»
Конечно. Вы можете сделать главный контейнер `column`, а внутри него один из блоков сделать `display: flex` с направлением `row`. Это база создания современных сеток.

## 🏗️ Практика: Гибкая карточка

Создадим компонент, который легко превращается из горизонтального в вертикальный.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <style>
    .card {
      display: flex;
      border: 1px solid #eee;
      border-radius: 12px;
      overflow: hidden;
      box-shadow: 0 4px 6px rgba(0,0,0,0.1);
      transition: all 0.3s ease;

      /* Попробуйте поменять на column прямо сейчас */
      flex-direction: row;
      max-width: 600px;
    }

    .card-image {
      background: #3498db;
      min-width: 200px;
      min-height: 200px;
      display: flex;
      align-items: center;
      justify-content: center;
      color: white;
      font-weight: bold;
    }

    .card-body {
      padding: 24px;
      background: white;
    }

    .card-body h3 {
      margin-top: 0;
      color: #333;
    }

    /* Пример адаптивности: на узких экранах переключаем в колонку */
    @media (max-width: 480px) {
      .card {
        flex-direction: column;
      }
    }
  </style>
</head>
<body>

  <div class="card">
    <div class="card-image">
      ![[Изображение товара]]
    </div>
    <div class="card-body">
      <h3>Современный гаджет</h3>
      <p>Это описание товара, которое прекрасно смотрится справа от картинки на десктопе и снизу на мобильном телефоне благодаря магии flex-direction.</p>
      <button>Купить сейчас</button>
    </div>
  </div>

</body>
</html>
```

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. `flex-direction` управляет вектором **Главной оси**.
> 2. `row` (ряд) и `column` (колонка) — фундаментальные режимы верстки.
> 3. При переключении в `column` горизонтальные и вертикальные свойства выравнивания меняются ролями.
> 4. `reverse` меняет визуальный порядок, но сохраняет порядок в коде (важно для доступности!).
> 5. Направление оси — это база, от которой зависит работа всех остальных свойств Flexbox.
