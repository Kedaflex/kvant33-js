# 🏗️ Практика: Разработка компонента «Тарифные планы»

← [[127 — 🏗️ Практика — Макет Holy Grail]] | [[129 — Введение в Grid]] →

> [!info] Цели урока
> - Спроектировать и сверстать современный блок тарифов (Pricing Tables).
> - Использовать `align-items: stretch` для идеального выравнивания высоты разнородных карточек.
> - Овладеть техникой «выталкивающего контента» через `flex-grow` для фиксации кнопок внизу.
> - Создать визуальную иерархию через масштабирование и акцентные цвета.
> - Обеспечить безупречную адаптивность: переход от горизонтального ряда к вертикальному стеку.

---

## 📖 Постановка задачи: Дизайн SaaS-интерфейса

В этом финальном практическом уроке мы создадим один из самых важных элементов любого коммерческого сайта — таблицу цен. Наша задача — сделать её не только красивой, но и технически совершенной.

**Что мы должны реализовать:**
1. **Геометрия:** Три карточки на десктопе должны быть **строго одинаковой высоты**, даже если в одной из них список преимуществ длиннее.
2. **Выравнивание:** Кнопки «Выбрать тариф» во всех карточках должны стоять ровно на одной горизонтальной линии.
3. **Акцент:** Центральная карточка («Популярный выбор») должна выделяться: быть чуть выше других и иметь яркую метку (badge).
4. **Адаптивность:** На смартфонах карточки должны выстраиваться в вертикальный ряд с сохранением всех стилей.

## 🎨 Шаг 2: Стилизация каркаса (Modern Minimalism)

Здесь мы применим магию `stretch` и вложенных флекс-контейнеров.

```css
.pricing-container {
  display: flex;
  justify-content: center;
  align-items: stretch; /* ГАРАНТИЯ: все карточки одной высоты! */
  gap: 30px;
  padding: 60px 20px;
  background-color: #f8fafc;
  font-family: 'Inter', sans-serif;
}

.pricing-card {
  /* Каждая карточка — это тоже ФЛЕКС-КОНТЕЙНЕР (колонка) */
  display: flex;
  flex-direction: column;
  flex: 1;
  max-width: 400px;
  padding: 40px;
  background: white;
  border-radius: 24px;
  border: 1px solid #e2e8f0;
  transition: transform 0.3s ease, box-shadow 0.3s ease;
  position: relative;
}

.pricing-card:hover {
  transform: translateY(-5px);
  box-shadow: 0 20px 40px rgba(0,0,0,0.08);
}
```

## 🚀 Шаг 3: Магия «пружины» внутри карточки

Самое сложное в таких карточках — выровнять кнопки внизу. Мы сделаем это через свойство `flex-grow` для списка функций.

```css
.card-header {
  text-align: center;
  margin-bottom: 30px;
}

.price {
  font-size: 48px;
  font-weight: 800;
  color: #1e293b;
}

.price span {
  font-size: 16px;
  color: #64748b;
}

.feature-list {
  list-style: none;
  padding: 0;
  margin: 0 0 40px 0;
  /* МАГИЯ: Список растягивается, заполняя всё свободное место,
     и выталкивает кнопку в самый низ карточки */
  flex-grow: 1;
}

.feature-list li {
  padding: 12px 0;
  color: #475569;
  border-bottom: 1px solid #f1f5f9;
}

.pricing-btn {
  width: 100%;
  padding: 16px;
  border-radius: 12px;
  border: 1px solid #cbd5e1;
  background: white;
  color: #1e293b;
  font-weight: 600;
  cursor: pointer;
  transition: 0.2s;
}

/* Стили для активной карточки */
.featured {
  border: 2px solid #3b82f6;
  background: #ffffff;
  transform: scale(1.05); /* Делаем чуть больше остальных */
}

.featured .btn-active {
  background: #3b82f6;
  color: white;
  border: none;
}
```

## 🔍 Глубокий разбор логики верстки

1. **`align-items: stretch` у родителя:** Это свойство — спасение. Оно говорит: «Я хочу, чтобы все мои дети (карточки) имели высоту самого высокого из них». Благодаря этому, даже если в одном тарифе 2 пункта, а в другом 10, карточки будут визуально идентичны.
2. **`display: flex; flex-direction: column` у карточки:** Это позволяет нам использовать внутри карточки все преимущества Flexbox.
3. **`flex-grow: 1` у списка функций:** Список работает как невидимая пружина. Если контента мало, он растягивается. Если много — занимает столько, сколько нужно. Но в любом случае он гарантирует, что кнопка под ним будет прижата к нижнему краю паддинга карточки.
4. **`transform: scale(1.05)`**: Мы используем масштабирование, чтобы создать визуальный акцент на «лучшем предложении».

## ❓ Часто задаваемые вопросы

### «Почему бы не использовать margin-top: auto для кнопки?»
Это отличный альтернативный вариант! `margin-top: auto` во флекс-контейнере сделает ровно то же самое — прижмет элемент к самому концу оси. Оба способа являются правильными и профессиональными.

### «Как сделать так, чтобы на мобилках Featured-карточка не была больше остальных?»
В медиа-запросе нужно будет обнулить `transform: scale(1.05)`. На маленьких экранах вертикальное увеличение может смотреться странно или перекрывать соседние блоки.

### «Можно ли добавить иконки галочек в список?»
Да, это стандарт индустрии. Вы можете добавить их через псевдоэлемент `::before` для каждого `li` или вставить `img` внутрь списка. Flexbox поможет выровнять иконку и текст через `display: inline-flex; align-items: center`.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. Flexbox позволяет создавать сложные компоненты с идеально выровненными элементами.
> 2. `align-items: stretch` решает проблему разной высоты колонок.
> 3. `flex-grow: 1` или `margin-top: auto` — лучшие способы прижать элемент (кнопку) к краю.
> 4. Вложенные флекс-контейнеры дают возможность управлять макетом на микро- и макро-уровнях.
> 5. Визуальные акценты (масштаб, тени) делают интерфейс понятным и привлекательным.
