## 📖 Что такое Holy Grail Layout?

В истории веб-дизайна существует структура, которую называют «Святым Граалем» (Holy Grail). Это классический макет, который вы видите на миллионах сайтов (от новостных порталов до панелей управления). Он состоит из:
1. **Header** (Шапка) — всегда сверху, на всю ширину.
2. **Центральный блок** (Main Body), который разделен на три колонки:
   - **Sidebar Left** (Навигация/Меню).
   - **Main Content** (Основная часть с текстом).
   - **Sidebar Right** (Реклама, виджеты или дополнительная информация).
3. **Footer** (Подвал) — всегда внизу.

**В чем была главная боль прошлого?**
До появления Flexbox было почти невозможно заставить все три центральные колонки иметь **одинаковую высоту**, если в одной из них текста больше, чем в других. Также огромной проблемой было удержание футера внизу экрана при малом количестве контента. Flexbox превратил это «хождение по мукам» в элегантную прогулку.

## 🛠️ Шаг 1: Семантическая HTML-разметка

Мы будем использовать современный подход с контейнером-оберткой, который обеспечит нам гибкость.

```html
<div class="site-wrapper">
  <header class="header">
    <div class="logo">FlexBrand</div>
    <nav class="top-nav">Верхнее меню</nav>
  </header>

  <div class="main-body">
    <nav class="sidebar-left">
      <ul>
        <li><a href="#">Главная</a></li>
        <li><a href="#">Статьи</a></li>
        <li><a href="#">Контакты</a></li>
      </ul>
    </nav>

    <main class="content">
      <h1>Добро пожаловать в Святой Грааль</h1>
      <p>Этот макет идеально сбалансирован. Центральная часть занимает всё доступное место, а боковые колонки имеют фиксированную ширину.</p>
      <p>Попробуйте добавить сюда очень много текста, и вы увидите, как боковые панели растянутся автоматически вслед за контентом.</p>
    </main>

    <aside class="sidebar-right">
      <div class="widget">Рекламный блок</div>
      <div class="widget">Погода</div>
    </aside>
  </div>

  <footer class="footer">
    &copy; 2023 Академия Флекса. Все права защищены.
  </footer>
</div>
```

## 🎨 Шаг 2: Вертикальный каркас (Sticky Footer)

Первым делом настроим нашу обертку `.site-wrapper`. Она должна превратиться в вертикальный стек.

```css
.site-wrapper {
  display: flex;
  flex-direction: column;
  /* Заставляем обертку быть минимум на всю высоту окна браузера */
  min-height: 100vh;
}

.header, .footer {
  background: #1a1a1a;
  color: white;
  padding: 30px;
  flex: 0 0 auto; /* Не растут, не сжимаются, база по контенту */
}

.main-body {
  display: flex; /* Включаем ВЛОЖЕННЫЙ флекс для колонок */
  flex: 1 0 auto; /* Растет, заполняя всё свободное место! Это прижмет футер вниз */
}
```

## 🚀 Шаг 3: Настройка трех колонок

Теперь поработаем с внутренним контейнером `.main-body`, который теперь является горизонтальным рядом.

```css
.sidebar-left, .sidebar-right {
  flex: 0 0 250px; /* Фиксированная ширина 250px */
  background: #f4f4f4;
  padding: 25px;
}

.sidebar-left { border-right: 1px solid #ddd; }
.sidebar-right { border-left: 1px solid #ddd; }

.content {
  flex: 1; /* Основной контент самый "жадный", забирает всё что осталось */
  padding: 40px;
  background: white;
}
```
**Магия этого этапа:** Поскольку `.main-body` — это флекс-контейнер, его дети по умолчанию имеют `align-items: stretch`. Это значит, что фоновый цвет сайдбаров будет тянуться до самого футера, даже если в них нет ни одного слова!

---

## 📱 Шаг 4: Трансформация для мобильных (Mobile First)

На экранах смартфонов три колонки превратятся в кашу. Нам нужно перестроить их в один вертикальный поток.

```css
@media (max-width: 900px) {
  .main-body {
    flex-direction: column; /* Магия смены направления */
  }

  .sidebar-left, .sidebar-right {
    flex: none; /* Убираем фиксированные 250px */
    width: 100%;
    order: 2; /* Уводим сайдбары вниз под контент */
  }

  .content {
    order: 1; /* Выводим основной контент на первое место */
  }
}
```

## 🔍 Глубокий анализ архитектуры

Что мы получили в итоге?
1. **Гибкость:** Контент всегда в центре, сайдбары по краям.
2. **Липкий футер:** Если на странице всего одна строчка текста, футер всё равно будет в самом низу экрана, а не «зависнет» посередине.
3. **Одинаковая высота:** Колонки визуально равны всегда.
4. **Умный порядок:** В мобильной версии мы использовали `order`, чтобы сначала показать пользователю статью (`content`), а уже потом навигацию и рекламу. Это лучший UX!

## ❓ Часто задаваемые вопросы

### «Зачем `min-height: 100vh` у обертки?»
Это критически важно. Если убрать это свойство, то при малом количестве контента `body` и `.site-wrapper` будут высотой всего в пару десятков пикселей. Футер поднимется вверх. `100vh` гарантирует, что наш макет всегда растянут на весь экран.

### «Можно ли сделать сайдбары разной ширины?»
Легко. Вы можете задать одному `flex: 0 0 200px`, а другому `flex: 0 0 350px`. Система останется стабильной.

### «Как сделать так, чтобы сайдбары сжимались на планшетах?»
Вместо `flex: 0 0 250px` используйте `flex: 0 1 250px`. Тогда при сужении экрана они начнут уменьшаться, прежде чем переключиться в режим колонки.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. Вложенные флекс-контейнеры — это стандарт индустрии для сложных макетов.
> 2. `flex: 1` на центральном блоке — секрет «липкого футера».
> 3. `align-items: stretch` (дефолт) — секрет колонок одинаковой высоты.
> 4. `order` — мощный инструмент для управления вниманием пользователя на разных устройствах.
> 5. Адаптивность достигается через простую смену `flex-direction`.
