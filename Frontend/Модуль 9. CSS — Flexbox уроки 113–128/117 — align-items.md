## 📖 Что такое align-items?

В мире Flexbox за каждое направление отвечает свой «министр». Если `justify-content` — это министр Главной оси, то `align-items` — это полномочный представитель **Поперечной оси (Cross Axis)**.

В стандартном режиме (`flex-direction: row`), который мы используем чаще всего, поперечная ось направлена вертикально (сверху вниз). Следовательно, `align-items` — это наш основной инструмент для управления «высотным» положением элементов. Оно определяет, будут ли ваши блоки «прилипать» к потолку контейнера, стоять на его полу или парить ровно посередине.

Важно помнить: это свойство задается **родителю** и действует на всех его детей одновременно.

---

## 🛠️ Разбор значений: От дефолта до эстетики

### 1. stretch (Значение по умолчанию)
Это, пожалуй, самая характерная черта Flexbox. Браузер берет самый высокий элемент в строке и заставляет всех остальных «дотянуться» до его высоты.
- **Как это работает:** Если у элементов не задана жесткая высота (`height`), они автоматически заполняют всё пространство поперечной оси.
- **Зачем это нужно:** Это идеальный способ сделать колонки в карточках или блоки в сетке одинаковой высоты без использования магических чисел в CSS.
- **Подвох:** Если вы явно задали элементу `height: 100px`, то `stretch` его не растянет — явное свойство всегда в приоритете.

### 2. flex-start
Элементы выравниваются по началу поперечной оси. В режиме `row` это означает, что они прижимаются к верхнему краю.
- **Поведение:** Элементы принимают свою естественную высоту (по контенту) и выстраиваются в одну линию по верхнему обрезу.

### 3. flex-end
Элементы «падают» на дно контейнера.
- **Поведение:** Выравнивание по нижнему краю (в режиме `row`). Это полезно, например, для подписей под картинками разной высоты, которые должны стоять на одной «земле».

### 4. center
Золотой стандарт современного веба. Элементы выравниваются по своей центральной горизонтальной линии относительно центра контейнера.
- **Кейс:** Если у вас есть иконка 24px и текст 16px, `center` сделает так, что они будут выглядеть гармонично, а не так, будто текст «сполз» вниз.

### 5. baseline
Самое «умное» и недооцененное значение. Оно выравнивает элементы не по их рамкам, а по **базовой линии шрифта**.
- **Пример:** У вас два слова в ряд. Одно написано гигантским шрифтом, другое — крошечным. Если вы используете `center`, их геометрические центры совпадут, но текст будет «скакать». Если `baseline` — буквы обоих слов будут стоять на одной невидимой строке. Это признак качественной, дорогой верстки.

## 🔄 align-items в режиме колонок (column)

Как мы уже знаем, при `flex-direction: column` оси меняются местами. Поперечная ось теперь горизонтальна.

Следовательно:
- `align-items: center` в колонке будет центрировать элементы **по горизонтали** (посередине ширины контейнера).
- `align-items: flex-end` прижмет их к правому краю.
- `align-items: stretch` заставит все блоки в колонке стать одинаковой ширины (во всю ширину родителя).

Это часто путает новичков: они пишут `align-items: center`, ожидая вертикального центрирования в колонке, а элементы просто прыгают вправо-влево. Всегда проверяйте направление главной оси!

## 📐 Сравнение: Как не запутаться?

Запомните простую мнемонику:
- **J**ustify — **H**orizontal (в обычном режиме). Буква J похожа на крючок, который тянет влево-вправо.
- **A**lign — **V**ertical. Буква А похожа на стрелку, указывающую вверх (или на гору с вершиной).

| Свойство | Направление при `row` | Направление при `column` |
| :--- | :--- | :--- |
| **justify-content** | Горизонтально (← →) | Вертикально (↑ ↓) |
| **align-items** | Вертикально (↑ ↓) | Горизонтально (← →) |

## 💡 Практические приемы и «Идеальное центрирование»

Десятилетиями верстальщики страдали, пытаясь отцентрировать блок внутри блока по вертикали. С Flexbox это решается тремя строчками:

```css
.hero-section {
  display: flex;           /* Включаем режим магии */
  justify-content: center; /* Центрируем по горизонтали */
  align-items: center;     /* Центрируем по вертикали */
  height: 100vh;           /* Даем родителю высоту (весь экран) */
}
```
Этот паттерн называется **"Center of the Universe"**. Он работает безотказно для любых элементов — будь то картинка, текст или целая форма.

### Почему вертикальное выравнивание не работает?
Если вы применили `align-items: center`, но визуально ничего не изменилось, в 99% случаев проблема в том, что **высота родительского контейнера равна высоте его содержимого**.
Браузер честно отцентрировал элементы, но так как лишнего места сверху и снизу нет, они остались там же, где и были. Всегда проверяйте `height` или `min-height` у родителя.

## ❓ Часто задаваемые вопросы

### «В чем разница между align-items и align-content?»
Этот вопрос возникнет у вас, как только мы дойдем до переноса строк (`flex-wrap`).
- `align-items` выравнивает элементы **внутри одной строки**.
- `align-content` выравнивает **сами строки** относительно друг друга, если их стало много.
Пока у вас одна строка элементов — забудьте про `align-content`, он вам не нужен.

### «Можно ли выровнять только один элемент по-другому?»
Да! Если все дети прижаты к верху (`flex-start`), но одну кнопку вы хотите опустить вниз, используйте свойство `align-self: flex-end` для этой конкретной кнопки. Мы разберем это подробнее в уроке 125.

## 🏗️ Лабораторная работа: Визуальный гид

Давайте создадим испытательный стенд для разных типов выравнивания.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <style>
    .demo-container {
      display: flex;
      border: 4px dashed #bdc3c7;
      background: #fdfdfd;
      height: 250px;
      margin-bottom: 40px;
      gap: 20px;
      padding: 10px;
    }

    .item {
      background: #3498db;
      color: white;
      padding: 20px;
      border-radius: 8px;
      font-family: sans-serif;
    }

    /* Разная высота для наглядности */
    .h-auto { height: auto; }
    .h-small { height: 60px; font-size: 12px; }
    .h-large { height: 120px; font-size: 24px; }

    /* Тестовые классы */
    .align-center   { align-items: center; }
    .align-start    { align-items: flex-start; }
    .align-end      { align-items: flex-end; }
    .align-stretch  { align-items: stretch; }
    .align-baseline { align-items: baseline; }
  </style>
</head>
<body>

  <h3>1. align-items: center (Гармония)</h3>
  <div class="demo-container align-center">
    <div class="item h-small">Малый</div>
    <div class="item h-large">Крупный</div>
    <div class="item h-auto">Авто-высота</div>
  </div>

  <h3>2. align-items: stretch (Сила Flexbox)</h3>
  <div class="demo-container align-stretch">
    <div class="item h-small">Я задал высоту 60px (не растянусь)</div>
    <div class="item h-auto">У меня нет высоты (я растянусь на все 250px!)</div>
  </div>

  <h3>3. align-items: baseline (Типографика)</h3>
  <div class="demo-container align-baseline">
    <div class="item h-small">Нижний край букв...</div>
    <div class="item h-large">...на одной линии</div>
  </div>

</body>
</html>
```

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. `align-items` — это управление по **Поперечной оси**.
> 2. `stretch` заставляет элементы занимать всю доступную высоту/ширину (если они не ограничены).
> 3. `center` — ключевой инструмент для вертикального выравнивания.
> 4. `baseline` выравнивает контент по смысловой линии текста.
> 5. Эффект от `align-items` виден только тогда, когда у родителя есть запас пространства по поперечной оси.
> 6. В режиме `column` это свойство отвечает за **горизонтальное** выравнивание.
