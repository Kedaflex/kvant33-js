## 📖 Что такое flex-grow?

До этого момента мы обсуждали, как выравнивать и двигать элементы, размеры которых нам известны. Но истинная магия Flexbox скрыта в самом его названии: **Flexible** (гибкий). Свойство `flex-grow` — это первый шаг к пониманию того, как Flexbox управляет пространством, которого «слишком много».

Представьте: у вас есть контейнер шириной 1000 пикселей и три блока внутри, каждый по 200 пикселей. Итого блоки занимают 600 пикселей. Остается 400 пикселей «пустоты».

`flex-grow` — это свойство, задаваемое **дочернему элементу**, которое определяет его способность «вырастать», чтобы «съесть» эту пустоту. Это своего рода коэффициент «жадности»: чем выше число, тем больше лишних пикселей заберет себе элемент.

---

## 🛠️ Математика жадности: Как браузер делит «пирог»

Свойство принимает безразмерное числовое значение (положительное). По умолчанию оно равно `0`.

### 1. flex-grow: 0 (Равнодушие)
Элемент не хочет расти. Он занимает ровно столько места, сколько требует его контент или заданная ширина (`width`). Если в контейнере есть свободное место, этот элемент его проигнорирует.

### 2. flex-grow: 1 (Справедливость)
Элемент заявляет свои права на свободное место. Если у всех элементов в контейнере стоит `flex-grow: 1`, они поделят все лишние пиксели **поровну**.

### 3. Пропорциональный рост
Если у элементов разные коэффициенты, браузер использует простую математику:
- **Шаг 1:** Считается сумма всех коэффициентов `flex-grow`. Например, у блока А — `1`, у блока Б — `3`. Итого: `1 + 3 = 4`.
- **Шаг 2:** Свободное место делится на эту сумму. Если лишних было 400px, то одна «доля» равна `400 / 4 = 100px`.
- **Шаг 3:** Блоки получают свои доли. Блок А получит `1 * 100 = 100px`. Блок Б получит `3 * 100 = 300px`.

**Важнейший момент:** Эти пиксели прибавляются к начальному размеру блока. Если блок А изначально был шире блока Б, то даже с меньшим коэффициентом он может остаться шире по итогу. `flex-grow` — это не про финальный размер, а про **размер добавки**.

## 📐 Взаимодействие с контентом: Ловушка для новичка

Многие думают, что если задать двум блокам `flex-grow: 1`, они станут абсолютно одинаковыми по ширине. Но это не всегда так.

Браузер сначала рассчитывает место под контент (текст, картинки). Если в левом блоке одно слово, а в правом — целый абзац, то «база» у них будет разной. Добавка в 100 пикселей к короткому слову и к длинному абзацу даст разный финальный результат.

**Как сделать их идеально равными?**
Нужно обнулить их «базу» (свойство `flex-basis: 0` или `width: 0`). Тогда браузер посчитает, что начальный размер у всех — ноль, и всё пространство контейнера станет «лишним». В этом случае коэффициенты сработают как чистые пропорции (например, 50% на 50%).

## 💡 Практические сценарии использования

### 1. Липкий подвал (Sticky Footer)
Это классическая задача: как сделать так, чтобы подвал сайта всегда был внизу, даже если на странице мало текста?
- Родитель (body): `display: flex; flex-direction: column; min-height: 100vh;`
- Шапка и подвал: `flex-grow: 0;` (не растут).
- Основной контент (main): `flex-grow: 1;`
**Результат:** Блок контента заберет всё свободное место на экране и физически «вытолкнет» подвал к самому нижнему краю окна.

### 2. Гибкая форма поиска
В шапке сайта часто есть поле ввода и кнопка. Мы хотим, чтобы кнопка была фиксированной, а поле ввода растягивалось на всё оставшееся место.
- Поле ввода (`input`): `flex-grow: 1;`
- Кнопка (`button`): `flex-grow: 0;`
**Результат:** Кнопка останется компактной, а поиск будет «резиновым».

## ❓ Часто задаваемые вопросы

### «Можно ли использовать дробные числа, например 0.5?»
Да, конечно. Вы можете использовать любые положительные числа. `flex-grow: 0.5` означает, что элемент заберет в два раза меньше лишнего места, чем сосед с коэффициентом `1`. Но для удобства чтения кода обычно используют целые числа.

### «Работает ли flex-grow в многострочном режиме (flex-wrap: wrap)?»
Да, но только в пределах **одной строки**. Если элементы перенеслись на вторую строку, они будут делить свободное место только этой второй строки между собой. Они не «видят» свободное место на этаже выше.

### «Что если у одного блока flex-grow: 1, а у другого 9999?»
Второй блок заберет себе практически всё свободное пространство, оставив первому лишь крохотную долю. Это иногда используется, чтобы гарантированно прижать один элемент к другому, но лучше использовать разумные числа (1, 2, 3).

## 🏗️ Лабораторная работа: Стенд пропорций

Создадим визуальный пример, где мы можем наглядно увидеть распределение долей.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <style>
    .layout-box {
      display: flex;
      background: #f4f7f6;
      border: 3px solid #2c3e50;
      width: 900px;
      margin-bottom: 40px;
      padding: 10px;
      gap: 10px;
    }

    .item {
      padding: 20px;
      color: white;
      text-align: center;
      font-family: sans-serif;
      font-weight: bold;
      border-radius: 6px;
    }

    /* Цвета для наглядности */
    .red    { background: #e74c3c; }
    .blue   { background: #3498db; }
    .green  { background: #2ecc71; }
    .purple { background: #9b59b6; }

    /* Коэффициенты жадности */
    .grow-1 { flex-grow: 1; }
    .grow-2 { flex-grow: 2; }
    .grow-3 { flex-grow: 3; }
    .fixed  { flex-grow: 0; width: 150px; }
  </style>
</head>
<body>

  <h3>Сценарий 1: Равные доли (1 : 1 : 1)</h3>
  <div class="layout-box">
    <div class="item red grow-1">Доля 1</div>
    <div class="item blue grow-1">Доля 1</div>
    <div class="item green grow-1">Доля 1</div>
  </div>

  <h3>Сценарий 2: Разные пропорции (1 : 2 : 3)</h3>
  <div class="layout-box">
    <div class="item red grow-1">1 часть</div>
    <div class="item blue grow-2">2 части</div>
    <div class="item green grow-3">3 части</div>
  </div>

  <h3>Сценарий 3: Фиксированный элемент + Резиновый</h3>
  <div class="layout-box">
    <div class="item purple fixed">Я всегда 150px</div>
    <div class="item red grow-1">А я заберу весь остаток</div>
  </div>

</body>
</html>
```

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. `flex-grow` управляет тем, как элементы забирают **свободное пространство** (лишние пиксели).
> 2. По умолчанию `flex-grow: 0` — элементы не расширяются больше своего размера.
> 3. Рост происходит пропорционально коэффициентам: элемент с `grow: 2` получит в два раза больше "прибавки", чем элемент с `grow: 1`.
> 4. Это свойство не задает итоговую ширину, оно лишь определяет размер добавки к начальному размеру.
> 5. В режиме `column` свойство заставляет элементы расти в высоту.
