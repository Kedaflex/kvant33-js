## 📖 Включение режима Flex

Все начинается с одного свойства, примененного к родительскому элементу. Как только вы пишете `display: flex`, происходит мгновенная трансформация.

### display: flex vs inline-flex
Это свойство определяет, как сам контейнер взаимодействует с внешним миром (своими соседями):
- **`display: flex`**: Контейнер ведет себя как блочный элемент. Он занимает всю доступную ширину родителя и заставляет следующие элементы переходить на новую строку.
- **`display: inline-flex`**: Контейнер ведет себя как строчно-блочный элемент. Его ширина подстраивается под содержимое, и он может стоять в одной строке с текстом или другими элементами.

**Важно:** Внутреннее поведение (влияние на дочерние элементы) у них абсолютно одинаковое.

---

## 📐 Главная ось (Main Axis)

Представьте, что внутри контейнера натянута невидимая струна. По умолчанию она идет горизонтально: от левого края к правому. Это и есть **Главная ось**.

Именно вдоль этой оси браузер выстраивает ваши элементы один за другим. Если вы не задали никаких дополнительных настроек, элементы «нанизываются» на эту струну.

### Начало и конец главной оси:
- **Main Start:** Точка, где начинается главная ось (по умолчанию левый край).
- **Main End:** Точка, где ось заканчивается (по умолчанию правый край).
- **Main Size:** Размер флекс-элемента вдоль главной оси (в данном случае — его ширина).

Когда мы говорим «выровнять элементы по центру», браузер смотрит именно на положение относительно этих точек на главной оси.

---

## ↕️ Поперечная ось (Cross Axis)

Перпендикулярно главной оси всегда располагается **Поперечная ось**. Если главная ось горизонтальная, то поперечная — вертикальная (сверху вниз).

Поперечная ось отвечает за то, как элементы располагаются в пределах высоты строки (если элементы стоят в ряд). Например, будут ли они прижаты к верху, к низу или растянуты на всю высоту.

### Начало и конец поперечной оси:
- **Cross Start:** Начало поперечной оси (верхний край).
- **Cross End:** Конец поперечной оси (нижний край).
- **Cross Size:** Размер флекс-элемента вдоль поперечной оси (его высота).

---

## 🔄 Великая трансформация: Смена направлений

Самая мощная и поначалу путающая особенность Flexbox заключается в том, что главная и поперечная оси **не привязаны жестко к горизонтали и вертикали.**

С помощью свойства `flex-direction` (которое мы разберем в следующем уроке) вы можете сказать браузеру: «Теперь главная ось идет сверху вниз».

Что произойдет в этот момент?
1. Главная ось станет вертикальной.
2. Поперечная ось автоматически повернется и станет горизонтальной.
3. Все свойства, которые раньше выравнивали элементы по горизонтали (например, `justify-content`), внезапно начнут выравнивать их по вертикали.

Это похоже на вращение всей системы координат. Именно поэтому во Flexbox мы почти не используем слова «лево», «право», «верх» или «низ». Мы используем «начало оси» и «конец оси». Это делает верстку независимой от направления письма (например, для арабских языков, где пишут справа налево, Flexbox просто разворачивает оси, и логика остается прежней).

## 🧪 Как элементы ведут себя по умолчанию?

Когда вы только включили `display: flex`, активируются значения свойств «по умолчанию»:
1. **Направление:** Элементы встают в ряд (слева направо).
2. **Растяжение:** Элементы не пытаются заполнить всю ширину контейнера, они занимают столько места, сколько требует их контент.
3. **Высота:** Это самое заметное — все элементы растягиваются по высоте, чтобы стать такими же высокими, как самый высокий элемент в контейнере (или как сам контейнер).
4. **Перенос:** Элементы будут пытаться втиснуться в одну строку, даже если им не хватает места. Они скорее сожмутся до неузнаваемости, чем перейдут на новую строку (если не включен перенос).

## ❓ Часто задаваемые вопросы

### «Почему мои блоки сжались, когда я включил flex?»
Потому что по умолчанию флекс-элементы имеют право сжиматься, если им тесно. Они игнорируют свою ширину (`width`), если контейнер слишком мал. Мы научимся управлять этим поведением с помощью свойств `flex-shrink` и `flex-basis`.

### «Влияет ли `display: flex` на текст внутри дочерних элементов?»
Нет. `display: flex` меняет поведение только прямых потомков (блоков). Текст внутри этих блоков продолжает подчиняться обычным правилам типографики, пока вы не сделаете сам этот блок флекс-контейнером.

### «Можно ли использовать маргины (`margin`) вместе с флексом?»
Да! Более того, во флексбоксе `margin: auto` работает магическим образом. Если вы зададите элементу `margin-left: auto`, он оттолкнется от всех соседей слева и уедет в крайнее правое положение. Это один из лучших способов прижать кнопку к правому краю меню.

## 🏗️ Лабораторная работа: Визуализация осей

Давайте создадим интерактивный стенд.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <style>
    .flex-wrapper {
      display: flex; /* Попробуйте изменить на inline-flex */
      border: 5px solid #2c3e50;
      height: 300px;
      padding: 10px;
      position: relative;
    }

    /* Визуализация осей через псевдоэлементы (просто для обучения) */
    .flex-wrapper::before {
      content: 'Главная ось →';
      position: absolute;
      top: -25px;
      left: 0;
      color: #e74c3c;
      font-weight: bold;
    }

    .flex-wrapper::after {
      content: 'Поперечная ось ↓';
      position: absolute;
      left: -120px;
      top: 0;
      transform: rotate(-90deg);
      transform-origin: top right;
      color: #27ae60;
      font-weight: bold;
    }

    .box {
      width: 100px;
      background: #3498db;
      color: white;
      margin: 5px;
      display: flex;
      align-items: center;
      justify-content: center;
      font-size: 24px;
    }

    .box:nth-child(2) {
      height: 150px; /* Этот блок растянет остальных по поперечной оси */
      background: #9b59b6;
    }
  </style>
</head>
<body>

  <div class="flex-wrapper">
    <div class="box">1</div>
    <div class="box">2</div>
    <div class="box">3</div>
  </div>

</body>
</html>
```

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. `display: flex` делает элемент флекс-контейнером, а его детей — флекс-элементами.
> 2. Вся логика Flexbox строится на двух осях: **Главной** и **Поперечной**.
> 3. По умолчанию главная ось горизонтальна, поперечная — вертикальна.
> 4. Флекс-элементы «растягиваются» по поперечной оси по умолчанию.
> 5. Концепция осей позволяет создавать интерфейсы, которые легко адаптируются к разным направлениям языка и размерам экранов.
