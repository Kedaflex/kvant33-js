## 📖 Индивидуальное выравнивание: align-self

До этого момента мы работали как дирижеры большого оркестра: задавали общие правила всему контейнеру через `align-items`. Все элементы слушались и выравнивались одинаково. Но что, если в вашем «хоре» один «голос» должен звучать иначе? Что, если все блоки должны быть прижаты к верхнему краю, а один — гордо стоять ровно посередине?

Для таких случаев существует свойство **`align-self`**. Оно задается **самому элементу** (ребенку) и обладает высшим приоритетом — оно полностью перечеркивает инструкции `align-items`, полученные от родителя, но только для этого конкретного блока.

### Значения align-self:
Они зеркально повторяют список `align-items`:
- `auto` (По умолчанию: элемент смиренно слушается родителя).
- `flex-start` / `flex-end`
- `center`
- `baseline`
- `stretch`

**Реальный пример:** В навигационной панели (шапке сайта) все ссылки выровнены по центру, а логотип прижат к верхнему краю, чтобы соответствовать специфическому дизайну бренда.

## 🔢 Смена порядка: Свойство order

Flexbox дает нам уникальную возможность «перекраивать» реальность. Свойство `order` позволяет изменить визуальный порядок элементов на экране, не прикасаясь к коду HTML.

По умолчанию у всех элементов на планете Земля `order` равен **`0`**.

### Правила игры:
1. Элементы выстраиваются на экране в порядке возрастания их значений `order`.
2. Если у элементов значения `order` одинаковы (например, у всех `0`), они отображаются в том порядке, в котором они записаны в HTML.
3. Вы можете использовать отрицательные числа! Чтобы гарантированно вытолкнуть элемент в самое начало, достаточно задать ему `order: -1`.

**Зачем это нужно?**
Представьте, что на десктопе у вас идет [Текст | Картинка]. Но на мобильном телефоне по правилам дизайна картинка **обязательно** должна быть сверху. Вместо того чтобы дублировать код или использовать сложные хаки, вы просто пишете в медиа-запросе для картинки `order: -1`. Магия!

## ⚠️ Темная сторона order: Доступность и UX

Свойство `order` — это мощное средство, которое накладывает на разработчика большую ответственность.

1. **Скринридеры (Читалки экрана):** Программы для людей с нарушениями зрения читают код так, как он написан в HTML. Если вы визуально поставили пятый пункт меню первым через `order`, скринридер всё равно прочитает его пятым. Это может полностью разрушить логику интерфейса для части пользователей.
2. **Навигация клавиатурой:** Когда пользователь перемещается по сайту клавишей `Tab`, фокус будет прыгать строго в порядке HTML-кода. Представьте замешательство человека, когда он нажимает Tab, и фокус с первого блока перескакивает в конец страницы, потому что вы визуально поменяли их местами.

**Золотое правило:** Используйте `order` только для адаптивной верстки (изменение визуального акцента на разных экранах) и никогда — для исправления ленивой верстки в HTML. Сначала сделайте логичный HTML-код, а потом украшайте его флексами.

## 💡 Секреты мастерства и часто путающие моменты

### Где мой justify-self?
Это, пожалуй, самый популярный вопрос на форумах. Если есть `align-self` (для поперечной оси), то где же `justify-self` (для главной)?
**Его не существует во Flexbox.** (Он есть в Grid, но это другая история).

**Почему?** Потому что в Flexbox главная ось — это единый поток. Если один элемент решит «выровняться по-своему» посередине потока, он неизбежно столкнется с соседями или собьет их позиции.

**Чем заменить?**
Если вам нужно прижать один элемент к противоположному краю главной оси, используйте магический `margin`. Например, `margin-left: auto` у последнего элемента в ряду прижмет его вправо, даже если остальные сгрудились слева. Это «чит-код», который работает идеально.

## ❓ Часто задаваемые вопросы

### «Влияет ли order на элементы внутри элемента?»
Нет. `order` меняет положение элемента только среди его «братьев» внутри одного и того же флекс-контейнера. На внутреннее содержимое элемента он никак не влияет.

### «Можно ли задать order через CSS-переменную?»
Да, это отличный способ управлять сложными интерфейсами или анимациями смены порядка.

### «Что если у одного блока order: 1, а у другого 999999?»
Блок с миллионом просто окажется после блока с единицей. Точные числа не важны, важна лишь их последовательность (больше/меньше).

## 🏗️ Лабораторная работа: Режиссура элементов

Посмотрите, как элементы нарушают иерархию и меняются местами.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <style>
    .stage {
      display: flex;
      height: 300px;
      background: #fdfdfd;
      border: 4px solid #2c3e50;
      align-items: flex-start; /* Базовое правило: все к верху */
      gap: 15px;
      padding: 20px;
    }

    .actor {
      width: 120px;
      padding: 20px;
      color: white;
      font-weight: bold;
      text-align: center;
      border-radius: 8px;
    }

    /* Первый в HTML, но последний на сцене */
    .actor-1 { background: #e74c3c; order: 10; }

    /* Второй в HTML, в центре сцены, первый по порядку */
    .actor-2 {
      background: #3498db;
      align-self: center;
      order: -1;
    }

    /* Третий в HTML, упал на дно, порядок по умолчанию (0) */
    .actor-3 {
      background: #2ecc71;
      align-self: flex-end;
      /* order: 0 (по умолчанию) */
    }
  </style>
</head>
<body>

  <h3>Сцена Flexbox: Индивидуальное поведение</h3>
  <div class="stage">
    <div class="actor actor-1">Актёр 1 (HTML 1, Order 10)</div>
    <div class="actor actor-2">Актёр 2 (HTML 2, Order -1, Center)</div>
    <div class="actor actor-3">Актёр 3 (HTML 3, Order 0, End)</div>
  </div>

  <p>Заметьте: Актёр 2 теперь первый слева, хотя в коде он второй.</p>

</body>
</html>
```

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. `align-self` — это личный выбор элемента, игнорирующий «мнение» родителя (`align-items`).
> 2. `order` переставляет блоки визуально, не меняя структуру документа.
> 3. Чем меньше число `order`, тем раньше (левее/выше) отобразится элемент.
> 4. `order` — главный инструмент для адаптивной перестановки контента на разных устройствах.
> 5. Никогда не используйте `order`, если это нарушает логическую последовательность чтения страницы.
