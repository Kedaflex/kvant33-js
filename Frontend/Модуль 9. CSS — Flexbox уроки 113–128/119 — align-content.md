## 📖 Что такое align-content?

В предыдущем уроке мы научились ломать «одномерность» флексбокса и переносить элементы на новые строки с помощью `flex-wrap`. Как только ваш контейнер превращается из одной шеренги в несколько рядов (этажей), в игру вступает новое правило.

Свойство `align-content` — это аналог `justify-content`, но для **Поперечной оси** и применительно не к отдельным элементам, а к **целым строкам** (флекс-линиям).

**Аналогия для запоминания:**
Представьте, что `align-items` — это то, как вы расставляете книги разного размера на **одной полке** (по нижнему краю или по центру полки).
А `align-content` — это то, как вы вешаете **сами полки** на стене: сдвинете их все к потолку, распределите равномерно по всей стене или соберете в центре.

---

## ⚠️ Строгий чеклист: Когда это работает?

Свойство `align-content` часто называют «самым неуловимым» во Flexbox, потому что новички часто пишут его там, где оно не может сработать. Для его активации должны совпасть два условия:

1. **Многострочность:** У вас должно быть как минимум две строки элементов. Это значит, что родителю обязан быть задан `flex-wrap: wrap` (или `wrap-reverse`), и элементам должно физически не хватать места в одной строке.
2. **Лишнее пространство:** У контейнера должна быть задана высота (`height` или `min-height`), которая **больше**, чем сумма высот всех его строк. Если контейнер «облегает» свои строки плотно, как вторая кожа, то лишнего места нет, и выравнивать их некуда.

Если хотя бы одно условие не соблюдено — свойство не будет иметь никакого эффекта.

## 🛠️ Разбор значений: Дизайн пространства

### 1. stretch (Значение по умолчанию)
Строки (флекс-линии) растягиваются, чтобы занять всё доступное пространство поровну.
- **Визуально:** Если у элементов не задана жесткая высота, они «распухают», чтобы заполнить контейнер без остатка.
- **Боль верстальщика:** Именно `stretch` создает те огромные, некрасивые пустые пространства между рядами карточек товаров, которые так бесят новичков.

### 2. flex-start
Все строки плотно прижимаются к началу контейнера (к верху в режиме `row`).
- **Кейс:** Это «лекарство» от лишних отступов. Если вы хотите, чтобы карточки в каталоге шли одна за другой с минимальным зазором — используйте именно это значение.

### 3. flex-end
Все строки «падают» на дно контейнера. Лишнее место остается сверху.

### 4. center
Все строки группируются ровно в центре контейнера по вертикали.
- **Кейс:** Идеально для «экранов-заглушек» или промо-блоков, где группа иконок или ссылок должна висеть в центре экрана.

### 5. space-between
Первая строка прилипает к верху, последняя — к низу, остальные распределяются равномерно.

### 6. space-around / space-evenly
Распределяют строки с равными промежутками вокруг них или между ними. Работает идентично логике `justify-content`, но на другом уровне иерархии.

## ⚖️ Великая битва: align-items vs align-content

Давайте разберемся в этом раз и навсегда, чтобы не тратить часы на отладку.

| Свойство | На что влияет | Когда использовать |
| :--- | :--- | :--- |
| **align-items** | Выравнивает **элементы внутри** каждой строки. | Всегда (даже если строка одна). |
| **align-content** | Выравнивает **сами строки** в контейнере. | Только когда строк **две и более**. |

**Пример:** У вас две строки карточек. В каждой строке одна карточка выше другой.
1. `align-items: center` сделает так, что в каждой строке карточки будут стоять по одной центральной линии.
2. `align-content: flex-start` сделает так, что вторая строка не уедет вниз контейнера, а встанет сразу под первой.

## ❓ Часто задаваемые вопросы

### «Я написал align-content: center, но ничего не произошло!»
Скорее всего, вы забыли задать `height` родителю. Если родитель имеет `height: auto`, он просто схлопывается по высоте контента, и «центра» для строк не существует.

### «Почему это свойство есть и во Flexbox, и в Grid?»
Это часть современного модуля CSS Box Alignment. Оно универсально. Хорошая новость в том, что выучив его один раз для Flexbox, вы уже будете знать, как оно работает в Grid.

### «Можно ли использовать оба свойства одновременно?»
Да, и это стандартная практика для сложных макетов. Вы выравниваете элементы внутри строк через `items`, а сами строки упаковываете через `content`.

## 🏗️ Лабораторная работа: Стенд выравнивания рядов

Поэкспериментируйте с высотой контейнера и значениями свойства.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <style>
    .playground {
      display: flex;
      flex-wrap: wrap;
      height: 500px; /* Фиксированная высота — ключ к работе свойства */
      width: 400px;
      border: 5px solid #2c3e50;
      background: #ecf0f1;
      margin-bottom: 50px;
      padding: 10px;
    }

    .box {
      width: 100px;
      height: 60px;
      background: #e74c3c;
      margin: 10px;
      color: white;
      display: flex;
      align-items: center;
      justify-content: center;
      font-weight: bold;
      border-radius: 4px;
    }

    /* Значения для тестирования */
    .start   { align-content: flex-start; }
    .end     { align-content: flex-end; }
    .center  { align-content: center; }
    .between { align-content: space-between; }
    .around  { align-content: space-around; }
    .stretch { align-content: stretch; }
  </style>
</head>
<body>

  <h3>1. align-content: flex-start (Плотная упаковка сверху)</h3>
  <div class="playground start">
    <div class="box">1</div><div class="box">2</div>
    <div class="box">3</div><div class="box">4</div>
    <div class="box">5</div>
  </div>

  <h3>2. align-content: space-between (Распределение по высоте)</h3>
  <div class="playground between">
    <div class="box">1</div><div class="box">2</div>
    <div class="box">3</div><div class="box">4</div>
    <div class="box">5</div>
  </div>

</body>
</html>
```

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. `align-content` — это «масштабное» выравнивание **рядов**, а не отдельных элементов.
> 2. Свойство требует `flex-wrap: wrap` и **запас высоты** в контейнере.
> 3. `flex-start` — ваш лучший друг для избавления от лишних дырок в многострочных сетках.
> 4. `stretch` (дефолт) растягивает строки, если у элементов нет фиксированного размера.
> 5. Если у вас одна строка — это свойство не делает ровным счетом ничего.
