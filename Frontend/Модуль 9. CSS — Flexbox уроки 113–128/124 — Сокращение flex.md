# ⚡ Сокращение flex: Три в одном и магия дефолтных значений

> - Изучить синтаксис и строгий порядок записи: `grow`, `shrink`, `basis`.

> - Овладеть магическими ключевыми словами: `initial`, `auto`, `none` и числовыми сокращениями.
> - Узнать, как сокращенная запись спасает от типичных ошибок распределения пространства.

## 📖 Почему сокращение — это не просто лень, а необходимость?

Мы потратили три урока, детально разбирая `flex-grow` (жадность), `flex-shrink` (сговорчивость) и `flex-basis` (точку отсчета). Вы можете продолжать писать их по отдельности, но спецификация W3C настоятельно рекомендует использовать универсальное свойство **`flex`**.

И это не просто ради экономии символов. У сокращенной записи есть два критических преимущества:
1. **Безопасность дефолтов:** Когда вы пишете свойства по отдельности, их значения по умолчанию могут конфликтовать или давать неожиданный результат. Свойство `flex` устанавливает пропущенные параметры в наиболее логичные и безопасные значения.
2. **Профессиональный стандарт:** Опытные разработчики сразу считывают «поведение» элемента по одной строке `flex: 1 0 200px`. Это делает код чище и понятнее для команды.

## 🛠️ Анатомия свойства flex: Порядок имеет значение

Синтаксис свойства `flex` всегда следует одному и тому же порядку. Запомните его как таблицу умножения:
`flex: [flex-grow] [flex-shrink] [flex-basis];`

**Примеры использования:**
- `flex: 1 1 300px;` (Элемент растет, сжимается, стартует с 300 пикселей).
- `flex: 2 0 10%;` (Растет в два раза быстрее, не сжимается вообще, база 10%).

### А что если я укажу не всё?
Здесь кроется самое интересное. Браузер — парень неглупый, он сам достроит картину:
- **Если указано одно число (например, `flex: 1`):** Оно воспринимается как `flex-grow`. При этом `flex-shrink` становится `1`, а `flex-basis` магическим образом становится **`0`**. (Это тот самый секрет идеально равных колонок!).
- **Если указана единица измерения (например, `flex: 200px`):** Она воспринимается как `flex-basis`. При этом `flex-grow` становится `1`, а `flex-shrink` тоже `1`.

## 💎 Полезные ключевые слова (Пресеты)

В CSS есть несколько «быстрых команд», которые заменяют часто используемые комбинации цифр.

### 1. flex: initial (Значение по умолчанию)
Это эквивалент `flex: 0 1 auto;`.
- Элемент не растет, но может сжиматься, если контейнеру тесно.
- Его размер определяется контентом.
- Это то, как ведет себя любой элемент, когда вы только включили `display: flex`.

### 2. flex: auto
Это эквивалент `flex: 1 1 auto;`.
- Настоящий «резиновый» блок. Он и растет, и сжимается.
- В отличие от `flex: 1`, он учитывает размер своего контента в качестве точки старта.

### 3. flex: none
Это эквивалент `flex: 0 0 auto;`.
- Элемент «заморожен». Он не растет и не сжимается.
- Идеально подходит для иконок, логотипов и кнопок, которые всегда должны выглядеть одинаково.

### 4. flex: [число] (например, flex: 1)
Это эквивалент `flex: [число] 1 0;`.
- Самый популярный способ создания сеток. База обнуляется, и элементы делят место строго пропорционально числам.

## 📐 Сравнение: Кто победит в верстке?

| Желаемое поведение | Длинный путь (Bad) | Коротки путь (Good) |
| :--- | :--- | :--- |
| **Идеально ровные колонки** | `grow: 1; shrink: 1; basis: 0;` | `flex: 1;` |
| **Фиксированный сайдбар** | `grow: 0; shrink: 0; basis: 250px;` | `flex: 0 0 250px;` |
| **Растянуть на всё место** | `grow: 1; shrink: 1; basis: auto;` | `flex: auto;` |
| **Не менять размер** | `grow: 0; shrink: 0; basis: auto;` | `flex: none;` |

## ❓ Часто задаваемые вопросы

### «Почему `flex: 1` делает колонки равными, а `flex: auto` — нет?»
Это тонкий момент.
- `flex: 1` устанавливает базу в **`0`**. Браузер думает, что элементы ничего не весят, и делит ВСЮ ширину поровну.
- `flex: auto` устанавливает базу в **`auto`**. Браузер сначала дает место под текст, а потом делит только ТО, ЧТО ОСТАЛОСЬ. Если в одном блоке текста много, он всё равно останется шире.

### «Можно ли опускать значения в середине?»
Нет, вы можете опускать только значения с конца. Нельзя написать `flex: 1 200px` (пропустив сжатие), браузер может неверно истолковать второе число. Всегда пишите либо одно значение, либо два (первые два), либо все три.

## 🏗️ Лабораторная работа: Быстрый прототип макета

Давайте соберем скелет страницы, используя только сокращенную запись `flex`.

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <style>
    .page-wrapper {
      display: flex;
      height: 400px;
      gap: 15px;
      background: #f0f2f5;
      padding: 15px;
      font-family: sans-serif;
    }

    .box {
      border-radius: 8px;
      color: white;
      padding: 20px;
      display: flex;
      align-items: center;
      justify-content: center;
    }

    /* Сайдбар: не растет, не сжимается, ровно 200px */
    .sidebar {
      flex: 0 0 200px;
      background: #2c3e50;
    }

    /* Контент: растет на все свободное место, база 0 */
    .main-content {
      flex: 1;
      background: #3498db;
    }

    /* Доп. инфо: по контенту, не сжимается */
    .extra-info {
      flex: none;
      background: #e74c3c;
    }
  </style>
</head>
<body>

  <div class="page-wrapper">
    <aside class="box sidebar">Sidebar (200px)</aside>
    <main class="box main-content">
      Main Content (Занимает всё оставшееся пространство)
    </main>
    <div class="box extra-info">
      Fixed Info
    </div>
  </div>

</body>
</html>
```

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. Сокращение `flex` — это путь профессионала. Оно чище и предсказуемее.
> 2. Порядок: **Grow → Shrink → Basis**.
> 3. `flex: 1` — лучший друг для создания идеально равных колонок (обнуляет базу).
> 4. `flex: none` — лучший друг для фиксированных элементов интерфейса.
> 5. Краткая запись сама подставляет безопасные значения для пропущенных параметров, оберегая вас от багов верстки.
