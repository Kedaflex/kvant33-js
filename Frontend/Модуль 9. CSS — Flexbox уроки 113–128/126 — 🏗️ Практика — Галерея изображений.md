## 📖 Постановка задачи: Галерея нового поколения

Сегодня мы создадим профессиональный компонент галереи. Наша цель — не просто выстроить картинки в ряд, а сделать «умную» систему, которая сама решает, сколько фотографий показать в строке в зависимости от ширины экрана.

**Требования к проекту:**
1. **Адаптивность:** На 4K мониторе — много колонок, на смартфоне — одна или две.
2. **Заполнение пустот:** Если в последнем ряду меньше элементов, чем в остальных, они должны красиво растягиваться, а не оставлять дырку.
3. **Эстетика:** Используем мягкие скругления (border-radius), глубокие тени при наведении и плавную анимацию.
4. **Интерактив:** Подпись к фото должна появляться плавно при наведении курсора.

## 🎨 Шаг 2: Стилизация в стиле минимализма

Превратим скучный список в премиальный интерфейс. Обратите внимание на использование переменных и плавных переходов.

```css
:root {
  --radius: 16px;
  --transition: 0.4s cubic-bezier(0.165, 0.84, 0.44, 1);
}

.gallery-container {
  display: flex;
  flex-wrap: wrap; /* Ключевое свойство для сетки */
  gap: 24px;       /* Современные отступы */
  padding: 40px;
  background-color: #f8f9fa;
}

.gallery-card {
  position: relative;
  /* ГИБКОСТЬ: Расти(1), Сжимайся(1), База(300px) */
  flex: 1 1 300px;
  height: 400px;
  border-radius: var(--radius);
  overflow: hidden;
  box-shadow: 0 10px 30px rgba(0,0,0,0.05);
  transition: var(--transition);
  cursor: pointer;
}

.gallery-card img {
  width: 100%;
  height: 100%;
  object-fit: cover; /* Спасает фото от искажений при растяжении блока */
  transition: var(--transition);
}

/* Эффекты при наведении */
.gallery-card:hover {
  transform: translateY(-8px);
  box-shadow: 0 20px 40px rgba(0,0,0,0.12);
}

.gallery-card:hover img {
  transform: scale(1.05);
}

.card-info {
  position: absolute;
  bottom: 0;
  left: 0;
  width: 100%;
  padding: 30px;
  background: linear-gradient(transparent, rgba(0,0,0,0.8));
  color: white;
  transform: translateY(20px);
  opacity: 0;
  transition: var(--transition);
}

.gallery-card:hover .card-info {
  transform: translateY(0);
  opacity: 1;
}
```

## 🔍 Глубокий разбор механики Flexbox

Почему этот макет считается «умным»? Давайте проанализируем связку свойств.

1. **`flex-wrap: wrap`**: Позволяет контейнеру превращаться из «строки» в «массив». Без него все ваши карточки сжались бы в тонкие полоски на одной линии.
2. **`flex-basis: 300px`**: Мы говорим браузеру: «В идеальном мире я хочу, чтобы карточка была шириной 300 пикселей». Браузер смотрит на ширину экрана (например, 1200px) и понимает, что в ряд влезет ровно 4 карточки (с учетом отступов).
3. **`flex-grow: 1`**: Это самое интересное. Если экран стал 1400px, то после размещения четырех карточек по 300px остается 200px лишнего места. Благодаря `flex-grow: 1` эти пиксели делятся поровну между карточками, и они становятся по 350px. Никаких пустых полей по краям!
4. **`object-fit: cover`**: Это свойство — лучший друг Flexbox. Так как ширина карточки меняется динамически, картинка внутри тоже растягивается. `cover` гарантирует, что она всегда будет заполнять весь блок, сохраняя свои пропорции и просто аккуратно обрезая лишнее.

## ❓ Часто задаваемые вопросы

### «Почему последняя карточка в ряду иногда становится огромной?»
Если у вас 4 карточки в верхнем ряду и всего одна в нижнем, то благодаря `flex-grow: 1` эта единственная карточка растянется на всю ширину контейнера. Если вам это не нравится, вы можете ограничить её рост через `max-width` или добавить пустые невидимые блоки в конец галереи.

### «Можно ли сделать одну фотографию в два раза шире остальных?»
Да! Просто добавьте этой конкретной карточке класс `.featured` и пропишите `.featured { flex-grow: 2; flex-basis: 600px; }`. Она будет забирать в два раза больше свободного места и стремиться к большему начальному размеру.

### «Как добавить подписи, которые видны всегда?»
Просто уберите `opacity: 0` и `transform: translateY(20px)` у класса `.card-info`. В минимализме часто используют подписи под фото, а не поверх них. В этом случае добавьте `flex-direction: column` самой карточке.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. Flexbox позволяет создавать сетки, которые сами рассчитывают количество колонок.
> 2. Свойство `flex: 1 1 300px` — это универсальная формула для адаптивной карточки.
> 3. `gap` — самый чистый способ управления расстоянием в галереях.
> 4. `object-fit: cover` критически важен для сохранения эстетики при изменении размеров блоков.
> 5. Современный минимализм строится на гармонии свободного пространства и плавных взаимодействий.
