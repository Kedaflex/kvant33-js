## 📖 Зачем нужны таблицы?

Таблицы в HTML предназначены для отображения **табличных данных**. Это любая информация, которую логично представить в виде сетки из строк и столбцов:
- Расписания
- Прайс-листы
- Статистика и отчеты
- Сравнение характеристик товаров

> [!warning] Историческая справка
> Раньше (в 2000-х годах) таблицы использовали для создания каркаса всего сайта (табличная вёрстка). Сейчас это **категорически запрещено**. Для расположения блоков на странице используются CSS Flexbox и Grid, а тег `<table>` — только для данных.

---

## 🏗️ Базовая структура таблицы

Для создания простейшей таблицы нужны три тега:

1.  **`<table>`** — главный контейнер всей таблицы.
2.  **`<tr>`** (Table Row) — строка таблицы.
3.  **`<td>`** (Table Data) — ячейка таблицы с данными.

### Как это работает?
Таблица строится **построчно**: сначала мы создаем «ряд» (`<tr>`), а внутри него размещаем «ячейки» (`<td>`).

```html
<table>
  <tr>
    <td>Ячейка 1.1</td>
    <td>Ячейка 1.2</td>
  </tr>
  <tr>
    <td>Ячейка 2.1</td>
    <td>Ячейка 2.2</td>
  </tr>
</table>
```

### Визуальная схема:
- `<table>` (Весь дом)
    - `<tr>` (Первый этаж)
        - `<td>` (Комната 1)
        - `<td>` (Комната 2)
    - `<tr>` (Второй этаж)
        - `<td>` (Комната 3)
        - `<td>` (Комната 4)

---

## 📌 Особенности тегов

| Тег | Что означает | Особенности |
|---|---|---|
| **`<table>`** | Таблица | Парный тег, оборачивает всё содержимое. |
| **`<tr>`** | Строка | Внутри `<table>` не может быть текста напрямую, только `<tr>` (или служебные теги, которые мы изучим позже). |
| **`<td>`** | Ячейка | Весь контент (текст, картинки, ссылки) пишется **только внутри** `<td>`. |

## 🛠️ Пример: Простой список покупок

Давай создадим таблицу из 3 строк и 2 столбцов:

```html
<table>
  <tr>
    <td>Яблоки</td>
    <td>1 кг</td>
  </tr>
  <tr>
    <td>Молоко</td>
    <td>2 пакета</td>
  </tr>
  <tr>
    <td>Хлеб</td>
    <td>1 батон</td>
  </tr>
</table>
```

> [!note] Где границы?
> Если ты запустишь этот код, ты увидишь просто текст, расставленный по сетке. По умолчанию у таблиц **нет видимых границ**. Мы научимся их добавлять в уроке по стилизации, но для отладки можно временно добавить атрибут `<table border="1">` (хотя в современном коде так не делают).

## 🧪 Практика

Создай файл `table.html` и напиши следующую разметку:

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Моя первая таблица</title>
</head>
<body>
    <h1>Простая таблица данных</h1>

    <table border="1"> <!-- Атрибут border только для того, чтобы мы видели сетку -->
        <tr>
            <td>Товар</td>
            <td>Цена</td>
        </tr>
        <tr>
            <td>Кофе</td>
            <td>350 руб</td>
        </tr>
        <tr>
            <td>Чай</td>
            <td>200 руб</td>
        </tr>
    </table>
</body>
</html>
```

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. Таблицы — только для **данных**, не для дизайна страницы.
> 2. Структура всегда: `table` → `tr` → `td`.
> 3. Контент пишется **только** внутри `<td>`.
> 4. Количество `<td>` в каждой строке `<tr>` должно быть одинаковым, чтобы таблица была ровной.
