## 📖 Что вообще такое «веб»?

**Веб** (Web) — это совокупность всех сайтов и веб-приложений, которые ты открываешь через браузер.

Каждый раз, когда ты заходишь на:
- google.com
- youtube.com
- vk.com
- любой интернет-магазин
- любой онлайн-сервис

...ты взаимодействуешь с результатом работы **веб-разработчиков**.

> [!tip] Факт
> По данным на 2024 год, в мире существует более **1.9 миллиарда** веб-сайтов. И каждый из них кто-то создал.

---

## 👤 Кто такой веб-разработчик?

**Веб-разработчик** — это специалист, который создаёт сайты и веб-приложения.

Но «создаёт сайты» — это очень широкое понятие. Давай разберёмся подробнее.

Когда ты открываешь любой сайт, ты видишь:
- **Текст** — заголовки, абзацы, списки
- **Картинки** — фотографии, иконки, логотипы
- **Кнопки** — которые можно нажимать
- **Формы** — в которые можно вводить данные
- **Анимации** — плавные переходы и эффекты
- **Данные** — которые откуда-то загружаются и куда-то сохраняются

За каждый из этих аспектов отвечают **разные технологии** и часто **разные специалисты**.

---

## 🔀 Три направления веб-разработки

| Направление | Что делает | Основные технологии |
|---|---|---|
| **Frontend** | То, что **видит** пользователь в браузере | HTML, CSS, JavaScript |
| **Backend** | То, что **работает на сервере**, «за кулисами» | Python, PHP, Node.js, Java, базы данных |
| **Fullstack** | И то, и другое **вместе** | Все технологии |

---

### 🎨 Frontend-разработчик (фронтенд)

**Frontend** — это всё, что пользователь видит и с чем взаимодействует в браузере.

Фронтенд-разработчик отвечает за:
- Как сайт **выглядит** (цвета, шрифты, расположение)
- Как сайт **реагирует** на действия пользователя (клики, наведение, ввод)
- Как сайт **адаптируется** под разные устройства (телефон, планшет, компьютер)

**Основные инструменты:**

| Технология | Для чего |
|---|---|
| **HTML** | Структура и содержание страницы |
| **CSS** | Внешний вид и оформление |
| **JavaScript** | Поведение и интерактивность |

> [!important] Именно это мы будем изучать
> Наш курс полностью посвящён **фронтенд-разработке**: HTML, CSS и JavaScript. Это фундамент, без которого невозможно двигаться дальше.

---

### ⚙️ Backend-разработчик (бэкенд)

**Backend** — это всё, что работает на сервере, «за кулисами».

Бэкенд-разработчик отвечает за:
- Хранение данных (базы данных)
- Обработку запросов (авторизация, оплата, поиск)
- Безопасность (защита данных пользователей)
- Логику приложения (что происходит, когда ты нажимаешь «Купить»)

**Основные инструменты:** Python, PHP, Java, Node.js, базы данных (MySQL, PostgreSQL, MongoDB)

---

### 🔄 Fullstack-разработчик (фулстек)

**Fullstack** — специалист, который умеет и то, и другое.

Такие разработчики могут создать сайт от начала до конца самостоятельно. Но обычно они знают всё понемногу, а не глубоко в каждой области.

---

## 🧱 Три кита фронтенда: HTML, CSS, JavaScript

Представь, что ты строишь дом:

| Технология | Аналогия с домом | Роль |
|---|---|---|
| **HTML** | 🧱 Стены, фундамент, крыша | **Структура.** Из чего состоит дом. Комнаты, двери, окна |
| **CSS** | 🎨 Обои, краска, мебель | **Внешний вид.** Как дом выглядит. Цвета, расположение мебели |
| **JavaScript** | ⚡ Электричество, водопровод | **Поведение.** Что дом умеет делать. Свет включается, вода течёт |

Теперь переведём это на язык веб-разработки:

---

### 🟠 HTML (HyperText Markup Language)

**Язык разметки**. Определяет **структуру** и **содержание** страницы.

HTML говорит браузеру:
- «Вот заголовок»
- «Вот абзац текста»
- «Вот картинка»
- «Вот ссылка»
- «Вот кнопка»

```html
<h1>Привет, мир!</h1>
<p>Это мой первый сайт.</p>
<button>Нажми меня</button>
```

```text
Не пугайся, если пока ничего не понятно. Через несколько уроков это будет казаться элементарным.
```

### 🔵 CSS (Cascading Style Sheets)

**Язык стилей**. Определяет **внешний вид** страницы.

CSS говорит браузеру:

- «Заголовок — синего цвета, размер 32 пикселя»
- «Кнопка — с закруглёнными углами и тенью»
- «На телефоне расположи элементы в одну колонку»



```CSS
h1 {
    color: blue;
    font-size: 32px;
}

button {
    background-color: green;
    border-radius: 8px;
}
```

---

### 🟡 JavaScript (JS)

**Язык программирования**. Определяет **поведение** страницы.

JavaScript говорит браузеру:

- «Когда пользователь нажмёт кнопку — покажи сообщение»
- «Загрузи данные с сервера и покажи их»
- «Проверь, правильно ли заполнена форма»



```JavaScript
let button = document.querySelector('button');

button.addEventListener('click', function() {
    alert('Привет! Ты нажал кнопку!');
});
```

## 🔍 Как они работают вместе?

Давай посмотрим на конкретном примере — **кнопка «Подписаться»**:

|Этап|Технология|Что происходит|
|---|---|---|
|1|**HTML**|Создаём кнопку с текстом «Подписаться»|
|2|**CSS**|Делаем кнопку зелёной, добавляем закруглённые углы, при наведении кнопка становится ярче|
|3|**JavaScript**|При клике отправляем данные на сервер, меняем текст на «Вы подписаны ✓», делаем кнопку неактивной|

> [!note] Важно  
> HTML, CSS и JavaScript **всегда** работают вместе. По отдельности они бесполезны. Поэтому мы изучаем их **все три** в одном курсе.

## ❓ Частые вопросы новичков

### «Нужно ли знать математику?»

Для фронтенда — **нет**. Базовая арифметика (сложение, умножение) — это максимум, что понадобится на начальном этапе.

### «Нужно ли знать английский?»

**Желательно**, но не обязательно. Код пишется на английском, но все ключевые слова — это простые английские слова (color, size, button). Ты их быстро запомнишь.

### «Сколько времени в день нужно уделять?»

**30–45 минут** на урок. Если можешь больше — отлично. Главное — **регулярность**, а не количество часов.

### «Я слишком стар/молод для этого?»

**Нет**. Программировать может научиться каждый. В веб-разработку приходят и в 14, и в 45 лет.

### «А вдруг у меня не получится?»

Получится. Абсолютно каждый разработчик когда-то не знал ничего. Главное — не бросать после первых трудностей.
