## 📖 Введение: Гибкость и точность

В прошлом уроке мы научились менять все теги одного типа. Но что, если на нашей странице 10 абзацев, и мы хотим сделать 3 из них красными, а остальные оставить черными? Селектор по тегу тут бессилен.

Здесь на сцену выходят **классы**. Это самый мощный, гибкий и часто используемый способ стилизации в современном вебе. Класс — это своего рода «ярлык» или «метка», которую мы вешаем на элемент в HTML, а затем обращаемся к этой метке в CSS.

---

## 🏗️ Как работают классы?

Работа с классами всегда состоит из двух шагов.

### Шаг 1: В HTML
Мы добавляем атрибут `class` к тегу. Внутри атрибута пишем любое имя (на английском).
```html
<p class="warning-text">Внимание! Это важное сообщение.</p>
```

### Шаг 2: В CSS
Чтобы обратиться к классу, мы пишем **точку `.`**, а затем имя класса.
```css
/* Точка в начале обязательна! */
.warning-text {
  color: red;
  font-weight: bold;
}
```

> [!tip] Важно
> Точка в CSS говорит браузеру: «Ищи не тег, а атрибут class с таким именем». В самом HTML внутри атрибута точка **не ставится**.

## 🚀 Преимущества классов

### 1. Многократное использование
Один и тот же класс можно повесить на любое количество элементов на странице. Например, вы создали класс `.btn` для красивой кнопки. Вы можете применить его к 10 разным кнопкам, и все они будут выглядеть одинаково.

### 2. Множественные классы на одном элементе
Это одна из самых крутых фишек. Вы можете дать элементу сразу несколько «характеристик», просто перечислив имена классов через пробел.

```html
<!-- Элемент получает стили и от .button, и от .button-large -->
<button class="button button-large">Купить</button>
```

### 3. Независимость от тегов
Класс можно повесить на `div`, на `p` и даже на `section`. Например, класс `.centered-text` будет работать одинаково хорошо везде.

## 🏷️ Правила именования классов

Как называть классы? Технически можно как угодно (кроме цифр в начале), но есть правила «хорошего тона»:

1. **Только латиница**: Используйте английские слова.
2. **Никаких пробелов**: Имя должно быть цельным. Для разделения слов используйте дефис `-` (чаще всего) или нижнее подчеркивание `_`.
3. **Смысл важнее вида**: Старайтесь называть классы по их функции, а не по внешнему виду.
   - Плохо: `.red-bold-text` (если цвет поменяется на синий, имя станет ложью).
   - Хорошо: `.error-message` (смысл сообщения не меняется от цвета).
4. **Регистр**: Пишите всё маленькими буквами (`lowercase`).

## ⚔️ Приоритет: Класс против Тега

Класс всегда «сильнее» тега. Если для тега `p` прописан синий цвет, а для класса `.red` — красный, то абзац с этим классом будет красным.

```css
p { color: blue; }    /* Слабый голос */
.red { color: red; }  /* Сильный голос */
```
```html
<p class="red">Я буду красным!</p>
```

Это позволяет нам задавать общие стили для всех тегов, а затем «точечно» изменять их там, где это нужно, с помощью классов.

---

## 🧩 Комбинированные селекторы (Уточнение)

Иногда нужно применить стиль к классу только в том случае, если он висит на определенном теге. Для этого тег и класс пишутся слитно (без пробела).

```css
/* Стиль применится только к <p class="alert"> */
/* На <div class="alert"> он не подействует */
p.alert {
  border: 1px solid red;
}
```
Это используется редко, но полезно знать, что такая возможность есть.

---

## 🧹 Организация кода с помощью классов

Именно на классах строятся современные методологии верстки (например, **BEM** — Блок, Элемент, Модификатор). Суть их в том, чтобы вообще не использовать селекторы по тегам для дизайна, а давать классы абсолютно каждому элементу. Это делает код предсказуемым и легким в поддержке.

### Пример профессиональной разметки:
```html
<article class="post">
  <h2 class="post-title">Заголовок статьи</h2>
  <p class="post-description">Краткое описание...</p>
  <button class="post-button">Читать</button>
</article>
```

## 🧪 Практика: Создаем систему уведомлений

Давайте создадим три типа сообщений, используя один общий класс для формы и разные классы для цвета.

```html
<div class="message message-info">
  Это обычная информация для пользователя.
</div>

<div class="message message-success">
  Поздравляем! Операция прошла успешно.
</div>

<div class="message message-error">
  Ошибка! Что-то пошло не так.
</div>
```

```css
/* Общие стили для всех сообщений */
.message {
  padding: 15px;
  margin-bottom: 10px;
  border-radius: 4px;
  font-family: sans-serif;
  border-left: 5px solid transparent;
}

/* Специфические стили для каждого типа */
.message-info {
  background-color: #e3f2fd;
  color: #0d47a1;
  border-left-color: #2196f3;
}

.message-success {
  background-color: #e8f5e9;
  color: #1b5e20;
  border-left-color: #4caf50;
}

.message-error {
  background-color: #ffebee;
  color: #b71c1c;
  border-left-color: #f44336;
}
```

## ⚠️ Типичные ошибки

1. **Забытая точка в CSS**: Написали `warning { ... }` вместо `.warning { ... }`. Браузер будет искать тег `<warning>`, которого не существует.
2. **Точка в HTML**: Написали `<div class=".box">`. Это создаст класс с именем ".box", и в CSS вам пришлось бы писать `..box`. Ошибка!
3. **Цифры в начале**: Имя `.1st-block` невалидно. Используйте `.block-1`.
4. **Слишком много классов**: Не нужно давать элементу 20 классов. Если вы чувствуете, что переборщили — скорее всего, вам нужно объединить стили в один осмысленный класс.

## ❓ Часто задаваемые вопросы

### «Есть ли лимит на количество классов на странице?»
Нет. Вы можете использовать тысячи классов. Браузеры оптимизированы для работы с ними.

### «Можно ли называть классы русскими буквами?»
Технически в современных браузерах это работает, но **никогда так не делайте**. Это считается признаком дилетантства. Только английский.

### «Что если у двух классов в одном элементе есть конфликтующие стили?»
Победит тот класс, который написан **ниже в CSS-файле**, а не тот, который написан первым в атрибуте `class`.

### «Как быстро придумать имя для класса?»
Используйте формулу: `сущность-деталь`. Например: `card-image`, `nav-link`, `footer-social-icon`.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **Класс** — основной способ стилизации элементов.
> 2. В HTML пишем `class="name"`, в CSS — `.name { ... }`.
> 3. Один класс — многим элементам. Много классов — одному элементу.
> 4. Классы **сильнее** тегов по приоритету.
> 5. Имена классов должны быть на латинице, без пробелов и со смыслом.
> 6. Дефис `-` — лучший разделитель слов в именах классов.
