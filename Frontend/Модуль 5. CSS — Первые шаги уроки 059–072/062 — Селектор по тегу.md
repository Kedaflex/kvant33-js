# 062 — Селектор по тегу

---

## 📖 Введение: Выбор по типу

Первый и самый простой способ сказать браузеру, какой элемент мы хотим стилизовать — это просто назвать его имя. Если мы хотим изменить все абзацы, мы используем селектор `p`. Если все заголовки — `h1`.

Этот тип селектора называется **селектором по типу** (Type Selector) или просто **селектором по тегу**. Он является фундаментом любого CSS-файла, так как позволяет задать общие правила для всего сайта разом.

---

## 🏗️ Как это работает?

Когда вы пишете в CSS имя тега без каких-либо дополнительных значков (точек или решеток), браузер ищет **все** совпадения в HTML-дереве.

```css
/* Все ссылки на сайте станут оранжевыми */
a {
  color: orange;
}

/* Все списки потеряют стандартные маркеры-точки */
ul {
  list-style-type: none;
}
```

### Главная особенность:
Селектор по тегу максимально охватный. Он не разбирает, находится ли абзац в шапке сайта, в подвале или внутри статьи. Если это тег `<p>`, стиль будет применен.

---

## 🏠 Глобальные селекторы: `html` и `body`

В начале работы над проектом разработчики часто используют селекторы по тегам для настройки «окружения».

### Тег `html`
Самый верхний элемент. Обычно здесь задают базовый размер шрифта или цвет фона всей страницы.
```css
html {
  font-size: 16px;
  background-color: #f0f0f0;
}
```

### Тег `body`
Здесь настраиваются параметры, которые должны наследоваться всеми остальными текстовыми элементами: семейство шрифтов, основной цвет текста, межстрочный интервал.
```css
body {
  font-family: 'Arial', sans-serif;
  color: #333;
  line-height: 1.6;
}
```

Благодаря **наследованию**, вам не нужно прописывать `font-family` для каждого абзаца или заголовка. Они «возьмут» этот шрифт у родительского элемента `body`.

---

## ⚙️ Когда использовать селекторы по тегу?

Этот метод идеально подходит для **базовой типографики** и **общих настроек**.

1. **Заголовки**: Установить единый шрифт и отступы для всех `h1`, `h2`, `h3`.
2. **Списки**: Убрать стандартные отступы у всех `ul` и `ol`.
3. **Изображения**: Сделать так, чтобы все картинки по умолчанию не вылезали за пределы экрана (`img { max-width: 100%; }`).
4. **Таблицы**: Задать общие границы для всех `table`.

---

## ⚔️ Ограничения и риски

Несмотря на простоту, у селекторов по тегу есть существенные минусы:

### 1. Отсутствие гибкости
Представьте, что вы сделали все `div` на сайте с красной рамкой. Но завтра вам понадобился `div` для невидимой обертки меню. Вам придется писать дополнительный код, чтобы «отменить» красную рамку для этого конкретного блока.

### 2. Низкий приоритет
Селекторы по тегу — самые «слабые» в иерархии CSS. Любой класс или ID легко перебьет эти стили. Это хорошо для базы, но плохо, если вы пытаетесь сделать что-то уникальное.

### 3. Производительность (в теории)
Если на странице 10 000 элементов `div`, браузеру придется проверить каждый из них. В современных реалиях это не критично, но в огромных проектах избыток селекторов по тегу считается плохим тоном.

> [!warning] Совет
> Никогда не стилизуйте универсальные контейнеры типа `div` или `span` напрямую по тегу. Это почти всегда приводит к проблемам в будущем. Используйте для них классы.

## 💡 Понятие "User Agent Styles"

Вы замечали, что даже если вы не написали ни одной строчки CSS, заголовки `h1` всё равно большие и жирные, а ссылки — синие?
Это работают **стили браузера по умолчанию** (User Agent Stylesheet). У каждого браузера (Chrome, Firefox) есть свой встроенный CSS-файл.

Используя селекторы по тегу, вы по сути **перезаписываете** эти стандартные настройки браузера под свои нужды.

---

## 🧹 Группировка для чистоты кода

Часто базовые настройки для разных тегов совпадают. Используйте группировку, о которой мы говорили в прошлом уроке.

```css
/* Устанавливаем общий шрифт для всех текстовых элементов */
h1, h2, h3, p, li, dt, dd {
  font-family: "Helvetica", sans-serif;
  margin-bottom: 1rem;
}
```

---

## 🧪 Практика: Создаем "Стиль по умолчанию"

Давайте напишем базовый набор правил, с которого можно начинать любой простой проект.

```css
/* Настройка всей страницы */
body {
  margin: 0; /* Убираем стандартные отступы у краев экрана */
  padding: 0;
  font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif;
  background-color: #fafafa;
  color: #2c3e50;
}

/* Стили для заголовков */
h1, h2 {
  color: #2980b9;
  text-transform: uppercase;
  letter-spacing: 2px;
}

/* Стили для ссылок */
a {
  color: #e67e22;
  text-decoration: none; /* Убираем подчеркивание */
}

/* Стили для списков */
ul {
  padding-left: 20px;
}

li {
  margin-bottom: 5px;
}
```

## ❓ Часто задаваемые вопросы

### «Что важнее: стиль для body или для p?»
Стиль для `p` важнее. Браузер всегда отдает приоритет более «близкому» селектору. `p` — это сам элемент, а `body` — его далекий предок.

### «Можно ли написать стиль для тега, которого не существует?»
Вы можете написать `my-tag { color: red; }`. Ошибки не будет, но так как в HTML нет такого тега, правило просто никогда не сработает (если только вы не создадите кастомный элемент на JavaScript).

### «Почему h1 не наследует размер шрифта от body?»
Заголовки наследуют *семейство* шрифта (`font-family`), но у них есть свой встроенный в браузер *размер* (`font-size`), который имеет более высокий приоритет, чем наследование. Чтобы изменить размер `h1`, нужно обратиться к нему напрямую.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **Селектор по тегу** выбирает все элементы данного типа на странице.
> 2. Идеален для **базовых настроек** (шрифты, общие отступы, цвета).
> 3. Позволяет переопределить стандартные стили браузера.
> 4. Имеет **низкий приоритет** (слабый селектор).
> 5. Не стоит использовать для универсальных контейнеров (`div`, `span`).
> 6. Хорошо работает в паре с наследованием (особенно стили для `body`).

## 🏁 Итог

Мы изучили самый широкий инструмент в CSS. Теперь вы умеете задавать правила игры для всей страницы сразу. Это экономит время и делает код системным.

Однако в реальной работе нам часто нужно сделать один абзац красным, а другой — синим. Для этого селекторы по тегам не подходят. В следующем уроке мы изучим «короля селекторов» — **Селектор по классу**, который дает нам полный контроль над каждым элементом.
