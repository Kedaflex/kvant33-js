# 067 — Фон — цвет и изображение

## 📖 Введение: Пространство за контентом

Каждый элемент в HTML можно представить как прозрачную коробку. CSS позволяет нам «покрасить» заднюю стенку этой коробки или наклеить на неё обои. Фон — это то, что создает глубину и атмосферу сайта.

Многие новички путают, когда нужно вставить картинку через HTML (`<img>`), а когда через CSS (`background-image`). Запомните простое правило: если картинка несет смысл (например, фото товара или портрет автора) — используйте `<img>`. Если картинка нужна только для украшения, текстуры или атмосферы — используйте свойства фона в CSS.

---

## 🎨 1. Фоновый цвет: `background-color`

Это свойство мы уже немного затрагивали. Оно заливает всю область элемента (включая внутренние отступы) сплошным цветом.

```css
.section {
  background-color: #f4f4f4;
}
```

### Зачем нужен цвет, если есть картинка?
Всегда задавайте фоновый цвет, даже если поверх него будет картинка. Почему?
1. Если картинка не загрузится (плохой интернет), пользователь увидит приятный цвет вместо пустоты.
2. Если картинка прозрачная, цвет будет просвечивать сквозь неё.

---

## 🖼️ 2. Фоновое изображение: `background-image`

Чтобы установить картинку на фон, используется функция `url()`.

```css
.hero {
  background-image: url('images/stars-bg.jpg');
}
```

### Пути к файлам (Важно!)
Если ваш CSS-файл лежит в папке `css/`, а картинки в папке `img/`, путь должен учитывать это:
`url('../img/stars-bg.jpg')` (две точки означают «выйти из текущей папки на уровень выше»).

---

## 🔄 3. Повторение фона: `background-repeat`

По умолчанию, если фоновая картинка меньше размера элемента, браузер начинает «замостить» её (как плитку в ванной) по горизонтали и вертикали.

Это свойство имеет четыре основных значения:
1. **`repeat`**: Повторять везде (по умолчанию). Идеально для бесшовных паттернов (текстура бумаги, песок, шум).
2. **`no-repeat`**: Показать картинку только один раз. Используется для больших фотографий.
3. **`repeat-x`**: Повторять только по горизонтали.
4. **`repeat-y`**: Повторять только по вертикали.

```css
.pattern {
  background-image: url('dot.png');
  background-repeat: repeat;
}

.banner {
  background-image: url('logo.png');
  background-repeat: no-repeat;
}
```

---

## ⚓ 4. Фиксация фона: `background-attachment`

Это свойство определяет, как ведет себя фон при прокрутке страницы.

- **`scroll`**: Фон прокручивается вместе с контентом (по умолчанию).
- **`fixed`**: Фон остается неподвижным, а текст «проезжает» поверх него. Это создает эффект параллакса — очень популярный прием в веб-дизайне.

```css
.fixed-bg {
  background-image: url('sky.jpg');
  background-attachment: fixed;
}
```

## 🧱 Комбинирование свойств

Вы можете задать элементу и цвет, и картинку одновременно.

```css
.header {
  background-color: #2c3e50; /* Синий цвет как запасной */
  background-image: url('pattern-overlay.png'); /* Прозрачный узор поверх цвета */
  background-repeat: repeat-x;
}
```

Такой подход делает дизайн более надежным и интересным.

---

## 🚀 Оптимизация фоновых изображений

Фоны часто бывают очень большими (Full HD и больше). Чтобы сайт не тормозил:
1. **Сжимайте картинки**: Используйте сервисы типа *TinyJPG*.
2. **Используйте правильные форматы**: `WebP` для фотографий, `SVG` для узоров и иконок.
3. **Размер имеет значение**: Не вставляйте картинку 4000px, если она будет отображаться в блоке 400px.

---

## 🧪 Практика: Создаем атмосферную секцию

Давайте создадим блок с красивым фоном, который не повторяется.

```html
<section class="nature-block">
  <h1>Красота дикой природы</h1>
  <p>Загляните в самые укромные уголки нашей планеты.</p>
</section>
```

```css
.nature-block {
  /* Задаем темный фон, чтобы белый текст всегда читался */
  background-color: #1a2a1a;

  /* Подключаем картинку */
  background-image: url('https://images.unsplash.com/photo-1441974231531-c6227db76b6e');

  /* Запрещаем повторение */
  background-repeat: no-repeat;

  /* Стили для контента внутри */
  color: white;
  padding: 100px 20px;
  text-align: center;
  font-family: sans-serif;
}
```

## ⚠️ Типичные ошибки

1. **Неправильный путь к картинке**: Самая частая проблема. Помните, что путь в CSS считается **относительно CSS-файла**, а не HTML-файла.
2. **Забытый фоновый цвет**: Если картинка тяжелая, пользователь увидит белый экран (или «дырку» в дизайне), пока она грузится.
3. **Огромные PNG**: Использование PNG для фотографий на фоне. PNG — тяжелый формат, используйте его только если нужна прозрачность. Для фото — только JPG или WebP.
4. **Конфликт текста и фона**: Поставили черный текст на темную картинку. Текст стал нечитаемым.

## ❓ Часто задаваемые вопросы

### «Можно ли сделать два фоновых изображения для одного элемента?»
Да! В современном CSS вы можете перечислить несколько `url()` через запятую.
`background-image: url('top-layer.png'), url('bottom-layer.jpg');`
Первая картинка в списке будет лежать «выше» всех остальных.

### «Почему моя фоновая картинка не отображается?»
Проверьте три вещи:
1. Правильный ли путь (`href`).
2. Есть ли у элемента высота (`height`) или контент внутри. Если элемент пустой и у него нет высоты, его размер равен 0, и фон просто негде рисовать.
3. Нет ли опечатки в названии файла (помните про регистр букв: `Image.jpg` и `image.jpg` — это разные файлы в интернете).

### «Что лучше: фон в CSS или тег img?»
Для декора (паттерны, градиенты, абстракции) — CSS. Для контента (фото котика в статье про котиков) — `img`. Это важно для SEO и скринридеров.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **`background-color`** — заливка цветом, важна как запасной вариант.
> 2. **`background-image`** — вставка изображения через `url()`.
> 3. **`background-repeat`** управляет «плиточностью» (repeat, no-repeat).
> 4. Пути в CSS считаются **от файла стилей**.
> 5. Декоративные картинки — в CSS, смысловые — в HTML через `<img>`.
