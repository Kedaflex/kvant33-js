## 📖 Введение: Язык визуальных команд

Если HTML — это разметка структуры, то CSS — это набор инструкций для браузера. Чтобы браузер нас понял, мы должны «говорить» с ним на строгом языке правил. У этого языка очень простой, но жесткий синтаксис. Ошибка в одном символе (например, пропущенная скобка) может привести к тому, что весь дизайн сайта «развалится».

В этом уроке мы разберем «анатомию» CSS-кода до мельчайших деталей, чтобы вы писали правила уверенно и без ошибок.

---

## 🏗️ Анатомия CSS-правила

Любая инструкция в CSS называется **правилом** (Rule). Оно состоит из двух больших частей: **Селектора** и **Блока объявлений**.

Рассмотрим пример:
```css
h1 {
  color: blue;
  font-size: 20px;
}
```

### 1. Селектор (Selector)
Это «адрес» или «имя» элемента, который мы хотим изменить. В данном примере `h1` — это селектор по тегу. Он говорит браузеру: «Найди на странице все заголовки первого уровня».

### 2. Блок объявлений (Declaration Block)
Это всё, что находится внутри **фигурных скобок `{ }`**. Блок содержит одну или несколько команд (объявлений), которые описывают, что именно нужно поменять у выбранного элемента.

---

## ⚙️ Из чего состоит объявление?

Каждое объявление внутри фигурных скобок состоит из двух частей, разделенных **двоеточием `:`**.

### Свойство (Property)
Это визуальный параметр, который мы хотим изменить (например, `color`, `width`, `margin`). Свойство — это всегда понятное английское слово. Существует более 500 различных CSS-свойств, но в повседневной работе вы будете постоянно использовать около 50 из них.

### Значение (Value)
Это конкретная настройка свойства.
- Если свойство — `color` (цвет), то значением может быть `red` или `blue`.
- Если свойство — `font-size` (размер шрифта), то значением может быть `20px` или `2em`.

> [!warning] Точка с запятой `;`
> Каждое объявление **обязано** заканчиваться точкой с запятой. Она сообщает браузеру: «Эта команда закончена, переходи к следующей». Если вы забудете её, браузер попытается «склеить» две команды в одну и не поймет ни одну из них.

---

## 🎨 Виды значений в CSS

Значения могут быть разными, и важно понимать, как их записывать.

1. **Ключевые слова**: Заранее определенные слова (например, `bold`, `center`, `solid`, `transparent`).
2. **Числовые значения с единицами измерения**: Число + буквы без пробела (например, `10px`, `50%`, `2rem`, `100vh`).
3. **Цветовые коды**: Шестнадцатеричные коды (`#ff0000`), RGB (`rgb(255, 0, 0)`) или названия цветов (`red`).
4. **Функции**: Специальные команды, например `calc(100% - 20px)` или `url("image.png")`.

---

## 🧹 Правила хорошего тона (Кодстайл)

Браузеру всё равно, как вы пишете код: в одну строчку или с огромными отступами. Но вам и вашим коллегам — нет.

### Плохой стиль (Трудно читать):
```css
h1{color:red;font-size:20px;text-align:center;}
```

### Хороший стиль (Профессиональный):
- Одно свойство на одной строке.
- Отступ (2 или 4 пробела) перед свойством.
- Пробел после двоеточия.
- Закрывающая фигурная скобка на новой строке.

```css
h1 {
  color: red;
  font-size: 20px;
  text-align: center;
}
```

## 🤝 Группировка селекторов

Часто нам нужно применить одни и те же стили к разным элементам. Чтобы не дублировать код, селекторы можно перечислить через **запятую**.

```css
/* Вместо этого: */
h1 { color: red; }
h2 { color: red; }
h3 { color: red; }

/* Пишем так: */
h1, h2, h3 {
  color: red;
  font-family: Arial, sans-serif;
}
```
Это делает CSS-файл короче, чище и его легче обновлять.

---

## 💬 Комментарии в CSS

Комментарии помогают объяснить, за что отвечает тот или иной блок кода. Браузер их полностью игнорирует.
В CSS комментарии пишутся между знаками `/*` и `*/`.

```css
/* Основные стили для шапки сайта */
header {
  background-color: #f4f4f4; /* Светло-серый фон */
}
```

> [!tip] Совет
> Используйте комментарии, чтобы разделять длинный CSS-файл на логические секции (например: `/* Typography */`, `/* Layout */`, `/* Components */`).

---

## ⚠️ Частые ошибки синтаксиса

1. **Пробел между числом и единицей измерения**: `20 px` — это ошибка. Нужно писать `20px`.
2. **Использование круглых скобок вместо фигурных**: `h1 ( color: blue; )` — не сработает.
3. **Пропущенное двоеточие**: `color red;` — браузер не поймет, где свойство, а где значение.
4. **Опечатки в названиях**: `colour: red;` (британский вариант) — не сработает, CSS понимает только американский английский (`color`).
5. **Забытая точка с запятой**: Самая коварная ошибка. Часто ломает не текущую строку, а следующую за ней.

## 🧪 Практика: Пишем и исправляем правила

Давайте попробуем написать правила для типичной карточки товара.

```css
/* Стили для всей карточки */
.product-card {
  border: 1px solid #ddd;
  padding: 15px;
  border-radius: 8px;
}

/* Группируем заголовки и цены */
.product-title, .product-price {
  font-family: sans-serif;
  color: #333;
}

/* Специфические стили для цены */
.product-price {
  font-weight: bold;
  font-size: 1.2em;
}

/* Стили для кнопки */
.buy-button {
  background-color: #ff5733;
  color: white;
  border: none;
  cursor: pointer; /* Делаем курсор "ручкой" при наведении */
}
```

## ❓ Часто задаваемые вопросы

### «Обязательно ли ставить точку с запятой после последнего свойства в блоке?»
Технически — нет, перед закрывающей скобкой её можно опустить. Но профессионалы **всегда** её ставят. Это защищает вас от ошибок, если вы позже решите добавить еще одну строку ниже.

### «Чувствителен ли CSS к регистру?»
Селекторы тегов (`h1`) и свойства (`color`) не чувствительны. Однако названия классов (`.myClass`) и ID могут быть чувствительны в зависимости от браузера и типа документа. Лучше всего всегда писать CSS строчными (маленькими) буквами.

### «Что будет, если я напишу свойство, которое браузер не знает?»
Браузер просто проигнорирует эту строку и перейдет к следующей. Он не выдаст ошибку на экране, но вы увидите предупреждение в консоли разработчика.

### «Можно ли писать CSS по-русски?»
Нет. Все свойства, ключевые слова и селекторы тегов — только на английском. Русскими могут быть только комментарии и (очень редко и не рекомендуется) названия классов.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **CSS-правило** = Селектор + Фигурные скобки.
> 2. **Объявление** = Свойство + Двоеточие + Значение + Точка с запятой.
> 3. **Точка с запятой** — обязательна в конце каждой строки.
> 4. **Запятая** используется для группировки нескольких селекторов.
> 5. Пишите код красиво: одна строка — одно свойство. Это экономит время при поиске багов.
> 6. Используйте **комментарии** `/* ... */` для организации кода.
