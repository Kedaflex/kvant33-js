## 📖 Введение: Один за всех

Представьте, что вы создали огромный сайт. Вы выбрали красивый оттенок синего для кнопок, заголовков, рамок и иконок. Вы использовали этот цвет 150 раз в своем CSS-файле. Вдруг заказчик говорит: «Синий — это скучно, давайте сделаем всё оранжевым».

Вам придется использовать поиск и замену, рискуя испортить другие элементы. Это неудобно и опасно. Именно для решения этой проблемы были придуманы **CSS-переменные** (официально они называются *Custom Properties*).

Переменная — это именованная «коробочка», в которую вы кладете значение один раз, а потом используете это имя по всему коду. Поменяли значение в коробочке — оно поменялось везде!

---

## 🏗️ 1. Объявление переменных

Чтобы переменные были доступны по всему сайту, их принято объявлять в специальном месте — псевдоклассе **`:root`**. Это, по сути, самый верхний уровень вашего HTML-документа.

### Синтаксис:
Имя переменной **всегда** должно начинаться с двух дефисов `--`.

```css
:root {
  --main-color: #3498db;
  --text-color: #333;
  --gap: 20px;
  --base-font: 'Arial', sans-serif;
}
```

---

## 🚀 2. Использование переменных

Чтобы «достать» значение из переменной, используется функция **`var()`**.

```css
h1 {
  color: var(--main-color);
  font-family: var(--base-font);
}

.card {
  padding: var(--gap);
  border: 1px solid var(--main-color);
}
```

### Значение "на всякий случай" (Fallback)
Если вы боитесь, что переменная может быть не определена, вы можете указать второе значение через запятую. Оно сработает как запасной вариант.
`color: var(--brand-color, red);`

---

## 🏠 3. Область видимости (Scoping)

Переменные, объявленные в `:root`, являются **глобальными** — их видят все. Но вы можете создать переменную и внутри конкретного блока. Тогда её будут видеть только этот блок и его «дети».

```css
.dark-section {
  --bg-color: black;
  --text-color: white;

  background-color: var(--bg-color);
  color: var(--text-color);
}
```
Снаружи этого блока переменная `--bg-color` не существует. Это позволяет создавать независимые темы для разных частей сайта.

---

## 🌓 4. Кейс: Темная тема за минуту

Переменные — это основа для создания темного режима. Вам не нужно переписывать все стили, вам нужно просто поменять значения переменных.

```css
:root {
  --bg: white;
  --text: black;
}

/* Если у тега body есть класс .dark-theme */
body.dark-theme {
  --bg: #1a1a1a;
  --text: #f0f0f0;
}

/* Сами стили не меняются! */
body {
  background-color: var(--bg);
  color: var(--text);
}
```
Теперь, просто переключая один класс в HTML, вы мгновенно меняете весь вид сайта.

---

## 🧮 5. Переменные и математика

Переменные отлично работают в паре с функцией `calc()`.

```css
:root {
  --base-size: 10px;
}

.box {
  /* Ширина будет 100px */
  width: calc(var(--base-size) * 10);
}
```

---

## 🏆 Преимущества использования переменных

1. **Читаемость**: Класс с `color: var(--danger-red)` гораздо понятнее, чем `color: #e74c3c`. Вы сразу понимаете, что это за цвет и зачем он здесь.
2. **Легкость правок**: Весь брендбук сайта (цвета, шрифты, скругления) собран в одном месте вверху файла.
3. **Чистота кода**: Вы не дублируете одни и те же HEX-коды сотни раз.
4. **Интеграция с JS**: JavaScript может очень легко менять CSS-переменные, создавая динамические эффекты (например, менять цвет фона в зависимости от времени суток).

## 🧪 Практика: Создаем цветовую схему проекта

Давайте настроим базовые переменные для нашего учебного проекта.

```css
/* 1. Настройка переменных */
:root {
  /* Основные цвета */
  --primary-color: #2ecc71;
  --secondary-color: #27ae60;
  --accent-color: #e67e22;

  /* Нейтральные тона */
  --light-grey: #ecf0f1;
  --dark-grey: #2c3e50;

  /* Размеры и отступы */
  --border-radius: 8px;
  --container-width: 1100px;
}

/* 2. Применение в коде */
body {
  background-color: var(--light-grey);
  color: var(--dark-grey);
}

.button-cta {
  background-color: var(--primary-color);
  border-radius: var(--border-radius);
  padding: 15px 30px;
  border: none;
  color: white;
}

.button-cta:hover {
  background-color: var(--secondary-color);
  cursor: pointer;
}

.container {
  max-width: var(--container-width);
  margin: 0 auto;
}
```

## ⚠️ Типичные ошибки

1. **Пропущенные дефисы**: Написать `var(main-color)` вместо `var(--main-color)`.
2. **Объявление вне селектора**: Попытаться объявить переменную просто так, в «воздухе», не внутри `:root` или класса.
3. **Неверный регистр**: CSS-переменные чувствительны к регистру! `--Main-Color` и `--main-color` — это разные переменные.
4. **Циклические ссылки**: Пытаться объявить `--a: var(--b)` и в то же время `--b: var(--a)`. Браузер запутается и проигнорирует обе.
5. **Использование var() как значения свойства**: Вы не можете написать `var(--my-property): blue;`. Имя свойства всегда должно быть жестким.

## ❓ Часто задаваемые вопросы

### «В чем разница между переменными в CSS и в SASS/SCSS?»
Переменные в препроцессорах (SASS) вычисляются в момент создания файла. CSS-переменные (Custom Properties) — «живые». Они работают прямо в браузере и могут меняться в зависимости от условий (например, в `@media` запросах), чего не умеет SASS.

### «Все ли браузеры поддерживают переменные?»
Все современные браузеры поддерживают их уже много лет (начиная с 2016-2017 гг.). Проблемы были только в Internet Explorer, но он официально не поддерживается.

### «Можно ли хранить в переменной сразу несколько значений?»
Да, например: `--border: 1px solid black;`. Затем вы можете использовать это как `border: var(--border);`.

### «Как переменные влияют на производительность?»
Если у вас тысячи переменных, которые постоянно меняются, это может немного нагрузить процессор. Но в обычном проекте вы никогда не заметите разницы.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **CSS-переменные** начинаются с `--` (например, `--my-var`).
> 2. Глобальные переменные живут в **`:root`**.
> 3. Доступ к значению — через функцию **`var()`**.
> 4. Переменные экономят время при изменении дизайна и делают код понятным.
> 5. Они позволяют легко реализовывать темы (темную/светлую).
> 6. Имена переменных чувствительны к регистру.
> 7. Это стандарт современной профессиональной верстки.
