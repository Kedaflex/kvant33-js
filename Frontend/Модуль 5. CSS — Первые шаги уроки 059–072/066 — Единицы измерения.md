## 📖 Введение: Размер имеет значение

В мире бумажной полиграфии всё просто: у вас есть лист формата А4, и его размеры никогда не меняются. В вебе всё иначе. Ваш сайт будут смотреть на крошечных часах, обычных смартфонах, ноутбуках и огромных 4K-мониторах.

Если вы зададите размер текста в «жестких» единицах, он может оказаться огромным на телефоне или микроскопическим на мониторе. Поэтому в CSS существует целый арсенал единиц измерения. Умение правильно их комбинировать — это и есть секрет качественной, адаптивной верстки.

---

## 📏 1. Абсолютные единицы (Пиксели)

Самая известная единица — это **пиксель (`px`)**.

### Что такое пиксель?
В CSS один пиксель — это не всегда одна точка на вашем физическом экране. Это логическая единица измерения.

- **Плюсы**: Предсказуемость. Вы точно знаете, что блок `100px` будет всегда одинакового размера. Идеально для тонких рамок (`border: 1px`).
- **Минусы**: Отсутствие гибкости. Если пользователь изменит размер шрифта в настройках своего браузера (например, из-за плохого зрения), ваш текст в `px` останется прежним. Это нарушение правил доступности.

---

## 🔄 2. Относительные единицы (em и rem)

Это «умные» единицы, которые зависят от размера шрифта.

### rem (Root em)
Зависит от размера шрифта **корневого элемента** (тега `<html>`).
- По умолчанию в браузерах `1rem = 16px`.
- Если вы напишете `font-size: 2rem`, текст станет `32px`.
- **Почему это круто?**: Если пользователь увеличит системный шрифт в браузере до `20px`, ваш `2rem` автоматически станет `40px`. Сайт «подстроится» под человека.

### em
Зависит от размера шрифта **родительского элемента**.
- Если у родителя `font-size: 20px`, то внутри него `1em = 20px`.
- **Опасность**: `em` накапливаются. Если вкладывать элементы друг в друга, размеры могут начать расти или уменьшаться в геометрической прогрессии.
- **Когда использовать?**: Для отступов (`padding`, `margin`) внутри кнопок или карточек, чтобы они пропорционально менялись вместе с текстом.

---

## 📊 3. Проценты (%)

Зависят от размера **родительского контейнера**.
- `width: 50%` — блок займет ровно половину ширины своего «папы».
- Идеально подходят для создания резиновых сеток (колонок сайта).

---

## 📱 4. Единицы области просмотра (Viewport units)

Эти единицы зависят напрямую от размера окна браузера.

1. **vw (Viewport Width)**: 1% от ширины окна. `100vw` — это вся ширина экрана.
2. **vh (Viewport Height)**: 1% от высоты окна. `100vh` — это вся высота экрана. Очень удобно для создания первых экранов лендингов, которые должны занимать ровно один экран.
3. **vmin / vmax**: Выбирают минимальное или максимальное значение из ширины и высоты. Полезно для элементов, которые всегда должны помещаться в экран (например, модальные окна).

---

## 🧮 5. Математика в CSS: `calc()`

Иногда нам нужно смешать разные единицы. Например: «сделай блок на всю ширину, но оставь 40 пикселей для боковой панели».

```css
.content {
  width: calc(100% - 40px);
}
```

Это невероятно мощный инструмент, который избавляет от сложных расчетов в голове.

---

## 🛡️ 6. Современный подход: `clamp()`

Это «король» адаптивной типографики. Функция позволяет задать три значения: минимальное, предпочтительное и максимальное.

```css
h1 {
  /* Минимум 24px, плавно растет как 5% от ширины экрана, но не больше 48px */
  font-size: clamp(24px, 5vw, 48px);
}
```
С этой строчкой ваш заголовок будет идеально смотреться и на iPhone, и на телевизоре без всяких дополнительных настроек.

---

## 🏆 Золотой стандарт выбора единиц

Профессиональные верстальщики следуют такому правилу:

1. **Шрифты**: Только `rem`.
2. **Внутренние отступы (padding)**: `rem` или `em`.
3. **Ширина контейнеров**: Проценты `%`, `vw` или `max-width` в пикселях.
4. **Тонкие линии и границы**: Пиксели `px`.
5. **Высота секций**: `vh` или `min-height`.

---

## 🧪 Практика: Гибкая карточка

Давайте создадим элемент, который использует разные единицы для разных целей.

```css
.card {
  /* Ширина зависит от экрана, но не больше 600px */
  width: 90%;
  max-width: 600px;

  /* Отступы зависят от базового шрифта */
  padding: 2rem;
  margin: 20px auto;

  border: 1px solid #ccc; /* Твердая граница */
  background-color: white;
}

.card-title {
  /* Адаптивный размер заголовка */
  font-size: clamp(1.5rem, 4vw, 2.5rem);
  margin-bottom: 1em; /* Отступ зависит от размера самого заголовка */
}

.card-text {
  font-size: 1rem; /* Стандартный размер (16px) */
  line-height: 1.5;
}
```

## ⚠️ Типичные ошибки

1. **Забытые единицы**: Написали `width: 100` вместо `width: 100%` или `100px`. В CSS число без единицы (кроме нуля) — это ошибка, и стиль не сработает.
2. **Слишком много vh на мобильных**: У мобильных браузеров есть выезжающие панели адреса, которые «отъедают» высоту. `100vh` может работать дергано. Для этого сейчас вводят новые единицы `dvh`, `lvh`, `svh`.
3. **Использование px для текста**: Это главная ошибка против доступности. Всегда используйте `rem`.
4. **Математические ошибки в calc()**: Помните, что вокруг знаков `+` и `-` в `calc()` **обязательно должны быть пробелы**. `calc(100%-20px)` не сработает!

## ❓ Часто задаваемые вопросы

### «Почему 1rem обычно равен 16px?»
Это стандарт, заложенный в браузеры еще в 90-х годах. Большинство людей считают такой размер текста комфортным для чтения. Но пользователь может поменять его в настройках на 20px или 24px.

### «В чем разница между 100% и 100vw?»
`100%` — это ширина внутри родителя (без учета его отступов). `100vw` — это физическая ширина всего окна браузера (включая полосу прокрутки). Если на странице есть скролл, `100vw` может вызвать горизонтальную прокрутку, будьте осторожны!

### «Нужно ли использовать пункты (pt)?»
Нет. `pt` (points) предназначены только для печати на бумаге. В вебе их использовать не следует.

### «Что такое ch?»
`1ch` равен ширине символа "0" в текущем шрифте. Очень удобно, если вы хотите ограничить ширину текстового блока так, чтобы в строке было не больше 60-80 символов (идеально для чтения).

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **px** — для жестких размеров и границ.
> 2. **rem** — лучший выбор для шрифтов (уважает настройки пользователя).
> 3. **%** — для гибких колонок и сеток.
> 4. **vh/vw** — для элементов, привязанных к размеру экрана.
> 5. **calc()** — для сложных вычислений прямо в CSS.
> 6. **clamp()** — для создания «идеальной» типографики.
> 7. Современная верстка должна быть **относительной**, а не абсолютной.
