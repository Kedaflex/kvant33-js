## 📖 Введение: Куда писать код?

Мы уже знаем, что CSS отвечает за внешний вид сайта. Но где именно мы должны писать эти правила? В HTML существует три разных места, куда можно «спрятать» или «подключить» стили.

Каждый из этих способов имеет свою область применения. Один хорош для быстрой правки, другой — для специфических настроек конкретной страницы, а третий — для создания серьезных, больших проектов. Давайте разберем их по порядку: от худшего к лучшему.

---

## 🏗️ 1. Встроенные стили (Inline Styles)

Это самый простой, но и самый «грязный» способ. Стили пишутся прямо внутри HTML-тега с помощью глобального атрибута **`style`**.

```html
<h1 style="color: red; font-size: 30px;">Красный заголовок</h1>
<p style="background-color: yellow;">Абзац с желтым фоном</p>
```

### 👍 Плюсы:
- Очень высокая скорость (не нужно создавать лишних файлов).
- Самый высокий приоритет (эти стили перебьют почти всё остальное).
- Удобно для динамических изменений через JavaScript.

### 👎 Минусы:
- **Каша в коде**: Смешивание смысла (HTML) и оформления (CSS).
- **Сложность поддержки**: Если у вас 100 заголовков, и вы хотите поменять их цвет, вам придется менять 100 атрибутов.
- **Повторение кода**: Нельзя написать одно правило для всех абзацев сразу.

> [!danger] Вердикт
> В реальной верстке старайтесь **избегать** этого способа. Он используется только в исключительных случаях.

---

## 🏗️ 2. Внутренние стили (Internal Styles)

Стили пишутся внутри HTML-файла, но не в тегах, а в специальном блоке **`<style>`**. Этот блок традиционно размещают внутри секции **`<head>`**.

```html
<head>
  <style>
    h1 { color: blue; }
    p { line-height: 1.5; color: #333; }
  </style>
</head>
```

### 👍 Плюсы:
- Стили отделены от тегов, код HTML становится чище.
- Можно одним правилом стилизовать сразу все элементы на странице (например, все `<li>`).
- Не нужно загружать дополнительные файлы с сервера.

### 👎 Минусы:
- Стили работают **только на одной странице**. Если у вас на сайте есть страница "О нас" и "Контакты", вам придется копировать блок `<style>` в оба файла.
- HTML-файл становится слишком длинным и тяжелым.

> [!tip] Вердикт
> Хорошо подходит для маленьких одностраничных сайтов (лендингов) или для специфических стилей, которые нужны только на одной конкретной странице.

---

## 🏗️ 3. Внешние стили (External Styles) — Профессиональный стандарт

Это именно тот способ, которым пользуются все профессиональные разработчики. Стили выносятся в отдельный файл с расширением **`.css`** (например, `style.css`), а затем этот файл подключается к HTML-странице с помощью тега **`<link>`**.

### Как это сделать:
1. Создаем файл `style.css`.
2. В HTML-файле внутри `<head>` пишем:

```html
<head>
  <link rel="stylesheet" href="style.css">
</head>
```

**Разбор тега `<link>`**:
- **`rel="stylesheet"`**: Сообщает браузеру, что мы подключаем именно таблицу стилей.
- **`href`**: Путь к файлу (так же, как у картинок или ссылок).

### 👍 Плюсы:
- **Идеальный порядок**: Полное разделение контента и оформления.
- **Глобальность**: Один файл стилей можно подключить к 1000 страниц. Изменили цвет в одном месте — он поменялся на всём сайте.
- **Кэширование**: Браузер скачивает CSS-файл один раз и запоминает его. При переходе на другую страницу сайта он загружается мгновенно.
- **Удобство работы**: Вы можете открыть HTML в одном окне редактора, а CSS — в другом.

### 👎 Минусы:
- Лишний запрос к серверу (но это решается современными технологиями оптимизации).

## ⚔️ Битва приоритетов: Кто победит?

Что произойдет, если вы подключите все три способа одновременно и в каждом укажете разный цвет для одного и того же заголовка?

1. **Inline (в теге)** — **Самый сильный**. Победит всех.
2. **Internal (в head)** и **External (ссылка)** — Они имеют равную силу. Побеждает тот, кто написан **ниже** в коде HTML.

### Пример:
```html
<head>
  <!-- Внешний файл говорит: цвет СИНИЙ -->
  <link rel="stylesheet" href="style.css">

  <style>
    /* Внутренний стиль говорит: цвет ЗЕЛЕНЫЙ */
    h1 { color: green; }
  </style>
</head>
<body>
  <!-- Инлайн стиль говорит: цвет КРАСНЫЙ -->
  <h1 style="color: red;">Какой я буду?</h1>
</body>
```
**Результат**: Заголовок будет **КРАСНЫМ**, потому что Inline-стиль — это «последнее слово» автора прямо у элемента.

## 📂 Организация файлов в проекте

Когда ваш проект растет, важно не сваливать всё в одну кучу. Обычно структуру делают так:

```text
📂 мой-проект/
├── 📄 index.html
├── 📄 about.html
├── 📂 css/
│   └── 📄 style.css
├── 📂 img/
│   └── 📄 logo.png
└── 📂 js/
    └── 📄 script.js
```

В этом случае путь в теге `<link>` будет выглядеть так:
`<link rel="stylesheet" href="css/style.css">`

---

## ⚠️ Типичные ошибки подключения

1. **Неверный путь**: Файл лежит в папке `css/style.css`, а в ссылке написано просто `style.css`. Итог — стили не работают.
2. **Опечатка в `rel`**: Написали `rel="style"` вместо `rel="stylesheet"`. Браузер проигнорирует файл.
3. **Подключение ПОСЛЕ использования**: Рекомендуется всегда подключать CSS в самом верху, в теге `<head>`. Если подключить его в конце `<body>`, пользователь на долю секунды увидит «голый» некрасивый сайт до того, как применятся стили. Это называется *FOUC (Flash of Unstyled Content)*.
4. **Забытая точка с запятой**: Внутри CSS-файла каждая строчка должна заканчиваться `;`. Если её забыть, всё, что написано ниже, может сломаться.

## 🧪 Практика: Подключаем первый стиль

Давайте создадим простую страницу и подключим к ней стили внешним способом.

**Файл `index.html`**:
```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Мой первый CSS</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <h1>Я учусь подключать стили</h1>
    <p>Если этот текст стал зеленым — я всё сделал правильно!</p>
</body>
</html>
```

**Файл `style.css`**:
```css
/* Обратите внимание: в CSS файле НЕ нужно писать теги <style> */
h1 {
    color: darkblue;
    text-decoration: underline;
}

p {
    color: green;
    font-size: 18px;
    font-weight: bold;
}
```

## ❓ Часто задаваемые вопросы

### «Можно ли подключить несколько CSS-файлов к одной странице?»
Да! Вы можете подключить сколько угодно файлов: один для шрифтов, другой для сетки, третий для главной страницы. Но помните: каждый `<link>` — это отдельный запрос к серверу, что может замедлить сайт. Профессионалы обычно объединяют их в один большой файл.

### «Что такое @import?»
Внутри одного CSS-файла можно подключить другой с помощью команды `@import url("another.css");`. Это удобно, но работает медленнее, чем тег `<link>`, поэтому использовать это не рекомендуется без особой нужды.

### «Почему мои изменения в CSS не видны в браузере?»
Скорее всего, сработал кэш. Браузер запомнил старую версию файла и не хочет скачивать новую. Нажмите `Ctrl + F5` (жесткая перезагрузка), чтобы сбросить кэш.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **Inline** (атрибут style) — для быстрых правок, плохой тон.
> 2. **Internal** (тег style в head) — для одной страницы.
> 3. **External** (файл .css и тег link) — **стандарт индустрии**.
> 4. Тег `<link>` должен быть внутри `<head>`.
> 5. Внешние стили позволяют управлять дизайном тысяч страниц из одного файла.
> 6. Не забывайте про правильные пути к файлам и атрибут `rel="stylesheet"`.
