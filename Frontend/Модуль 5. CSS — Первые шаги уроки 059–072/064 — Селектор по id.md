## 📖 Введение: Личный номер элемента

Мы уже умеем стилизовать группы элементов с помощью классов. Но иногда на странице есть элемент, который существует в единственном экземпляре и должен иметь совершенно уникальный стиль. Для таких случаев в HTML существует атрибут **`id`** (идентификатор).

Идентификатор — это как номер паспорта. Он должен быть уникальным для каждого гражданина (элемента) в пределах одного города (HTML-страницы). В CSS селектор по ID обладает огромной мощью, но именно эта мощь делает его опасным инструментом в руках новичка.

---

## 🏗️ Как работают селекторы по ID?

Как и в случае с классами, работа идет в два этапа.

### Шаг 1: В HTML
Добавляем атрибут `id` к тегу. Значение должно быть уникальным — его нельзя повторять для других тегов на этой же странице.
```html
<header id="main-header">...</header>
```

### Шаг 2: В CSS
Чтобы обратиться к ID, мы используем символ **решетки `#`**, а затем пишем имя идентификатора.
```css
/* Решетка в начале обязательна! */
#main-header {
  background-color: #333;
  color: white;
  padding: 20px;
}
```

## ⚖️ Ключевые отличия ID от Классов

Это самый важный раздел урока. Запомните эти правила:

1. **Уникальность**: Класс можно использовать много раз. ID — **только один раз на странице**.
2. **Количество на элемент**: У элемента может быть 10 классов, но **только один ID**.
3. **Сила (Приоритет)**: ID намного сильнее класса. Если у элемента есть класс, говорящий «будь красным», и ID, говорящий «будь синим» — элемент будет синим.

### Пример битвы приоритетов:
```css
p { color: green; }      /* Вес: 1 (самый слабый) */
.my-class { color: red; } /* Вес: 10 (сильнее тега) */
#my-id { color: blue; }    /* Вес: 100 (сильнее класса) */
```
```html
<p id="my-id" class="my-class">Какого я цвета?</p>
```
**Результат**: Текст будет **СИНИМ**. Даже если вы переставите правила в CSS-файле местами, ID всё равно победит, потому что он «тяжелее».

## 🚀 Когда использование ID — это хорошо?

В современной верстке (особенно при использовании методологии БЭМ) стараются вообще не использовать ID для оформления. Но есть случаи, где они незаменимы:

### 1. Якорные ссылки (Навигация)
Если вы хотите, чтобы при клике на ссылку страница «открутилась» к конкретному блоку, вам нужен ID.
```html
<a href="#contacts">Перейти к контактам</a>
...
<section id="contacts">Здесь наши контакты</section>
```

### 2. Связка Label и Input
В формах мы используем ID, чтобы связать текстовую подпись с полем ввода.
```html
<label for="user-name">Ваше имя:</label>
<input type="text" id="user-name">
```

### 3. JavaScript
Программистам на JS гораздо проще и быстрее найти на странице один уникальный элемент по его ID, чем искать его среди сотен классов.

## ⚠️ Почему ID — это часто плохо для CSS?

Многие новички начинают вешать ID на всё подряд, потому что это кажется удобным. Вот почему так делать **нельзя**:

1. **Невозможность переиспользования**: Вы написали крутой стиль для `#header`. Завтра вы захотели такой же стиль для подвала (`footer`). Но вы не можете дать им один и тот же ID. Вам придется либо копировать код, либо переделывать всё на классы.
2. **Проблемы с приоритетом**: Если вы стилизовали элемент через ID, вы больше не сможете легко изменить его через классы. Вам придется либо использовать `!important` (что очень плохо), либо создавать еще более сложный селектор.
3. **Хрупкость кода**: Код на ID сложнее поддерживать. Классы позволяют создавать гибкие «кирпичики», из которых строится сайт. ID — это «монолит», который трудно сдвинуть.

> [!abstract] Золотое правило
> Используйте **классы** для 99% задач по дизайну. Используйте **ID** для навигации, форм и JavaScript, но старайтесь **не писать для них стили** в CSS.

---

## 🎨 Правила именования ID

Такие же, как у классов:
- Только латиница.
- Без пробелов.
- Не начинать с цифр.
- Используйте дефис `-` для разделения слов.
- Имя должно быть уникальным не только по смыслу, но и по написанию.

## 🧪 Практика: Навигация по странице

Давайте создадим простую структуру с якорными ссылками и минимальным стилем для уникальных блоков.

```html
<nav>
  <ul>
    <li><a href="#home">Главная</a></li>
    <li><a href="#about">О нас</a></li>
    <li><a href="#projects">Проекты</a></li>
  </ul>
</nav>

<section id="home">
  <h1>Добро пожаловать!</h1>
</section>

<section id="about">
  <h2>Наша история</h2>
</section>

<section id="projects">
  <h2>Что мы сделали</h2>
</section>
```

```css
/* Используем ID только для того, чтобы задать разный фон уникальным секциям */
#home {
  background-color: #f1f1f1;
  height: 500px;
}

#about {
  background-color: #e1e1e1;
  height: 500px;
}

#projects {
  background-color: #d1d1d1;
  height: 500px;
}

/* Общий стиль для всех секций задаем через ТЕГ или КЛАСС */
section {
  padding: 50px;
  border-bottom: 2px solid #ccc;
}
```

## ⚠️ Типичные ошибки

1. **Дублирование ID**: Написали `id="box"` дважды. Браузер не выдаст ошибку, но якорные ссылки и JavaScript будут работать непредсказуемо (обычно они найдут только первый элемент).
2. **Смешивание `#` и `.`**: Написали в CSS `.my-id` для элемента с `id="my-id"`. Стиль не применится.
3. **Использование ID для каждого элемента**: Попытка заменить классы идентификаторами. Это превратит ваш CSS-файл в бесконечную простыню неповторяющегося кода.

## ❓ Часто задаваемые вопросы

### «Что если я всё-таки использовал один ID дважды?»
Валидатор W3C выдаст ошибку. Браузер применит стили CSS к обоим элементам (чаще всего), но якорная ссылка `<a href="#id">` приведет пользователя только к первому элементу. JavaScript функция `getElementById` также вернет только первый найденный элемент.

### «Можно ли дать элементу и ID, и класс одновременно?»
Да, конечно. Это стандартная практика. Класс используется для общего оформления (например, размер и шрифт), а ID — для уникальных правок или как метка для скриптов.

### «Почему ID сильнее классов?»
Так устроена спецификация CSS. Изначально предполагалось, что ID — это нечто очень важное и уникальное, поэтому его голос должен быть громче всех остальных.

### «Как быстро найти все ID в моем коде?»
В редакторе VS Code можно нажать `Ctrl + F` и искать символ `#`. Но будьте внимательны — этот же символ используется для цветовых кодов.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **ID** — это уникальный идентификатор элемента на странице.
> 2. В HTML пишем `id="name"`, в CSS — `#name { ... }`.
> 3. ID должен быть **строго один** на всю страницу.
> 4. Селектор по ID обладает **очень высоким приоритетом** (сильнее тегов и классов).
> 5. Главные сферы применения ID: **якоря**, **формы** и **JavaScript**.
> 6. Старайтесь **не использовать ID для стилизации**, если можно обойтись классами.
