## 📖 Почему формат имеет значение?

Одну и ту же фотографию можно сохранить в разных форматах. И результат будет **очень разным**:

| Формат | Размер файла | Качество | Прозрачность |
|---|---|---|---|
| BMP (без сжатия) | 5 000 КБ | Идеальное | ❌ |
| PNG | 1 200 КБ | Идеальное | ✅ |
| JPEG | 200 КБ | Хорошее (с потерями) | ❌ |
| WebP | 120 КБ | Хорошее (с потерями) | ✅ |
| AVIF | 80 КБ | Хорошее (с потерями) | ✅ |

Размер файла напрямую влияет на **скорость загрузки** сайта. Выбор правильного формата — это баланс между **качеством** и **размером**.

---

## 🎨 Два типа графики

Прежде чем разбирать форматы, нужно понять два фундаментально разных подхода к хранению изображений.

### Растровая графика (Raster)

Изображение состоит из **пикселей** — крошечных цветных точек. Как мозаика.

| Свойство | Описание |
|---|---|
| Из чего состоит | Сетка пикселей (точек) |
| При увеличении | **Размывается**, видны отдельные пиксели |
| Для чего подходит | Фотографии, сложные изображения с множеством цветов |
| Форматы | JPEG, PNG, GIF, WebP, AVIF |

### Векторная графика (Vector)

Изображение описывается **математическими формулами** — линиями, кривыми, фигурами.

| Свойство | Описание |
|---|---|
| Из чего состоит | Математические формулы (точки, линии, кривые) |
| При увеличении | **Остаётся чётким** при любом масштабе |
| Для чего подходит | Логотипы, иконки, схемы, простые иллюстрации |
| Форматы | SVG |

### Сравнение при увеличении

| Параметр | Растр (JPEG/PNG) | Вектор (SVG) |
|---|---|---|
| Оригинал | Чёткий | Чёткий |
| Увеличение ×2 | Слегка размытый | Чёткий |
| Увеличение ×10 | Пиксельная каша | Чёткий |
| Увеличение ×100 | Квадраты | Чёткий |

## 📋 Основные форматы для веба

### 🟡 JPEG (JPG)

| Характеристика | Значение |
|---|---|
| **Полное название** | Joint Photographic Experts Group |
| **Расширение** | `.jpg` или `.jpeg` (одно и то же) |
| **Тип** | Растровый |
| **Сжатие** | С потерями (lossy) |
| **Прозрачность** | ❌ Не поддерживает |
| **Анимация** | ❌ Не поддерживает |
| **Количество цветов** | ~16.7 миллионов |
| **Лучше всего для** | Фотографии, сложные изображения |

#### Как работает сжатие JPEG

JPEG **удаляет** часть информации, которую человеческий глаз почти не замечает. Чем сильнее сжатие — тем меньше файл, но хуже качество.

| Качество | Размер файла | Визуальное качество |
|---|---|---|
| 100% | Большой (~1 МБ) | Идеальное |
| 80% | Средний (~200 КБ) | Отличное (потери почти не видны) |
| 60% | Маленький (~100 КБ) | Хорошее (небольшие артефакты) |
| 20% | Очень маленький (~30 КБ) | Плохое (заметные артефакты, «квадратики») |

> [!tip] Рекомендуемое качество JPEG
> Для веба оптимально **70–85%**. Это даёт хороший баланс между качеством и размером файла.

#### Когда использовать JPEG

| Подходит ✅ | Не подходит ❌ |
|---|---|
| Фотографии людей, природы, еды | Логотипы, иконки |
| Фоновые изображения | Текст на картинке |
| Изображения с множеством цветов и градиентов | Изображения с прозрачным фоном |
| Баннеры, hero-изображения | Простые иллюстрации с чёткими линиями |

---

### 🔵 PNG

| Характеристика | Значение |
|---|---|
| **Полное название** | Portable Network Graphics |
| **Расширение** | `.png` |
| **Тип** | Растровый |
| **Сжатие** | Без потерь (lossless) |
| **Прозрачность** | ✅ Полная (включая полупрозрачность) |
| **Анимация** | ❌ Не поддерживает (есть APNG, но редко используется) |
| **Количество цветов** | ~16.7 миллионов + альфа-канал |
| **Лучше всего для** | Изображения с прозрачностью, скриншоты, графика с текстом |

#### Прозрачность — главное преимущество PNG

PNG поддерживает **альфа-канал** — каждый пиксель может быть полностью прозрачным, полупрозрачным или непрозрачным.

```html
<!-- Логотип с прозрачным фоном -->
<img src="logo.png" alt="Логотип" width="200" height="80">
```
Логотип в PNG будет отображаться **без белого квадрата** вокруг — фон будет просвечивать.

#### Когда использовать PNG

|Подходит ✅|Не подходит ❌|
|---|---|
|Логотипы с прозрачным фоном|Фотографии (файл будет огромным)|
|Скриншоты интерфейса|Большие фоновые изображения|
|Иллюстрации с чёткими линиями|Баннеры и hero-секции|
|Изображения с текстом|Анимации|
|Иконки (если не SVG)|—|

---

### 🟢 SVG

|Характеристика|Значение|
|---|---|
|**Полное название**|Scalable Vector Graphics|
|**Расширение**|`.svg`|
|**Тип**|Векторный|
|**Сжатие**|Текстовый формат (можно сжимать gzip)|
|**Прозрачность**|✅ Поддерживает|
|**Анимация**|✅ Поддерживает (CSS/JS-анимации)|
|**Масштабирование**|Идеальное при любом размере|
|**Лучше всего для**|Логотипы, иконки, схемы, простые иллюстрации|
#### Особенности SVG

SVG — это фактически **код** (XML-разметка). Файл SVG можно открыть в текстовом редакторе и увидеть:

```XML

<svg width="100" height="100" xmlns="http://www.w3.org/2000/svg">
  <circle cx="50" cy="50" r="40" fill="#3498db" />
</svg>
```

Это рисует **синий круг**. Математически. Без единого пикселя.

#### Три способа вставить SVG в HTML

|Способ|Код|Особенности|
|---|---|---|
|Через `<img>`|`<img src="icon.svg" alt="Иконка">`|Простой, но нельзя стилизовать CSS|
|Инлайн (встроенный)|`<svg>...</svg>` прямо в HTML|Можно стилизовать через CSS и JS|
|Через CSS `background`|`background-image: url('icon.svg')`|Для фоновых изображений|

```HTML

<!-- Способ 1: через <img> — самый простой -->
<img src="logo.svg" alt="Логотип компании" width="200" height="60">

<!-- Способ 2: инлайн — можно стилизовать -->
<svg width="50" height="50">
  <rect width="50" height="50" rx="10" fill="#e74c3c" />
</svg>
```

#### Когда использовать SVG

|Подходит ✅|Не подходит ❌|
|---|---|
|Логотипы|Фотографии|
|Иконки|Сложные изображения с градиентами и текстурами|
|Схемы, графики, диаграммы|Растровые изображения|
|Иллюстрации с чёткими линиями|—|
|Карты|—|
|Анимированные элементы интерфейса|—|

### 🟠 WebP

|Характеристика|Значение|
|---|---|
|**Полное название**|Web Picture|
|**Расширение**|`.webp`|
|**Тип**|Растровый|
|**Сжатие**|С потерями и без потерь (оба режима)|
|**Прозрачность**|✅ Поддерживает|
|**Анимация**|✅ Поддерживает|
|**Разработчик**|Google|
|**Лучше всего для**|Замена JPEG и PNG с меньшим размером файла|

#### Главное преимущество WebP

WebP даёт **такое же качество**, как JPEG/PNG, но с **меньшим размером файла**:

|Сравнение|JPEG|WebP|Экономия|
|---|---|---|---|
|Фотография (lossy)|200 КБ|120 КБ|**40%**|
|Скриншот (lossless)|800 КБ|500 КБ|**37%**|

#### Поддержка браузерами

|Браузер|Поддержка WebP|
|---|---|
|Chrome|✅ С 2014 года|
|Firefox|✅ С 2019 года|
|Safari|✅ С 2020 года (macOS Big Sur)|
|Edge|✅ С 2018 года|

WebP поддерживается **всеми современными браузерами**. Можно смело использовать.

#### Когда использовать WebP

|Подходит ✅|Не подходит ❌|
|---|---|
|Фотографии (замена JPEG)|Когда нужна поддержка очень старых браузеров|
|Изображения с прозрачностью (замена PNG)|Логотипы и иконки (лучше SVG)|
|Анимации (замена GIF)|Печатная полиграфия|

---

### 🟣 AVIF

|Характеристика|Значение|
|---|---|
|**Полное название**|AV1 Image File Format|
|**Расширение**|`.avif`|
|**Тип**|Растровый|
|**Сжатие**|С потерями и без потерь|
|**Прозрачность**|✅ Поддерживает|
|**Анимация**|✅ Поддерживает|
|**Лучше всего для**|Максимальное сжатие при отличном качестве|

#### AVIF vs WebP

|Сравнение|WebP|AVIF|Экономия|
|---|---|---|---|
|Фотография|120 КБ|80 КБ|**33%** ещё меньше|

AVIF — самый эффективный формат на 2024 год, но поддержка **чуть хуже**, чем у WebP.

|Браузер|Поддержка AVIF|
|---|---|
|Chrome|✅ С 2020|
|Firefox|✅ С 2021|
|Safari|✅ С 2023 (macOS Ventura / iOS 16)|
|Edge|✅ С 2021|

> [!note] На будущее  
> AVIF — формат будущего. Пока для начинающих достаточно JPEG + PNG + SVG + WebP. AVIF можно добавить позже через `<picture>`.

---

### ⚪ GIF

|Характеристика|Значение|
|---|---|
|**Полное название**|Graphics Interchange Format|
|**Расширение**|`.gif`|
|**Тип**|Растровый|
|**Сжатие**|Без потерь|
|**Прозрачность**|⚠️ Только полная (1 бит: прозрачно или нет, без полупрозрачности)|
|**Анимация**|✅ Поддерживает|
|**Количество цветов**|Максимум **256**|
|**Лучше всего для**|Простые анимации (мемы, реакции)|

#### Почему GIF устарел

|Проблема GIF|Лучшая альтернатива|
|---|---|
|Только 256 цветов|JPEG / WebP / AVIF|
|Большой размер анимаций|WebP-анимации, видео MP4|
|Нет полупрозрачности|PNG / WebP|

> [!tip] Если нужна анимация  
> Вместо GIF используй короткое **видео** (MP4) или **анимированный WebP**. Они в 5–10 раз меньше по размеру при лучшем качестве.

>

> - **Логотипы, иконки, схемы** → SVG
> - Если сомневаешься → **WebP**

## 🛠️ Оптимизация изображений

Даже правильный формат не спасёт, если файл **слишком большой**. Перед добавлением на сайт изображения нужно **оптимизировать**.

### Что значит «оптимизировать»?

|Действие|Описание|
|---|---|
|**Уменьшить разрешение**|Если изображение 4000×3000, а на сайте показывается 800×600 — уменьши до 1600×1200 (×2 для Retina)|
|**Сжать файл**|Удалить ненужные метаданные, применить алгоритмы сжатия|
|**Выбрать правильный формат**|JPEG для фото, SVG для логотипов|
|**Конвертировать в WebP**|Если поддержка браузерами не проблема|

### Бесплатные инструменты оптимизации

#### Онлайн-сервисы

|Сервис|URL|Возможности|
|---|---|---|
|**Squoosh**|squoosh.app|Лучший инструмент. Сжатие, конвертация, сравнение. От Google|
|**TinyPNG**|tinypng.com|Сжатие PNG и JPEG. Простой интерфейс|
|**iLoveIMG**|iloveimg.com|Сжатие, изменение размера, конвертация|

#### Десктопные программы

|Программа|Платформа|Возможности|
|---|---|---|
|**GIMP**|Windows / Mac / Linux|Бесплатный редактор (аналог Photoshop)|
|**Paint.NET**|Windows|Простой редактор с плагинами|
|**Preview**|Mac|Встроенный инструмент, «Экспорт» с настройкой качества|

### Рекомендуемые размеры для веба

|Тип изображения|Рекомендуемая ширина|Размер файла|
|---|---|---|
|Hero-секция (фон)|1920px|До 200 КБ|
|Фото в статье|800–1200px|До 150 КБ|
|Миниатюра (thumbnail)|300–400px|До 50 КБ|
|Логотип|200–300px|До 20 КБ|
|Иконка|24–64px|До 5 КБ|

> [!tip] Retina-дисплеи  
> Для чётких картинок на Retina-экранах (MacBook, iPhone) сохраняй изображения с **двойным разрешением**: если на сайте картинка 400×300, файл должен быть 800×600. Браузер сожмёт до 400×300, но на Retina будет чётко.

## 🔍 Как узнать формат и размер через DevTools

|Шаг|Действие|
|---|---|
|1|Открой DevTools → вкладка **Network**|
|2|Обнови страницу (F5)|
|3|Отфильтруй по типу **Img** (кнопка в панели фильтров)|
|4|Увидишь список всех изображений на странице|
|5|Для каждого отображается **имя**, **тип**, **размер файла**, **время загрузки**|

## ❓ Частые вопросы

### «Чем `.jpg` отличается от `.jpeg`?»

**Ничем.** Это один и тот же формат. Расширение `.jpg` появилось потому, что старые версии Windows поддерживали только 3-символьные расширения. Используй любое, но будь последовательным.

### «Можно ли конвертировать JPEG в PNG, чтобы улучшить качество?»

**Нет.** Если изображение уже было сжато в JPEG (с потерями), потерянную информацию **не вернуть**. Конвертация в PNG просто сделает файл больше без улучшения качества.

### «Почему SVG не подходит для фотографий?»

Фотография содержит миллионы пикселей с уникальными цветами. Описать это математическими формулами — **невозможно** (точнее, файл будет огромным). SVG эффективен только для **простых фигур и линий**.

### «Стоит ли переводить все картинки в WebP?»

Для новых проектов — **да**, если не нужна поддержка очень старых браузеров. WebP почти всегда лучше JPEG и PNG по соотношению размер/качество.

### «Где взять SVG-иконки?»

Бесплатные SVG-иконки: **heroicons.com**, **lucide.dev**, **feathericons.com**, **icons.getbootstrap.com**.

### «Как конвертировать JPEG в WebP?»

Используй **squoosh.app** — открой JPEG, выбери формат WebP, настрой качество, скачай.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 
> 1. **Растровая графика** (JPEG, PNG, WebP) — пиксели, размывается при увеличении
> 2. **Векторная графика** (SVG) — формулы, чёткая при любом масштабе
> 3. **JPEG** — фотографии, сжатие с потерями, нет прозрачности
> 4. **PNG** — прозрачность, без потерь, скриншоты, логотипы (если не SVG)
> 5. **SVG** — логотипы, иконки, схемы. Масштабируется идеально
> 6. **WebP** — современная замена JPEG и PNG, меньше размер, поддерживает прозрачность
> 7. **AVIF** — формат будущего, ещё лучше WebP
> 8. **GIF** — устарел, используй WebP или MP4 для анимаций
> 9. Всегда **оптимизируй** картинки перед добавлением на сайт (squoosh.app)
> 10. Фото → **JPEG/WebP**, чёткие линии → **PNG/WebP**, логотипы → **SVG**
