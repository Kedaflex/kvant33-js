## 📖 Когда порядок имеет значение

В прошлом уроке мы изучили `<ul>` — список, в котором порядок **не важен**. Но бывают ситуации, когда порядок **критически важен**:

| Ситуация | Почему порядок важен |
|---|---|
| Рецепт приготовления | Нельзя добавить соус раньше, чем обжарить мясо |
| Инструкция по установке | Шаги выполняются последовательно |
| Рейтинг фильмов | 1-е место важнее 2-го |
| Результаты соревнований | Золото, серебро, бронза |
| Учебный план | Тема 1 изучается до темы 2 |
| Юридические пункты | Пункт 1, пункт 2, подпункт 2.1 |

Для таких случаев существует **`<ol>`** — нумерованный (упорядоченный) список.

---

## 🔢 Тег `<ol>` — нумерованный список

**`<ol>`** (ordered list — упорядоченный список) создаёт список, в котором каждый элемент автоматически получает **номер**.

### Базовый синтаксис

```html
<ol>
  <li>Разогрей духовку до 180°C</li>
  <li>Смешай муку и сахар</li>
  <li>Добавь яйца и масло</li>
  <li>Выпекай 30 минут</li>
</ol>
```

**Результат в браузере:**

1. Разогрей духовку до 180°C
2. Смешай муку и сахар
3. Добавь яйца и масло
4. Выпекай 30 минут

---

### Как выглядит по умолчанию

|Свойство|Значение по умолчанию|
|---|---|
|Нумерация|Арабские цифры (1, 2, 3...)|
|Начало|С 1|
|Направление|По возрастанию|
|Отступ слева|~40px (padding-left)|
|Элемент `<ol>`|Блочный|
|Элемент `<li>`|Блочный|

---

## 📏 Правила использования

Правила такие же, как у `<ul>`:

|Правило|Подробности|
|---|---|
|`<li>` — только внутри `<ol>`|Нельзя использовать `<li>` без родительского списка|
|Внутри `<ol>` — только `<li>`|Нельзя помещать `<p>`, `<div>` и т.д. напрямую в `<ol>`|
|Внутри `<li>` — что угодно|Текст, ссылки, картинки, другие списки|
|Вложенный список — внутри `<li>`|Не рядом с `<li>`, а внутри него|

---

## ⚙️ Атрибуты `<ol>`

У `<ol>` есть три полезных атрибута, которых нет у `<ul>`.

---

### 📌 Атрибут `start` — начальный номер

По умолчанию нумерация начинается с **1**. Атрибут `start` позволяет начать с **любого числа**.

```HTML

<!-- Нумерация с 1 (по умолчанию) -->
<ol>
  <li>Первый</li>
  <li>Второй</li>
  <li>Третий</li>
</ol>

<!-- Нумерация с 5 -->
<ol start="5">
  <li>Пятый</li>
  <li>Шестой</li>
  <li>Седьмой</li>
</ol>
```

**Результат второго списка:**

5. Пятый
6. Шестой
7. Седьмой

#### Когда полезен `start`

|Ситуация|Пример|
|---|---|
|Продолжение списка после перерыва|Список разбит текстом или картинкой|
|Топ-10 с конца|`start="10"` и `reversed` для обратного отсчёта|
|Нумерация глав|Глава 3 начинается с пункта 3.1|

```HTML

<h2>Шаги 1–3</h2>
<ol>
  <li>Открой редактор</li>
  <li>Создай файл</li>
  <li>Напиши код</li>
</ol>

<p>Теперь сохрани файл и перейди к следующим шагам:</p>

<!-- Продолжаем с 4 -->
<ol start="4">
  <li>Сохрани файл</li>
  <li>Открой в браузере</li>
  <li>Проверь результат</li>
</ol>
```

---

### 📌 Атрибут `reversed` — обратный порядок

**Булев атрибут**. Нумерация идёт **от большего к меньшему**.

```HTML

<h2>Топ-5 языков программирования</h2>
<ol reversed>
  <li>C++</li>
  <li>Java</li>
  <li>C</li>
  <li>JavaScript</li>
  <li>Python</li>
</ol>
```

**Результат:**

5. C++
6. Java
7. C
8. JavaScript
9. Python

#### Комбинация `reversed` + `start`

```HTML

<!-- Обратный отсчёт с 10 -->
<ol reversed start="10">
  <li>Пункт десять</li>
  <li>Пункт девять</li>
  <li>Пункт восемь</li>
</ol>
```

**Результат:**

10. Пункт десять
11. Пункт девять
12. Пункт восемь

---

### 📌 Атрибут `type` — тип нумерации

Атрибут `type` меняет **формат** нумерации.

|Значение `type`|Формат|Пример нумерации|
|---|---|---|
|`1`|Арабские цифры (по умолчанию)|1, 2, 3, 4, 5|
|`a`|Строчные латинские буквы|a, b, c, d, e|
|`A`|Заглавные латинские буквы|A, B, C, D, E|
|`i`|Строчные римские цифры|i, ii, iii, iv, v|
|`I`|Заглавные римские цифры|I, II, III, IV, V|

```HTML

<!-- Арабские цифры (по умолчанию) -->
<ol type="1">
  <li>Первый</li>
  <li>Второй</li>
  <li>Третий</li>
</ol>

<!-- Строчные буквы -->
<ol type="a">
  <li>Пункт a</li>
  <li>Пункт b</li>
  <li>Пункт c</li>
</ol>

<!-- Заглавные римские цифры -->
<ol type="I">
  <li>Раздел I</li>
  <li>Раздел II</li>
  <li>Раздел III</li>
</ol>
```

#### Типичные сценарии

|Формат|Где используется|
|---|---|
|`1` (арабские)|Инструкции, рецепты, обычные списки|
|`a` / `A` (буквы)|Подпункты в документах (пункт a, пункт b)|
|`i` / `I` (римские)|Юридические документы, оглавления, классические тексты|

> [!note] CSS vs атрибут type  
> Тип нумерации можно также задать через CSS-свойство `list-style-type`. В профессиональной разработке **предпочтительнее CSS**, а `type` используется, когда нумерация имеет **семантическое** значение (например, юридические пункты).

---

## 📌 Атрибут `value` у `<li>` — конкретный номер

Можно задать **конкретный номер** отдельному элементу списка. Последующие элементы продолжат нумерацию от этого числа.

```HTML

<ol>
  <li>Первый (1)</li>
  <li>Второй (2)</li>
  <li value="10">Десятый (10)</li>
  <li>Одиннадцатый (11)</li>
  <li>Двенадцатый (12)</li>
</ol>
```

**Результат:**

1. Первый (1)
2. Второй (2)
3. Десятый (10)
4. Одиннадцатый (11)
5. Двенадцатый (12)

Нумерация «перепрыгнула» с 2 на 10, а потом продолжилась: 11, 12.

#### Когда полезен `value`

|Ситуация|Пример|
|---|---|
|Пропущенные пункты|Некоторые пункты удалены, но нумерация должна остаться|
|Нестандартная нумерация|Пункты 1, 2, 5, 10, 20|

## 🪆 Вложенные нумерованные списки

Нумерованные списки, как и маркированные, можно **вкладывать**. Вложенный `<ol>` начинает нумерацию **заново с 1**.

```HTML

<ol>
  <li>Подготовка
    <ol>
      <li>Купить ингредиенты</li>
      <li>Помыть овощи</li>
      <li>Подготовить посуду</li>
    </ol>
  </li>
  <li>Приготовление
    <ol>
      <li>Нарезать овощи</li>
      <li>Обжарить на сковороде</li>
      <li>Добавить специи</li>
    </ol>
  </li>
  <li>Подача
    <ol>
      <li>Выложить на тарелку</li>
      <li>Украсить зеленью</li>
    </ol>
  </li>
</ol>
```

**Результат:**

1. Подготовка  
    　1. Купить ингредиенты  
    　2. Помыть овощи  
    　3. Подготовить посуду
2. Приготовление  
    　1. Нарезать овощи  
    　2. Обжарить на сковороде  
    　3. Добавить специи
3. Подача  
    　1. Выложить на тарелку  
    　2. Украсить зеленью

---

### Вложенные списки с разными типами нумерации

Классическая схема для документов:

```HTML

<ol type="I">
  <li>Общие положения
    <ol type="1">
      <li>Область применения
        <ol type="a">
          <li>Для физических лиц</li>
          <li>Для юридических лиц</li>
        </ol>
      </li>
      <li>Термины и определения</li>
    </ol>
  </li>
  <li>Права и обязанности
    <ol type="1">
      <li>Права пользователя</li>
      <li>Обязанности пользователя</li>
    </ol>
  </li>
</ol>
```

**Результат:**

I. Общие положения  
　1. Область применения  
　　a. Для физических лиц  
　　b. Для юридических лиц  
　2. Термины и определения  
II. Права и обязанности  
　3. Права пользователя  
　4. Обязанности пользователя

---

### Комбинация `<ol>` и `<ul>`

Нумерованные и маркированные списки можно **свободно комбинировать**:

```HTML

<ol>
  <li>Установи VS Code</li>
  <li>Установи расширения:
    <ul>
      <li>Live Server</li>
      <li>Prettier</li>
      <li>Auto Rename Tag</li>
    </ul>
  </li>
  <li>Создай рабочую папку</li>
  <li>Создай первый файл</li>
</ol>
```

**Результат:**

1. Установи VS Code
2. Установи расширения:  
    　• Live Server  
    　• Prettier  
    　• Auto Rename Tag
3. Создай рабочую папку
4. Создай первый файл

Здесь **порядок шагов** важен (нумерованный), но **порядок расширений** — нет (маркированный).

---

## ⚖️ `<ul>` vs `<ol>` — как выбрать?

|Вопрос|Ответ|Тег|
|---|---|---|
|Если поменять пункты местами, смысл изменится?|Да|`<ol>`|
|Если поменять пункты местами, смысл изменится?|Нет|`<ul>`|
|Пункты — это шаги?|Да|`<ol>`|
|Пункты — это рейтинг/ранжирование?|Да|`<ol>`|
|Пункты — просто набор равноправных элементов?|Да|`<ul>`|

### Примеры

|Список|Тег|Почему|
|---|---|---|
|Рецепт (шаги приготовления)|`<ol>`|Порядок важен|
|Список покупок|`<ul>`|Порядок не важен|
|Топ-10 книг|`<ol>`|Рейтинг|
|Навыки в резюме|`<ul>`|Набор без порядка|
|Инструкция по установке|`<ol>`|Шаги|
|Меню навигации|`<ul>`|Набор ссылок|
|Результаты матча (1-е, 2-е, 3-е место)|`<ol>`|Ранжирование|
|Теги статьи (Python, Backend, API)|`<ul>`|Набор без порядка|

## 🧪 Практика

Создай файл и набери вручную:

```HTML

<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Нумерованные списки</title>
  </head>
  <body>

    <h1>Нумерованные списки в HTML</h1>

    <!-- Простой нумерованный список -->
    <h2>Рецепт блинов</h2>
    <ol>
      <li>Смешай 2 стакана муки и 2 яйца</li>
      <li>Добавь 2 стакана молока</li>
      <li>Добавь 2 столовые ложки сахара и щепотку соли</li>
      <li>Перемешай до однородной массы</li>
      <li>Разогрей сковороду с маслом</li>
      <li>Наливай тесто тонким слоем и жарь с двух сторон</li>
    </ol>

    <!-- Атрибут start -->
    <h2>Продолжение инструкции</h2>
    <p>Первые 3 шага ты уже выполнил. Продолжай:</p>
    <ol start="4">
      <li>Открой файл в браузере</li>
      <li>Проверь результат</li>
      <li>Исправь ошибки</li>
      <li>Повтори шаги 4–6 до идеала</li>
    </ol>

    <!-- Атрибут reversed -->
    <h2>Топ-5 языков программирования (2024)</h2>
    <ol reversed>
      <li>C#</li>
      <li>C++</li>
      <li>Java</li>
      <li>JavaScript</li>
      <li>Python</li>
    </ol>

    <!-- Атрибут type -->
    <h2>Разные типы нумерации</h2>

    <h3>Заглавные римские цифры</h3>
    <ol type="I">
      <li>Введение</li>
      <li>Основная часть</li>
      <li>Заключение</li>
    </ol>

    <h3>Строчные буквы</h3>
    <ol type="a">
      <li>Вариант A</li>
      <li>Вариант B</li>
      <li>Вариант C</li>
      <li>Вариант D</li>
    </ol>

    <!-- Атрибут value -->
    <h2>Нестандартная нумерация (value)</h2>
    <ol>
      <li>Введение</li>
      <li>Основы HTML</li>
      <li value="5">Основы CSS (пункты 3–4 пропущены)</li>
      <li>Flexbox</li>
      <li value="10">JavaScript (перепрыгнули на 10)</li>
      <li>DOM</li>
    </ol>

    <!-- Вложенные нумерованные списки -->
    <h2>План курсовой работы</h2>
    <ol type="I">
      <li>Введение
        <ol type="1">
          <li>Актуальность темы</li>
          <li>Цели и задачи</li>
        </ol>
      </li>
      <li>Теоретическая часть
        <ol type="1">
          <li>Обзор литературы</li>
          <li>Основные понятия
            <ol type="a">
              <li>Определение А</li>
              <li>Определение Б</li>
              <li>Определение В</li>
            </ol>
          </li>
        </ol>
      </li>
      <li>Практическая часть
        <ol type="1">
          <li>Методика исследования</li>
          <li>Результаты</li>
        </ol>
      </li>
      <li>Заключение</li>
      <li>Список литературы</li>
    </ol>

    <!-- Комбинация ol и ul -->
    <h2>Утренняя рутина</h2>
    <ol>
      <li>Проснуться в 7:00</li>
      <li>Зарядка (15 минут):
        <ul>
          <li>Отжимания</li>
          <li>Приседания</li>
          <li>Планка</li>
        </ul>
      </li>
      <li>Душ</li>
      <li>Завтрак. Варианты:
        <ul>
          <li>Овсянка с фруктами</li>
          <li>Яичница с тостами</li>
          <li>Йогурт с мюсли</li>
        </ul>
      </li>
      <li>Начать работу</li>
    </ol>

  </body>
</html>
```

### Что проверить в браузере

|Что проверить|Ожидаемый результат|
|---|---|
|Рецепт блинов|Нумерация 1–6|
|`start="4"`|Нумерация начинается с 4|
|`reversed`|Топ-5 с обратным отсчётом (5, 4, 3, 2, 1)|
|`type="I"`|Римские цифры (I, II, III)|
|`type="a"`|Строчные буквы (a, b, c, d)|
|`value`|Нумерация перепрыгивает (1, 2, 5, 6, 10, 11)|
|Вложенный список|Три уровня (I → 1 → a)|
|Комбинация `<ol>` + `<ul>`|Нумерованные шаги с маркированными подпунктами|

## ⚠️ Типичные ошибки

### Ошибка 1: `<ol>` когда порядок не важен

```HTML

<!-- ❌ Порядок продуктов не важен — используй <ul> -->
<ol>
  <li>Молоко</li>
  <li>Хлеб</li>
  <li>Яйца</li>
</ol>

<!-- ✅ Правильно -->
<ul>
  <li>Молоко</li>
  <li>Хлеб</li>
  <li>Яйца</li>
</ul>
```

---

### Ошибка 2: Ручная нумерация вместо `<ol>`

```HTML

<!-- ❌ Неправильно: нумерация вручную -->
<p>1. Открой редактор</p>
<p>2. Создай файл</p>
<p>3. Напиши код</p>

<!-- ❌ Тоже неправильно: нумерация в <ul> -->
<ul>
  <li>1. Открой редактор</li>
  <li>2. Создай файл</li>
  <li>3. Напиши код</li>
</ul>

<!-- ✅ Правильно: используй <ol> -->
<ol>
  <li>Открой редактор</li>
  <li>Создай файл</li>
  <li>Напиши код</li>
</ol>
```

Если ты вручную пишешь номера — почти наверняка нужен `<ol>`.

---

### Ошибка 3: Вложенный список рядом с `<li>`

```HTML

<!-- ❌ Неправильно -->
<ol>
  <li>Пункт 1</li>
  <ol>
    <li>Подпункт</li>
  </ol>
  <li>Пункт 2</li>
</ol>

<!-- ✅ Правильно -->
<ol>
  <li>Пункт 1
    <ol>
      <li>Подпункт</li>
    </ol>
  </li>
  <li>Пункт 2</li>
</ol>
```

---

### Ошибка 4: `type` с неправильным значением

```HTML

<!-- ❌ Неправильно: type="roman" не существует -->
<ol type="roman">
  <li>Пункт</li>
</ol>

<!-- ✅ Правильно: type="I" для римских цифр -->
<ol type="I">
  <li>Пункт</li>
</ol>
```

Допустимые значения `type`: `1`, `a`, `A`, `i`, `I`. Никаких других.

## 📊 Сводка атрибутов `<ol>`

|Атрибут|Тип|Значение по умолчанию|Назначение|Пример|
|---|---|---|---|---|
|`start`|Число|`1`|Начальный номер|`<ol start="5">`|
|`reversed`|Булев|Нет (по возрастанию)|Обратный порядок|`<ol reversed>`|
|`type`|Строка|`1`|Формат нумерации|`<ol type="a">`|

|Атрибут `<li>`|Тип|Назначение|Пример|
|---|---|---|---|
|`value`|Число|Задаёт конкретный номер для этого пункта|`<li value="10">`|

## 📊 Сравнение `<ul>` и `<ol>`

|Характеристика|`<ul>`|`<ol>`|
|---|---|---|
|Название|Unordered list|Ordered list|
|Перевод|Неупорядоченный|Упорядоченный|
|Маркер|● ○ ■ (точки)|1. 2. 3. (номера)|
|Порядок|Не важен|Важен|
|Атрибуты|Нет специфических|`start`, `reversed`, `type`|
|Применение|Навигация, перечисления|Инструкции, рейтинги|
|`<li>`|Внутри `<ul>`|Внутри `<ol>`|
|Вложенность|✅|✅|

## ❓ Частые вопросы

### «Можно ли начать нумерацию с 0?»

Да: `<ol start="0">`. Это валидный HTML.

### «Можно ли начать с отрицательного числа?»

Да: `<ol start="-3">`. Нумерация пойдёт: -3, -2, -1, 0, 1...

### «Как убрать номера у `<ol>` через CSS?»

```CSS

ol {
  list-style: none;
  padding-left: 0;
}
```

Но если ты убираешь нумерацию, возможно, тебе нужен `<ul>`, а не `<ol>`.

### «Как стилизовать номера (цвет, размер)?»

Через CSS-псевдоэлемент `::marker`:

```CSS

ol li::marker {
  color: red;
  font-weight: bold;
  font-size: 1.2em;
}
```

Мы изучим это в модуле CSS.

### «Можно ли использовать `type` у `<li>` внутри `<ol>`?»

Нет, атрибут `type` существует только у `<ol>`. У `<li>` внутри `<ol>` есть только `value`.

### «Если я разорву `<ol>` текстом посередине, нумерация сбросится?»

Да, каждый новый `<ol>` начинает нумерацию заново (с 1 по умолчанию). Используй `start`, чтобы продолжить:

```HTML

<ol>
  <li>Шаг 1</li>
  <li>Шаг 2</li>
</ol>

<p>Примечание: перед шагом 3 сделай перерыв.</p>

<ol start="3">
  <li>Шаг 3</li>
  <li>Шаг 4</li>
</ol>
```

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 
> 1. **`<ol>`** — нумерованный (упорядоченный) список. Порядок **важен**

> 6. Если порядок пунктов **можно изменить** без потери смысла — используй `<ul>`
> 7. Если порядок **важен** (шаги, рейтинг, хронология) — используй `<ol>`
> 8. Не пиши номера вручную — пусть `<ol>` делает это автоматически
