## 📖 Зачем нужны списки?

Списки — один из самых часто используемых элементов на веб-страницах. Они встречаются **повсюду**:

| Где | Пример |
|---|---|
| Навигация сайта | Меню со ссылками |
| Перечисление характеристик | Характеристики товара |
| Инструкции | Шаги выполнения задачи |
| Содержание | Оглавление статьи |
| Подвал сайта | Списки ссылок в footer |
| Карточка товара | Список преимуществ |

> [!tip] Откровение
> Если ты откроешь DevTools на любом сайте, ты обнаружишь, что **навигационное меню** — это почти всегда список `<ul>` со ссылками внутри `<li>`. Даже если визуально оно выглядит как горизонтальная строка кнопок.

## 🔵 Тег `<ul>` — маркированный список

**`<ul>`** (unordered list — неупорядоченный список) создаёт список, в котором **порядок элементов не важен**.

Каждый пункт списка оборачивается в тег **`<li>`** (list item — элемент списка).

### Базовый синтаксис

```html
<ul>
  <li>Яблоки</li>
  <li>Бананы</li>
  <li>Апельсины</li>
</ul>
```

**Результат в браузере:**

• Яблоки  
• Бананы  
• Апельсины

---

### Как это выглядит по умолчанию

|Свойство|Значение по умолчанию|
|---|---|
|Маркер|Чёрный кружок (disc)|
|Отступ слева|~40px (padding-left)|
|Между пунктами|Небольшой вертикальный отступ|
|Элемент `<ul>`|Блочный (занимает всю ширину)|
|Элемент `<li>`|Блочный (каждый пункт с новой строки)|

---

## 📏 Правила использования

### Правило 1: `<li>` — только внутри `<ul>`

Тег `<li>` **не может** существовать сам по себе. Он всегда должен быть внутри `<ul>` (или `<ol>`, `<menu>`).

```HTML

<!-- ❌ Неправильно: <li> без родительского списка -->
<li>Пункт 1</li>
<li>Пункт 2</li>

<!-- ✅ Правильно: <li> внутри <ul> -->
<ul>
  <li>Пункт 1</li>
  <li>Пункт 2</li>
</ul>
```

---

### Правило 2: Внутри `<ul>` — только `<li>`

Прямые дочерние элементы `<ul>` могут быть **только** `<li>`. Нельзя помещать другие теги непосредственно внутрь `<ul>`.

```HTML

<!-- ❌ Неправильно: <p> напрямую внутри <ul> -->
<ul>
  <p>Пункт 1</p>
  <p>Пункт 2</p>
</ul>

<!-- ❌ Неправильно: текст напрямую внутри <ul> -->
<ul>
  Просто текст
  <li>Пункт</li>
</ul>

<!-- ✅ Правильно: только <li> внутри <ul> -->
<ul>
  <li>Пункт 1</li>
  <li>Пункт 2</li>
</ul>
```

---

### Правило 3: Внутри `<li>` — почти всё что угодно

А вот внутри `<li>` можно размещать **любой контент**:

|Можно внутри `<li>`|Пример|
|---|---|
|Текст|`<li>Простой текст</li>`|
|Ссылка|`<li><a href="#">Ссылка</a></li>`|
|Форматирование|`<li><strong>Жирный</strong> текст</li>`|
|Изображение|`<li><img src="icon.png" alt="Иконка"></li>`|
|Абзац|`<li><p>Абзац текста</p></li>`|
|Другой список|`<li><ul><li>Вложенный</li></ul></li>`|
|Несколько элементов|`<li><h3>Заголовок</h3><p>Описание</p></li>`|

```HTML

<!-- Список ссылок -->
<ul>
  <li><a href="index.html">Главная</a></li>
  <li><a href="about.html">О нас</a></li>
  <li><a href="contacts.html">Контакты</a></li>
</ul>

<!-- Список с форматированием -->
<ul>
  <li><strong>HTML</strong> — язык разметки</li>
  <li><strong>CSS</strong> — язык стилей</li>
  <li><strong>JavaScript</strong> — язык программирования</li>
</ul>

<!-- Список с изображениями -->
<ul>
  <li><img src="icon-html.png" alt=""> HTML</li>
  <li><img src="icon-css.png" alt=""> CSS</li>
  <li><img src="icon-js.png" alt=""> JavaScript</li>
</ul>
```

## 🪆 Вложенные (многоуровневые) списки

Списки можно **вкладывать** друг в друга — создавать подпункты внутри пунктов.

### Важное правило

Вложенный `<ul>` размещается **внутри `<li>`**, а не рядом с ним.

```HTML

<!-- ✅ Правильно: вложенный <ul> ВНУТРИ <li> -->
<ul>
  <li>Фрукты
    <ul>
      <li>Яблоки</li>
      <li>Бананы</li>
    </ul>
  </li>
  <li>Овощи
    <ul>
      <li>Морковь</li>
      <li>Картофель</li>
    </ul>
  </li>
</ul>

<!-- ❌ Неправильно: вложенный <ul> РЯДОМ с <li> -->
<ul>
  <li>Фрукты</li>
  <ul>
    <li>Яблоки</li>
    <li>Бананы</li>
  </ul>
</ul>
```

### Маркеры вложенных уровней

Браузер автоматически меняет маркер для каждого уровня вложенности:

|Уровень|Маркер по умолчанию|Название|
|---|---|---|
|1 (основной)|●|disc (закрашенный круг)|
|2 (вложенный)|○|circle (пустой круг)|
|3 (вложенный во вложенный)|■|square (квадрат)|
|4+|■|square (далее не меняется)|

### Пример трёхуровневого списка

```HTML

<ul>
  <li>Веб-разработка
    <ul>
      <li>Frontend
        <ul>
          <li>HTML</li>
          <li>CSS</li>
          <li>JavaScript</li>
        </ul>
      </li>
      <li>Backend
        <ul>
          <li>Python</li>
          <li>Node.js</li>
          <li>PHP</li>
        </ul>
      </li>
    </ul>
  </li>
  <li>Мобильная разработка
    <ul>
      <li>Android (Kotlin)</li>
      <li>iOS (Swift)</li>
      <li>Кроссплатформенные (Flutter, React Native)</li>
    </ul>
  </li>
</ul>
```

**Результат:**

● Веб-разработка  
　○ Frontend  
　　■ HTML  
　　■ CSS  
　　■ JavaScript  
　○ Backend  
　　■ Python  
　　■ Node.js  
　　■ PHP  
● Мобильная разработка  
　○ Android (Kotlin)  
　○ iOS (Swift)  
　○ Кроссплатформенные (Flutter, React Native)

> [!warning] Не злоупотребляй вложенностью  
> Три уровня — это обычно максимум. Если у тебя 4–5 уровней вложенности, стоит пересмотреть структуру: возможно, лучше разбить на несколько отдельных списков с заголовками.

## 🎯 Когда использовать маркированный список

### Подходит `<ul>`

|Ситуация|Почему `<ul>`|
|---|---|
|Список покупок|Порядок не важен|
|Навигационное меню|Набор равноправных ссылок|
|Список навыков (резюме)|Порядок не принципиален|
|Характеристики товара|Перечисление без порядка|
|Список тегов / категорий|Набор равноправных элементов|
|Ссылки в подвале сайта|Группа ссылок|

### Не подходит `<ul>`

|Ситуация|Что использовать|
|---|---|
|Пошаговая инструкция|`<ol>` (нумерованный список)|
|Рейтинг (от лучшего к худшему)|`<ol>` (порядок важен)|
|Термин и определение|`<dl>` (список определений)|
|Табличные данные|`<table>`|

> [!tip] Простое правило  
> Если поменять пункты местами и **смысл не изменится** — используй `<ul>`.  
> Если порядок **важен** — используй `<ol>` (следующий урок).

## 🔧 Маркированный список для навигации

Один из самых распространённых паттернов в вебе — **навигация как список**:

```HTML

<nav>
  <ul>
    <li><a href="index.html">Главная</a></li>
    <li><a href="catalog.html">Каталог</a></li>
    <li><a href="about.html">О нас</a></li>
    <li><a href="contacts.html">Контакты</a></li>
  </ul>
</nav>
```

### Почему навигация — это список?

|Причина|Пояснение|
|---|---|
|**Семантика**|Меню — это набор (список) ссылок|
|**Доступность**|Экранная читалка скажет: «Навигация, список из 4 пунктов»|
|**Стилизация**|CSS легко превратит вертикальный список в горизонтальное меню|

> [!note] Как превратить в горизонтальное меню?  
> Через CSS (модуль 9, Flexbox). Маркеры убираются свойством `list-style: none`, пункты выстраиваются в строку через `display: flex`. Мы это сделаем в уроке [[112 — 🏗️ Практика — Навигационная панель]].

## 🎨 CSS-стилизация маркеров (обзор)

Хотя CSS — тема будущих модулей, полезно знать, что маркеры списка можно полностью настроить.

### Убрать маркеры

```CSS

ul {
  list-style: none;
  padding: 0;
}
```

### Сменить тип маркера

```CSS

ul {
  list-style-type: square;    /* квадрат */
  list-style-type: circle;    /* пустой круг */
  list-style-type: disc;      /* закрашенный круг (по умолчанию) */
  list-style-type: "→ ";      /* кастомный символ */
}
```

### Использовать картинку как маркер

```CSS

ul {
  list-style-image: url('icon-check.png');
}
```

### Использовать эмодзи как маркер

```CSS

ul {
  list-style-type: "✅ ";
}
```

> [!note] Мы подробно изучим стилизацию списков в модуле CSS. Пока сосредоточься на правильном HTML.

## ⚠️ Типичные ошибки

### Ошибка 1: `<li>` без `<ul>`

```HTML

<!-- ❌ Неправильно -->
<li>Пункт 1</li>
<li>Пункт 2</li>

<!-- ✅ Правильно -->
<ul>
  <li>Пункт 1</li>
  <li>Пункт 2</li>
</ul>
```

---

### Ошибка 2: Посторонние элементы внутри `<ul>`

```HTML

<!-- ❌ Неправильно: <p> напрямую в <ul> -->
<ul>
  <li>Пункт 1</li>
  <p>Это абзац внутри списка</p>
  <li>Пункт 2</li>
</ul>

<!-- ✅ Правильно: <p> внутри <li> -->
<ul>
  <li>Пункт 1</li>
  <li><p>Это абзац внутри пункта списка</p></li>
  <li>Пункт 2</li>
</ul>
```

---

### Ошибка 3: Вложенный список рядом с `<li>`, а не внутри

```HTML

<!-- ❌ Неправильно: вложенный <ul> НЕ внутри <li> -->
<ul>
  <li>Фрукты</li>
  <ul>
    <li>Яблоки</li>
  </ul>
  <li>Овощи</li>
</ul>

<!-- ✅ Правильно: вложенный <ul> ВНУТРИ <li> -->
<ul>
  <li>Фрукты
    <ul>
      <li>Яблоки</li>
    </ul>
  </li>
  <li>Овощи</li>
</ul>
```

---

### Ошибка 4: Список для одного пункта

```HTML

<!-- ❌ Бессмысленно: список из одного пункта -->
<ul>
  <li>Единственный пункт</li>
</ul>

<!-- ✅ Если пункт один — используй обычный элемент -->
<p>Единственный пункт</p>
```

Список из одного элемента **не имеет смысла** — это не список.

---

### Ошибка 5: Использование `<ul>` для оформления отступов

```HTML

<!-- ❌ Неправильно: список ради визуального отступа -->
<ul>
  <li>Это просто текст, который я хочу сдвинуть вправо</li>
</ul>

<!-- ✅ Правильно: отступ через CSS -->
<p style="margin-left: 40px;">Этот текст сдвинут вправо через CSS.</p>
```

## ❓ Частые вопросы

### «Можно ли убрать маркеры?»

Да, через CSS:

```CSS

ul {
  list-style: none;
  padding-left: 0;
}
```

Это часто делают для **навигационных меню** — маркеры там не нужны.

### «Можно ли сделать горизонтальный список?»

Да, через CSS Flexbox:

```CSS

ul {
  display: flex;
  list-style: none;
  gap: 20px;
}
```

Мы подробно изучим это в модуле Flexbox.

### «Сколько пунктов может быть в списке?»

Ограничений нет. Но если пунктов больше 15–20, стоит подумать о разбиении на несколько списков с подзаголовками.

### «Обязательно ли закрывать `<li>`?»

Формально в HTML5 закрывающий тег `</li>` **необязателен** — браузер может определить конец элемента по началу следующего `<li>` или `</ul>`. Но это **плохая практика**. Всегда закрывай `</li>`.

```HTML

<!-- Работает, но плохая практика -->
<ul>
  <li>Пункт 1
  <li>Пункт 2
  <li>Пункт 3
</ul>

<!-- Правильная практика -->
<ul>
  <li>Пункт 1</li>
  <li>Пункт 2</li>
  <li>Пункт 3</li>
</ul>
```

### «Чем `<ul>` отличается от `<ol>`?»

`<ul>` — **неупорядоченный** (маркированный) список. Порядок не важен.  
`<ol>` — **упорядоченный** (нумерованный) список. Порядок важен.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 
> 1. **`<ul>`** — маркированный (неупорядоченный) список. Порядок пунктов **не важен**
> 2. **`<li>`** — элемент списка, только внутри `<ul>` (или `<ol>`)
> 3. Внутри `<ul>` могут быть **только `<li>`**
> 4. Внутри `<li>` может быть **что угодно**: текст, ссылки, картинки, другие списки
> 5. Вложенные списки размещаются **внутри `<li>`**, а не рядом
> 6. **Навигация** — почти всегда `<nav>` + `<ul>` + `<li>` + `<a>`
> 7. Не более **3 уровней** вложенности
> 8. Если порядок важен — используй `<ol>` (следующий урок)
