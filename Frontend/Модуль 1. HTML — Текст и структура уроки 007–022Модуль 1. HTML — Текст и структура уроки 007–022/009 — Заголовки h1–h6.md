## 📖 Зачем нужны заголовки?

Открой любую книгу, статью или учебник. Ты увидишь:
- **Название книги** — самый крупный текст
- **Названия глав** — поменьше
- **Названия разделов** — ещё меньше
- **Подразделы** — ещё мельче

Эта **иерархия** помогает читателю понять структуру текста, быстро найти нужный раздел, оценить содержание.

В HTML заголовки выполняют **ту же роль**. Они создают иерархическую структуру страницы.

---

## 🔢 Шесть уровней заголовков

HTML предоставляет **6 уровней** заголовков — от `<h1>` (самый важный) до `<h6>` (наименее важный).

```html
<h1>Заголовок 1 уровня</h1>
<h2>Заголовок 2 уровня</h2>
<h3>Заголовок 3 уровня</h3>
<h4>Заголовок 4 уровня</h4>
<h5>Заголовок 5 уровня</h5>
<h6>Заголовок 6 уровня</h6>
```
### Как выглядят заголовки по умолчанию

Браузер отображает заголовки с **разными размерами и жирностью** по умолчанию (без CSS):

|Тег|Размер по умолчанию|Визуальное описание|
|---|---|---|
|`<h1>`|32px (2em)|Самый крупный и жирный|
|`<h2>`|24px (1.5em)|Крупный, жирный|
|`<h3>`|~18.7px (1.17em)|Средний, жирный|
|`<h4>`|16px (1em)|Как обычный текст, но жирный|
|`<h5>`|~13.3px (0.83em)|Меньше обычного текста, жирный|
|`<h6>`|~10.7px (0.67em)|Самый мелкий, жирный|

> [!note] Размеры — не главное  
> Размеры заголовков по умолчанию можно (и нужно) менять через CSS. Визуальный размер — это **оформление**. Гораздо важнее **семантика** — какой уровень важности несёт заголовок.

## 🏗️ Иерархия заголовков

Заголовки образуют **структуру документа**, похожую на оглавление книги.

### Аналогия с книгой

|Уровень|Книга|HTML|
|---|---|---|
|1|Название книги|`<h1>`|
|2|Глава|`<h2>`|
|3|Раздел внутри главы|`<h3>`|
|4|Подраздел|`<h4>`|
|5|Пункт подраздела|`<h5>`|
|6|Подпункт|`<h6>`|

### Пример: статья на сайте

```HTML
<h1>Путешествие по Италии</h1>

  <h2>Рим</h2>
    <h3>Колизей</h3>
    <h3>Ватикан</h3>
    <h3>Фонтан Треви</h3>

  <h2>Флоренция</h2>
    <h3>Галерея Уффици</h3>
    <h3>Собор Санта-Мария-дель-Фьоре</h3>
      <h4>История собора</h4>
      <h4>Купол Брунеллески</h4>

  <h2>Венеция</h2>
    <h3>Площадь Сан-Марко</h3>
    <h3>Гондолы</h3>
```

### Оглавление этой статьи выглядело бы так

|Уровень|Заголовок|
|---|---|
|h1|Путешествие по Италии|
|∟ h2|Рим|
|∟ h3|Колизей|
|∟ h3|Ватикан|
|∟ h3|Фонтан Треви|
|∟ h2|Флоренция|
|∟ h3|Галерея Уффици|
|∟ h3|Собор Санта-Мария-дель-Фьоре|
|∟ h4|История собора|
|∟ h4|Купол Брунеллески|
|∟ h2|Венеция|
|∟ h3|Площадь Сан-Марко|
|∟ h3|Гондолы|

> [!tip] Мысленный тест  
> Если ты можешь построить **оглавление** из своих заголовков — значит, структура правильная. Если оглавление выглядит хаотично — нужно пересмотреть иерархию.

## 📏 Правила использования заголовков

### Правило 1: Один `<h1>` на страницу

На каждой странице должен быть **только один** `<h1>`. Это **главный заголовок**, который описывает всю страницу.

|Правильно ✅|Неправильно ❌|
|---|---|
|Один `<h1>` на странице|Два или более `<h1>`|

```HTML
<!-- Правильно ✅ -->
<h1>Рецепты итальянской кухни</h1>
<h2>Паста карбонара</h2>
<h2>Пицца маргарита</h2>

<!-- Неправильно ❌ -->
<h1>Рецепты итальянской кухни</h1>
<h1>Паста карбонара</h1>
<h1>Пицца маргарита</h1>
```

> [!note] Техническое примечание  
> Браузер **не сломается** от двух `<h1>`. Но с точки зрения семантики и SEO — это неправильно. Google ожидает один `<h1>`, который чётко описывает содержимое страницы.

---

### Правило 2: Не пропускай уровни

Заголовки должны идти **последовательно**. После `<h1>` идёт `<h2>`, после `<h2>` — `<h3>`. Нельзя перепрыгивать через уровень.

|Правильно ✅|Неправильно ❌|
|---|---|
|h1 → h2 → h3|h1 → h3 (пропущен h2)|
|h2 → h3 → h3|h2 → h4 (пропущен h3)|
|h3 → h2 (возврат на уровень выше — ОК)|h1 → h4 (пропущены h2 и h3)|

```HTML
<!-- Правильно ✅ -->
<h1>Мой блог</h1>
  <h2>Статья первая</h2>
    <h3>Введение</h3>
    <h3>Основная часть</h3>
  <h2>Статья вторая</h2>

<!-- Неправильно ❌ -->
<h1>Мой блог</h1>
  <h3>Статья первая</h3>   <!-- Пропущен h2! -->
    <h5>Введение</h5>      <!-- Пропущены h4! -->
```

---

### Правило 3: Заголовок — для структуры, не для размера

Новички часто выбирают уровень заголовка **по размеру шрифта**: «мне нужен текст поменьше, возьму `<h4>`». Это **неправильный подход**.

|Цель|Правильное решение|Неправильное решение|
|---|---|---|
|Текст поменьше|Используй правильный уровень заголовка + измени размер через CSS|Используй `<h4>` вместо `<h2>`, потому что он меньше|
|Крупный текст|Используй `<p>` + измени размер через CSS|Используй `<h1>` для обычного абзаца, потому что он крупный|

```HTML
<!-- Неправильно ❌: h4 выбран ради размера -->
<h1>Мой сайт</h1>
<h4>Добро пожаловать на мой сайт</h4>

<!-- Правильно ✅: h2 по иерархии, размер настроим в CSS -->
<h1>Мой сайт</h1>
<h2>Добро пожаловать на мой сайт</h2>
```

---

### Правило 4: Заголовок — не замена для жирного текста

Если нужно выделить текст жирным шрифтом — используй `<strong>`, а не заголовок.

|Задача|Правильно ✅|Неправильно ❌|
|---|---|---|
|Выделить важное слово|`<p><strong>Важно!</strong> Прочитай это.</p>`|`<h3>Важно!</h3><p>Прочитай это.</p>`|
|Подпись к картинке|`<figcaption>Фото кота</figcaption>`|`<h5>Фото кота</h5>`|

---

### Правило 5: Заголовки — блочные элементы

Заголовки занимают **всю ширину** страницы и начинаются с **новой строки**. Они не могут стоять в одной строке с другим текстом (без CSS).

```HTML
<!-- Каждый заголовок начнётся с новой строки -->
<h2>Первый</h2>
<h2>Второй</h2>
<p>Текст абзаца.</p>
```

Заголовок **не может** быть внутри `<p>`:

```HTML
<!-- Неправильно ❌ -->
<p>Текст <h2>Заголовок</h2> продолжение</p>

<!-- Правильно ✅ -->
<p>Текст перед заголовком.</p>
<h2>Заголовок</h2>
<p>Текст после заголовка.</p>
```

## 🔍 Зачем заголовки нужны — три причины

### 1. SEO (поисковая оптимизация)

Поисковые системы (Google, Яндекс) используют заголовки, чтобы **понять**, о чём страница.

|Что анализирует Google|Как использует|
|---|---|
|`<h1>`|Главная тема страницы|
|`<h2>`|Основные разделы|
|`<h3>` – `<h6>`|Подразделы и детали|

Страница с правильной иерархией заголовков **ранжируется выше** в поисковой выдаче, чем страница без заголовков или с хаотичной структурой.

---

### 2. Доступность (accessibility)

**Экранные читалки** (программы для слабовидящих) позволяют пользователю «перемещаться по заголовкам» — как по оглавлению. Пользователь может быстро перейти к нужному разделу.

|Действие пользователя|Что делает экранная читалка|
|---|---|
|Нажимает клавишу «H»|Перемещается к следующему заголовку|
|Нажимает «1»|Перемещается к следующему `<h1>`|
|Нажимает «2»|Перемещается к следующему `<h2>`|
|Запрашивает оглавление|Показывает список всех заголовков на странице|

> [!important] Около 15% населения имеют нарушения зрения  
> Правильные заголовки — это не опция, а забота о людях. Доступность — важная часть профессиональной веб-разработки.

---

### 3. Читаемость кода

Заголовки делают HTML-код **понятнее** для разработчика. Ты сразу видишь структуру страницы, когда читаешь код.

```HTML
<!-- Сразу понятна структура -->
<h1>Интернет-магазин</h1>
  <h2>Каталог</h2>
    <h3>Электроника</h3>
    <h3>Одежда</h3>
  <h2>О нас</h2>
  <h2>Контакты</h2>
```

## ⚠️ Типичные ошибки

### Ошибка 1: Несколько `<h1>`

```HTML
<!-- ❌ Неправильно -->
<h1>Новости</h1>
<h1>Главная новость дня</h1>
<h1>Спорт</h1>
```

```HTML
<!-- ✅ Правильно -->
<h1>Новости</h1>
<h2>Главная новость дня</h2>
<h2>Спорт</h2>
```

---
### Ошибка 2: Пропуск уровней

```HTML
<!-- ❌ Неправильно: после h1 сразу h3 -->
<h1>Мой блог</h1>
<h3>Первая статья</h3>
<h3>Вторая статья</h3>
```

```HTML
<!-- ✅ Правильно -->
<h1>Мой блог</h1>
<h2>Первая статья</h2>
<h2>Вторая статья</h2>
```

---

### Ошибка 3: Заголовок ради размера

```HTML
<!-- ❌ Неправильно: h6 использован для мелкого текста -->
<h6>Автор: Иван Петров</h6>
```

```HTML
<!-- ✅ Правильно: обычный абзац с классом -->
<p class="author">Автор: Иван Петров</p>
```

---

### Ошибка 4: Пустой заголовок

```HTML
<!-- ❌ Бессмысленно -->
<h2></h2>

<!-- ❌ Заголовок только из пробелов -->
<h2>   </h2>
```

Пустые заголовки не несут информации и сбивают экранные читалки.

---

### Ошибка 5: Заголовок внутри `<p>`

```HTML
<!-- ❌ Неправильно -->
<p>Текст <h2>Заголовок</h2> ещё текст</p>
```

Браузер «сломает» эту конструкцию, потому что `<h2>` — блочный элемент, а `<p>` не может содержать блочные элементы. Браузер принудительно закроет `<p>` перед `<h2>`.

## 🔍 Проверка заголовков в DevTools

Ты можешь проверить структуру заголовков любого сайта.

### Способ 1: Вкладка Elements

|Шаг|Действие|
|---|---|
|1|Открой любой сайт|
|2|Нажми `F12` → **Elements**|
|3|Нажми `Ctrl + F` (поиск внутри Elements)|
|4|Набери `<h1` — найдёшь все `<h1>` на странице|
|5|Набери `<h2` — найдёшь все `<h2>`|

### Способ 2: Через консоль

|Шаг|Действие|
|---|---|
|1|Открой DevTools → **Console**|
|2|Введи команду ниже и нажми Enter|

```JavaScript
document.querySelectorAll('h1, h2, h3, h4, h5, h6').forEach(h => console.log(h.tagName, h.textContent));
```

Это выведет **все заголовки** на странице с их уровнями. Попробуй на разных сайтах — увидишь, как профессионалы строят структуру.

## 💡 Когда использовать `<h5>` и `<h6>`?

На практике `<h5>` и `<h6>` используются **очень редко**. Большинство страниц обходятся тремя уровнями: `<h1>`, `<h2>`, `<h3>`.

|Уровень|Как часто используется|Где нужен|
|---|---|---|
|`<h1>`|На каждой странице|Главный заголовок|
|`<h2>`|Почти на каждой странице|Основные разделы|
|`<h3>`|Часто|Подразделы|
|`<h4>`|Иногда|Глубокая вложенность (длинные статьи)|
|`<h5>`|Редко|Очень детализированный контент|
|`<h6>`|Почти никогда|Экстремально глубокая вложенность|

> [!tip] Совет  
> Если тебе «не хватает» уровней заголовков — скорее всего, стоит разбить страницу на **несколько отдельных страниц**. Документ с заголовками до h6 обычно слишком длинный и сложный.

## ❓ Частые вопросы

### «А если мне нужен заголовок 7 уровня?»

Его не существует. `<h7>` — невалидный тег. Если нужна более глубокая вложенность — пересмотри структуру страницы.

### «Можно ли стилизовать `<h2>` так, чтобы он выглядел как `<h1>`?»

Да, через CSS можно задать **любой** размер и стиль для любого заголовка. Визуальный размер и семантический уровень — разные вещи.

```CSS
/* h2 будет выглядеть как h1, но семантически останется h2 */
h2 {
    font-size: 32px;
}
```

### «Я видел сайты с несколькими `<h1>`. Это ошибка?»

Формально HTML5 допускает несколько `<h1>`, если они находятся в разных секциях (`<section>`, `<article>`). Но на практике большинство SEO-специалистов и руководств рекомендуют **один `<h1>` на страницу**.

### «Нужно ли оборачивать заголовок и его контент в контейнер?»

Пока необязательно. Но в модуле 4 мы изучим семантические теги (`<section>`, `<article>`), которые группируют заголовок с его содержимым. Это улучшает структуру.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 
> 1. В HTML **6 уровней** заголовков: от `<h1>` (самый важный) до `<h6>` (наименее важный)
> 2. На странице должен быть **один `<h1>`** — главный заголовок
> 3. **Не пропускай уровни**: после h1 → h2, после h2 → h3, не h1 → h3
> 4. Выбирай уровень заголовка по **значимости**, а не по размеру шрифта
> 5. Заголовки важны для **SEO**, **доступности** и **читаемости** кода
> 6. Если из заголовков можно составить **логичное оглавление** — структура правильная
> 7. `<h5>` и `<h6>` используются **очень редко** — обычно хватает h1–h3

## 🏁 Итог

Заголовки — один из самых важных инструментов HTML. Они создают **скелет** страницы, помогают поисковикам понять контент и делают сайт доступным для всех пользователей. Запомни: **один h1, последовательные уровни, выбор по смыслу, а не по размеру**.

В следующем уроке мы изучим **абзацы, переносы строк и горизонтальные линии** — теги `<p>`, `<br>` и `<hr>`.
