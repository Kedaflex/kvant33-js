## 📖 Введение: Сила букв

Вы когда-нибудь задумывались, что более 90% информации в интернете — это текст? Мы читаем новости, статьи, документацию, посты в соцсетях. Веб-дизайнер Оливер Райхенштейн однажды сказал: «Веб-дизайн на 95% состоит из типографики». И он был прав.

Шрифт — это не просто способ отображения символов. Это голос вашего сайта. Он может быть кричащим и дерзким, строгим и академичным или уютным и неформальным. Неправильно подобранный шрифт может сделать даже самый гениальный текст нечитаемым, а правильный — превратить обычную страницу в произведение искусства.

В этом модуле мы станем «дирижерами букв». И начнем мы с самого фундаментального свойства — `font-family`, которое определяет, каким именно шрифтом будет написан ваш контент.

---

## 🛠️ Свойство font-family

Свойство `font-family` (семейство шрифтов) указывает приоритетный список одного или нескольких названий семейств шрифтов для выбранного элемента.

В отличие от многих других свойств CSS, где мы указываем одно конкретное значение, в `font-family` мы почти всегда указываем список. Почему? Потому что мы не можем на 100% гарантировать, что конкретный шрифт установлен на компьютере или смартфоне пользователя.

### Синтаксис:
```css
p {
  font-family: "Helvetica Neue", Helvetica, Arial, sans-serif;
}
```

Браузер читает этот список слева направо:
1. «Так, автор хочет **Helvetica Neue**. Она есть в системе? Нет.»
2. «Окей, смотрю следующий — **Helvetica**. Есть? Нет.»
3. «Ладно, ищем **Arial**. О, есть! Использую его.»
4. Если бы и Arial не было, браузер применил бы стандартный шрифт из семейства **sans-serif** (без засечек), который настроен в операционной системе по умолчанию.

## 🏛️ Пять базовых семейств (Generic Families)

В мире CSS существует пять стандартных категорий, к которым относится любой шрифт. Указание такой категории в конце вашего списка — это «страховочный трос», который гарантирует, что браузер выберет хотя бы похожий по стилю шрифт.

### 1. Serif (С засечками)
Шрифты с небольшими штрихами на концах основных линий букв. Они выглядят классически, формально и традиционно. Считается, что засечки помогают глазу «цепляться» за строку при чтении длинных печатных текстов.
- **Примеры**: Times New Roman, Georgia, Garamond.
- **В CSS**: `font-family: Georgia, serif;`

### 2. Sans-serif (Без засечек)
Шрифты с прямыми концами линий. Выглядят современно, чисто и минималистично. Они лучше читаются на экранах с низким разрешением, так как засечки могут превращаться в визуальный шум.
- **Примеры**: Arial, Helvetica, Verdana, Roboto.
- **В CSS**: `font-family: Arial, sans-serif;`

### 3. Monospace (Моноширинные)
Шрифты, в которых каждая буква имеет одинаковую ширину (как на печатной машинке). Незаменимы для отображения программного кода, таблиц и ASCII-арта.
- **Примеры**: Courier New, Consolas, Monaco.
- **В CSS**: `font-family: "Courier New", monospace;`

### 4. Cursive (Курсивные/Рукописные)
Шрифты, имитирующие почерк или каллиграфию. Обычно используются для декоративных заголовков или подписей. Читать длинный текст таким шрифтом — мучение.
- **Примеры**: Comic Sans MS, Brush Script.
- **В CSS**: `font-family: "Comic Sans MS", cursive;`

### 5. Fantasy (Декоративные)
Художественные шрифты для спецэффектов. Они очень специфичны и редко используются в серьезном интерфейсном дизайне.
- **Примеры**: Impact, Papyrus.
- **В CSS**: `font-family: Impact, fantasy;`

## 🛡️ Безопасные веб-шрифты (Web Safe Fonts)

До появления технологии `@font-face` (которую мы изучим позже) разработчики были ограничены коротким списком шрифтов, которые с огромной вероятностью установлены у всех пользователей (Windows, macOS, Linux).

Их называют **Web Safe Fonts**. Вот основные из них:
- **Arial / Helvetica**: Стандарт де-факто для интернета.
- **Times New Roman / Times**: Классика для текстов.
- **Georgia**: Отличный шрифт с засечками, специально разработанный для экранов.
- **Verdana**: Широкий и очень читабельный шрифт без засечек.
- **Courier New**: Стандарт для кода.

> [!warning] Важно
> Даже если вы используете супер-современный шрифт, загружаемый извне, ваш стек всегда должен заканчиваться «безопасным» шрифтом и базовым семейством. Это профессиональный стандарт.

## 📚 Правила создания шрифтового стека

Хороший шрифтовой стек — это не просто случайный набор слов. Это продуманная стратегия.

### 1. Кавычки имеют значение
Если название шрифта состоит из нескольких слов, разделенных пробелами (например, *Times New Roman*), его **обязательно** нужно брать в кавычки. Если название состоит из одного слова (*Arial*), кавычки не обязательны, но и не запрещены.

```css
/* Правильно */
font-family: "Open Sans", Arial, sans-serif;

/* Ошибка (может не сработать) */
font-family: Open Sans, Arial, sans-serif;
```

### 2. Порядок от частного к общему
Сначала указывайте самый редкий и красивый шрифт, который вы хотите видеть. Затем — более распространенные аналоги. В самом конце — ключевое слово семейства.

```css
/* Пример идеального стека для заголовков */
h1 {
  font-family: "My Awesome Font", "Helvetica Neue", Helvetica, Arial, sans-serif;
}
```

### 3. Учитывайте платформы
Разные операционные системы имеют свои предустановленные шрифты.
- В Windows есть **Segoe UI**.
- В macOS есть **San Francisco**.
- В Android есть **Roboto**.

Современные сайты часто используют системные шрифты для максимальной скорости загрузки:
```css
body {
  font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Helvetica, Arial, sans-serif;
}
```

## 🔍 Как браузер ищет шрифт?

Важно понимать, что браузер ищет шрифт **целиком**, а не по отдельным буквам. Если в вашем любимом шрифте нет кириллических символов (русских букв), браузер не будет использовать его для русского текста.

Он увидит: «Ага, в шрифте "SuperFont" нет буквы 'Ж'. Пойду-ка я поищу ее в следующем шрифте в списке». В итоге вы можете получить «эффект заложника», когда латиница отображается одним шрифтом, а кириллица — другим, стандартным.

> [!tip] Совет
> Всегда проверяйте, поддерживает ли выбранный вами шрифт нужные языки (глифы).

## ❓ Часто задаваемые вопросы

### «Почему нельзя просто указать один шрифт и всё?»
Потому что у пользователя может быть отключен интернет (если шрифт внешний), или он может использовать специфическую ОС (например, дистрибутив Linux), где вашего шрифта просто нет. Без списка ваша страница превратится в непредсказуемое месиво.

### «Влияет ли font-family на скорость загрузки?»
Если вы используете системные шрифты (те, что уже есть на компьютере), то скорость загрузки мгновенная. Если вы подключаете внешние файлы (Google Fonts), то чем больше шрифтов в списке, тем медленнее может быть первая отрисовка страницы.

### «Можно ли использовать разные шрифты для разных языков?»
Напрямую через `font-family` — нет. Браузер сам переключится на следующий шрифт в списке, если в первом нет нужных символов. Но лучше выбирать универсальные шрифты.

### «Что такое системный шрифт?»
Это шрифт, который используется в интерфейсе самой операционной системы (кнопки в окнах, меню Пуск и т.д.). Использование системных шрифтов делает сайт «родным» для устройства пользователя.

## 🧪 Практика: Экспериментируем со стеками

Давайте попробуем создать несколько разных «настроений» для одного и того же текста.

```html
<!-- HTML структура -->
<section class="demo">
  <h2 class="modern">Современный заголовок</h2>
  <h2 class="classic">Классический заголовок</h2>
  <h2 class="code">Заголовок-код</h2>
</section>
```

```css
/* CSS стили */
.modern {
  /* Чистый, современный вид без засечек */
  font-family: "Segoe UI", Roboto, Helvetica, Arial, sans-serif;
}

.classic {
  /* Строгий, книжный вид с засечками */
  font-family: "Times New Roman", Georgia, serif;
}

.code {
  /* Вид как в редакторе кода */
  font-family: "Consolas", "Monaco", "Courier New", monospace;
}
```

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **font-family** — главное свойство для выбора шрифта.
> 2. Всегда указывайте **список (стек)**, а не одно название.
> 3. Заканчивайте любой список **базовым семейством** (serif, sans-serif и т.д.).
> 4. Названия с пробелами берите в **кавычки**.
> 5. Соблюдайте порядок: от самого желанного шрифта к самому обычному.
> 6. Учитывайте системные шрифты разных платформ для лучшего UX.
