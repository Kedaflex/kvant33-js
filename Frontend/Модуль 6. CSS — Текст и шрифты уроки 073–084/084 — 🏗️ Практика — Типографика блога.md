# 084 — 🏗️ Практика — Типографика блога

## 📖 Введение: Создаем идеальный лонгрид

Вы когда-нибудь замечали, что на одних сайтах читать длинные статьи легко и приятно, а на других — глаза устают уже через пару абзацев? Секрет не в содержании, а в **типографике**. Профессионально оформленный текст имеет свой ритм, свои акценты и достаточное количество «воздуха».

Сегодня мы превратим «голый» HTML-код статьи в стильный и удобный для чтения блог-пост. Мы пройдем весь путь: от подбора шрифтов до настройки микро-отступов между буквами.

---

## 🏗️ Шаг 1: Подготовка контента

Для начала нам нужен качественный текст. Создайте файл `index.html` со следующей структурой:

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Искусство Типографики | Блог</title>
    <!-- Google Fonts подлючим на следующем шаге -->
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <article class="blog-post">
        <header class="post-header">
            <span class="category">Дизайн</span>
            <h1>Почему типографика — это 95% дизайна</h1>
            <p class="post-meta">Опубликовано 15 октября • 5 мин на чтение</p>
        </header>

        <section class="post-content">
            <p class="intro">В мире современного веб-дизайна мы часто увлекаемся яркими картинками, сложными анимациями и бесконечными градиентами. Но мы забываем о главном инструменте передачи информации.</p>

            <p>Типографика — это не просто выбор шрифта. Это искусство расположения текста так, чтобы он был не только читабельным, но и передавал нужное настроение. Хороший дизайнер знает, что шрифт имеет свой голос.</p>

            <blockquote>
                "Веб-дизайн — это типографика на 95%."
                <cite>— Оливер Райхенштейн</cite>
            </blockquote>

            <h2>Базовые правила для веба</h2>
            <p>Чтобы ваш текст было удобно читать с любого экрана, нужно соблюдать несколько простых, но критически важных правил:</p>
            <ul>
                <li><strong>Контраст:</strong> Текст должен четко выделяться на фоне.</li>
                <li><strong>Интерлиньяж:</strong> Не зажимайте строки, дайте им дышать.</li>
                <li><strong>Длина строки:</strong> Не заставляйте глаз бегать от края до края монитора.</li>
            </ul>

            <figure>
                <img src="https://images.unsplash.com/photo-1516051662687-567d7c4e8f6a" alt="Печатная машинка">
                <figcaption>Классическая типографика вдохновляет современных веб-разработчиков.</figcaption>
            </figure>

            <p>Помните, что каждый выбор — от <code>font-family</code> до <code>letter-spacing</code> — влияет на то, как пользователь воспримет вашу информацию.</p>
        </section>
    </article>
</body>
</html>
```

## 🎨 Шаг 2: Шрифтовая пара (Google Fonts)

Зайдем на Google Fonts и выберем два шрифта:
1. **Playfair Display** (Serif) — для элегантных заголовков. Выберем веса 700 и 900.
2. **Source Sans 3** (Sans-serif) — для максимально чистого и понятного чтения. Выберем веса 400 и 600.

Подключим их в HTML:
```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Playfair+Display:wght@700;900&family=Source+Sans+3:ital,wght@0,400;0,600;1,400&display=swap" rel="stylesheet">
```

## 🛠️ Шаг 3: Настройка "Базы" и Вертикального Ритма

В файле `style.css` сначала зададим общие правила.

```css
:root {
    --text-main: #2c3e50;
    --text-muted: #7f8c8d;
    --accent: #e67e22;
    --bg-body: #ffffff;
}

body {
    background-color: var(--bg-body);
    /* Используем сокращенную запись для базы */
    font: 400 1.125rem/1.65 'Source Sans 3', sans-serif;
    color: var(--text-main);
    margin: 0;
    padding: 0;
}

/* Ограничиваем ширину для удобного чтения */
.blog-post {
    max-width: 700px;
    margin: 60px auto;
    padding: 0 20px;
}
```

## 📐 Шаг 4: Оформление заголовков и мета-данных

Сделаем заголовки заметными и характерными.

```css
.category {
    text-transform: uppercase;
    letter-spacing: 0.2em;
    font-size: 0.75rem;
    font-weight: 600;
    color: var(--accent);
}

.post-header h1 {
    font-family: 'Playfair Display', serif;
    font-size: 3.5rem;
    line-height: 1.1;
    margin: 15px 0;
    font-weight: 900;
}

.post-meta {
    font-size: 0.9rem;
    color: var(--text-muted);
    font-style: italic;
    border-bottom: 1px solid #eee;
    padding-bottom: 20px;
    margin-bottom: 40px;
}

.post-content h2 {
    font-family: 'Playfair Display', serif;
    font-size: 2rem;
    margin-top: 40px;
    color: #1a1a1a;
}
```

## 📝 Шаг 5: Работа с текстом и цитатами

Добавим изюминку нашему тексту.

```css
/* Выделяем первый абзац (лид) */
.intro {
    font-size: 1.3rem;
    line-height: 1.5;
    color: #444;
    margin-bottom: 30px;
}

/* Оформляем цитату */
blockquote {
    margin: 40px 0;
    padding: 0 30px;
    border-left: 4px solid var(--accent);
    font-style: italic;
    font-size: 1.5rem;
    color: #555;
}

blockquote cite {
    display: block;
    font-size: 1rem;
    font-style: normal;
    text-transform: uppercase;
    letter-spacing: 1px;
    margin-top: 10px;
    color: var(--text-muted);
}

/* Подпись к картинке */
figcaption {
    text-align: center;
    font-size: 0.9rem;
    color: var(--text-muted);
    margin-top: 10px;
}

/* Оформление кода в тексте */
code {
    background: #f4f4f4;
    padding: 2px 5px;
    border-radius: 4px;
    font-family: monospace;
    font-size: 0.95em;
}
```

## 🪄 Шаг 6: Финальный штрих — Буквица

Буквица (drop cap) — это большая первая буква в начале статьи. Она сразу привлекает внимание и задает тон всему чтению.

```css
/* Выбираем первую букву первого параграфа в контенте */
.post-content p:first-of-type::first-letter {
    font-family: 'Playfair Display', serif;
    font-size: 5rem;
    font-weight: 900;
    float: left; /* Чтобы текст обтекал букву */
    line-height: 1;
    margin-right: 15px;
    color: var(--accent);
    text-transform: uppercase;
}
```

## 💡 Анализ проделанной работы

Что мы получили в итоге?
1. **Иерархия**: Заголовки (Playfair) сразу отличаются от текста (Source Sans).
2. **Ритм**: Межстрочный интервал `1.65` делает чтение длинного текста легким.
3. **Фокус**: Ширина в `700px` — это стандарт индустрии (Medium, Substack), который не дает глазу устать.
4. **Акценты**: Оранжевый цвет направляет взгляд на категорию, цитату и буквицу.
5. **Чистота**: Использование `rem` гарантирует, что типографика будет хорошо выглядеть на любом устройстве.

## 🧪 Самопроверка

Откройте ваш результат в браузере и проверьте по списку:
1. **Читабельность**: Текст не кажется слишком тесным? (Проверьте `line-height`).
2. **Контраст**: Легко ли отличить цитату от обычного текста?
3. **Шрифты**: Загрузились ли Google Fonts? (Если нет, сработал ли ваш стек — `serif/sans-serif`).
4. **Адаптивность**: Уменьшите ширину окна. Текст должен подстраиваться под размер экрана без горизонтальной прокрутки.

## ❓ Часто задаваемые вопросы

### «Почему мы использовали float для буквицы?»
Это классический способ. Он заставляет текст «обтекать» большую букву. Есть и более современные методы (например, `initial-letter`), но они пока хуже поддерживаются браузерами.

### «Можно ли использовать три разных шрифта?»
Технически — да. Но на практике это почти всегда перегружает дизайн. Пара «Заголовок + Текст» — это самый безопасный и профессиональный выбор.

### «Зачем мы использовали :first-of-type?»
Чтобы буквица появилась только в самом первом абзаце статьи, а не в каждом параграфе. Это делает акцент уникальным.
