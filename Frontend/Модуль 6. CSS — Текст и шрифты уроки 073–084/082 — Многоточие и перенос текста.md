## 📖 Введение: Когда текст не влезает

В идеальном мире дизайнера заголовки всегда короткие, а слова — удобной длины. В реальном мире фронтенд-разработчика пользователи вводят «Длинношеееееее», ссылки бывают на три строки, а названия товаров в сетке карточек могут занимать в два раза больше места, чем планировалось.

Если мы ничего не предпримем, текст либо «вывалится» из своего блока, перекрывая другие элементы, либо просто некрасиво обрежется. Сегодня мы научимся профессионально управлять такими ситуациями: заставлять слова переноситься там, где нужно, или элегантно прятать лишнее за тремя точками.

---

## 🚫 Запрет переноса: white-space

По умолчанию браузер переносит текст на новую строку, как только он доходит до края контейнера. Свойство `white-space` управляет этим поведением.

### Основные значения:
1. **normal**: (По умолчанию). Текст переносится, лишние пробелы схлопываются.
2. **nowrap**: **Самое важное значение**. Весь текст будет идти одной длинной строкой, даже если он выйдет за границы экрана.
3. **pre**: Текст отображается точно так же, как он написан в HTML (со всеми пробелами и переносами строк). Поведение тега `<pre>`.
4. **pre-wrap**: Сохраняет все переносы и пробелы, но при этом переносит текст на новую строку, если он не влезает.

## 💬 Эффект многоточия: text-overflow

Чтобы создать то самое знаменитое многоточие в конце длинной строки, нам нужно использовать «святую троицу» свойств. Одно без другого работать не будет.

```css
.truncate {
  width: 200px;           /* 1. Ограничиваем ширину */
  white-space: nowrap;    /* 2. Запрещаем перенос строк */
  overflow: hidden;       /* 3. Прячем то, что не влезло */
  text-overflow: ellipsis; /* 4. Добавляем магическое многоточие */
}
```

**Как это работает:**
Браузер видит, что текст не влезает в одну строку, обрезает его и в самый конец (перед границей блока) вставляет символ `…`. Это идеальный способ оформления заголовков в списках или имен пользователей в шапке сайта.

## ✂️ Разрыв слов: word-break и overflow-wrap

Что делать, если у вас есть очень длинное слово (например, немецкое "Rindfleischetikettierungsüberwachungsaufgabenübertragungsgesetz" или просто очень длинная ссылка), которое не влезает в ширину мобильного экрана? Оно просто «проткнет» ваш дизайн насквозь.

### 1. word-break: break-all
Это свойство разрешает разрывать слово в **любом** месте, даже если это некрасиво с точки зрения грамматики.
```css
.break {
  word-break: break-all;
}
```

### 2. overflow-wrap: break-word (ранее word-wrap)
Более «умное» свойство. Оно сначала пытается перенести слово на новую строку целиком, и только если оно не влезает даже на целую строку, разрывает его.
**Это рекомендуемый вариант для большинства случаев.**

## 📚 Многострочное многоточие (Line Clamp)

Сделать многоточие в конце первой строки легко. Но что, если мы хотим показать ровно три строки текста, а всё, что дальше — обрезать?

Долгое время это было невозможно чистым CSS, но сейчас есть стандартный (хоть и странный по синтаксису) способ:

```css
.line-clamp {
  display: -webkit-box;
  -webkit-line-clamp: 3;    /* Количество видимых строк */
  -webkit-box-orient: vertical;
  overflow: hidden;
}
```
Несмотря на префикс `-webkit-`, этот код сейчас поддерживается во всех современных браузерах (включая Firefox и Edge) и является стандартом де-факто для обрезания анонсов статей в блогах.

## 🧪 Практические советы

1. **Для кнопок**: Всегда используйте `white-space: nowrap` для кнопок, если не хотите, чтобы текст внутри них внезапно разделился на две строки на узких экранах.
2. **Для ссылок**: Если вы выводите ссылки, оставленные пользователями, всегда добавляйте `overflow-wrap: break-word;`, иначе одна длинная ссылка может «развалить» всю вашу сетку.
3. **Подсказки**: Помните, что скрывая текст за многоточием, вы скрываете информацию. Хорошим тоном считается добавление атрибута `title` к такому элементу:
   ```html
   <div class="truncate" title="Полное длинное название товара">
     Полное длинное название...
   </div>
   ```
   Тогда при наведении мышки пользователь увидит полный текст.

## ❓ Часто задаваемые вопросы

### «Почему text-overflow: ellipsis не работает у меня?»
Проверьте, задана ли блоку ширина (`width` или `max-width`) и стоит ли `overflow: hidden`. Если блок может бесконечно расширяться, тексту просто нет смысла обрезаться.

### «Можно ли заменить многоточие на другой символ?»
Да, в свойстве `text-overflow` можно написать строку: `text-overflow: "---";`. Но многоточие `ellipsis` — это встроенное ключевое слово, которое работает наиболее стабильно.

### «В чем разница между word-break и word-wrap?»
`word-break` — это про то, как мы разрезаем слова внутри строк. `word-wrap` (теперь `overflow-wrap`) — это про то, переносим ли мы длинное слово, которое переполняет контейнер. Для большинства задач «спасения верстки» лучше подходит `overflow-wrap: break-word`.

## 🧪 Практика: Карточка анонса статьи

```html
<div class="card">
  <h2 class="card-title">Очень длинный заголовок новости, который точно не влезет в одну строку</h2>
  <p class="card-text">Здесь находится текст анонса. Мы хотим показать только небольшую часть, чтобы пользователь заинтересовался и нажал кнопку. Если текста будет слишком много, он просто скроется за многоточием, сохраняя чистоту нашего дизайна.</p>
</div>
```

```css
.card {
  width: 300px;
  padding: 20px;
  border: 1px solid #ddd;
  font-family: sans-serif;
}

.card-title {
  font-size: 1.2rem;
  margin-bottom: 10px;
  /* Заголовок в одну строку с многоточием */
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}

.card-text {
  font-size: 0.9rem;
  line-height: 1.5;
  color: #666;
  /* Анонс ровно в 3 строки */
  display: -webkit-box;
  -webkit-line-clamp: 3;
  -webkit-box-orient: vertical;
  overflow: hidden;
}
```

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 1. **`white-space: nowrap`** — запрещает перенос текста на новую строку.
> 2. **`text-overflow: ellipsis`** — работает только вместе с `nowrap` и `overflow: hidden`.
> 3. **`overflow-wrap: break-word`** — спасает верстку от аномально длинных слов и ссылок.
> 4. **`-webkit-line-clamp`** — современный способ обрезать многострочный текст.
> 5. Скрывая текст, всегда думайте о пользователе: добавляйте `title` или возможность увидеть текст полностью.
> 6. Управление переполнением — критически важный навык для создания надежных интерфейсов.
