## 📖 Что такое браузер?

**Браузер** — это программа, которая умеет:
1. **Отправлять запросы** к серверам (мы разобрали это в прошлом уроке)
2. **Получать файлы** (HTML, CSS, JS, картинки)
3. **Превращать код в визуальную страницу**, которую ты видишь на экране

По сути, браузер — это **переводчик**. Он берёт код, написанный разработчиком, и превращает его в красивую, интерактивную страницу.

## 🌐 Какие браузеры бывают?

| Браузер | Разработчик | Движок | Доля рынка (2024) |
|---|---|---|---|
| **Google Chrome** | Google | Blink | ~65% |
| **Safari** | Apple | WebKit | ~18% |
| **Firefox** | Mozilla | Gecko | ~3% |
| **Edge** | Microsoft | Blink | ~5% |
| **Opera** | Opera Software | Blink | ~2% |
| **Яндекс Браузер** | Яндекс | Blink | ~3% (в РФ больше) |

> [!tip] Что использовать для обучения?
> Рекомендуется **Google Chrome**. У него лучшие инструменты разработчика, и большинство учебных материалов показывают примеры именно в Chrome. Но подойдёт любой современный браузер.

## ⚙️ Что такое движок браузера?

**Движок** (engine) — это «мозг» браузера. Именно он читает HTML, CSS, JavaScript и отрисовывает страницу.

| Движок | Браузеры | Особенности |
|---|---|---|
| **Blink** | Chrome, Edge, Opera, Яндекс Браузер | Самый популярный, разрабатывается Google |
| **WebKit** | Safari | Предшественник Blink, используется Apple |
| **Gecko** | Firefox | Независимый движок от Mozilla |

### Почему это важно знать?

Разные движки могут **немного по-разному** отображать одну и ту же страницу. Иногда CSS-свойство работает в Chrome, но не работает в Safari. Поэтому профессиональные разработчики проверяют свои сайты в нескольких браузерах.

> [!note] Пока не переживай
> На этапе обучения достаточно одного браузера. Тестирование в разных браузерах — это задача для более продвинутого уровня.

## 🧩 Из чего состоит браузер?

Браузер — это не просто «окно с сайтом». Внутри него работает множество компонентов.

| Компонент | Что делает | Аналогия |
|---|---|---|
| **Пользовательский интерфейс** | Адресная строка, кнопки «Назад/Вперёд», вкладки | 🚗 Руль и педали автомобиля |
| **Сетевой модуль** | Отправляет HTTP-запросы, загружает файлы | 📡 Антенна, принимающая сигнал |
| **HTML-парсер** | Читает HTML-код и строит DOM-дерево | 📋 Читает чертёж дома |
| **CSS-парсер** | Читает CSS и понимает, как всё должно выглядеть | 🎨 Читает дизайн-проект |
| **JavaScript-движок** | Выполняет JS-код (в Chrome это V8) | ⚡ Электрик, подключающий проводку |
| **Модуль рендеринга** | Объединяет всё и рисует страницу на экране | 🖼️ Художник, рисующий финальную картину |

## 🔄 Как браузер превращает код в страницу?

Это называется **рендеринг** (rendering) — процесс отрисовки страницы. Давай разберём его по шагам.

### Шаг 1: Загрузка файлов

| Что происходит | Подробности |
|---|---|
| Браузер получает **HTML-файл** | Это первый файл, который приходит от сервера |
| Браузер начинает **читать HTML** сверху вниз | Построчно, как книгу |
| Встречает ссылки на **CSS и JS файлы** | И запрашивает их тоже |
| Встречает ссылки на **картинки** | И начинает их загрузку |

---

### Шаг 2: Построение DOM-дерева

Браузер читает HTML и строит **DOM** (Document Object Model) — **древовидную структуру** страницы.

Допустим, у нас есть такой HTML:

```html
<!DOCTYPE html>
<html>
  <head>
    <title>Мой сайт</title>
  </head>
  <body>
    <h1>Привет!</h1>
    <p>Это абзац текста.</p>
  </body>
</html>
```
Браузер превращает это в дерево:

|Уровень|Элемент|Родитель|
|---|---|---|
|1|`document`|—|
|2|`html`|document|
|3|`head`|html|
|4|`title` → «Мой сайт»|head|
|3|`body`|html|
|4|`h1` → «Привет!»|body|
|4|`p` → «Это абзац текста.»|body|

> [!note] DOM — это очень важно  
> Когда мы будем изучать JavaScript, мы будем **работать с DOM напрямую**: находить элементы, менять их, добавлять новые. Но это будет позже — в модуле 20.

---

### Шаг 3: Построение CSSOM

Параллельно браузер читает CSS и строит **CSSOM** (CSS Object Model) — структуру стилей.



```CSS
h1 {
    color: blue;
    font-size: 32px;
}

p {
    color: gray;
}
```

|Элемент|Стили|
|---|---|
|`h1`|цвет: синий, размер шрифта: 32px|
|`p`|цвет: серый|

---

### Шаг 4: Объединение в Render Tree

Браузер объединяет **DOM** и **CSSOM** в **Render Tree** (дерево рендеринга). Это дерево содержит только **видимые** элементы с их стилями.

|Элемент|Видимый?|Стили|
|---|---|---|
|`html`|✅|—|
|`head`|❌ (не отображается)|—|
|`body`|✅|—|
|`h1`|✅|синий, 32px|
|`p`|✅|серый|

> [!note] Скрытые элементы  
> Элементы с `display: none` **не попадают** в Render Tree. Элемент `<head>` тоже не отображается — он содержит служебную информацию.

---

### Шаг 5: Layout (расположение)

Браузер вычисляет **размеры и позиции** каждого элемента на экране.

|Элемент|Ширина|Высота|Позиция (X, Y)|
|---|---|---|---|
|`body`|1920px|auto|0, 0|
|`h1`|1920px|44px|0, 0|
|`p`|1920px|24px|0, 44|

Этот процесс ещё называют **Reflow** — браузер «раскладывает» элементы на странице, как мебель в комнате.

---

### Шаг 6: Paint (отрисовка)

Наконец, браузер **рисует пиксели** на экране: цвета, тени, границы, текст, картинки.

---

### Весь процесс рендеринга — сводка

|Шаг|Название|Что делает|
|---|---|---|
|1|**Загрузка**|Скачивает HTML, CSS, JS, картинки с сервера|
|2|**Парсинг HTML**|Строит DOM-дерево из HTML-кода|
|3|**Парсинг CSS**|Строит CSSOM из CSS-кода|
|4|**Render Tree**|Объединяет DOM + CSSOM (только видимые элементы)|
|5|**Layout**|Вычисляет размеры и позиции элементов|
|6|**Paint**|Рисует пиксели на экране|

> [!tip] Это происходит очень быстро  
> Весь процесс рендеринга занимает **миллисекунды**. Но если страница тяжёлая (много картинок, сложный CSS, тяжёлый JS), рендеринг может замедлиться, и пользователь увидит «подвисание».

## 🔧 DevTools — инструменты разработчика

**DevTools** (Developer Tools) — это набор инструментов, встроенных в каждый браузер. Они позволяют **заглянуть под капот** любого сайта.

### Как открыть DevTools?

|Способ|Windows / Linux|Mac|
|---|---|---|
|Горячая клавиша|`F12`|`Cmd + Option + I`|
|Горячая клавиша (альт.)|`Ctrl + Shift + I`|`Cmd + Option + I`|
|Через меню|Меню → Дополнительные инструменты → Инструменты разработчика|Аналогично|
|Правый клик|Правый клик на странице → «Просмотреть код»|Правый клик → «Посмотреть код элемента»|

> [!important] Попробуй прямо сейчас!  
> Открой любой сайт, нажми `F12` (или `Ctrl + Shift + I`) и посмотри, что произошло. Появилась панель с кодом — это DevTools.

---

### Основные вкладки DevTools

|Вкладка|Что показывает|Когда нужна|
|---|---|---|
|**Elements**|HTML-структура страницы и CSS-стили|Просмотр и редактирование разметки и стилей|
|**Console**|Место для выполнения JavaScript-кода|Отладка JS, вывод сообщений|
|**Sources**|Все загруженные файлы (HTML, CSS, JS)|Просмотр исходного кода|
|**Network**|Все сетевые запросы (загрузка файлов)|Анализ скорости загрузки|
|**Application**|Хранилища (LocalStorage, cookies)|Работа с сохранёнными данными|
|**Performance**|Производительность страницы|Оптимизация (продвинутый уровень)|

> [!note] Пока нам нужны только две вкладки  
> **Elements** — чтобы смотреть HTML и CSS.  
> **Console** — чтобы писать JavaScript.  
> Остальные вкладки мы освоим по мере прохождения курса.

---

### Вкладка Elements — подробнее

Это самая полезная вкладка для начинающего. Она показывает:

**Левая панель** — HTML-код страницы в виде дерева. Можно раскрывать и сворачивать элементы.

**Правая панель** — CSS-стили выбранного элемента.

#### Что можно делать во вкладке Elements?

|Действие|Как сделать|Зачем|
|---|---|---|
|**Выбрать элемент**|Нажми на иконку 🔍 (левый верхний угол DevTools), затем кликни на любой элемент на странице|Увидеть его HTML и CSS|
|**Изменить текст**|Двойной клик на текст в HTML-дереве|Поэкспериментировать (изменения временные)|
|**Изменить стиль**|В правой панели измени значение CSS-свойства|Попробовать другой цвет, размер|
|**Удалить элемент**|Выдели элемент → нажми `Delete`|Посмотреть, как страница выглядит без него|

> [!warning] Изменения в DevTools — временные!  
> Всё, что ты меняешь через DevTools, **исчезнет** после обновления страницы. Ты не ломаешь сайт — ты меняешь его только **у себя на экране**. Экспериментируй смело!

---

### Вкладка Console — подробнее

**Console** — это место, где можно выполнять JavaScript-код прямо в браузере.

Попробуй прямо сейчас:

1. Открой DevTools (`F12`)
2. Перейди на вкладку **Console**
3. Введи следующую команду и нажми `Enter`:



```JavaScript
alert('Привет! Я написал свой первый JavaScript!');
```

Ты должен увидеть всплывающее окно с текстом. Поздравляю — ты только что **выполнил JavaScript-код**!

Ещё несколько команд для экспериментов:

|Команда|Что произойдёт|
|---|---|
|`alert('Привет!')`|Покажет всплывающее окно|
|`console.log('Тест')`|Выведет текст в консоль|
|`document.title`|Покажет заголовок текущей страницы|
|`document.title = 'Мой сайт'`|Изменит заголовок вкладки (временно)|
|`document.body.style.backgroundColor = 'red'`|Покрасит фон страницы в красный|

> [!tip] Не бойся экспериментировать  
> Консоль — твоя песочница. Здесь можно пробовать любой код. Сломать что-то невозможно — просто обнови страницу.

> [!note] Мы будем активно использовать этот режим

## 🔍 Просмотр исходного кода страницы

Помимо DevTools, есть ещё один способ посмотреть HTML-код любого сайта.

|Способ|Как сделать|Что увидишь|
|---|---|---|
|**Ctrl + U** (Windows) / **Cmd + U** (Mac)|Нажми на любой странице|Весь HTML-код страницы в отдельной вкладке|
|**Правый клик → «Просмотр кода страницы»**|Правый клик мышкой|То же самое|

Это покажет **исходный HTML-код**, который сервер отправил браузеру. Выглядит страшно, но через несколько уроков ты начнёшь его понимать.

## 📊 Как сайт загружается — вкладка Network

Вкладка **Network** показывает все файлы, которые браузер загрузил для отображения страницы.

### Как посмотреть?

1. Открой DevTools (`F12`)
2. Перейди на вкладку **Network**
3. Обнови страницу (`F5` или `Ctrl + R`)
4. Увидишь список всех загруженных файлов

|Столбец|Что показывает|
|---|---|
|**Name**|Имя файла|
|**Type**|Тип файла (html, css, js, png, jpg)|
|**Size**|Размер файла|
|**Time**|Сколько времени заняла загрузка|

> [!tip] Попробуй на разных сайтах  
> Открой вкладку Network на `google.com` — там мало файлов. Потом открой на каком-нибудь новостном сайте — там будут **сотни** файлов. Вот почему некоторые сайты загружаются медленно.

## ❓ Частые вопросы

### «Почему один и тот же сайт может выглядеть по-разному в разных браузерах?»

Потому что разные браузеры используют разные **движки рендеринга**. Они по-разному интерпретируют некоторые CSS-свойства. Это называется **кросс-браузерная совместимость**, и разработчики тратят немало времени на решение этой проблемы.

### «Что такое кэш браузера?»

**Кэш** — это временное хранилище. Когда ты открываешь сайт, браузер **сохраняет файлы** (картинки, CSS, JS) на твоём компьютере. При повторном посещении он берёт их из кэша, а не скачивает заново. Поэтому второй раз сайт открывается быстрее.

### «Как очистить кэш?»

|Браузер|Горячие клавиши|
|---|---|
|Chrome / Edge|`Ctrl + Shift + Delete`|
|Firefox|`Ctrl + Shift + Delete`|
|Safari|`Cmd + Option + E`|

Или при обновлении страницы: `Ctrl + Shift + R` (жёсткое обновление без кэша).

### «Можно ли через DevTools взломать сайт?»

**Нет**. DevTools изменяет страницу только **на твоём экране**. Сервер ничего не знает о твоих изменениях. Это как перерисовать картинку на своём мониторе — оригинал не меняется.

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 
> 1. **Браузер** — это программа, которая превращает код (HTML, CSS, JS) в визуальную страницу
> 2. Внутри браузера работает **движок рендеринга** (Blink, WebKit, Gecko)
> 3. Процесс рендеринга: **HTML → DOM** + **CSS → CSSOM** → **Render Tree** → **Layout** → **Paint**
> 4. **DevTools** (`F12`) — главный инструмент веб-разработчика
> 5. Вкладка **Elements** — для просмотра HTML и CSS
> 6. Вкладка **Console** — для выполнения JavaScript
> 7. Изменения в DevTools **временные** — они исчезают после обновления страницы
