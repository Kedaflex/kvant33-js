## 📖 Каждый HTML-файл имеет одинаковую структуру

Неважно, это сайт Google, маленький блог или твой учебный проект — **любой** HTML-документ состоит из одних и тех же обязательных частей.

Вот минимальный правильный HTML-файл:

```html
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Название страницы</title>
  </head>
  <body>
    <!-- Здесь будет содержимое страницы -->
  </body>
</html>
```

Выглядит сложнее, чем в прошлом уроке. Но сейчас мы разберём **каждую строку**, и всё станет понятно.

## 📐 Общая схема

|Часть|Роль|Аналогия|
|---|---|---|
|`<!DOCTYPE html>`|Объявление типа документа|📋 Обложка книги: «Это книга, а не журнал»|
|`<html>`|Корневой элемент, оболочка всего|📦 Коробка, в которой лежит всё остальное|
|`<head>`|Служебная информация|🧠 Мозг: думает, но не виден снаружи|
|`<body>`|Видимое содержимое|👤 Тело: то, что видят люди|

## 1️⃣ `<!DOCTYPE html>` — объявление типа документа

HTML

```
<!DOCTYPE html>
```

### Что делает?

Говорит браузеру: **«Этот документ написан на HTML5»**.

### Зачем нужен?

Без `<!DOCTYPE>` браузер переходит в **quirks mode** (режим совместимости) — он начинает отображать страницу по старым правилам, как в 2001 году. Это может приводить к неожиданным результатам.

|С `<!DOCTYPE html>`|Без `<!DOCTYPE html>`|
|---|---|
|✅ Стандартный режим (standards mode)|⚠️ Режим совместимости (quirks mode)|
|Браузер следует современным правилам|Браузер имитирует поведение старых версий|
|Всё отображается предсказуемо|Возможны баги в отображении|

### Что нужно запомнить?

|Факт|Подробности|
|---|---|
|Всегда первая строка|`<!DOCTYPE html>` пишется **до** всего остального|
|Это не тег|Это **объявление** (declaration), а не тег|
|Не имеет закрывающей части|Просто одна строка|
|Регистр не важен|`<!DOCTYPE html>`, `<!doctype html>` — оба варианта работают|
|Всегда одинаковый|В HTML5 пишется именно так, без вариантов|

> [!tip] Интересный факт  
> В старых версиях HTML объявление DOCTYPE было длинным и страшным:
> 
> HTML
> 
> ```
> <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN"
>     "http://www.w3.org/TR/html4/loose.dtd">
> ```
> 
> В HTML5 его упростили до `<!DOCTYPE html>`. Спасибо разработчикам стандарта.

## 2️⃣ `<html>` — корневой элемент

HTML

```
<html lang="ru">
  ...
</html>
```

### Что делает?

Оборачивает **весь** HTML-документ. Это **корень** дерева — все остальные элементы находятся внутри `<html>`.

### Атрибут `lang`

|Атрибут|Значение|Зачем|
|---|---|---|
|`lang="ru"`|Язык содержимого — русский|Помогает браузерам, поисковикам и экранным читалкам понять язык страницы|
|`lang="en"`|Язык содержимого — английский|Для англоязычных страниц|
|`lang="uk"`|Язык содержимого — украинский|Для украиноязычных страниц|

### Зачем указывать язык?

|Кому помогает|Как именно|
|---|---|
|**Браузеру**|Предлагает перевод страницы, если язык не совпадает с языком пользователя|
|**Поисковикам**|Google понимает, на каком языке страница, и показывает её нужной аудитории|
|**Экранным читалкам**|Программы для слабовидящих людей правильно произносят текст с нужным акцентом|
|**CSS**|Можно стилизовать элементы в зависимости от языка|

> [!important] Всегда указывай lang  
> Это занимает 8 символов, но улучшает доступность и SEO. Привыкай писать `<html lang="ru">`.

## 3️⃣ `<head>` — служебная информация

HTML

```
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Название страницы</title>
</head>
```

### Что делает?

Содержит **мета-информацию** о странице. Ничего из `<head>` **не отображается** на самой странице (кроме `<title>`, который показывается во вкладке браузера).

### Что может быть внутри `<head>`?

|Элемент|Назначение|Обязательный?|
|---|---|---|
|`<meta charset="UTF-8">`|Кодировка символов|✅ Да|
|`<meta name="viewport" ...>`|Настройка для мобильных устройств|✅ Да|
|`<title>`|Заголовок вкладки|✅ Да|
|`<link>`|Подключение CSS-файлов|Позже|
|`<script>`|Подключение JS-файлов|Позже|
|`<meta name="description">`|Описание для поисковиков|Позже|
|`<link rel="icon">`|Иконка сайта (favicon)|Позже|

> [!note] Не всё сразу  
> Сейчас мы разберём три обязательных элемента. Остальные (`<link>`, `<script>`, и другие мета-теги) будем добавлять постепенно по мере прохождения курса.

---

### 📌 `<meta charset="UTF-8">`

HTML

```
<meta charset="UTF-8">
```

**Кодировка** — это таблица соответствия между числами и символами. Компьютер хранит всё в числах, а кодировка говорит, какое число соответствует какой букве.

|Кодировка|Поддерживает|Проблемы|
|---|---|---|
|**UTF-8**|Все языки мира: русский, китайский, арабский, эмодзи 🎉|Нет проблем|
|Windows-1251|Только кириллицу|Латиница и другие языки — крокозябры|
|ASCII|Только латиницу|Русский текст — крокозябры|

#### Что будет без `<meta charset="UTF-8">`?

Русский текст может превратиться в **крокозябры**:

|С charset="UTF-8"|Без charset (или неправильная кодировка)|
|---|---|
|Привет, мир!|ÐŸÑ€Ð¸Ð²ÐµÑ‚, Ð¼Ð¸Ñ€!|
|Всё отображается правильно|Нечитаемые символы|

> [!warning] Всегда добавляй эту строку  
> `<meta charset="UTF-8">` должен быть **первым** элементом внутри `<head>`. Браузер должен узнать кодировку **до** того, как начнёт читать текст.

### 📌 `<meta name="viewport">`

HTML

```
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

Этот тег нужен для **правильного отображения на мобильных устройствах**.

#### Что он делает?

|Параметр|Значение|Зачем|
|---|---|---|
|`width=device-width`|Ширина страницы = ширина устройства|Без этого мобильный браузер показывает десктопную версию, сжатую до маленького экрана|
|`initial-scale=1.0`|Начальный масштаб = 100%|Страница не приближена и не отдалена|

#### Разница на мобильном

|С viewport|Без viewport|
|---|---|
|Текст читаемый, нормального размера|Текст крошечный, страница «сжата»|
|Элементы адаптируются к экрану|Видна десктопная версия целиком, нужно зумить|

> [!note] Пока просто добавляй  
> Мы подробнее разберём viewport и адаптивную вёрстку в модуле 11. Сейчас просто включай эту строку в каждый файл.

### 📌 `<title>` — заголовок страницы



```HTML
<title>Название страницы</title>
```

#### Где отображается `<title>`?

|Место|Как выглядит|
|---|---|
|**Вкладка браузера**|Текст на вкладке|
|**Результаты поиска Google**|Синяя кликабельная строка|
|**Закладки**|Название закладки при сохранении|
|**Социальные сети**|Заголовок при репосте ссылки|

#### Правила хорошего `<title>`

|Правило|Хороший пример ✅|Плохой пример ❌|
|---|---|---|
|Описывай содержание страницы|`Каталог товаров — МойМагазин`|`Страница 1`|
|50–60 символов максимум|`Курсы веб-разработки для начинающих`|`Лучшие курсы по веб-разработке HTML CSS JavaScript для начинающих с нуля бесплатно 2024 год`|
|Уникальный для каждой страницы|Своё название на каждой странице|Одинаковый title на всех страницах|

## 4️⃣ `<body>` — видимая часть страницы



```HTML
<body>
    <!-- Здесь всё, что видит пользователь -->
</body>
```

### Что делает?

Содержит **всё**, что отображается на странице: текст, изображения, кнопки, формы, видео — абсолютно всё.

### Правила

|Правило|Подробности|
|---|---|
|Только один `<body>`|В документе может быть **только один** тег `<body>`|
|Внутри `<html>`|`<body>` находится внутри `<html>`, после `<head>`|
|Весь контент — здесь|Никакой видимый контент не должен быть за пределами `<body>`|

## 5️⃣ Комментарии в HTML

В коде из примера ты видел строку:

HTML

```
<!-- Здесь будет содержимое страницы -->
```

Это **комментарий**. Браузер его **полностью игнорирует**. Комментарии нужны для заметок разработчика.

|Синтаксис|Пример|
|---|---|
|`<!-- текст комментария -->`|`<!-- TODO: добавить навигацию -->`|

### Зачем нужны комментарии?

|Причина|Пример|
|---|---|
|Пометить секцию кода|`<!-- Шапка сайта -->`|
|Оставить заметку для себя|`<!-- Исправить отступы -->`|
|Временно «выключить» код|`<!-- <p>Этот абзац пока не нужен</p> -->`|
|Объяснить сложный участок|`<!-- Используем Grid из-за бага в Firefox -->`|

> [!tip] Горячая клавиша  
> Выдели строку или блок кода и нажми `Ctrl + /` — VS Code автоматически обернёт его в комментарий. Нажми ещё раз — комментарий уберётся.

## ⚡ Emmet — быстрое создание шаблона

**Emmet** — это инструмент, встроенный в VS Code, который позволяет писать HTML **в разы быстрее** с помощью сокращений.

### Самое главное сокращение

Набери в пустом `.html` файле символ `!` и нажми `Tab` (или `Enter`):



```text
!
```

VS Code мгновенно развернёт это в полный шаблон:

```HTML
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
  </head>
  <body>
    
  </body>
</html>
```

> [!important] Используй это в каждом новом файле  
> Вместо того чтобы набирать шаблон вручную каждый раз — просто набери `!` и нажми `Tab`. Затем поменяй `lang="en"` на `lang="ru"` и напиши свой `<title>`.

### Что нужно изменить после генерации?

|Что менять|Было|Стало|
|---|---|---|
|Язык|`lang="en"`|`lang="ru"`|
|Заголовок|`<title>Document</title>`|`<title>Твоё название</title>`|

---

### Другие полезные сокращения Emmet

Попробуй набрать эти сокращения в `.html` файле (внутри `<body>`) и нажать `Tab`:

|Сокращение|Результат|
|---|---|
|`h1`|`<h1></h1>`|
|`p`|`<p></p>`|
|`ul>li*3`|`<ul>` с тремя `<li></li>` внутри|
|`h2+p`|`<h2></h2>` и `<p></p>` друг за другом|
|`a`|`<a href=""></a>`|
|`img`|`<img src="" alt="">`|
|`div.container`|`<div class="container"></div>`|
|`p*5`|Пять тегов `<p></p>`|
|`p{Текст абзаца}`|`<p>Текст абзаца</p>`|

> [!note] Emmet — это бонус  
> Необязательно учить все сокращения сейчас. Ты будешь осваивать их постепенно. Главное — запомни `!` + `Tab` для создания шаблона.

## 🔍 Вложенность и отступы

HTML-код имеет **иерархическую структуру** — одни теги находятся внутри других. Для читаемости мы используем **отступы**.

### Правила вложенности

|Правило|Пример|
|---|---|
|`<head>` и `<body>` — внутри `<html>`|`<html>` → `<head>` + `<body>`|
|Мета-теги и `<title>` — внутри `<head>`|`<head>` → `<meta>` + `<title>`|
|Весь контент — внутри `<body>`|`<body>` → `<h1>` + `<p>` + ...|
|Каждый уровень вложенности = +1 отступ|Дочерний элемент сдвинут вправо|

### Правильные отступы



```HTML
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <title>Мой сайт</title>
  </head>
  <body>
    <h1>Заголовок</h1>
    <p>Абзац текста.</p>
    <ul>
      <li>Первый пункт</li>
      <li>Второй пункт</li>
    </ul>
  </body>
</html>
```

### Неправильные отступы (всё работает, но читать невозможно)

```HTML
<!DOCTYPE html>
<html lang="ru">
<head>
<meta charset="UTF-8">
<title>Мой сайт</title>
</head>
<body>
<h1>Заголовок</h1>
<p>Абзац текста.</p>
<ul>
<li>Первый пункт</li>
<li>Второй пункт</li>
</ul>
</body>
</html>
```

> [!warning] Браузеру всё равно  
> Браузер проигнорирует все отступы и пробелы — страница будет выглядеть одинаково. Но **тебе и другим разработчикам** не всё равно. Аккуратные отступы делают код **читаемым**.

### Табуляция vs пробелы

|Настройка|Описание|
|---|---|
|**2 пробела**|Стандарт для HTML/CSS/JS (используем в курсе)|
|**4 пробела**|Альтернативный вариант|
|**Tab**|Клавиша Tab вставляет отступ (VS Code заменяет на пробелы)|

VS Code по умолчанию использует 4 пробела. Чтобы изменить на 2:

|Шаг|Действие|
|---|---|
|1|Нажми на надпись `Spaces: 4` в правом нижнем углу Status Bar|
|2|Выбери **«Indent Using Spaces»**|
|3|Выбери **2**|

## 🧪 Практика: создаём правильный шаблон

Давай создадим страницу с полным правильным шаблоном.

### Шаг 1: Создай новый проект

|Шаг|Действие|
|---|---|
|1|В папке `web-course` создай папку `003-template`|
|2|Внутри создай файл `index.html`|
|3|Набери `!` и нажми `Tab`|
|4|Поменяй `lang="en"` на `lang="ru"`|
|5|Поменяй `<title>Document</title>` на `<title>Правильный шаблон</title>`|

### Шаг 2: Добавь контент в `<body>`



```HTML
<body>
  <h1>Структура HTML-документа</h1>
  <p>Каждый HTML-файл состоит из одних и тех же частей:</p>

  <h2>DOCTYPE</h2>
  <p>Объявление <strong>DOCTYPE</strong> говорит браузеру, что это HTML5.</p>

  <h2>Элемент html</h2>
  <p>Корневой элемент. Всё остальное — <em>внутри него</em>.</p>

  <h2>Элемент head</h2>
  <p>Служебная информация, невидимая на странице.</p>
  <ul>
    <li>Кодировка (charset)</li>
    <li>Viewport для мобильных</li>
    <li>Заголовок вкладки (title)</li>
  </ul>

  <h2>Элемент body</h2>
  <p>Всё, что видит пользователь.</p>

  <!-- Это комментарий. Браузер его не покажет. -->
</body>
```

### Шаг 3: Сохрани и проверь

Сохрани файл (`Ctrl + S`) и открой через Live Server. Убедись, что всё отображается правильно.

## ❓ Частые вопросы

### «Что будет, если поменять порядок `<head>` и `<body>`?»

Браузер всё равно отобразит страницу, потому что он умеет «исправлять» ошибки. Но это **неправильный HTML**. Всегда соблюдай порядок: сначала `<head>`, потом `<body>`.

### «Можно ли не писать `<html>`, `<head>`, `<body>`?»

Технически браузер добавит их сам. Но это **плохая практика**. Явно указывай все обязательные теги — это делает код понятным и предсказуемым.

### «Зачем мне запоминать шаблон, если есть Emmet `!`?»

Emmet — отличный инструмент, но ты должен **понимать**, что генерирует. На собеседовании могут попросить написать структуру HTML-документа на бумаге. Понимание важнее, чем механическое использование.

### «Сколько файлов может быть на одном сайте?»

Сколько угодно. Простой сайт может состоять из одного `index.html`. Крупный сайт — из сотен файлов. Каждая страница — это обычно отдельный HTML-файл (или генерируется программно).

## 📋 Финальный шаблон (эталон)

Этот шаблон используй для каждого нового HTML-файла в курсе:



```HTML
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Название страницы</title>
  </head>
  <body>

  </body>
</html>
```

|Строка|Нужно менять?|Что менять|
|---|---|---|
|`<!DOCTYPE html>`|❌ Никогда|Всегда одинаковый|
|`<html lang="ru">`|⚠️ Иногда|Язык страницы, если не русский|
|`<meta charset="UTF-8">`|❌ Никогда|Всегда UTF-8|
|`<meta name="viewport" ...>`|❌ Никогда|Всегда одинаковый|
|`<title>...</title>`|✅ Всегда|Уникальный заголовок для каждой страницы|

## 🔑 Ключевые мысли урока

> [!abstract] Запомни
> 
> 1. Каждый HTML-файл начинается с `<!DOCTYPE html>`
> 2. `<html lang="ru">` — корневой элемент с указанием языка
> 3. `<head>` — служебная информация (кодировка, viewport, title)
> 4. `<meta charset="UTF-8">` — обязателен, иначе крокозябры
> 5. `<meta name="viewport">` — обязателен для мобильных
> 6. `<title>` — заголовок вкладки, важен для SEO
> 7. `<body>` — всё видимое содержимое страницы
> 8. `<!-- комментарий -->` — заметки в коде, браузер их не показывает
> 9. `!` + `Tab` в VS Code — мгновенная генерация шаблона через Emmet
> 10. Правильные **отступы** делают код читаемым
