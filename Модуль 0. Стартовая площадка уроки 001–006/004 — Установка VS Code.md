## 📖 Зачем нужен редактор кода?

HTML, CSS и JavaScript — это обычный **текст**. Технически, писать код можно даже в Блокноте (Notepad). Но это всё равно что забивать гвозди микроскопом — можно, но неудобно.

**Редактор кода** — это программа, специально созданная для написания кода. Она умеет:

| Функция | Что даёт | Пример |
|---|---|---|
| **Подсветка синтаксиса** | Разные части кода окрашены в разные цвета | Теги — одним цветом, атрибуты — другим |
| **Автодополнение** | Предлагает варианты по мере ввода | Набираешь `<h` → предлагает `<h1>`, `<h2>`... |
| **Автозакрытие тегов** | Закрывает теги автоматически | Набираешь `<div>` → сам добавляет `</div>` |
| **Подсветка ошибок** | Подчёркивает ошибки в коде | Забыл закрыть тег — подсветит красным |
| **Форматирование** | Выравнивает код одной кнопкой | Превращает хаос в читаемый код |
| **Терминал** | Встроенная командная строка | Не нужно переключаться между окнами |
| **Расширения** | Тысячи плагинов для любых задач | Live Server, Prettier, и другие |

## 🆚 Почему именно VS Code?

Существует много редакторов кода. Вот самые популярные:

| Редактор | Цена | Плюсы | Минусы |
|---|---|---|---|
| **VS Code** | Бесплатный | Быстрый, огромное количество расширений, популярный | Потребляет больше памяти, чем простые редакторы |
| Sublime Text | Условно бесплатный | Очень быстрый, лёгкий | Меньше расширений, просит купить лицензию |
| WebStorm | Платный (~$70/год) | Мощные инструменты из коробки | Платный, тяжёлый |
| Notepad++ | Бесплатный | Лёгкий, простой | Только Windows, мало возможностей |
| Vim | Бесплатный | Невероятно быстрый | Сложнейший интерфейс, не для новичков |

> [!important] Мы используем VS Code
> **Visual Studio Code** (VS Code) — самый популярный редактор кода в мире. Его используют более **70% веб-разработчиков**. Он бесплатный, кроссплатформенный и имеет огромную экосистему расширений.

> [!warning] Не путай!
> **VS Code** (Visual Studio Code) и **Visual Studio** — это **разные программы**. Visual Studio — тяжёлая IDE для C#/C++. Нам нужен именно VS Code — лёгкий редактор.

## 📥 Установка VS Code

### Windows

| Шаг | Действие |
|---|---|
| 1 | Перейди на сайт [https://code.visualstudio.com](https://code.visualstudio.com) |
| 2 | Нажми большую кнопку **«Download for Windows»** |
| 3 | Скачается файл `VSCodeUserSetup-x64-*.exe` |
| 4 | Запусти скачанный файл |
| 5 | Нажимай **«Далее»** на каждом шаге установки |
| 6 | На шаге **«Выберите дополнительные задачи»** отметь галочки (см. таблицу ниже) |
| 7 | Нажми **«Установить»** |
| 8 | Дождись завершения и нажми **«Готово»** |

#### Галочки на шаге «Дополнительные задачи»

| Опция | Поставить? | Зачем |
|---|---|---|
| Создать значок на рабочем столе | ✅ Да | Быстрый доступ |
| Добавить действие «Открыть с помощью Code» в контекстное меню файла | ✅ Да | Правый клик → открыть файл в VS Code |
| Добавить действие «Открыть с помощью Code» в контекстное меню папки | ✅ Да | Правый клик → открыть папку в VS Code |
| Зарегистрировать Code как редактор по умолчанию | ✅ Да | Файлы кода будут открываться в VS Code |
| Добавить в PATH | ✅ Да | Можно открывать VS Code из командной строки |

### macOS

| Шаг | Действие |
|---|---|
| 1 | Перейди на [https://code.visualstudio.com](https://code.visualstudio.com) |
| 2 | Нажми **«Download for Mac»** |
| 3 | Скачается файл `VSCode-darwin-*.zip` |
| 4 | Открой скачанный `.zip` файл (распакуется автоматически) |
| 5 | Перетащи **Visual Studio Code.app** в папку **Applications** |
| 6 | Открой VS Code из папки Applications |
| 7 | При первом запуске macOS спросит разрешение — нажми **«Открыть»** |

#### Добавление в PATH (macOS)

Чтобы открывать VS Code из терминала командой `code`:

| Шаг | Действие |
|---|---|
| 1 | Открой VS Code |
| 2 | Нажми `Cmd + Shift + P` (откроется командная палитра) |
| 3 | Набери `shell command` |
| 4 | Выбери **«Shell Command: Install 'code' command in PATH»** |
| 5 | Готово! Теперь в терминале можно писать `code .` для открытия папки |

### Настройка Live Server

**Live Server** — самое важное расширение для нашего курса. Оно запускает локальный сервер и **автоматически обновляет страницу** в браузере каждый раз, когда ты сохраняешь файл.

#### Как использовать Live Server?

|Шаг|Действие|
|---|---|
|1|Открой HTML-файл в VS Code|
|2|Нажми правой кнопкой мыши в редакторе|
|3|Выбери **«Open with Live Server»**|
|4|Откроется браузер со страницей|
|5|Теперь при каждом сохранении (`Ctrl + S`) страница обновляется автоматически|

Или ещё проще: нажми кнопку **«Go Live»** в правом нижнем углу Status Bar.

> [!note] Зачем Live Server, если можно просто открыть HTML-файл?  
> Можно, но тогда после каждого изменения тебе придётся **вручную обновлять** страницу в браузере (`F5`). С Live Server обновление происходит **автоматически**. Экономит кучу времени.

---

### Настройка Prettier

**Prettier** — расширение для автоматического форматирования кода.

#### Включаем автоформатирование при сохранении

|Шаг|Действие|
|---|---|
|1|Открой настройки VS Code: `Ctrl + ,` (или `Cmd + ,` на Mac)|
|2|В поисковой строке набери `format on save`|
|3|Поставь галочку напротив **«Editor: Format On Save»**|
|4|В поисковой строке набери `default formatter`|
|5|В выпадающем списке **«Editor: Default Formatter»** выбери **Prettier**|

Теперь при каждом сохранении (`Ctrl + S`) код будет автоматически форматироваться.

#### До и после Prettier

**До (некрасивый код):**



```HTML
<div><h1>Привет</h1><p>Текст</p>
    <a href="https://example.com"
>Ссылка</a></div>
```

**После (отформатированный код):**



```HTML
<div>
  <h1>Привет</h1>
  <p>Текст</p>
  <a href="https://example.com">Ссылка</a>
</div>
```

## 📂 Создаём рабочую папку для курса

Давай создадим структуру папок, в которой будем хранить все файлы курса.

|Шаг|Действие|
|---|---|
|1|Открой **Проводник** (Windows) или **Finder** (Mac)|
|2|Перейди в удобное место (например, Рабочий стол или папку Документы)|
|3|Создай папку с названием `web-course`|
|4|Внутри создай папку `001-hello-world`|
|5|Открой VS Code|
|6|Нажми **File → Open Folder** (или `Ctrl + K, Ctrl + O`)|
|7|Выбери папку `web-course`|

### Рекомендуемая структура папок



```text
📂 web-course/
├── 📂 001-hello-world/
├── 📂 002-structure/
├── 📂 003-tags/
└── ... (будем создавать по мере прохождения курса)
```

> [!tip] Правила именования папок и файлов
> 
> |Правило|Хорошо ✅|Плохо ❌|
> |---|---|---|
> |Латинские буквы|`my-project`|`мой-проект`|
> |Дефис вместо пробела|`hello-world`|`hello world`|
> |Нижний регистр|`index.html`|`Index.HTML`|
> |Без спецсимволов|`page-1`|`page#1!`|
